# Gap Register

This register turns broad partial-conformance areas into named backlog items that can be moved milestone by milestone.

## Closed Structural Gaps

| Gap ID | Linked requirement area | Resolution |
| --- | --- | --- |
| `GAP-DOCS-001` | `REQ-DOCS-MIRROR-*` | Closed. The local standards mirror is now sourced from the single archive declared in `docs/posix-manifest.txt`, and `docs/fetch-posix-docs.sh` populates it directly. |
| `GAP-SH-002` | `REQ-SH-STARTUP-*`, `REQ-SH-OPERANDS-*` | Closed (Milestone 1). `$-` `i` flag is now fixed at startup in the `interactive` field, not recomputed dynamically from terminal state. |
| `GAP-SH-004` | `REQ-SH-INTERACTIVE-*`, `REQ-JOBS-SIGNALS-*` | Closed (Milestone 1). Interactive shells now ignore SIGQUIT and SIGTERM at startup, install SIGINT handler, and SIGINT during `read_line` discards the current input and re-prompts. |
| `GAP-EXPAND-004` | `REQ-EXPAND-CMDSUB-*` | Closed (Milestone 1). `capture_output` no longer returns `Err` on non-zero child exit. It always returns `Ok(text)` and sets `self.last_status` to the child's exit code. |
| `GAP-EXEC-005` | `REQ-EXEC-STATUS-*` | Closed (Milestone 1). Removed the erroneous immediate `remove` after `insert` in `wait_on_job_index`. Job status is now correctly consumed on the first subsequent `wait` call. |
| `GAP-EXPAND-001` | `REQ-EXPAND-FIELDS-*`, `REQ-EXPAND-PARAMS-*` | Closed (Milestone 2). `"$@"` now produces separate fields via the `Segment::AtBreak` / `Expansion::AtFields` model. Zero positionals produce zero fields. Prefix/suffix joining works correctly. `"$*"` uses `positional_params()` and joins with IFS[0]. |
| `GAP-EXPAND-003` | `REQ-EXPAND-PARAMS-*` | Closed (Milestone 2). `${...}` brace scanning now respects quotes, backslash escapes, and nested `${...}`, `$(...)`, `$((...))`, and backtick constructs via `scan_to_closing_brace`. Backtick command substitution (`` `cmd` ``) is recognized by both the tokenizer and expander, including inside double-quotes. The tokenizer also correctly keeps `$(...)`, `$((...))`, and `${...}` as single word tokens when unquoted. |
| `GAP-EXEC-001` | `REQ-EXEC-ERRORS-*` | Closed (Milestone 3). Prefix variable assignments before non-special builtins and functions are now temporary (save/restore via `save_vars`/`restore_vars`). Special-builtin assignments remain permanent per POSIX. Assignment values are expanded via `expand_word_text` which performs tilde, parameter, command substitution, arithmetic expansion, and quote removal — but no field splitting or pathname expansion. |
| `GAP-EXEC-003` | `REQ-EXEC-SEARCH-*`, `REQ-EXEC-UTILITY-*` | Closed (Milestone 3). `resolve_command_path` now checks both `is_regular_file()` and `is_executable()` via `stat_path`. Pre-exec access check uses both `F_OK` and `X_OK`. `exec_replace` takes separate `file` and `argv` parameters so `argv[0]` is the command name as typed. Exec failure exit codes: ENOENT → 127, EACCES/other → 126. ENOEXEC fallback sets `child_shell.shell_name` to the original command name. |
| `GAP-EXEC-004` | `REQ-EXEC-ASYNC-*` | Closed (Milestone 3). Background `&` commands redirect stdin from `/dev/null` via `stdin_override` threaded through `spawn_and_or` → `spawn_pipeline`. Job start message now prints `[%d] %d\n` (job id and last PID). AND-OR lists with `&` (e.g. `cmd1 && cmd2 &`) execute the full AND-OR list asynchronously in a forked subshell via `execute_and_or`. |
| `GAP-EXPAND-002` | `REQ-EXPAND-TILDE-*`, `REQ-EXPAND-QUOTE-*`, `REQ-EXPAND-ARITH-*` | Closed (Milestone 5). Tilde expansion now handles bare `~`, `~user` via `getpwnam`, and tilde after `:` in assignment contexts. Double-quote backslash now only escapes `$`, `` ` ``, `"`, `\`, `}`, and newline per POSIX. Arithmetic expansion supports the full POSIX operator set: comparison, bitwise, logical, ternary, assignment, unary, parentheses, variable references (bare and `$`-prefixed), octal/hex constants, and parameter/command expansion within `$((...))`. Assignment side effects persist. |
| `GAP-EXEC-002` | `REQ-EXEC-ENV-*`, `REQ-EXPAND-CMDSUB-*`, `REQ-EXEC-GROUP-*` | Closed (Milestone 6). Eliminated the render+reparse round-trip: `execute_nested_program` now directly executes the parsed AST via `execute_list_item` instead of serializing to text and re-parsing. Subshell trap reset added at all five fork sites (`spawn_and_or`, `fork_and_execute_command`, `execute_command` Subshell variant, `spawn_prepared` ENOEXEC fallback, `capture_output`): `TrapAction::Command` entries are removed and OS signal handlers reset to default; `TrapAction::Ignore` entries are preserved per POSIX 2.13. Alias test expectations updated to reflect correct POSIX parse-time alias resolution (aliases defined inside compound command bodies are not visible to later commands in the same body). |
| `GAP-BUILTIN-001` | `REQ-BUILTIN-CD-*` | Closed (Milestone 7). `cd` now supports `-L` (logical, default), `-P` (physical), and `-e` (error if PWD cannot be determined after `-P` chdir). Last-one-wins for `-L`/`-P`. Logical mode prepends `$PWD` and canonicalizes `.`/`..` components via `canonicalize_logical_path`. Physical mode uses `get_cwd()` after `chdir`. Combined flags like `-LP`, `-PL`, `-Pe` are parsed correctly. |
| `GAP-BUILTIN-003` | `REQ-BUILTIN-READ-*` | Closed (Milestone 7). `read` with no variable names now reads into `REPLY` per POSIX. Backslash continuation, `-r`, `-d`, IFS splitting, and readonly error reporting were already in place. |
| `GAP-BUILTIN-004` | `REQ-BUILTIN-TRAP-*` | Closed (Milestone 7). Added 12 new signal names (ILL, FPE, KILL, USR1, SEGV, USR2, PIPE, CHLD, TSTP, TTIN, TTOU, BUS, SYS, CONT) with numeric codes and SIG prefix support. `ignored_on_entry` tracking added: `Shell::probe_ignored_signals` queries the initial disposition of all trappable signals at startup via `query_signal_disposition`, and `set_trap` silently refuses to modify signals ignored on entry in non-interactive shells per POSIX. |
| `GAP-BUILTIN-005` | `REQ-BUILTIN-UMASK-*` | Closed (Milestone 7). Added `s` (setuid/setgid) and `X` (conditional execute) permission letters to `symbolic_permission_bits`. Since `umask` only manages the 0o777 permission mask, `s` contributes zero bits (accepted without error). |
| `GAP-JOBS-001` | `REQ-JOBS-CONTROL-*` | Closed (Milestone 8). `set -m` now has full runtime effect: shell puts itself into its own process group and takes the terminal foreground at startup when interactive. Stopped-job tracking via `WUNTRACED`/`wifstopped`, `JobState` enum (`Running`/`Stopped`/`Done`), complete POSIX job-id grammar (`%%`, `%+`, `%-`, `%N`, `%string`, `%?string`) with `current_job_id`/`previous_job_id` resolution. `fg` and `bg` check `set -m`, send `SIGCONT` to stopped jobs, and manage terminal foreground process group. `jobs` supports `-l` (long) and `-p` (pid-only) modes with `+`/`-` current/previous markers and stopped/running/done state display. `kill` builtin added with `-l`, `-s`, numeric shorthand, `%job` specifiers, and `--` separator. |
| `GAP-JOBS-002` | `REQ-JOBS-SIGNALS-*` | Closed (Milestone 8). When job control is disabled (`set +m`), asynchronous lists inherit `SIG_IGN` for `SIGINT` and `SIGQUIT` in a forked wrapper process; when enabled, normal signal inheritance applies and the wrapper is not needed for single pipelines. Interactive shells with `set -m` ignore `SIGTSTP`, `SIGTTIN`, and `SIGTTOU`. |
| `GAP-JOBS-003` | `REQ-SYS-JOBCONTROL-*`, `REQ-JOBS-CONTROL-*` | Closed (Milestone 8). Terminal attribute save/restore via `tcgetattr`/`tcsetattr` added to `SystemInterface`. When a foreground job is stopped (`SIGTSTP`), the shell saves terminal attributes in the job's `saved_termios` field. When `fg` resumes a job, saved attributes are restored via `set_terminal_attrs` before handing off the terminal foreground. `setup_job_control` at startup calls `setpgid` and `tcsetpgrp` to take ownership of the terminal. |
| `GAP-SH-001` | `REQ-SH-OPTIONS-*` | Closed (Milestone 8). `set -m` now has full runtime effect: enables job control when set, defaults to on for interactive shells. All POSIX single-letter options are now fully implemented. |

## High-Priority Open Gaps

| Gap ID | Linked requirement area | Current gap |
| --- | --- | --- |
| `GAP-SH-003` | `REQ-SH-INTERACTIVE-*` | Command history list semantics are still simplified, and command-line / vi-mode editing remain unimplemented. |
| `GAP-BUILTIN-002` | `REQ-BUILTIN-SET-*` | `set` now covers `-a`, `-b`, `-C`, `-e`, `-f`, `-h`, `-m`, `-n`, `-u`, `-v`, `-x`, named `-o`/`+o` for all 11 option names, option-state reporting (`set -o`/`set +o`), and `nounset` expansion failures. `-m` now has full runtime effect. Remaining: `ignoreeof`, `nolog`, and finer Issue 8 edge cases. |
| `GAP-BUILTIN-006` | `REQ-BUILTIN-*`, `REQ-SYS-ACCOUNTING-*` | Mirrored utility pages such as `hash`, `getopts`, `ulimit`, and `fc` still need either implementation or an explicit conformance decision before a full POSIX claim is credible. |

## Milestone Mapping

Each milestone targets roughly equal implementation effort.

- **Milestone 0** (complete): structural mirror and matrix work; `GAP-DOCS-001` closed.
- **Milestone 1** (complete): quick correctness fixes; `GAP-SH-002`, `GAP-SH-004`, `GAP-EXPAND-004`, `GAP-EXEC-005` closed.
- **Milestone 2** (complete): core expansion fixes; `GAP-EXPAND-001`, `GAP-EXPAND-003` closed. `"$@"` separate-field semantics via `Segment`/`Expansion` enums, quote-aware `${...}` brace scanning via `scan_to_closing_brace`, backtick command substitution in tokenizer and expander, `$*` IFS join via `positional_params()`.
- **Milestone 3** (complete): execution model fixes; `GAP-EXEC-001`, `GAP-EXEC-003`, `GAP-EXEC-004` closed. Temporary prefix assignments via `save_vars`/`restore_vars`, assignment expansion without field splitting via `expand_word_text`, command search `X_OK`/`is_executable()`, separate `file`/`argv` in `exec_replace` for correct `argv[0]`, exit 126/127 distinction, ENOEXEC `$0` fix, background stdin from `/dev/null`, job message `[%d] %d\n`, AND-OR with `&` via subshell fork.
- **Milestone 4** (complete): Shell options (errexit, xtrace): `GAP-SH-001` (partial), `GAP-BUILTIN-002` (partial). Added `-e` (errexit) with full POSIX exception rules (condition suppression in `if`/`while`/`until`/`elif`, negated pipeline suppression, non-final AND-OR command suppression), `-x` (xtrace) with PS4 expansion, `-b` (notify stub), `-h` (hashall stub), `-m` (monitor stub). Fixed combined `-c` in multi-char option loop (e.g. `sh -ec '...'`). `set` builtin now delegates to `set_short_option` instead of duplicating match arms. `$-` includes `b`, `e`, `h`, `x` when active. `set -o`/`set +o` listing covers all 11 named options.
- **Milestone 5** (complete): Expansion completeness: `GAP-EXPAND-002` closed. Fixed double-quote backslash to only escape `$`, `` ` ``, `"`, `\`, `}`, and newline. Tilde expansion now handles `~user` via `getpwnam` and tilde after `:` in assignment values. Full arithmetic parser with recursive descent covering all POSIX operators (comparison, bitwise, shift, logical, ternary, assignment), variable references (bare `x` and `$x`), octal/hex constants, unary `+`/`-`/`~`/`!`, parameter and command expansion within `$((...))`, and side-effect persistence for assignment operators. Added `sys::home_dir_for_user` via `getpwnam` to `SystemInterface`. 32 new unit tests and 10 new integration tests. Production-code line coverage remains at 100.00%.
- **Milestone 6** (complete): Execution environment and subshells: `GAP-EXEC-002` closed. Eliminated the render+reparse round-trip in `execute_nested_program` — compound command bodies (if, while, for, case, brace group, subshell) now execute directly from the parsed AST. Added `Shell::reset_traps_for_subshell` which removes `TrapAction::Command` entries and resets OS signal handlers to default while preserving `TrapAction::Ignore` per POSIX 2.13. Trap reset installed at all five fork sites: `spawn_and_or`, `fork_and_execute_command`, Subshell variant in `execute_command`, ENOEXEC fallback in `spawn_prepared`, and `capture_output`. Alias integration tests updated for correct POSIX parse-time resolution semantics. 1 new unit test and 6 new integration tests. Production-code line coverage remains at 100.00% (5124/5124).
- **Milestone 7** (complete): Builtin completeness: `GAP-BUILTIN-001`, `GAP-BUILTIN-003`, `GAP-BUILTIN-004`, `GAP-BUILTIN-005` closed. `cd` now supports `-L` (logical, default), `-P` (physical), and `-e` with full POSIX 10-step algorithm: logical mode prepends `$PWD` and canonicalizes `.`/`..` via `canonicalize_logical_path`; physical mode uses `get_cwd()` after `chdir`. `read` with no variable names reads into `REPLY`. Added 18 trappable signals (up from 6), SIG prefix support, numeric signal codes, and `ignored_on_entry` tracking via `probe_ignored_signals`/`query_signal_disposition` — non-interactive shells silently refuse to trap signals ignored on entry per POSIX. `umask` now accepts `s` and `X` in symbolic mode. 5 new unit tests and 7 new integration tests.
- **Milestone 8** (complete): Job control: `GAP-JOBS-001`, `GAP-JOBS-002`, `GAP-JOBS-003`, `GAP-SH-001` (`-m` portion) closed. `set -m` with full runtime effect (shell PGRP setup, terminal foreground ownership), `WUNTRACED`/`wifstopped`-based stopped-job detection, `JobState` enum for job lifecycle tracking, terminal attribute save/restore via `tcgetattr`/`tcsetattr`, complete POSIX job-id grammar (`%%`, `%+`, `%-`, `%N`, `%string`, `%?string`), enhanced `jobs` (`-l`/`-p`, state display, `+`/`-` markers), `fg`/`bg` with terminal handoff and `SIGCONT`, `kill` builtin with `-l`/`-s`/numeric shorthand/`%job`/`--`, async signal inheritance (`SIG_IGN` for `SIGINT`/`SIGQUIT` when `set +m`). 49 new unit tests and coverage at 99.94% (5449/5452).
- **Milestone 9** — Missing builtins and interactive editing: `GAP-BUILTIN-006`, `GAP-SH-003`. Implement `hash`, `getopts`, `ulimit`, `fc`, command history semantics, and vi-mode line editing.
