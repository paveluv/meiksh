## Context

Action code "U" (Unspecified) in the "Actions when Creating a File that Already Exists" table: the effect is unspecified unless specified by the utility description. This applies to certain combinations of existing and new file types where the standard intentionally leaves behavior undefined. For `meiksh`, the shell may encounter this with unusual file type combinations (e.g., redirecting output where the target is a socket or certain special files). The shell need not guarantee any specific behavior in these cases but should avoid crashing. See POSIX Utilities Chapter 1, Section 1.1.1.4 action code U (utilities/V3_chap01.html line 682).
