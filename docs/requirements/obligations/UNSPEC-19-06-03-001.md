## Extracted Text

> If the commands string consists solely of redirections, the results are unspecified.

## Context

**Behavior:** In the `$(commands)` form, if the entire command string is only redirections with no actual command (e.g., `$(> file)`, `$(2>&1)`, `$(< input)`), the result is **unspecified**. This is a standalone extraction of the same rule documented in SHALL-19-06-03-005.

**Implementation guidance:** The implementation may choose to: execute the redirections in the subshell (with no command, producing no output), produce an empty string, or report an error. The chosen behavior should be documented. A reasonable default is to execute the redirections (which may create files as side effects) and produce empty output.

**Spec reference:** See 2.6.3 Command Substitution in utilities/V3_chap02.html, line 1129.
