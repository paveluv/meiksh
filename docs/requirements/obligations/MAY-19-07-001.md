## Extracted Text

> The shell may support an additional format used for redirection:

## Context

The shell MAY support an additional redirection format using braces to capture an automatically-assigned file descriptor number. See 2.7 Redirection in utilities/V3_chap02.html.

**Optional format:** `{location}redir-op word` — where `location` is non-empty and indicates a location (such as a shell variable name) where the shell stores the integer fd number it allocates.

**Decision for meiksh:** This is a MAY obligation. The implementation can choose to support this (e.g., bash supports `{varname}>file` which opens the file on a new fd and assigns the fd number to `varname`) or to not support it. If not supported, this syntax should be treated as a regular word/error.

**If supported:** The behavior is implementation-defined (see IMPLDEF-19-07-001). Common implementations assign the next available fd >= 10, open the file, and store the fd number in the named variable.
