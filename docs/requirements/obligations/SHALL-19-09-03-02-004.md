## Extracted Text

> If, and only if, job control is disabled, the standard input for the subshell in which an asynchronous AND-OR list is executed shall initially be assigned to an open file description that behaves as if /dev/null had been opened for reading only. This initial assignment shall be overridden by any explicit redirection of standard input within the AND-OR list.

## Context

**Behavior:** If and ONLY if job control is disabled, the stdin for the subshell executing an async AND-OR list MUST initially be assigned to a file description that behaves as if `/dev/null` had been opened for reading only. This assignment is overridden by any explicit stdin redirection within the AND-OR list.

**Applies to:** `cmd &` when job control is disabled (non-interactive mode or `set +m`). When job control is enabled, stdin is NOT redirected to /dev/null — the background job shares the terminal's stdin.

**Implementation detail:** When forking the background subshell with job control disabled: redirect fd 0 to `/dev/null` (or equivalent) in the child before executing the AND-OR list. If the AND-OR list itself contains `< file` or similar stdin redirection, that overrides the /dev/null assignment.

**Spec reference:** Section 2.9.3.1, lines 2165-2166 in `utilities/V3_chap02.html`.
