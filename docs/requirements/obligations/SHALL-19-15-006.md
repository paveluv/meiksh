## Context

**`--` argument handling for special built-ins.**

Some special built-ins are described as conforming to XBD 12.2 Utility Syntax Guidelines (see basedefs/V1_chap12.html#tag_12_02). For those that do conform, standard option parsing rules apply, including `--` to end option processing.

For special built-ins that are **not** described as conforming to the syntax guidelines, the general requirement from 1.4 Utility Description Defaults (see utilities/V3_chap01.html#tag_18_04) that `--` be recognized as a first argument to be discarded **does not apply**. A conforming application **shall not** use `--` with those non-conforming special built-ins.

Implementation: for each special built-in, check its specification to determine if it conforms to XBD 12.2. If it does, handle `--`. If not, do not treat `--` specially — it may be treated as a regular argument or an error depending on the specific built-in.
