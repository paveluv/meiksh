## Extracted Text

> In locales other than the POSIX locale, a character may have a state-dependent encoding. There are two types of these encodings:: A locking-shift encoding (where the state of the character is determined by a shift code that may affect more than the single character following it) cannot be defined with the current character set description file format. Use of a locking-shift encoding with any of the standard utilities in the Shell and Utilities volume of POSIX.1-2024 or with any of the functions in the System Interfaces volume of POSIX.1-2024 that do not specifically mention the effects of state-dependent encoding is implementation-defined.

## Context

Locking-shift encodings (where a shift code changes the interpretation of all subsequent characters until another shift code) are implementation-defined for the shell. The shell MAY choose not to support locking-shift encodings, but if it does, the behavior must be documented. In practice, locking-shift encodings (like ISO-2022 variants) are rare in modern systems, and meiksh can document that locking-shift encodings are not supported.
