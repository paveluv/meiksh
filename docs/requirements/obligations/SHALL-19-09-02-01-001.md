## Extracted Text

> The exit status of a pipeline shall depend on whether or not the pipefail option (see set) is enabled and whether or not the pipeline begins with the ! reserved word, as described in the following table. The pipefail option determines which command in the pipeline the exit status is derived from; the ! reserved word causes the exit status to be the logical NOT of the exit status of that command. The shell shall use the pipefail setting at the time it begins execution of the pipeline, not the setting at the time it sets the exit status of the pipeline. (For example, in command1 | set -o pipefail the exit status of command1 has no effect on the exit status of the pipeline, even if the shell executes set -o pipefail in the current shell environment.)

## Context

The shell must determine the exit status of a pipeline according to a four-way matrix based on two boolean conditions:

1. **Whether `pipefail` is enabled** (see `set -o pipefail` in 2.26 set, utilities/V3_chap02.html#tag_19_26)
2. **Whether the pipeline begins with `!`**

The exit status rules are:

| pipefail | `!` prefix | Exit status derived from |
|----------|-----------|--------------------------|
| off | no | Last (rightmost) command in the pipeline |
| off | yes | Logical NOT of the last command's exit status (0→1, non-zero→0) |
| on | no | Last (leftmost-to-rightmost) command that exits with non-zero status, or zero if all succeed |
| on | yes | Logical NOT of the pipefail-derived status |

Critical timing rule: the shell must use the `pipefail` setting as it was **at the time execution of the pipeline begins**, not at the time the pipeline's exit status is determined. Example: `command1 | set -o pipefail` — even if `set -o pipefail` runs in the current shell environment, the exit status of `command1` has no effect on the pipeline exit status because `pipefail` was not enabled when the pipeline began.

The `!` reserved word performs logical NOT: if the derived exit status is zero, the pipeline exit status is 1; if non-zero, the pipeline exit status is 0.

See 2.9.2 Pipelines in utilities/V3_chap02.html#tag_19_09_02 for the full pipeline semantics.
