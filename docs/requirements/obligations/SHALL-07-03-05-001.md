## Extracted Text

> The LC_TIME category shall define the interpretation of the conversion specifications supported by the date utility and shall affect the behavior of the strftime(), wcsftime(), strptime(), and nl_langinfo() functions. Since the interfaces for C-language access and locale definition differ significantly, they are described separately.

## Context

The LC_TIME locale category defines time and date formatting rules, including day names, month names, AM/PM strings, and date/time format strings. It affects the `date` utility and C library functions like `strftime()`. The shell is affected because: (1) the shell must correctly propagate the LC_TIME environment variable to child processes (especially `date`); (2) the `printf` builtin's `%(datefmt)T` extension (if supported) may use `strftime()` which depends on LC_TIME; (3) the PS1/PS4 prompt expansions may include time formatting that depends on LC_TIME. The shell must ensure LC_TIME is correctly inherited from the environment and exported to child processes.

See: `basedefs/V1_chap07.html` §7.3.5 LC_TIME.
