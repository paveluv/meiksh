## Extracted Text

> Whenever an error is detected and the first character in the optstring operand is not a <colon> (':'), a diagnostic message shall be written to standard error with the following information in an unspecified format:: If an option is found that was not specified in optstring, this error is identified and the invalid option character shall be identified in the message.

## Context

**Behavior:** When an unrecognized option character is found (verbose mode, no leading `:`), the diagnostic message to stderr must:
1. Identify the error type (unknown/illegal option).
2. Include the invalid option character in the message.

**Example formats (historical, not mandated):**
- `"script: illegal option -- x"`
- `"script: unknown option: -x"`

The exact format is unspecified, but the option character must be identifiable by the user.

**Spec reference:** POSIX.1-2024, `getopts` STDERR, lines 181-182 of `utilities/getopts.html`.
