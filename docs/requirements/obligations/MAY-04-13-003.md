## Extracted Text

> Network byte order may not be convenient for processing actual values. For this, it is more sensible for values to be stored as ordinary integers. This is known as "host byte order". In host byte order:: Bits might not be allocated to bytes in any obvious order at all.

## Context

This obligation concerns byte ordering details for host integer representations. It is not relevant to a POSIX shell implementation — the shell does not perform low-level byte-order manipulation. No shell action is required.

See: `basedefs/V1_chap04.html` §4.13 Host and Network Byte Orders.
