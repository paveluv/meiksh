## Extracted Text

> Expands to the positional parameters, starting from one, initially producing one field for each positional parameter that is set. When the expansion occurs in a context where field splitting will be performed, any empty fields may be discarded and each of the non-empty fields shall be further split as described in 2.6.5 Field Splitting. When the expansion occurs in a context where field splitting will not be performed, the initial fields shall be joined to form a single field with the value of each parameter separated by the first character of the IFS variable if IFS contains at least one character, or separated by a <space> if IFS is unset, or with no separation if IFS is set to a null string.

## Context

This defines the expansion behavior of the special parameter `$*` (asterisk).

**Required behavior — outside double-quotes (field-splitting context):**
- `$*` expands to the positional parameters starting from `$1`, initially producing one field per set positional parameter.
- Empty fields may be discarded, and each non-empty field is further split according to IFS (see 2.6.5 Field Splitting).

**Required behavior — in a non-field-splitting context (including inside double-quotes `"$*"`):**
- The initial fields (one per positional parameter) must be joined into a single field.
- The separator between each parameter's value is:
  - The first character of `IFS`, if `IFS` contains at least one character.
  - A `<space>` if `IFS` is unset.
  - No separator (direct concatenation) if `IFS` is set to the null/empty string.

**Key distinction from `$@`:** Inside double-quotes, `"$*"` always produces exactly one field (joined), whereas `"$@"` produces one field per positional parameter.

**Conditions:** Applies in all execution contexts. The IFS-based joining behavior is the primary mechanism for stringifying positional parameters.

**Testing notes:** Verify with various IFS values: default (`<space><tab><newline>`), custom single-character (e.g., `IFS=,`), multi-byte first character, empty IFS, and unset IFS.

See 2.5.2 "Special Parameters" (`*`) and 2.6.5 "Field Splitting" in `utilities/V3_chap02.html`.
