## Extracted Text

> If any of the errors shown as "shall exit" or "may exit" occur in a subshell environment, the shell shall (respectively, may) exit from the subshell environment with a non-zero status and continue in the environment from which that subshell environment was invoked.

## Context

Defines how the error exit rules apply within subshell environments. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required:**
- If any error shown as "shall exit" in the table occurs in a subshell environment, the shell SHALL exit from the subshell with a non-zero status and continue in the parent environment.
- If any error shown as "may exit" in the table occurs in a subshell environment, the shell MAY exit from the subshell with a non-zero status and continue in the parent environment.
- In neither case does the parent shell exit — only the subshell exits.

**Key implication:** Subshell environments (created by `(...)`, command substitution `$(...)`, pipe segments, etc.) are isolated. An error that would kill a non-interactive shell script only kills the subshell, and the parent continues.

**Example:** `(false_syntax_here)` — the subshell exits with non-zero status, but the parent script continues.
