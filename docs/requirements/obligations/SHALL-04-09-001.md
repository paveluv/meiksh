## Extracted Text

> Uppercase and lowercase letters shall retain their unique identities between conforming implementations.

## Context

The shell must treat uppercase and lowercase letters in filenames as distinct characters. When the shell performs pathname resolution, command lookup, or any file operation, it must preserve the case of letters exactly as specified. For example, `File.txt` and `file.txt` must refer to different directory entries. This applies to all shell operations that deal with filenames: redirection targets, command names, operands to builtins like `cd`, `.` (dot), and `source`, as well as pathname expansion (globbing). The shell itself must never fold or normalize letter case in filenames.

See: `basedefs/V1_chap04.html` §4.9 Filenames.
