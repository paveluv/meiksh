## Extracted Text

> Otherwise, the exit status of command shall be that of the simple command specified by the arguments to command.

## Context

**Behavior:** When `command` successfully invokes a simple command (exit codes 126 and 127 do not apply), the exit status of `command` must be the exit status of the invoked command. The `command` utility is transparent with respect to exit status — it passes through whatever the executed command returned.

**Example:** `command ls /nonexistent` — if `ls` exits with status 2, then `command` also exits with status 2.

**Applies to:** Successful invocation of any command via `command [-p] command_name [args...]`.

**Spec reference:** POSIX.1-2024, `utilities/command.html`, section EXIT STATUS (tag_20_22_14), lines 210-211.
