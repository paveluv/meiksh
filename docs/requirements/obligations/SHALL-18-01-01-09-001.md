## Extracted Text

> When the current working directory (see XBD 3.94 Current Working Directory) is to be changed, unless the utility or function description states otherwise, the operation shall succeed unless a call to the chdir() function defined in the System Interfaces volume of POSIX.1-2024 would fail when invoked with the new working directory pathname as its argument.

## Context

When the shell changes the current working directory (via `cd` or any other mechanism), the operation **shall** succeed if and only if a `chdir()` system call with the target pathname would succeed. This means the shell must not change directories to a path that `chdir()` would reject (e.g., non-existent directory, insufficient permissions, path component not a directory). If the `chdir()` equivalent would fail, the shell must report an error and retain the previous working directory. This applies to the `cd` builtin and any internal directory-change operations unless their specific descriptions state otherwise. Reference: XCU 1.1.1.9, XBD 3.94 (Current Working Directory).
