## Extracted Text

> In either case, execution of the utility in the specified environment shall be performed as follows:: If the command name contains at least one <slash>: If the named utility exists, the shell shall execute the utility with actions equivalent to calling the execl() function defined in the System Interfaces volume of POSIX.1-2024 with the path and arg0 arguments set to the command name, and the remaining execl() arguments set to the command arguments (if any) and the null terminator. If the execl() function fails due to an error equivalent to the [ENOEXEC] error, the shell shall execute a command equivalent to having a shell invoked with the command name as its first operand, with any remaining arguments passed to the new shell. The shell may apply a heuristic check to determine if the file to be executed could be a script and may bypass this command execution if it determines that the file cannot be a script. In this case, it shall write an error message, and the command shall fail with an exit status of 126. Note: A common heuristic for rejecting files that cannot be a script is locating a NUL byte prior to a <newline> byte within a fixed-length prefix of the file. Since sh is required to accept input files with unlimited line lengths, the heuristic check cannot be based on line length. It is unspecified whether environment variables that were passed to the shell when it was invoked, but were not used to initialize shell variables (see 2.5.3 Shell Variables) because they had invalid names, are included in the environment passed to execl() and (if execl() fails as described above) to the new shell. If the named utility does not exist, the command shall fail with an exit status of 127 and the shell shall write an error message.

## Context

**Behavior:** Non-built-in execution for commands WITH at least one slash (direct path commands):

1. **If utility exists:** Call `execl()` with `path` AND `arg0` both set to the command name, remaining args = command arguments.
2. **ENOEXEC fallback:** If `execl()` fails with ENOEXEC, execute a command equivalent to invoking `sh` with the command name as its first operand, passing remaining arguments. (Note: unlike the no-slash case, `$0` in the new shell is always the command name.)
3. **Script heuristic (MAY):** Same as the no-slash case — may reject non-script files with exit status 126.
4. **Invalid env vars (UNSPECIFIED):** Same as the no-slash case.
5. **If utility does not exist:** Fail with exit status 127 and write error message.

**Applies to:** Commands like `./script.sh`, `/usr/bin/foo`, `path/to/cmd`.

**Implementation detail:** Fork, then call exec with the literal command name as the path. On ENOEXEC, fall back to shell execution. On ENOENT, return 127.

**Spec reference:** Section 2.9.1.6, lines 1982-2005 in `utilities/V3_chap02.html`. Cross-refs: 2.5.3 Shell Variables.
