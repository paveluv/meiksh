## Extracted Text

> The following exit values shall be returned:: One of the name operands specified did not have an alias definition, or an error occurred.

## Context

An exit status >0 from `alias` specifically indicates that one of the `alias-name` operands specified did not have an alias definition (when querying), or some other error occurred. For example, `alias foo` when `foo` is not defined must return >0. If multiple operands are given and some succeed and some fail, the exit status **shall** still be >0 to indicate that at least one error occurred. Reference: alias utility EXIT STATUS.
