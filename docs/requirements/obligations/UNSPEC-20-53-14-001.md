## Extracted Text

> An option, specified or unspecified by optstring, was found.

## Context

**Behavior (unspecified aspect):** The "unspecified by optstring" part indicates that when an option character not in `optstring` is encountered, `getopts` still returns exit status 0 (an option was "found"). The behavior for the unknown option is defined elsewhere (set `name` to `?`, etc.), but the exit status 0 applies regardless of whether the option was recognized.

**Implementation guidance:** The implementation must return 0 for both recognized and unrecognized option characters. The distinction between the two is communicated through the value of `name` and `OPTARG`, not through the exit status.

**Spec reference:** POSIX.1-2024, `getopts` EXIT STATUS, line 205 of `utilities/getopts.html`.
