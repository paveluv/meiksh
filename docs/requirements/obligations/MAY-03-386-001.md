## Extracted Text

> A named collection of messages objects (one messages object per supported language) for internationalization and localization purposes. A text domain is often named after the application or library that provides the collection, but may have a more general name if it is intended to be shared by multiple applications or libraries.

## Context

**Definition only — no shell relevance.** Defines "text domain" as a named collection of messages objects for i18n/l10n. Often named after the application but may be shared.

**Shell impact:** None. If `meiksh` uses gettext for its own message localization, it would set a text domain, but this is an internal implementation detail. No action required for shell command language compliance.
