## Extracted Text

> When the -l option is specified, the symbolic name of each signal shall be written in the following format:

## Context

**Behavior:** When `kill -l` is specified without an `exit_status` operand (list-all mode), each supported signal name must be written to stdout in this format:

```
"%s%c", <signal_name>, <separator>
```

Where `<signal_name>` is uppercase without the `SIG` prefix, and `<separator>` is either a space or newline (see SHALL-20-64-10-003 for separator rules).

**Output:** One entry per signal, with signal names like `HUP`, `INT`, `QUIT`, `KILL`, `TERM`, etc.
