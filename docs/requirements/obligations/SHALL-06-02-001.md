## Extracted Text

> The POSIX locale shall contain 256 single-byte characters including the characters in Portable Character Set and Non-Portable Control Characters, which have the properties listed in 7.3.1 LC_CTYPE. It is unspecified whether characters not listed in those two tables are classified as punct or cntrl, or neither. Other locales shall contain the characters in Portable Character Set and may contain any or all of the control characters identified in Non-Portable Control Characters; the presence, meaning, and representation of any additional characters are locale-specific.

## Context

The POSIX locale defines exactly 256 single-byte characters. For the shell:

1. **POSIX locale (`LC_ALL=C` or `LANG=C`)**: All 256 byte values are valid single-byte characters. Pattern matching, character classes, and string operations work byte-by-byte. This is the simplest case for shell processing.
2. **Other locales (e.g., UTF-8)**: Characters may be multibyte. The shell must handle multibyte characters correctly for pattern matching, `${#var}` (string length in characters, not bytes), character class matching `[[:alpha:]]`, etc.
3. **Portable character set**: Guaranteed to be present in all locales. The shell's syntax characters are all in this set.

The shell's character handling must respect the current locale's `LC_CTYPE` category for character classification and multibyte processing.

See: `basedefs/V1_chap06.html` §6.2 Character Encoding; `basedefs/V1_chap07.html` §7.3.1 LC_CTYPE.
