## Extracted Text

> The following commands shall be recognized in command mode:: Convert, if the current character is a lowercase letter, to the equivalent uppercase letter and vice versa, as prescribed by the current locale. The current cursor position then shall be advanced by one character. If the cursor was positioned on the last character of the line, the case conversion shall occur, but the cursor shall not advance. If the '~' command is preceded by a count, that number of characters shall be converted, and the cursor shall be advanced to the character position after the last character converted. If the count is larger than the number of characters after the cursor, this shall not be considered an error; the cursor shall advance to the last character on the line.

## Context

**Behavior:** In vi command mode, `~` shall toggle the case of the current character (lowercase to uppercase and vice versa, per the current locale) and advance the cursor by one position.

**Edge cases:**
- If cursor is on the last character: case conversion occurs, but cursor does NOT advance.
- With a count: convert that many characters and advance cursor to the position after the last converted character.
- If count exceeds remaining characters: NOT an error; cursor advances to the last character on the line (all remaining characters are converted).

**Implementation:** For each character in range, check `islower()`/`isupper()` per locale, apply `toupper()`/`tolower()`, advance cursor. Clamp at end of line.
