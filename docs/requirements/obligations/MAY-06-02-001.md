## Context

In non-POSIX locales, characters may use state-dependent encodings where shift codes change the interpretation of subsequent bytes. For the shell, this means:

1. **Single-shift encodings**: Each non-initial-state character is preceded by a shift code. The shell must support these if the OS provides them.
2. **Locking-shift encodings**: A shift code affects all subsequent characters until another shift code. Support is implementation-defined.

In practice, modern systems use stateless encodings (UTF-8, Latin-1) rather than state-dependent ones (like ISO-2022-JP). The shell should primarily focus on UTF-8 multibyte support. If a state-dependent encoding is active, the shell must handle it correctly for character counting, pattern matching, and string operations.
