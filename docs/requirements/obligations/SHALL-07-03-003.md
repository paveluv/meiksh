## Extracted Text

> The locale definition file shall contain one or more locale category source definitions, and shall not contain more than one definition for the same locale category. If the file contains source definitions for more than one category, implementation-defined categories, if present, shall appear after the categories defined by 7.1 General. A category source definition contains either the definition of a category or a copy directive. For a description of the copy directive, see localedef. In the event that some of the information for a locale category, as specified in this volume of POSIX.1-2024, is missing from the locale source definition, the behavior of that category, if it is referenced, is unspecified.

## Context

**Behavior:** Locale definition file structure requirements:
1. Must contain one or more locale category definitions (e.g., `LC_CTYPE`, `LC_COLLATE`, etc.).
2. Must not have duplicate definitions for the same category.
3. Implementation-defined categories must appear after standard categories.
4. Missing category information produces unspecified behavior for that category.

**Shell relevance:** The shell does not parse locale definition files. This constrains `localedef` input. The shell relies on the resulting compiled locale being correct and complete. If a locale has missing category data, the shell's behavior for that category is unspecified (e.g., missing `LC_CTYPE` could break character classification).
