## Extracted Text

> If a file that does not exist is to be written, it shall be created as described below, unless the utility description states otherwise.

## Context

If a file that does not exist is to be written, it shall be created as described in Section 1.1.1.4, unless the utility description states otherwise. For `meiksh`, this applies when the shell performs output redirection (`>`, `>>`) to a file that does not exist: the shell must create the file with the default attributes specified in the following obligations (user ID, group ID, permissions, timestamps, etc.). The `noclobber` option (`set -C`) modifies this behavior for `>` (but not `>>`). See POSIX Utilities Chapter 1, Section 1.1.1.4 (utilities/V3_chap01.html line 84).
