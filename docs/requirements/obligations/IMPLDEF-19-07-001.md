> where location is non-empty and indicates a location where an integer value can be stored, such as the name of a shell variable. If this format is supported its behavior is implementation-defined.

## Context

If the shell supports the `{location}redir-op word` redirection format (see MAY-19-07-001), the behavior of that format is implementation-defined. See 2.7 Redirection in utilities/V3_chap02.html.

**Decision for meiksh:** If the `{var}` redirection syntax is supported, the implementation must document:
- What constitutes a valid `location` (typically a shell variable name)
- How the file descriptor number is chosen (typically the lowest available fd >= 10)
- Whether the fd is automatically closed when the variable goes out of scope
- How this interacts with `{var}<&-` or `{var}>&-` for closing

**If not supporting this format:** No implementation-defined behavior needs to be documented; the braces would be treated as regular characters/syntax.
