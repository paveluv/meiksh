## Extracted Text

> The following exit values shall be returned:

## Context

**Behavior:** Framing statement for getopts exit status values. The `getopts` utility uses three distinct exit status ranges:
- **0:** An option was found (whether valid or invalid — error conditions like unknown option or missing argument still return 0).
- **1:** End of options was encountered.
- **>1:** A getopts processing error occurred (e.g., cannot set a readonly variable).

This three-way distinction is important: scripts typically use `while getopts ... ; do ... done` which loops while exit status is 0 and stops when it becomes 1.

**Spec reference:** POSIX.1-2024, `getopts` EXIT STATUS, line 201 of `utilities/getopts.html`.
