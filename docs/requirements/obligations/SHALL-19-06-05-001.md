## Extracted Text

> After parameter expansion ( 2.6.2 Parameter Expansion), command substitution ( 2.6.3 Command Substitution), and arithmetic expansion ( 2.6.4 Arithmetic Expansion), if the shell variable IFS (see 2.5.3 Shell Variables) is set and its value is not empty, or if IFS is unset, the shell shall scan each field containing results of expansions and substitutions that did not occur in double-quotes for field splitting; zero, one or multiple fields can result.

## Context

**Behavior:** Field splitting is performed as a post-expansion step, after parameter expansion, command substitution, and arithmetic expansion. The shell must scan fields that contain **unquoted** expansion results and split them into zero, one, or more fields based on the IFS variable.

**Conditions for field splitting to occur:**
- IFS is **set and non-empty**, OR IFS is **unset** (in which case it defaults to space+tab+newline per SHALL-19-06-05-006)
- The field contains results from expansions that were **not** inside double-quotes

**Conditions where field splitting does NOT occur:**
- IFS is set to the empty string (`IFS=''`) — see SHALL-19-06-05-003
- The expansion occurred inside double-quotes (`"$var"`, `"$(cmd)"`, `"$((expr))"`)
- The field contains no expansion results (purely literal text)

**Spec reference:** See 2.6.5 Field Splitting in utilities/V3_chap02.html, lines 1203–1208. See 2.5.3 Shell Variables (#tag_19_05_03) for IFS definition.
