## Extracted Text

> The rules described so far in 2.14.1 Patterns Matching a Single Character and 2.14.2 Patterns Matching Multiple Characters are qualified by the following rules that apply when pattern matching notation is used for filename expansion:: If a specified pattern does not contain any '*', '?' or '[' characters that will be treated as special, the pattern string shall be left unchanged.

## Context

**Pattern without glob characters: left unchanged.**

If a pattern does not contain any `*`, `?`, or `[` characters that will be treated as special (i.e., they are all quoted, escaped, or the pattern has none), the pattern string **shall** be left unchanged. No filesystem matching is attempted.

This means a word like `hello` or `he\*llo` (where `*` is escaped) is passed through as-is, not matched against the filesystem. This is the base case for pathname expansion — most words are not glob patterns and just pass through untouched.

See 2.14.3 in utilities/V3_chap02.html.
