## Extracted Text

> If a \xXX or \ddd escape sequence yields a byte whose value is 0, it is unspecified whether that null byte is included in the result or if that byte and any following regular characters and escape sequences up to the terminating unescaped single-quote are evaluated and discarded.

## Context

**Behavior:** If a `\xXX` or `\ddd` escape sequence inside `$'...'` yields a byte with value 0 (null byte), it is unspecified whether:
- (a) The null byte is included in the result string, OR
- (b) The null byte AND all following regular characters and escape sequences up to the terminating `'` are discarded.

**Applies to:** All shell modes during `$'...'` escape processing.

**Implementation choice:** Since most shell implementations represent strings as C strings (null-terminated), option (b)—truncating at the null—is the common behavior. The implementation should choose one approach and document it. Option (a) would require non-C-string internal representation.

**Examples of affected input:** `$'\x00rest'`, `$'\0rest'`, `$'\000rest'`

**Spec reference:** Section 2.2.4, line 227 in `utilities/V3_chap02.html`.
