## Extracted Text

> An ERE matching a single character repeated by an '*', '?', or an interval expression shall not match a null expression unless this is the only match for the repetition or it is necessary to satisfy the exact or minimum number of occurrences for the interval expression.

## Context

When a single-character ERE is repeated using `*`, `?`, or an interval expression, the ERE engine must prefer matching actual characters over the empty string. A null (empty) match is only permitted when: (1) it is the only possible match for the repetition (no non-null match exists), or (2) matching the empty string is required to satisfy the minimum count of an interval expression (e.g., `a{0}` must match the empty string). This prevents pathological behavior where `a*` could endlessly match empty strings at every position. The engine must advance past null matches to avoid infinite loops. This rule does not apply to `+` since `+` requires at least one occurrence.
