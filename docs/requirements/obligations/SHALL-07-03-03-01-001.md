## Extracted Text

> The monetary formatting definitions for the POSIX locale follow; the code listing depicting the localedef input, the table representing the same information with the addition of localeconv() and nl_langinfo() formats. All values shall be unavailable in the POSIX locale.

## Context

In the POSIX locale, all LC_MONETARY values are unavailable (integer keywords return -1, string keywords return empty string). This means: no currency symbol, no decimal point, no thousands separator, no sign strings, etc. The shell is affected because when `LC_MONETARY=POSIX` or `LC_MONETARY=C`, programs invoked by the shell cannot format monetary values using locale data — there are no monetary formatting rules defined. The shell must ensure this default state is correctly maintained for the POSIX locale.

See: `basedefs/V1_chap07.html` §7.3.3.1 LC_MONETARY Category in the POSIX Locale.
