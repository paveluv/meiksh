## Extracted Text

> The <signal.h> header shall also define the following symbolic constants:: [CX] Causes extra information to be passed to signal handlers at the time of receipt of a signal.

## Context

This is the [CX] extension defining the semantics of `SA_SIGINFO`: when set in `sa_flags`, the signal handler is invoked with three arguments (signal number, `siginfo_t*`, and `void*` context), providing extended information such as the sending process PID (`si_pid`), user ID (`si_uid`), signal code (`si_code`), and signal-specific data. The handler must be installed via the `sa_sigaction` member of `struct sigaction` rather than `sa_handler`. For `meiksh`, this enables richer handling of `SIGCHLD` (obtaining child exit status, PID, etc.) and `SIGINT`/`SIGTERM` (identifying the sender). See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 532–533).
