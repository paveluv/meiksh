## Extracted Text

> The exit status of a command shall be determined as follows:: If the command is not found, the exit status shall be 127.

## Context

**Behavior:** When a command name is produced after word expansion but no matching utility, builtin, or function can be found via the command search algorithm (see 2.9.1.4), the shell MUST set the exit status to exactly 127. The shell must also write a diagnostic error message to standard error.

**Applies to:** Both simple commands with PATH lookup (no slash in name) and direct-path commands (with slash) where the named file does not exist. Applies in all shell modes (interactive and non-interactive).

**Error handling:** The shell must write a "command not found" style diagnostic to stderr. In interactive mode, the shell continues; in non-interactive mode, behaviour depends on 2.8.1 Consequences of Shell Errors.

**Implementation detail:** After the command search in 2.9.1.4 step 1e fails, or after a slash-containing path does not exist, set `$?` to 127 before returning from command execution.

**Spec reference:** Section 2.8.2, line 1701 in `utilities/V3_chap02.html`.
