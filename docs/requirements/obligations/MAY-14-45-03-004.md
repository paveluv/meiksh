## Extracted Text

> [CX] Inclusion of the <signal.h> header may make visible all symbols from the <time.h> header.

## Context

Inclusion of `<signal.h>` may make visible all symbols from `<time.h>` ([CX] extension). This is permitted because `siginfo_t` and `sigevent` structures may reference types from `<time.h>` (e.g., `timer_t`, `struct timespec`). For `meiksh`, this means that after `#include <signal.h>`, types and functions from `<time.h>` might be available without an explicit include — but portable code should not rely on this and should include `<time.h>` explicitly if needed. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 1168–1170).
