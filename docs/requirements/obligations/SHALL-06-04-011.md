## Extracted Text

> In the first format, the line in the character set mapping definition shall define a single symbolic name and a corresponding encoding. A symbolic name is one or more characters from the set shown with visible glyphs in Portable Character Set, enclosed between angle brackets. A character following an escape character is interpreted as itself; for example, the sequence "<\\\>>" represents the symbolic name "\>" enclosed between angle brackets.

## Context

**Behavior:** In charmap files, the first line format defines a single symbolic name (e.g., `<A>`) mapped to an encoding value. Symbolic names consist of Portable Character Set visible-glyph characters enclosed in angle brackets. The escape character can be used to include literal special characters within symbolic names.

**Shell relevance:** The shell does not parse charmap files. This defines charmap syntax for `localedef`. No shell implementation action needed, but the shell relies on these mappings being correct for character classification.
