## Extracted Text

> The getopts utility shall retrieve options and option-arguments from a list of parameters. It shall support the Utility Syntax Guidelines 3 to 10, inclusive, described in XBD 12.2 Utility Syntax Guidelines.

## Context

The `getopts` builtin must parse command-line options from a parameter list according to POSIX Utility Syntax Guidelines 3-10 (XBD 12.2). These guidelines include: options are single alphanumeric characters preceded by `-` (G3), options without arguments can be grouped (G5), option-arguments follow the option character (G6-G7), `--` terminates option processing (G10), and others. `getopts` is an intrinsic utility that must be built into the shell. It processes one option per invocation and is designed to be called in a loop. The parameters parsed default to the positional parameters (`$@`) but can be overridden by explicit `param` operands. Reference: POSIX.1-2024 §getopts, DESCRIPTION (tag_20_53_03); XBD 12.2.
