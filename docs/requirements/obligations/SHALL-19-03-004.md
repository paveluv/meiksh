## Extracted Text

> If the end of input is recognized, the current token (if any) shall be delimited.

## Context

**Behavior:** Token recognition Rule 1: If end-of-input (EOF) is recognized, the current token being built (if any) is delimited (completed). This is the highest-priority rule.

**Applies to:** All shell modes. Triggered when the input stream ends (end of file, end of `-c` string, etc.).

**Implementation detail:** When the lexer reaches EOF, it must finalize any in-progress token and pass it to the parser. If no token is in progress, no token is emitted. The parser should then check for incomplete constructs (unclosed quotes, missing `fi`/`done`/`esac`, etc.) and report syntax errors as appropriate.

**Edge cases:**
- EOF while inside a quoted string → syntax error (unclosed quote)
- EOF while building an operator token → the partial operator is delimited and classified
- EOF with no current token → nothing emitted

**Spec reference:** Section 2.3, rule 1, lines 256-257 in `utilities/V3_chap02.html`.
