## Extracted Text

> The relationship between the actual date and time in Coordinated Universal Time, as determined by the International Earth Rotation Service, and the system's current value for seconds since the Epoch is unspecified.

## Context

The shell uses "seconds since the Epoch" in the `SECONDS` variable (which tracks elapsed time since shell invocation) and indirectly through utilities like `date`. The relationship between the system clock and actual UTC is unspecified — the shell cannot assume the system clock is perfectly synchronized. This means the shell must use the system's time as-is without attempting to validate or correct it. The `SECONDS` variable typically uses monotonic elapsed time, not wall-clock time, so clock adjustments may or may not affect it (implementation-defined).
