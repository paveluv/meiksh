## Extracted Text

> The environment variables LANG , LC_ALL , LC_COLLATE , LC_CTYPE , LC_MESSAGES , LC_MONETARY , LC_NUMERIC , LC_TIME , and NLSPATH provide for the support of internationalized applications. The standard utilities shall make use of these environment variables as described in this section and the individual ENVIRONMENT VARIABLES sections for the utilities. See 7.1 General for the consequences of setting these variables to locales with different character sets.

## Context

**Behavior:** The standard utilities shall make use of the internationalization environment variables `LANG`, `LC_ALL`, `LC_COLLATE`, `LC_CTYPE`, `LC_MESSAGES`, `LC_MONETARY`, `LC_NUMERIC`, `LC_TIME`, and `NLSPATH` as described in section 8.2 and in each utility's ENVIRONMENT VARIABLES section.

**Shell relevance:** The shell is both a standard utility and an executor of other standard utilities. The shell must: (1) honor these variables for its own locale-sensitive behavior (pattern matching uses `LC_COLLATE` and `LC_CTYPE`; messages use `LC_MESSAGES`; character classification uses `LC_CTYPE`), and (2) propagate them to child processes. See section 7.1 for consequences of setting these to locales with different character sets.
