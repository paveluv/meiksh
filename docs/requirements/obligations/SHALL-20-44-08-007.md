## Extracted Text

> The following environment variables shall affect the execution of fc:: Determine a decimal number representing the limit to the number of previous commands that are accessible. If this variable is unset, an unspecified default greater than or equal to 128 shall be used. The maximum number of commands in the history list is unspecified, but shall be at least 128. An implementation may choose to access this variable only when initializing the history file, as described under HISTFILE . Therefore, it is unspecified whether changes made to HISTSIZE after the history file has been initialized are effective.

## Context

This is the full `HISTSIZE` specification for `fc`. Key implementation requirements: (1) `HISTSIZE` is a decimal number limiting the number of accessible previous commands. (2) If unset, the default must be >= 128 (the specific default is unspecified). (3) The maximum history size must be at least 128 regardless. (4) The implementation may only read `HISTSIZE` at history initialization time (same timing as `HISTFILE` initialization). (5) Changes to `HISTSIZE` after initialization may or may not take effect (implementation choice). For meiksh, a reasonable default like 500 or 1000 when `HISTSIZE` is unset would be appropriate. Reference: POSIX.1-2024 §fc, ENVIRONMENT VARIABLES (tag_20_44_08).
