## Extracted Text

> A ';' separator or a ';' or <newline> terminator shall cause the preceding AND-OR list to be executed sequentially; an '&' separator or terminator shall cause asynchronous execution of the preceding AND-OR list.

## Context

**Behavior:** A `;` separator or a `;` or newline terminator causes the preceding AND-OR list to be executed sequentially (synchronously — wait for completion before proceeding). An `&` separator or terminator causes asynchronous execution of the preceding AND-OR list (run in background).

**Applies to:** All command lists. Examples: `cmd1; cmd2` executes sequentially; `cmd1 & cmd2` runs cmd1 in background, then cmd2 in foreground; `cmd1 &` runs cmd1 in background.

**Implementation detail:** The parser identifies `;`, newline, and `&` as list terminators/separators. The executor must fork for `&` (asynchronous) and wait for `;`/newline (synchronous). A trailing `&` with no following command still runs the preceding AND-OR list asynchronously.

**Spec reference:** Section 2.9.3, lines 2108-2109 in `utilities/V3_chap02.html`.
