## Extracted Text

> The eval utility shall construct a command string by concatenating arguments together, separating each with a <space> character. The constructed command string shall be tokenized (see 2.3 Token Recognition), parsed (see 2.10 Shell Grammar), and executed by the shell in the current environment. It is unspecified whether the commands are parsed and executed as a program (as for a shell script) or are parsed as a single compound_list that is executed after the entire constructed command string has been parsed.

## Context

**Behavior:** The `eval` special built-in must:
1. Concatenate all arguments together, separating each with a single space character, to form a command string.
2. Tokenize the constructed string (per 2.3 Token Recognition).
3. Parse the tokens (per 2.10 Shell Grammar).
4. Execute the resulting commands in the **current** shell environment.

It is **unspecified** whether the constructed command string is parsed/executed incrementally or as a single compound_list after complete parsing.

**Example:** `eval "echo hello" "world"` constructs the string `echo hello world` and executes it.

**Exit status:** The exit status of the last command executed by `eval`, or zero if no commands are executed.

**Applies to:** All shell modes. This is a special built-in.
