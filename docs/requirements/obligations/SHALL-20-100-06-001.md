## Extracted Text

> If the -d delim option is not specified, or if it is specified and delim is not the null string, the standard input shall contain zero or more bytes (which need not form valid characters) and shall not contain any null bytes.

## Context

**Behavior required:** When using the default newline delimiter OR a non-null `-d delim`:
- The standard input may contain zero or more bytes.
- The bytes need not form valid characters (arbitrary byte sequences are allowed).
- The input MUST NOT contain null bytes (`\0`). If null bytes are present, behavior is undefined.

**Condition:** When `-d` is not specified, or when `-d delim` is specified with a non-empty `delim`.

**Implementation detail:** The `read` builtin reads arbitrary bytes (not just valid UTF-8 or locale characters) but assumes no null bytes in the input stream. The implementation may use C string functions that rely on null termination. If null bytes appear in the input under these conditions, the implementation need not handle them correctly.
