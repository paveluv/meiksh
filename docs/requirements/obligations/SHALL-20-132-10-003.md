## Context

**Behavior:** In the symbolic output format (`-S`), each of the three permission fields (owner, group, other) shall be a combination of the letters r, w, x. The presence of a letter means the corresponding permission bit is **clear** (not set) in the mask — i.e., the permission is **allowed** for newly created files.

**Inverted semantics:** This is the opposite of the mask's octal representation. The symbolic output shows what is *permitted*, not what is *masked*. For example:
- Mask 022 (blocks group-write, other-write) → `u=rwx,g=rx,o=rx` (r, w, x present for owner; r, x for group/other)
- Mask 077 (blocks all group/other) → `u=rwx,g=,o=` (empty strings for group/other)

**Empty fields:** If all bits are set in the mask for a category, that field shall be empty (e.g., `o=`).
