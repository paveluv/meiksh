## Extracted Text

> Variable assignments shall be performed as follows:: If the command name is a special built-in utility, variable assignments shall affect the current execution environment before the utility is executed and remain in effect when the command completes; if an assigned variable is further modified by the utility, the modifications made by the utility shall persist. Unless the set -a option is on (see set), it is unspecified: Whether or not the variables gain the export attribute during the execution of the special built-in utility Whether or not export attributes gained as a result of the variable assignments persist after the completion of the special built-in utility

## Context

**Behavior:** When the command name is a special built-in utility (break, colon, continue, dot, eval, exec, exit, export, readonly, return, set, shift, times, trap, unset), variable assignments MUST: (1) affect the current execution environment BEFORE the utility executes, (2) remain in effect after the command completes, and (3) if the utility further modifies the assigned variable, those modifications persist.

Two things are UNSPECIFIED unless `set -a` is on: (1) whether the variables gain the export attribute during execution, and (2) whether export attributes gained persist after the command completes.

**Applies to:** `VAR=value special_builtin` commands. Example: `VAR=hello eval 'echo $VAR'` — VAR is set before eval runs, and persists after.

**Implementation detail:** Set the variables in the current shell variable table before invoking the special builtin. Do NOT create a temporary scope. If `set -a` is active, the variables should gain the export attribute.

**Spec reference:** Section 2.9.1.2, lines 1776-1783 in `utilities/V3_chap02.html`. Cross-ref: `set` builtin, 2.26 Declaration Utility.
