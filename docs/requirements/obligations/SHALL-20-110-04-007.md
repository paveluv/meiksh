## Extracted Text

> Specify that the shell is interactive; see below. An implementation may treat specifying the -i option as an error if the real user ID of the calling process does not equal the effective user ID or if the real group ID does not equal the effective group ID.

## Context

The `-i` option specifies that the shell is **interactive**. When `-i` is given, the shell enters interactive mode regardless of whether stdin/stderr are terminals. Interactive mode implies:
- The shell reads the `ENV` file at startup (if [UP] is supported)
- SIGINT is caught (for line editing), SIGQUIT and SIGTERM are ignored
- Job control may be enabled (if `-m` is also set or is the default for interactive)
- The shell writes prompts (PS1, PS2) to stderr
- Syntax errors and certain other errors do not cause exit

An implementation **may** (optional, see MAY-20-110-04-001) treat `-i` as an error if the real user ID does not equal the effective user ID, or if the real group ID does not equal the effective group ID. This is a security measure to prevent setuid/setgid shells from being forced into interactive mode.

The shell is also considered interactive without `-i` under the separate rule later in the OPTIONS section when stdin and stderr are terminals.
