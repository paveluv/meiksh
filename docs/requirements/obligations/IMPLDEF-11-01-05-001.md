## Extracted Text

> A terminal device associated with a terminal device file may operate in full-duplex mode, so that data may arrive even while output is occurring. Each terminal device file has an input queue associated with it, into which incoming data is stored by the system before being read by a process. The system may impose a limit, {MAX_INPUT}, on the number of bytes that may be stored in the input queue. The behavior of the system when this limit is exceeded is implementation-defined.

## Context

**Behavior (implementation-defined):** Terminal devices may operate in full-duplex mode (simultaneous input and output). Each terminal has an input queue. The system may impose a MAX_INPUT limit on the input queue size. The behavior when MAX_INPUT is exceeded is implementation-defined (data may be lost, input may be suspended, etc.).

**Conditions:** Applies to all terminal device I/O. The shell reads input from the terminal's input queue.

**Shell relevance:** The shell must be aware that the terminal input queue may have a limited size. For long command lines or rapid input, data could be lost if MAX_INPUT is exceeded. This is primarily a kernel concern, but the shell should handle partial reads gracefully.
