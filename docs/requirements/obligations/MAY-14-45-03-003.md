## Extracted Text

> For some implementations, the value of si_addr may be inaccurate.

## Context

For some implementations, the value of `si_addr` in `siginfo_t` may be inaccurate. This applies to signals like `SIGILL`, `SIGFPE`, `SIGSEGV`, and `SIGBUS` where `si_addr` is supposed to indicate the faulting address. The standard acknowledges that on some architectures, the exact faulting address may not be precisely determinable. For `meiksh`, this is not directly relevant since the shell does not typically handle faulting-address signals programmatically beyond default termination behavior. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html line 1121).
