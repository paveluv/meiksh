## Extracted Text

> This variable shall represent an absolute pathname of the current working directory. It shall not contain any components that are dot or dot-dot. The value is set by the cd utility, and by the sh utility during initialization.

## Context

**Behavior:** The `PWD` environment variable shall represent an absolute pathname of the current working directory. It shall not contain any `.` or `..` components. The value is set by the `cd` built-in utility, and by the `sh` utility during initialization.

**Shell relevance:** The shell must: (1) initialize `PWD` during startup to an absolute pathname of the current working directory (without `.` or `..` components), (2) update `PWD` every time the `cd` built-in successfully changes the working directory, (3) ensure `PWD` never contains `.` or `..` path components (canonical form), (4) export `PWD` so child processes can read it. The `pwd` built-in's default (`-L`) mode prints the value of `PWD`.

**Spec reference:** Section 8.3 "Other Environment Variables", lines 717-719 in `basedefs/V1_chap08.html`; see also `cd` and `sh` utility descriptions.
