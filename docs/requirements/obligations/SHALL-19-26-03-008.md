## Extracted Text

> (Uppercase C.) Prevent existing regular files from being overwritten by the shell's '>' redirection operator (see 2.7.2 Redirecting Output); the ">|" redirection operator shall override this noclobber option for an individual file.

## Context

**Behavior:** The `-C` (`noclobber`) option: prevents the `>` redirection operator from overwriting existing regular files. When enabled:
1. `> file` must fail if `file` already exists and is a regular file.
2. The `>|` redirection operator overrides noclobber for an individual file, allowing the overwrite.

**Implementation:** When processing `>` redirection with `-C` set, use `O_EXCL` flag (or equivalent) to prevent overwriting existing files. The `>|` operator bypasses this check.

**Applies to:** All shell modes.

**Spec reference:** `set` DESCRIPTION in `utilities/V3_chap02.html`, lines 4797-4799. Cross-ref: 2.7.2 Redirecting Output.
