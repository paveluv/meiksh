## Context

**Behavior:** The special control character values **shall** be stored in the `c_cc` array of the termios structure. Each entry is indexed by a symbolic constant (VINTR, VQUIT, VERASE, VKILL, VEOF, VEOL, VSUSP, VSTART, VSTOP, VMIN, VTIME). The array provides different semantics depending on whether canonical or non-canonical mode is active:
- Canonical mode: VEOF, VEOL, VERASE, VKILL are meaningful.
- Non-canonical mode: VMIN, VTIME are meaningful.
- Both modes: VINTR, VQUIT, VSUSP, VSTART, VSTOP are meaningful (when ISIG/IXON is set).

**Conditions:** The `c_cc` array is part of `struct termios` and is accessed via `tcgetattr()`/`tcsetattr()`.

**Shell relevance:** The shell must use the `c_cc` array to configure special characters. Key uses:
- Set VMIN=1, VTIME=0 for non-canonical line editing
- Read/modify VINTR, VQUIT, VSUSP values for signal characters
- Set `_POSIX_VDISABLE` to disable specific special characters
- The `stty` builtin displays and modifies these values

**Spec reference:** POSIX.1-2024, §11.2.6 Special Control Characters (tag_11_02_06), `basedefs/V1_chap11.html` line 1205.
