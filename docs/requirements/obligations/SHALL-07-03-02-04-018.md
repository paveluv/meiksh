## Extracted Text

> The UNDEFINED means that all characters not specified in this definition (explicitly or via the ellipsis) shall be ignored when comparing primary weights, and have individual secondary weights based on their ordinal encoded values.

## Context

In an LC_COLLATE example, the UNDEFINED symbol causes all characters not explicitly specified in the collation definition to receive IGNORE as their primary weight (they are skipped during primary-level comparison) and unique secondary weights based on their code point values. This establishes a two-level collation where unspecified characters are invisible at the primary level but ordered by code point at the secondary level. The shell is affected because this determines the collation behavior for characters not explicitly addressed in the locale's collation definition, impacting pattern matching bracket expression ranges and equivalence classes.

See: `basedefs/V1_chap07.html` §7.3.2.4 Collation Order.
