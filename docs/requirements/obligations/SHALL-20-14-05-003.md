## Extracted Text

> The following operands shall be supported:: An absolute or relative pathname of the directory that shall become the new working directory. The interpretation of a relative pathname by cd depends on the -L option and the CDPATH and PWD environment variables. If directory is an empty string, cd shall write a diagnostic message to standard error and exit with non-zero status. If directory consists of a single '-' (<hyphen-minus>) character, the cd utility shall behave as if directory contained the value of the OLDPWD environment variable, except that after it sets the value of PWD it shall write the new value to standard output. The behavior is unspecified if OLDPWD does not start with a <slash> character.

## Context

**Behavior:** The `directory` operand for `cd` has several specific behaviors the shell must implement:

1. **Absolute pathname:** If `directory` starts with `/`, it is used directly (step 3 of the algorithm sets curpath to the operand).
2. **Relative pathname:** Interpretation depends on `-L` option, `CDPATH`, and `PWD`. Relative paths starting with `.` or `..` skip CDPATH search (step 4). Other relative paths search CDPATH first (step 5), falling back to the operand as-is (step 6).
3. **Empty string operand:** If `directory` is `""`, the shell must write a diagnostic to stderr and exit with non-zero status. The directory must NOT be changed.
4. **Hyphen operand (`-`):** The shell must behave as if the value of `$OLDPWD` was given as the directory operand, with the additional requirement that after setting PWD, the new value must be written to stdout.
5. **Unspecified:** If OLDPWD does not start with `/` when `-` is used, behavior is unspecified (implementation may choose any behavior).

**Error handling:** Empty string must produce a diagnostic on stderr with non-zero exit.
