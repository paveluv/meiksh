## Extracted Text

> An empty operand shall be interpreted as the collating element itself.

## Context

When a weight operand is empty (omitted) in the LC_COLLATE collation order section, it defaults to the collating element's own position in the collation sequence. This is the most common case — a character's weight at a given level equals its own collation position. The shell is affected because this default weight assignment is used pervasively in locale definitions and determines the ordering that governs bracket expression ranges and equivalence classes in pattern matching.

See: `basedefs/V1_chap07.html` §7.3.2.4 Collation Order.
