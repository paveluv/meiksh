## Extracted Text

> This section uses the term "invalid" for certain constructs or conditions. Invalid REs shall cause the utility or function using the RE to generate an error condition. When invalid is not used, violations of the specified syntax or semantics for REs produce undefined results: this may entail an error, enabling an extended syntax for that RE, or using the construct in error as literal characters to be matched. For example, the BRE construct "\{1,2,3\}" does not comply with the grammar. A conforming application cannot rely on it producing an error nor matching the literal characters "\{1,2,3\}".

## Context

**Behavior:** The term "invalid" in section 9 means the RE construct shall cause the utility or function to generate an error condition. When "invalid" is NOT used, violations produce undefined results — which may mean an error, an extended syntax interpretation, or treating the construct as literal characters.

**Shell relevance:** When the shell encounters an invalid bracket expression in a glob pattern or `case` pattern (e.g., an unclosed `[.` collating symbol), it must treat this as an error. However, constructs that are merely "undefined" (not "invalid") may be handled in any way. The shell should prefer generating an error for clearly malformed bracket expressions while being permissive for ambiguous cases.
