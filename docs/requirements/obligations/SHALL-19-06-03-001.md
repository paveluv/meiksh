> Command substitution allows the output of one or more commands to be substituted in place of the commands themselves. Command substitution shall occur when command(s) are enclosed as follows:

## Context

**Behavior:** The shell must recognize two syntactic forms for command substitution:
1. `$(commands)` — the modern, nestable form
2. `` `commands` `` — the backquoted (legacy) form

When either form is encountered during word expansion, the shell must execute the enclosed commands and substitute the resulting standard output in place of the command substitution expression.

**Conditions:** Command substitution is recognized during word expansion in all contexts where expansions are performed (unquoted, double-quoted, here-documents). It is NOT recognized inside single-quotes.

**Inputs/Outputs:**
- Input: a word or portion of a word containing `$(...)` or `` `...` ``
- Output: the enclosed commands are executed and their stdout replaces the command substitution expression
