## Extracted Text

> When used in a context where assignment is defined to occur and at no other time, the value (representing a word or field) shall be assigned as the value of the variable denoted by varname. Assignment context occurs in the cmd_prefix portion of a shell simple command, as well as in arguments of a recognized declaration utility.

## Context

Variable assignment in the shell occurs only in specific contexts:

1. **Simple command prefix**: Words of the form `name=value` before the command name in a simple command (e.g., `VAR=hello cmd`). These set the variable for the duration of that command (or permanently if no command name follows).
2. **Declaration utility arguments**: Arguments to declaration utilities like `export`, `readonly`, `local` (non-POSIX), and `typeset` (non-POSIX) that match the `name=value` pattern.

Outside these contexts, a word like `name=value` is NOT treated as an assignment — it's just a regular word/argument. The shell's parser must distinguish assignment context from non-assignment context. For example, in `echo FOO=bar`, `FOO=bar` is a regular argument to `echo`, not an assignment.

The variable name (`varname`) must be a valid shell Name (letter or underscore followed by letters, digits, or underscores). The value undergoes tilde expansion, parameter expansion, command substitution, and arithmetic expansion (but NOT field splitting or pathname expansion when in assignment context).

See: `basedefs/V1_chap04.html` §4.26 Variable Assignment; `utilities/V3_chap02.html` §2.9.1 Simple Commands.
