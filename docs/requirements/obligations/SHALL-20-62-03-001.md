## Extracted Text

> If the current shell execution environment (see 2.13 Shell Execution Environment) is not a subshell environment, the jobs utility shall display the status of background jobs that were created in the current shell execution environment; it may also do so if the current shell execution environment is a subshell environment.

## Context

**Behavior:** The `jobs` utility shall display the status of background jobs created in the current shell execution environment when that environment is NOT a subshell. In a subshell environment, displaying job status is optional (implementation may choose to do so).

**Condition:** The distinction is between the main shell execution environment and subshell environments (created by `(...)`, command substitution, etc.).

**What is displayed:** Status of background jobs — both running and completed — that were created via asynchronous commands (`cmd &`) in the current shell execution environment.

**Spec reference:** POSIX.1-2024, `jobs` DESCRIPTION, lines 38-39 of `utilities/jobs.html`; see also Section 2.13 Shell Execution Environment.
