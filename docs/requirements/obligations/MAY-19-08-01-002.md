## Extracted Text

> An expansion error is one that occurs when the shell expansions defined in 2.6 Word Expansions are carried out (for example, "${x!y}", because '!' is not a valid operator); an implementation may treat these as syntax errors if it is able to detect them during tokenization, rather than during expansion.

## Context

Defines what an "expansion error" is and allows early detection. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Definition:**
- An expansion error is one that occurs during the shell expansions defined in 2.6 Word Expansions (tag_19_06).
- Example: `${x!y}` is an expansion error because `!` is not a valid parameter expansion operator.

**Optional behavior (MAY):**
- An implementation MAY treat expansion errors as syntax errors if it is able to detect them during tokenization rather than during expansion.
- This means the shell could reject `${x!y}` at parse time (syntax error) rather than waiting until the expansion is evaluated at runtime.

**Decision for meiksh:** Detecting expansion errors early (during parsing) is generally better for user experience and allows failing fast. However, it changes the error category from "expansion error" to "syntax error", which has identical exit/no-exit consequences in the table, so the practical difference is minimal.
