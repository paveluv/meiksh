## Extracted Text

> The following options shall be supported:: (The letter ell.) List the commands rather than invoking an editor on them. The commands shall be written in the sequence indicated by the first and last operands, as affected by -r, with each command preceded by the command number.

## Context

This is the full description of `-l` for `fc`. In list mode: (1) Commands from the history are written to stdout rather than being placed in a temporary file for editing. (2) The listing order follows the `first` and `last` operands — if `first` is newer than `last`, the commands are listed in reverse. (3) The `-r` option further reverses the order. (4) Each listed command is preceded by its command number (history line number), unless `-n` is also specified. (5) The output format per command is `"%d\t%s\n", <line number>, <command>`. Multi-line commands use `"\t%s\n"` for continuation lines. Reference: POSIX.1-2024 §fc, OPTIONS (tag_20_44_04).
