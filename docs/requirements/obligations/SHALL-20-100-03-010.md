## Extracted Text

> The specified var operands shall be processed in the order they appear on the command line, and the output fields generated by the field splitting algorithm shall be used in the order they were generated, by repeating the following checks until neither is true:

## Context

**Behavior required:** After field splitting produces a list of output fields, the assignment to variables follows a left-to-right pairing:

- Variables are processed in command-line order (left to right).
- Output fields from field splitting are consumed in generation order (left to right).
- The assignment loop repeats the checks defined in SHALL-20-100-03-011 and SHALL-20-100-03-012 until neither check's condition is true.

**Condition:** After field splitting has been performed on the `read` input.

**Implementation detail:** This establishes the variable assignment algorithm as a loop with two checks (detailed in the next obligations):
1. If more than one var remains and fields remain → assign next field to next var.
2. If exactly one var remains and there is unsplit remainder → assign remainder to that var.
The loop runs until neither condition is met.
