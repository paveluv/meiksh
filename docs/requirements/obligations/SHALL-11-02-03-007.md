## Extracted Text

> Carriage-return delay type 1 shall be dependent on the current column position, type 2 shall be about 0.10 seconds, and type 3 shall be about 0.15 seconds. If OFILL is set, delay type 1 shall transmit two fill characters, and type 2 four fill characters.

## Context

**Behavior:** Carriage-return delay types (CRDLY mask in `c_oflag`):
- CR1 (type 1): delay is dependent on the current column position.
- CR2 (type 2): delay is about 0.10 seconds.
- CR3 (type 3): delay is about 0.15 seconds.
When OFILL is set: type 1 shall transmit two fill characters, type 2 shall transmit four fill characters.

**Relevance to meiksh:** The `stty` builtin must support setting CR0, CR1, CR2, CR3 via the CRDLY mask. The actual delay behavior is handled by the OS terminal driver.

**Spec reference:** POSIX.1-2024, §11.2.3 Output Modes (tag_11_02_03), `basedefs/V1_chap11.html` line 801.
