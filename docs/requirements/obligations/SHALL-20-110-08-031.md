## Extracted Text

> The following environment variables shall affect the execution of sh:: Establish a string formatted as described in XBD 8. Environment Variables, used to effect command interpretation; see 2.9.1.4 Command Search and Execution.

## Context

**Behavior:** PATH establishes a colon-separated list of directory pathnames used for command search and execution:

1. **Format:** Defined in XBD 8 (basedefs/V1_chap08.html#tag_08): colon-separated directory pathnames. An empty component (leading/trailing colon or `::`) indicates the current working directory.
2. **Usage:** When a simple command name (without slashes) is to be executed, the shell searches the directories listed in PATH, in order, for an executable file with the matching name. See 2.9.1.4 Command Search and Execution (utilities/V3_chap02.html#tag_19_09_01_04) for the full search algorithm.
3. **Built-in precedence:** Special built-ins are found before PATH is searched. Regular built-ins are found during the PATH search (at the point where the built-in's "located" directory would be found).

**Applies to:** All shell modes (interactive and non-interactive).
