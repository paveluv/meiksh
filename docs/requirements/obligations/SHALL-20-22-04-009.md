## Extracted Text

> Write a string to standard output that indicates the pathname or command that will be used by the shell, in the current shell execution environment (see 2.13 Shell Execution Environment), to invoke command_name, but do not invoke command_name. Executable utilities, regular built-in utilities, command_names including a <slash> character, and any implementation-provided functions that are found using the PATH variable (as described in 2.9.1.4 Command Search and Execution), shall be written as absolute pathnames. Shell functions, special built-in utilities, regular built-in utilities not associated with a PATH search, and shell reserved words shall be written as just their names. An alias shall be written as a command line that represents its alias definition. Otherwise, no output shall be written and the exit status shall reflect that the name was not found.

## Context

**Behavior:** Duplicate extraction of the `-v` option description (same as SHALL-20-22-04-006). The `command -v` output rules:

- **Absolute pathnames** for: executable utilities, regular built-ins found via PATH, command_names with `/`, implementation functions found via PATH.
- **Just the name** for: shell functions, special built-ins, regular built-ins not found via PATH, reserved words.
- **Alias definition** for: aliases (written as `alias name='definition'`).
- **No output** (and exit >0) for: names not found in any category.

The lookup uses the current shell execution environment (section 2.13), meaning it reflects currently defined functions, aliases, and the current PATH.
