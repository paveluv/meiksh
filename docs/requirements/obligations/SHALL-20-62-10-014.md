## Extracted Text

> If the -l option is specified:: For job-control background jobs, a field containing the process group ID shall be inserted before the <state> field. Also, more processes in a process group may be output on separate lines, using only the process ID and <command> fields.

## Context

**Behavior:** When `jobs -l` is specified, the output format for job-control background jobs is extended:
1. A process group ID field shall be inserted before the `<state>` field in the output line.
2. Additional processes within the same process group may optionally be listed on separate lines showing only PID and command.

**Output format (approximate):** `[%d] %c %d %s %s\n`, job-number, current, pgid, state, command — with optional additional lines: `%d %s\n`, pid, command.

**Spec reference:** POSIX.1-2024, `jobs` STDOUT, lines 171-174 of `utilities/jobs.html`.
