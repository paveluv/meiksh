## Extracted Text

> The wait utility shall wait for one or more child processes whose process IDs are known in the current shell execution environment (see 2.13 Shell Execution Environment) to terminate.

## Context

**Behavior:** The `wait` builtin shall block until one or more child processes terminate. It only waits for processes whose PIDs are "known" to the current shell execution environment (Section 2.13) — i.e., processes that were started as asynchronous commands (background jobs) in the current shell.

**Known process IDs:** A process ID is "known" if it was recorded by the shell when an asynchronous command was started (via `&`). The shell maintains a list of known PIDs in its execution environment. A PID that was never started by this shell instance is "unknown."

**Conditions:** `wait` operates differently depending on whether operands are provided. With no operands, it waits for all known PIDs. With operands, it waits for the specified PIDs/job IDs.
