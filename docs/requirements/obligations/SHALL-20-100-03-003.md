## Extracted Text

> If standard input is a terminal device and the invoking shell is interactive, read shall prompt for a continuation line when it reads an input line ending with a <backslash> <newline>, unless the -r option is specified.

## Context

**Behavior required:** When ALL of these conditions are met:
1. Standard input is a terminal device (isatty(0) is true)
2. The invoking shell is interactive
3. The `-r` option was NOT specified
4. The input line ends with `\<newline>` (backslash continuation)

Then `read` must display the PS2 continuation prompt on stderr and read another line of input.

**Condition:** Interactive shell, terminal stdin, no `-r`, and the current physical line ends with `\<newline>`.

**Implementation detail:** After detecting a trailing `\<newline>`:
1. Remove the `\<newline>` pair
2. Write the value of `PS2` to stderr
3. Continue reading the next physical line from stdin
4. Repeat until a line does not end with `\<newline>` (or EOF)

If stdin is not a terminal or the shell is not interactive, line continuation still occurs (the `\<newline>` is removed and reading continues) but no prompt is displayed.

**Spec reference:** Section DESCRIPTION of `utilities/read.html`, lines 45-46.
