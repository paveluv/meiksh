## Extracted Text

> The shell is not required to write a diagnostic message, but the utility itself shall write a diagnostic message if required to do so.

## Context

Clarifies the diagnostic message responsibility for non-special utility errors (note 3 from the table). See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required:**
- The shell is NOT required to write its own diagnostic message when a non-special utility encounters an error.
- However, the utility itself SHALL write a diagnostic message if required to do so by its own specification.

**Implementation note:** For non-special built-in errors and external command errors, the shell typically just sets `$?` to the command's exit status. The shell does not need to add its own error message on top of whatever the utility already printed. For shell-generated errors (redirection failures, command-not-found), the shell DOES write diagnostics (per the "yes" entries in the table's "Shell Diagnostic Message Required" column).
