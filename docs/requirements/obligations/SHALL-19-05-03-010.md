## Extracted Text

> The following variables shall affect the execution of the shell:: Provide a default value for the internationalization variables that are unset or null. (See XBD 8.2 Internationalization Variables for the precedence of internationalization variables used to determine the values of locale categories.)

## Context

This defines the semantics of the `LANG` variable.

**Required behavior:**
- `LANG` provides a default value for internationalization variables (`LC_*`) that are unset or null.
- When a locale category (e.g., `LC_CTYPE`, `LC_MESSAGES`) is not set or is set to an empty string, the shell must fall back to the value of `LANG` for that category.
- The precedence order is: `LC_ALL` overrides everything; specific `LC_*` variables override `LANG`; `LANG` is the fallback.

**Conditions:** Applies whenever the shell needs to determine locale settings (character classification, collation, messaging, etc.).

See 2.5.3 "Shell Variables" (`LANG`) and XBD 8.2 "Internationalization Variables" in `basedefs/V1_chap08.html`.
