## Extracted Text

> The <signal.h> header shall also define the following symbolic constants:: [CX] Do not generate SIGCHLD when children stop [XSI] or stopped children continue.

## Context

This is the [CX]/[XSI] extension defining the semantics of `SA_NOCLDSTOP`: when this flag is set in `sa_flags` for a `SIGCHLD` handler via `sigaction()`, the system does not generate `SIGCHLD` when children stop. With the XSI extension, it also suppresses `SIGCHLD` when stopped children continue (i.e., receive `SIGCONT`). For `meiksh` job control, the shell generally needs `SIGCHLD` on stop/continue events to update job status, so this flag should typically be omitted from the shell's `SIGCHLD` handler configuration. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 517–520).
