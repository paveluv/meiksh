## Extracted Text

> The following value may be a constant within an implementation or may vary from one pathname to another.

## Context

The `{LINE_MAX}` value (and similar pathname-associated limits) **may** be a constant across the entire implementation or **may** vary from one pathname to another (e.g., different filesystems may have different limits). The shell should use `pathconf()` or equivalent when determining per-path limits rather than assuming a single global constant. This affects how the shell handles line-length limits for input files on different filesystems. Reference: XCU 1.2.
