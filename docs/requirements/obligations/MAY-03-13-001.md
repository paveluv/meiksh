## Extracted Text

> Memory associated with a thread, established upon request by the implementation for a thread, separate from the thread signal stack, in which signal handlers responding to signals sent to that thread may be executed.

## Context

**Definition only — no shell relevance.** Defines "alternate signal stack" as a separate memory area per thread where signal handlers may execute, established via `sigaltstack()`. This is a C-level threading/signal concept.

**Shell impact:** None. The shell may internally use `sigaltstack()` for its own signal handling robustness (e.g., handling stack overflow signals), but this is an internal implementation detail not mandated by the shell specification. No action required.
