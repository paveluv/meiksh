## Extracted Text

> Non-Interactive Shell: Redirection error with special built-in utilities -- shall exit

## Context

A non-interactive shell SHALL exit when a redirection error occurs with a special built-in utility. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required:**
- When the shell is non-interactive and a redirection attached to a special built-in fails (e.g., `exec >nonexistent_dir/file`, `: >readonly_file`), the shell SHALL write a diagnostic to stderr and exit.
- A shell diagnostic message IS required.

**What constitutes a redirection error:** Failure to open/create the target file (see SHALL-19-07-005), invalid fd in duplication, etc.

**Contrast:** Redirection errors with compound commands, functions, and non-special utilities do NOT cause exit in non-interactive shells (see SHALL-19-08-01-010 through SHALL-19-08-01-015).
