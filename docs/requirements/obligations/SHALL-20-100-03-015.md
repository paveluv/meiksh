## Extracted Text

> If there are still one or more unprocessed var operands, each of the variables names by those operands shall be assigned an empty string.

## Context

**Behavior required:** After the assignment loop completes (both checks in SHALL-20-100-03-011/012 are false), if any var operands have not been assigned a value, each of those variables must be set to the empty string.

**Condition:** After the assignment loop ends, when there are more var operands than output fields and no unsplit remainder.

**Example:** `read a b c` with input `"one"`:
- `a="one"` (from field splitting)
- `b=""` (no more fields, unprocessed var)
- `c=""` (no more fields, unprocessed var)

**Implementation detail:** After the assignment loop, iterate over any remaining unassigned var operands and set each to `""`. This ensures all named variables are always set after `read`, even if the input was shorter than expected.
