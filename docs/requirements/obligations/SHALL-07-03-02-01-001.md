## Extracted Text

> The <collating-symbol> operand shall be a symbolic name, enclosed between angle brackets ('<' and '>'), and shall not duplicate any symbolic name in the current charmap file (if any), or any other symbolic name defined in this collation definition. The string operand is a string of two or more characters that collates as an entity. A <collating-element> defined via this keyword is only recognized with the LC_COLLATE category.

## Context

**Behavior:** The `collating-element` keyword defines multi-character collating elements:
- The symbolic name must be unique — no duplicates from the charmap file or other collation definitions.
- The string operand is 2+ characters that will be treated as a single collating unit.
- These definitions are scoped to LC_COLLATE only.

Example: `collating-element <ch> from "ch"` defines "ch" as a single collating element.

**Shell relevance:** Multi-character collating elements defined here become available in the shell's bracket expressions via `[.xx.]` notation. For example, if "ch" is a collating element, `[[.ch.]]` matches the two-character sequence "ch" as a single unit. The shell must query the locale's collation data to resolve `[.xx.]` expressions.
