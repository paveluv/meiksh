## Extracted Text

> Expands to the shortest representation of the decimal number of positional parameters. The command name (parameter 0) shall not be counted in the number given by '#' because it is a special parameter, not a positional parameter.

## Context

This defines the expansion behavior of the special parameter `$#`.

**Required behavior:**
- `$#` must expand to the shortest decimal string representation of the count of positional parameters currently set.
- "Shortest representation" means no leading zeros: if there are 3 positional parameters, `$#` expands to `3`, not `03` or `003`.
- The command name (parameter `$0`) is NOT counted. `$0` is a special parameter, not a positional parameter, so it is excluded from the count.
- When no positional parameters are set, `$#` expands to `0`.

**Conditions:** Applies in all execution contexts. The count reflects the current scope: inside a function, `$#` counts the function's positional parameters; in a subshell, it inherits the parent's positional parameters unless `set` is called.

**Inputs/outputs:** Read-only; cannot be assigned. Updated implicitly by `set --`, function calls, and shell invocation.

See 2.5.2 "Special Parameters" (`#`) in `utilities/V3_chap02.html`.
