## Extracted Text

> The <signal.h> header shall define the following macros which shall expand to integer constant expressions that need not be usable in #if preprocessing directives:: SIG_BLOCK

## Context

The `<signal.h>` header shall define the macro `SIG_BLOCK` as an integer constant expression. This macro is used as the `how` argument to `sigprocmask()` (and `pthread_sigmask()`). When `SIG_BLOCK` is specified, the resulting process signal mask is the union of the current mask and the signal set pointed to by the `set` argument — i.e., signals in `set` are added to the blocked set. The constant need not be usable in `#if` preprocessing directives. For `meiksh`, this is relevant when the shell calls `sigprocmask()` to block signals during critical sections (e.g., job control, trap handling). The shell must use the platform-provided `SIG_BLOCK` value from `<signal.h>`. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 496–502).
