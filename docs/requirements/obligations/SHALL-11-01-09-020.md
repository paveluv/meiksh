## Extracted Text

> A special character is recognized not only by its value, but also by its context; for example, an implementation may support multi-byte sequences that have a meaning different from the meaning of the bytes when considered individually. Implementations may also support additional single-byte functions. These implementation-defined multi-byte or single-byte functions shall be recognized only if the IEXTEN flag is set; otherwise, data is received without interpretation, except as required to recognize the special characters defined in this section.

## Context

**Behavior:**
1. Implementations **may** support multi-byte sequences or additional single-byte functions as special characters.
2. These implementation-defined functions **shall** be recognized only when IEXTEN is set in `c_lflag`.
3. When IEXTEN is **not** set, data is received without interpretation except for the standard special characters (INTR, QUIT, ERASE, KILL, EOF, EOL, SUSP, START, STOP, CR, NL).

**Conditions:** IEXTEN must be set in `c_lflag` for implementation-defined special character functions to be active.

**Shell relevance:** The shell should be aware that with IEXTEN set, the terminal driver may consume additional characters that would otherwise be delivered as input. When the shell's line editor is active in non-canonical mode, IEXTEN behavior may interfere. The shell may choose to clear IEXTEN during line editing to ensure all characters are delivered. meiksh must document any implementation-defined special characters it supports.

**Spec reference:** POSIX.1-2024, §11.1.9 Special Characters (tag_11_01_09), `basedefs/V1_chap11.html` line 266.
