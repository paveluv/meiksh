## Extracted Text

> The <signal.h> header shall also define the following symbolic constants:: [CX] Causes certain functions to become restartable.

## Context

This is the [CX] extension defining the semantics of `SA_RESTART`: when set in `sa_flags`, certain functions (such as `read()`, `write()`, `wait()`, `waitpid()`, `ioctl()`, and others) that are interrupted by the signal are automatically restarted instead of returning -1 with `errno` set to `EINTR`. For `meiksh`, the shell should use `SA_RESTART` for `SIGCHLD` handlers so that `read()` calls for user input and `waitpid()` calls are not spuriously interrupted. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 529–530).
