## Extracted Text

> Implementations shall provide a character set description file for at least one coded character set supported by the implementation. These files are referred to elsewhere in POSIX.1-2024 as charmap files. It is implementation-defined whether or not users or applications can provide additional character set description files.

## Context

The system must provide at least one charmap file describing a supported character set. The shell does not directly read charmap files — locale and character set information is accessed through C library functions (`setlocale()`, `nl_langinfo()`, etc.) or their Rust equivalents. However, the `localedef` utility (if the shell implements it) uses charmap files. For meiksh, this is an OS-level requirement: the shell relies on the OS providing correct locale/charmap data. No direct shell action is required.

See: `basedefs/V1_chap06.html` §6.4 Character Set Description File.
