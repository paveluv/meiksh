## Extracted Text

> The command utility shall cause the shell to treat the arguments as a simple command, suppressing the shell function lookup that is described in 2.9.1.4 Command Search and Execution, item 1c.

## Context

**Behavior:** When `command` is used as a prefix, the shell must:
1. Treat the remaining arguments as a simple command.
2. Skip the shell function lookup step (item 1c in the Command Search and Execution algorithm, section 2.9.1.4). Normally, the shell checks if the command name matches a defined function before searching PATH — `command` suppresses this.

This means `command foo` will never invoke a function named `foo`, even if one is defined. It will instead search for `foo` as a built-in or external utility.

**Use case:** Allows a function named `ls` to call the real `ls` utility via `command ls` without infinite recursion.

**Spec reference:** POSIX.1-2024, `utilities/command.html`, section DESCRIPTION (tag_20_22_03), lines 39-40. See also 2.9.1.4 Command Search and Execution.
