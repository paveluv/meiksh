## Extracted Text

> The following environment variable shall affect the execution of exec:: Determine the search path when looking for the utility given as the utility operand; see XBD 8.3 Other Environment Variables.

## Context

**Behavior:** When `exec` is invoked with a utility operand, the shell shall use PATH to determine the search path for locating the named utility. The format and semantics of PATH are defined in XBD 8.3 Other Environment Variables (basedefs/V1_chap08.html#tag_08_03).

**Applies to:** All shell modes when `exec utility [argument...]` is invoked.

**Details:** PATH is a colon-separated list of directory pathnames. The shell searches these directories in order for the named utility. The rules for PATH searching (empty components meaning current directory, absolute vs. relative pathnames, etc.) are specified in XBD 8.3. If the utility operand contains a slash, PATH is not used; the operand is used as-is.

**Error handling:** If the utility cannot be found via PATH, the exit status shall be as described in 2.9.1.6 (126 for found-but-not-executable, 127 for not-found).

**Cross-reference:** See `basedefs/V1_chap08.html#tag_08_03` for the normative definition of PATH format and search semantics.

**Spec reference:** Section 2.21 "exec" ENVIRONMENT VARIABLES, lines 3911-3912 in `utilities/V3_chap02.html`.
