## Extracted Text

> When time is used in any of the following circumstances, via a simple command for which the word time is the command name (see 2.9.1.1 Order of Processing), and none of the characters in the word time is quoted, the results (including parsing of later words) are unspecified:: The simple command for which the word time is the command name includes one or more redirections (see 2.7 Redirection) or is (directly) part of a pipeline (see 2.9.2 Pipelines).

## Context

**Behavior (unspecified):** When unquoted `time` is the command name of a simple command that includes redirections (see 2.7) or is directly part of a pipeline (see 2.9.2), the results are unspecified. Examples: `time a | b`, `time a >/dev/null`.

**Workaround:** Users can wrap in a group command: `{ time a; } | b` or use `command time a >/dev/null`.

**Spec reference:** `utilities/time.html` lines 61-64.
