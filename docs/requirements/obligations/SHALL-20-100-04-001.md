## Extracted Text

> The read utility shall conform to XBD 12.2 Utility Syntax Guidelines.

## Context

**Behavior required:** The `read` builtin must conform to the POSIX Utility Syntax Guidelines defined in XBD 12.2. This means:
- Options must be single letters preceded by `-`.
- Options that take arguments must accept the argument as the next argument or immediately following the option letter.
- `--` terminates option processing.
- Options can be grouped (e.g., `-rd x` for `-r -d x`).

**Condition:** All invocations of `read`.

**Implementation detail:** The `read` builtin's option parser must handle standard POSIX option parsing, including `-d delim` and `-r`, following the utility syntax guidelines.

**Spec reference:** Section OPTIONS of `utilities/read.html`, line 106; XBD 12.2.
