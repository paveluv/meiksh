## Extracted Text

> Certain block-mode terminals may be unable to support shell command line editing. If a terminal is unable to provide either edit mode, it need not be possible to set -o vi when using the shell on this terminal.

## Context

**Behavior (optional):** Certain block-mode terminals may be unable to support shell command line editing. If a terminal cannot provide either edit mode, it need not be possible to `set -o vi` when using the shell on that terminal.

**Implementation:** This is a permissive clause. If meiksh detects a terminal that doesn't support character-at-a-time input (block-mode), it may refuse to enable vi mode. In practice, modern terminal emulators support character mode, so this is mainly relevant for legacy hardware terminals.

**Applies to:** Interactive shell on block-mode terminals.

**Spec reference:** Command Line Editing section in `utilities/sh.html`, lines 322-324.
