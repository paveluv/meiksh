## Extracted Text

> When a file that does not exist is created, the following features defined in the System Interfaces volume of POSIX.1-2024 shall apply unless the utility or function description states otherwise:: The user ID of the file shall be set to the effective user ID of the calling process.

## Context

When a new file is created, the user ID of the file shall be set to the effective user ID of the calling process. For `meiksh`, when the shell creates files via output redirection (`> file`), the created file's owner will be the effective user ID of the shell process. This is handled by the underlying `open()` / `creat()` system calls. The shell does not need special logic — the OS enforces this. See POSIX Utilities Chapter 1, Section 1.1.1.4 item 1 (utilities/V3_chap01.html line 89).
