## Extracted Text

> The <signal.h> header shall also define the following symbolic constants:: [CX] Causes signal dispositions to be set to SIG_DFL on entry to signal handlers.

## Context

This is the [CX] extension defining the semantics of `SA_RESETHAND`: when this flag is set in `sa_flags` for a `sigaction()` call, the signal disposition is automatically reset to `SIG_DFL` upon entry to the signal handler. This provides one-shot signal handling — after the handler runs once, subsequent deliveries of that signal will invoke the default action. For `meiksh`, this may be used for signals where the shell wants to handle a signal once (e.g., `SIGTERM` cleanup) and then allow the default action on redelivery. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 525–526).
