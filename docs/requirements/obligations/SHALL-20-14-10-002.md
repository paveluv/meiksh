## Extracted Text

> If a non-empty directory name from CDPATH is not used, and the directory argument is not '-', there shall be no output.

## Context

**Behavior:** The cd built-in must produce NO stdout output when:
1. CDPATH was not used (either CDPATH is unset/empty, or the operand started with `.`/`..`/`/`, or no CDPATH entry matched), AND
2. The directory operand is not `-`.

This is the common case: `cd /some/dir` or `cd ../foo` should produce no output. Only CDPATH-resolved changes and `cd -` produce stdout output.

**Applies to:** Most normal cd invocations.

**Spec reference:** POSIX.1-2024, `utilities/cd.html`, section STDOUT (tag_20_14_10), line 222.
