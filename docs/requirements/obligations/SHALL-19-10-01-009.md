## Extracted Text

> The reduction shall then proceed based upon the token identifier type yielded by the rule applied. When more than one rule applies, the highest numbered rule shall apply (which in turn may refer to another rule). (Note that except in rule 7, the presence of an '=' in the token has no effect.)

## Context

**Rule priority:** When multiple grammar rules could apply to a TOKEN at a given parser state, the **highest-numbered** rule takes precedence. For example, if both Rule 1 (reserved word) and Rule 7 (assignment) could apply, Rule 7 wins. This priority ordering is critical for correct parsing of constructs like `for x in ...` where multiple rules may be candidates.

After the winning rule determines the token type, the parser proceeds with that type for grammar reduction.

**Exception:** The presence of `=` in a token has no effect on classification except in Rule 7 (assignment detection). In all other rules, `=` is treated as an ordinary character.
