## Extracted Text

> If the implementation supports a character encoding of this type, any of the standard utilities that describe character (versus byte) or text-file manipulation shall have the following characteristics:: The utility shall process the statefully encoded data as a concatenation of state-independent characters. The presence of redundant locking shifts shall not affect the comparison of two statefully encoded strings.

## Context

**Behavior:** When processing state-dependent encodings, the shell (and all standard utilities) must treat statefully encoded data as a sequence of state-independent characters. Redundant locking shift codes must not affect string comparison — two strings that decode to the same character sequence must compare as equal regardless of redundant shift codes.

**Shell relevance:** This affects the shell's string comparison operations: `[[ str1 == str2 ]]`, `[ str1 = str2 ]`, case pattern matching, and `${var#pattern}` / `${var%pattern}` matching. The shell must normalize or ignore redundant shift codes during comparison. In practice, this is handled by the C library's `wcscmp()` or equivalent after converting from multi-byte to wide characters.
