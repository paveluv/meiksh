## Context

**Behavior:** When a charmap file uses range notation (`<name1>...<name2> \xHH\xHH`), the encoding value given is for the first name in the range. Subsequent names get sequentially increasing encodings. For multi-byte encodings, bytes are treated as unsigned octets with carry propagation between bytes (like incrementing a big-endian integer). Ranges that would produce null bytes in non-first positions should be avoided.

**Shell relevance:** The shell does not parse charmap files. This is charmap range encoding semantics consumed by `localedef`. No shell implementation action needed.
