## Extracted Text

> If ISIG is set, each input character shall be checked against the special control characters INTR, QUIT, and SUSP. If an input character matches one of these control characters, the function associated with that character shall be performed. If ISIG is not set, no checking shall be done. Thus these special input functions are possible only if ISIG is set.

## Context

**Behavior:** When ISIG is set in `c_lflag`, each input character shall be checked against the special control characters INTR, QUIT, and SUSP. If an input character matches one of these, the associated signal function shall be performed (SIGINT for INTR, SIGQUIT for QUIT, SIGTSTP for SUSP). When ISIG is not set, no such checking is done and these characters are read as normal input.

**Relevance to meiksh:** The `stty` builtin must support `isig`/`-isig`. ISIG is critical for interactive shells — it enables Ctrl-C (INTR), Ctrl-\ (QUIT), and Ctrl-Z (SUSP) signal generation. The shell's line editor typically keeps ISIG set so that interrupt handling works. If the shell disables ISIG, it must handle these signals itself.

**Spec reference:** POSIX.1-2024, §11.2.5 Local Modes (tag_11_02_05), `basedefs/V1_chap11.html` line 1192.
