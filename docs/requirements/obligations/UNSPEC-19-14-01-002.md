## Extracted Text

> For the shell only, it is unspecified whether or not a <backslash> character inside a bracket expression preserves the literal value of the following character.

## Context

**Duplicate of UNSPEC-19-14-01-001: backslash inside bracket expressions in non-shell-quoting contexts is unspecified for the shell.**

Same obligation as UNSPEC-19-14-01-001. Whether `\` inside `[...]` in a non-shell-quoting pattern context escapes the next character is implementation-defined for the shell. See UNSPEC-19-14-01-001 for details.
