## Extracted Text

> If the first character of optstring is a <colon>, the shell variable specified by name shall be set to the <colon> character and the shell variable OPTARG shall be set to the option character found.

## Context

**Behavior:** This is a duplicate extraction of the same requirement as SHALL-20-53-03-011. When a required option-argument is missing and `optstring` starts with `:`:
1. The variable named by `name` shall be set to `:`.
2. `OPTARG` shall be set to the option character that required the missing argument.

**Condition:** The first character of `optstring` must be `:` (silent error mode). No diagnostic is written to stderr in this mode.

**Key distinction:** This silent mode allows scripts to handle missing-argument errors programmatically via the `:` value in `name`, rather than relying on stderr diagnostics.

**Spec reference:** POSIX.1-2024, `getopts` DESCRIPTION, lines 69-71 of `utilities/getopts.html`.
