## Extracted Text

> The system may implement certain utilities as shell functions (see XCU 2.9.5 Function Definition Command) or built-in utilities, but only an application that is aware of the command search order (as described in XCU 2.9.1.4 Command Search and Execution) or of performance characteristics can discern differences between the behavior of such a function or built-in utility and that of an executable file.

## Context

The shell MAY implement standard utilities (like `echo`, `test`, `printf`, etc.) as built-in commands or shell functions rather than external executables. This is a key design decision for meiksh. When doing so, the behavior must be indistinguishable from the external utility except through:

1. **Command search order**: Builtins are found before external commands in `PATH`. The `command -v` and `type` builtins can reveal whether something is a builtin.
2. **Performance**: Builtins execute without `fork()`/`exec()`, making them faster.

The shell must ensure that any utility implemented as a builtin produces the same observable behavior (output, exit status, side effects) as the external version would. References to XCU §2.9.5 (function definitions) and §2.9.1.4 (command search) are already extracted.

See: `basedefs/V1_chap04.html` §4.25 Utility; `utilities/V3_chap02.html` §2.9.5, §2.9.1.4.
