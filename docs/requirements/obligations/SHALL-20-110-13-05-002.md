## Extracted Text

> If the current line is not the edit line, any command that modifies the current line shall cause the content of the current line to replace the content of the edit line, and the current line shall become the edit line. This replacement cannot be undone (see the u and U commands below). The modification requested shall then be performed to the edit line. When the current line is the edit line, the modification shall be done directly to the edit line.

## Context

**Behavior:** In vi command mode, when a modifying command is executed and the current line is NOT the edit line (i.e., the user has navigated to a history entry), the content of the current (history) line shall first replace the edit line's content, making the current line the new edit line. This replacement cannot be undone with `u` or `U`. The requested modification is then performed on the (now current) edit line. When the current line is already the edit line, modifications are applied directly.

**Conditions:** Applies whenever the user navigates to a history line (via `k`, `j`, `G`, `/`, `?`, etc.) and then issues a modification command (insert, delete, change, etc.).

**Implementation:** Track which line is the "edit line" (the original input line) vs. a history line. On any modifying command, if on a history line: (1) copy history line content to the edit line buffer, (2) mark as edit line, (3) clear undo history for this copy operation, (4) apply the modification.
