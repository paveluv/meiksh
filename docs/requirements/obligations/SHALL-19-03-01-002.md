## Extracted Text

> After a token has been categorized as type TOKEN (see 2.10.1 Shell Grammar Lexical Conventions), including (recursively) any token resulting from an alias substitution, the TOKEN shall be subject to alias substitution if all of the following conditions are true:: The TOKEN does not contain any quoting characters.

## Context

**Behavior:** Alias substitution condition 1: The TOKEN must not contain any quoting characters. This means no single-quotes, double-quotes, backslashes, or dollar-single-quote sequences appear anywhere in the token text. If any quoting character is present, the token is exempt from alias substitution.

**Applies to:** All shell modes during alias processing.

**Implementation detail:** After tokenization, check whether the token's raw text includes any of: `'`, `"`, `\`, or the `$'` sequence. If any are found, skip alias substitution for this token. This means `\ls` (escaped) or `'ls'` (quoted) will NOT be alias-expanded, which is the standard way users bypass aliases.

**Example:** If `alias ls='ls --color'` is defined:
- `ls` → alias expanded (no quoting)
- `\ls` → NOT expanded (contains `\`)
- `"ls"` → NOT expanded (contains `"`)
