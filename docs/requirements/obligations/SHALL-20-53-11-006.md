## Extracted Text

> If an option is found that was not specified in optstring, this error is identified and the invalid option character shall be identified in the message.

## Context

**Behavior:** Duplicate extraction of SHALL-20-53-11-003. When an unrecognized option is encountered (verbose mode), the stderr diagnostic must identify the invalid option character. The message format is unspecified but the character must be present.

**Spec reference:** POSIX.1-2024, `getopts` STDERR, lines 181-182 of `utilities/getopts.html`.
