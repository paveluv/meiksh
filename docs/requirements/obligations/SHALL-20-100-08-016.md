## Extracted Text

> Provide the prompt string that an interactive shell shall write to standard error when a line ending with a <backslash> <newline> is read and the -r option was not specified.

## Context

**Behavior required:** Same text as SHALL-20-100-08-015, extracted as a standalone obligation. The PS2 variable provides the continuation prompt for `read` when backslash-newline continuation occurs in an interactive shell.

**Condition:** Interactive shell, terminal stdin, no `-r`, line continuation via `\<newline>`.

**Implementation detail:** See SHALL-20-100-08-015 for full details. The shell writes the expanded value of PS2 to fd 2 (stderr) before reading the continuation line.

**Spec reference:** Section ENVIRONMENT VARIABLES of `utilities/read.html`, lines 170-171.
