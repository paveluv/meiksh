## Extracted Text

> After field splitting, if set -f is not in effect, each field in the resulting command line shall be expanded using the algorithm described in 2.14 Pattern Matching Notation, qualified by the rules in 2.14.3 Patterns Used for Filename Expansion.

## Context

After field splitting (see 2.6.5 in utilities/V3_chap02.html), if the `set -f` (noglob) option is NOT in effect, the shell must perform pathname expansion on each resulting field. The algorithm is defined in 2.14 Pattern Matching Notation (tag_19_14) and further qualified by 2.14.3 Patterns Used for Filename Expansion (tag_19_14_03).

**Behavior required:**
- Each field from field splitting is treated as a potential glob pattern.
- Pattern metacharacters (`*`, `?`, `[...]`) are expanded against filenames in the filesystem according to the pattern matching rules in 2.14.
- If a pattern matches no files, the behavior depends on whether the implementation treats it as a literal (POSIX default: the pattern word is left unchanged).
- If `set -f` is in effect, this entire step is skipped and fields pass through unchanged.

**Ordering:** This step occurs after field splitting and before quote removal. The expansion applies to each field individually, not to the command line as a whole.

**Key references:**
- 2.14 Pattern Matching Notation (tag_19_14 in utilities/V3_chap02.html) for the matching algorithm
- 2.14.3 Patterns Used for Filename Expansion (tag_19_14_03) for filesystem-specific qualifications (e.g., dot-files, slashes)
- `set -f` (the `noglob` option) to disable this step
