## Extracted Text

> Non-Interactive Shell: Special built-in utility error -- shall exit1

## Context

A non-interactive shell SHALL exit when a special built-in utility error occurs, with a caveat about the `command` utility. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required:**
- When the shell is non-interactive and a special built-in utility encounters an error (e.g., `break` with invalid argument, `set` with invalid option, `export` with read-only variable), the shell SHALL exit.
- The footnote (note 1) qualifies this: the shell exits ONLY if the special built-in is executed directly. If it is executed via the `command` utility (e.g., `command set --invalid`), the shell SHALL NOT exit (see SHALL-19-08-01-023).
- A shell diagnostic message is NOT required (note 2) — the special built-in itself is responsible for writing its own diagnostic.

**Special built-ins:** `break`, `:`, `continue`, `.`, `eval`, `exec`, `exit`, `export`, `readonly`, `return`, `set`, `shift`, `times`, `trap`, `unset`.
