## Extracted Text

> In a locale definition file, the operands are strings, formatted as indicated by the grammar in 7.4 Locale Definition Grammar. For some keywords, the strings can contain only integers. Keywords that are not provided, or integer keywords set to -1, can be used to indicate that the value is not available in the locale. String values set to the empty string ("") can be used to indicate that the value is available and is an empty string, or that the value is not available. The following keywords shall be recognized:: Set to a value indicating the separation of the currency_symbol, the sign string, and the value for a negative formatted monetary quantity.

## Context

The `n_sep_by_space` value is an integer (0, 1, or 2) controlling space separation between the currency symbol, sign string, and value for negative monetary quantities, using the same interpretation as `p_sep_by_space`. The shell does not directly format monetary values, but must correctly propagate the LC_MONETARY environment variable.

See: `basedefs/V1_chap07.html` §7.3.3 LC_MONETARY.
