## Extracted Text

> The following rules are used to construct patterns matching multiple characters from patterns matching a single character:: The concatenation of one or more patterns matching a single character with one or more <asterisk> characters is a valid pattern. In such patterns, each <asterisk> shall match a string of zero or more characters, matching the greatest possible number of characters that still allows the remainder of the pattern to match the string.

## Context

**`*` in combination with single-character patterns: greedy matching.**

When `*` is concatenated with one or more single-character patterns (e.g., `a*.c`, `*foo*`), each `*` **shall** match a string of zero or more characters, using **greedy** matching — it matches the greatest possible number of characters that still allows the remainder of the pattern to match the string.

Examples:
- Pattern `a*b` against `"aXXbYYb"`: `*` greedily matches `XXbYY` (the last `b` in the string satisfies the trailing `b`).
- Pattern `*.*` against `"file.tar.gz"`: the first `*` matches `file.tar` and `.` matches `.` and the final `*` matches `gz`.

This greedy behavior is significant for parameter expansion patterns like `${var##pattern}` (longest prefix) vs `${var#pattern}` (shortest prefix), though the greedy/non-greedy distinction there is controlled by `#` vs `##`, not by `*` itself.

See 2.14.2 in utilities/V3_chap02.html.
