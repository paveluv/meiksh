## Extracted Text

> An unsigned positive integer value that defines the minimum number of bytes in a character for the encoded character set. [XSI] On XSI-conformant systems, <mb_cur_min> shall always be 1.

## Context

**Behavior:** The `<mb_cur_min>` declaration in a charmap file specifies the minimum number of bytes per character. On XSI-conformant systems, this value must always be 1, meaning every character set must include single-byte characters.

**Shell relevance:** The shell does not read charmap files directly. However, the guarantee that `mb_cur_min` is 1 on XSI systems means the shell can assume that all Portable Character Set members (used for shell syntax) are single-byte, which simplifies tokenization. The `javascript:open_code('XSI')` ref is a UI artifact, not a file to extract.
