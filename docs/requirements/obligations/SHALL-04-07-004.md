## Extracted Text

> Implementations may provide additional or alternate file access control mechanisms, or both. An additional access control mechanism shall only further restrict the access permissions defined by the file permission bits. An alternate file access control mechanism shall:: Be disabled for a file after the file permission bits are changed for that file with chmod(). The disabling of the alternate mechanism need not disable any additional mechanisms supported by an implementation.

## Context

**Requirement — low shell relevance.** An alternate access control mechanism SHALL be disabled for a file after `chmod()` changes its permission bits. Disabling the alternate mechanism need not disable additional mechanisms.

**Shell impact:** The `chmod` utility (external) and the shell's `umask` builtin interact with file permissions. After `chmod` is called on a file, the shell can assume alternate ACLs are disabled for that file. This is an OS-level guarantee. No direct shell action required.
