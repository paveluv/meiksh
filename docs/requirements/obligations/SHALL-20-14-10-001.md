## Extracted Text

> If a non-empty directory name from CDPATH is used, or if the operand '-' is used, and the absolute pathname of the new working directory can be determined, that pathname shall be written to the standard output as follows:

## Context

**Behavior:** The cd built-in must write the new working directory to stdout under two conditions:

1. **CDPATH match:** A non-empty directory name from CDPATH was used to resolve the target directory (i.e., step 5 succeeded with a non-null CDPATH entry).
2. **Hyphen operand:** The operand was `-` (changing to OLDPWD).

In either case, if the absolute pathname of the new working directory can be determined, it must be written to stdout in the format: `"%s\n", <new directory>`.

**Output format:** A single line containing the absolute pathname of the new working directory, followed by a newline.

**Conditions:** The absolute pathname must be determinable. If it cannot be determined, see UNSPEC-20-14-10-001.

**Spec reference:** POSIX.1-2024, `utilities/cd.html`, section STDOUT (tag_20_14_10), line 214.
