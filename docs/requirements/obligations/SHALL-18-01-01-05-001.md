## Context

When a directory that is the root directory or current working directory of any process is removed, the effect is implementation-defined. If file access permissions deny access, the removal shall fail. Otherwise, when a file is successfully removed, a series of effects occur (directory entry removal, link count decrement, space reclamation or preservation, timestamp updates). For `meiksh`, file removal is not directly performed by shell operators but by utilities like `rm`, `rmdir`, and `unlink`. However, the shell must understand that removing a directory being used as CWD by a running process has implementation-defined behavior — this affects the shell's own `cd` and `pwd` operations if the current directory is deleted. See POSIX Utilities Chapter 1, Section 1.1.1.5 (utilities/V3_chap01.html lines 694–725).
