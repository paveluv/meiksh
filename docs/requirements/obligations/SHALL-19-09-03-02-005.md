## Extracted Text

> If the shell is interactive and the asynchronous AND-OR list became a background job, the job number and the process ID associated with the job shall be written to standard error using the format:

## Context

**Behavior:** When the shell is interactive and the async AND-OR list became a background job, the shell MUST write the job number and process ID to stderr using the format: `"[%d] %d\n", <job_number>, <process_id>`. Example output: `[1] 12345`.

**Applies to:** Interactive shells where the `&` list became a background job (job control enabled, or implementation chose to assign a job number with job control disabled).

**Implementation detail:** Immediately after forking the background job, print the job notification to stderr. The job number is the shell's internal job identifier (typically starting at 1 and incrementing). The process ID is the PID of the background subshell.

**Spec reference:** Section 2.9.3.1, line 2168 in `utilities/V3_chap02.html`.
