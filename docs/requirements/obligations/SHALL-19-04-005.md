## Extracted Text

> This recognition shall only occur when none of the characters is quoted and when the word is used as:: The third word in a case command (only in is valid in this case)

## Context

**Behavior:** Reserved word recognition position 3: The third word in a `case` command. In `case WORD in`, the third word position is where `in` must appear. ONLY `in` is valid in this position as a reserved word.

**Applies to:** All shell modes during parsing.

**Grammar context:** `case WORD in ... esac`. The parser, after seeing `case` (first word) and the case expression WORD (second word), expects the reserved word `in` as the third word. Any other reserved word in this position is a syntax error.

**Implementation detail:** After recognizing `case` and consuming the next WORD token, check if the following token is the unquoted word `in`. If so, classify it as the `In` reserved word token. If it's any other word, report a syntax error.
