## Context

When `fc` is used in edit mode (not `-l`), the following behavior is required: (1) The edited command text must be appended to the end of the history list. (2) The edited command must then be re-executed by the shell. (3) The `fc` command itself (that initiated the editing) must NOT be recorded in the history list. (4) If the editor exits with a non-zero status, both the history entry and the re-execution must be suppressed — the command is silently discarded. (5) Any variable assignments or redirections on the `fc` command line apply to BOTH the `fc` invocation itself AND the re-executed command. For example, `fc -s -- -1 2>/dev/null` suppresses stderr for both `fc` and the re-executed command. Reference: POSIX.1-2024 §fc, DESCRIPTION (tag_20_44_03).
