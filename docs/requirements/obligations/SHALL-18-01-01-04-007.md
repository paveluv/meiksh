## Extracted Text

> When a file that does not exist is created, the following features defined in the System Interfaces volume of POSIX.1-2024 shall apply unless the utility or function description states otherwise:: If the file is a directory, it shall be an empty directory; otherwise, the file shall have length zero.

## Context

When a new file is created, if it is a directory, it shall be an empty directory; otherwise (regular file, etc.) the file shall have length zero. For `meiksh`, when the shell creates a file via `> file` redirection, the new file is created with zero length (which is the default behavior of `open()` with `O_CREAT | O_TRUNC`). The shell does not create directories as part of redirection; that would be done by utilities like `mkdir`. See POSIX Utilities Chapter 1, Section 1.1.1.4 item 5 (utilities/V3_chap01.html line 102).
