## Extracted Text

> The kill utility shall conform to XBD 12.2 Utility Syntax Guidelines, [XSI] except that in the last two SYNOPSIS forms, the -signal_number and -signal_name options are usually more than a single character.

## Context

**Behavior:** The `kill` utility **shall** conform to the Utility Syntax Guidelines (XBD 12.2), with one noted exception: the XSI-extension forms `-signal_number` and `-signal_name` (e.g., `-9`, `-KILL`) are typically more than a single character, which deviates from the standard single-character option convention.

**Conditions:** The standard form uses `-s signal_name` and `-l` (list). The `-signal_number` and `-signal_name` short forms are XSI extensions.

**Shell relevance:** The `kill` builtin must support:
- Standard options per Utility Syntax Guidelines: `-s signal_name`, `-l` [exit_status]
- XSI extension: `-signal_number` (e.g., `-9`), `-signal_name` (e.g., `-KILL`, `-HUP`)
- The `-l` option with no argument lists all signal names
- The `-l` option with an exit_status argument outputs the signal name corresponding to that exit status
- `--` to separate options from operands

**Spec reference:** POSIX.1-2024, `kill` OPTIONS (tag_20_64_04), `utilities/kill.html` lines 60–63. See also XBD 12.2.
