## Extracted Text

> When both the -l option and exit_status operand are specified, the symbolic name of the corresponding signal shall be written in the following format: "%s\n", <signal_name>

## Context

**Behavior:** When `kill -l exit_status` is invoked (with an `exit_status` operand), the output format is:

```
"%s\n", <signal_name>
```

A single signal name followed by a newline. This differs from the list-all format (SHALL-20-64-10-002/003) in that exactly one signal name is written, always terminated by a newline.

**Example:** `kill -l 9` outputs `KILL\n`. A `kill -l exit_status` call based on a process terminated by a signal must also produce one corresponding signal name followed by `\n`.
