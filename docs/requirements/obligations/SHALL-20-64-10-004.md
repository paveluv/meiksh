## Extracted Text

> When both the -l option and exit_status operand are specified, the symbolic name of the corresponding signal shall be written in the following format:

## Context

**Behavior:** When `kill -l exit_status` is invoked (with an `exit_status` operand), the output format is:

```
"%s\n", <signal_name>
```

A single signal name followed by a newline. The signal name is uppercase, without the `SIG` prefix. This differs from the list-all format (SHALL-20-64-10-002/003) in that exactly one signal name is written, always terminated by a newline.

**Example:** `kill -l 9` outputs `KILL\n`. `kill -l 137` (where 137 = 128+9) also outputs `KILL\n`.
