> Explanation of codes: A Automatically included; see text. — Permitted. x Mutually-exclusive. * See note 2. The <space>, which is part of the space and blank classes, cannot belong to punct or graph, but shall automatically belong to the print class. Other space or blank characters can be classified as any of punct, graph, or print.

## Context

**Behavior:** The LC_CTYPE character class mutual-exclusion and inclusion rules summary:
- <space> is in `space` and `blank` classes, automatically in `print`, but forbidden from `punct` and `graph`.
- Other `space`/`blank` characters (not the literal <space>) can additionally be in `punct`, `graph`, or `print`.
- The mutual-exclusion table ensures no character can be in conflicting classes (e.g., `cntrl` and `print` are mutually exclusive, `digit` and `alpha` are mutually exclusive).

**Shell relevance:** These rules guarantee consistent character classification for the shell's bracket expression matching. For example:
- `[[:print:]]` always includes space but `[[:graph:]]` never does.
- No character matches both `[[:cntrl:]]` and `[[:print:]]`.
- The shell can rely on these invariants when implementing pattern matching.
