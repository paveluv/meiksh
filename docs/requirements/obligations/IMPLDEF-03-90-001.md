## Extracted Text

> The time spent executing a process or thread, including the time spent executing system services on behalf of that process or thread. The value of the CPU-time clock for a process is implementation-defined. With this definition the sum of all the execution times of all the threads in a process might not equal the process execution time, even in a single-threaded process, because implementations may differ in how they account for time during context switches or for other reasons.

## Context

**Definition — moderate shell relevance.** Defines "CPU time (execution time)" as time spent executing a process/thread, including system service time on its behalf. The CPU-time clock value is implementation-defined, and thread CPU times may not sum to process CPU time.

**Shell impact:** The `times` builtin reports cumulative user and system CPU time for the shell process and its children. The `time` reserved word reports wall-clock and CPU time for a pipeline. Both rely on OS timing facilities (`times()` syscall, `clock_gettime()`). The shell must correctly retrieve and format these values.
