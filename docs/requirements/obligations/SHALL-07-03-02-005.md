## Extracted Text

> A collation sequence definition shall define the relative order between collating elements (characters and multi-character collating elements) in the locale. This order is expressed in terms of collation values; that is, by assigning each element one or more collation values (also known as collation weights). This does not imply that implementations shall assign such values, but that ordering of strings using the resultant collation definition in the locale behaves as if such assignment is done and used in the collation process. At least the following capabilities are provided:: One-to-many mapping. A single character is mapped into a string of collating elements.

## Context

**Behavior:** LC_COLLATE must support one-to-many character mapping — where a single character is treated as a sequence of collating elements for sorting purposes. For example, the German sharp-s (ß) might be mapped to the collation sequence "ss".

**Shell relevance:** One-to-many mappings primarily affect string comparison (`strcoll()`) and sorting. For the shell's pattern matching, this could mean that a single character matches differently than expected in range expressions. The shell relies on the C library's collation functions to handle these mappings correctly.
