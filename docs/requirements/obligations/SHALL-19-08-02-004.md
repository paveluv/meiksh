## Extracted Text

> The exit status of a command shall be determined as follows:: Otherwise, if the command terminated due to the receipt of a signal, the shell shall assign it an exit status greater than 128. The exit status shall identify, in an implementation-defined manner, which signal terminated the command. Note that shell implementations are permitted to assign an exit status greater than 255 if a command terminates due to a signal.

## Context

**Behavior:** When a command is terminated by a signal, the shell MUST assign an exit status greater than 128. The specific value must identify which signal terminated the command, but the encoding is implementation-defined. The spec explicitly permits exit statuses greater than 255 for signal deaths.

**Applies to:** Any child process that was terminated by a signal (WIFSIGNALED(status) is true from wait()). This includes both external commands and subshell processes.

**Implementation detail (IMPLDEF):** The conventional encoding used by most shells is `128 + signal_number` (e.g., SIGTERM=15 → exit status 143). meiksh should document its chosen encoding. The spec allows values > 255, but the conventional range is 129-192 on most systems. The shell must call `wait()`/`waitpid()` and check `WIFSIGNALED()` before `WEXITSTATUS()`.
