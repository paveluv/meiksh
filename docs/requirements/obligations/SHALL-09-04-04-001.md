## Extracted Text

> When not inside a bracket expression, a <period> ('.') is an ERE that shall match any character in the supported character set except NUL.

## Context

Outside a bracket expression, the period `.` must match exactly one character—any character in the supported character set except the NUL character (`\0`). This is the ERE wildcard. Key points: (1) Only NUL is excluded; newline characters ARE matched by `.` unless a utility-specific option (like `REG_NEWLINE`) changes this behavior. (2) Inside a bracket expression, `.` is literal. (3) The period matches exactly one character, never zero or more than one. The ERE engine must implement this as a single-character atom that accepts all characters except `\0`.
