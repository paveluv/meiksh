## Extracted Text

> The <signal.h> header shall also define the following symbolic constants:: [XSI] Causes implementations not to create zombie processes or status information on child termination. See sigaction().

## Context

This is the [XSI] extension defining the semantics of `SA_NOCLDWAIT`: when set in `sa_flags` for `SIGCHLD`, the implementation does not create zombie processes and does not preserve termination status information when children exit. This means `waitpid()` will block until all children have terminated and then return -1 with `errno` set to `ECHILD`. For `meiksh`, the shell must NOT use this flag because the shell relies on `waitpid()` to collect child exit statuses for `$?`, pipeline status, and job state tracking. See `sigaction()` for full details. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 536–538).
