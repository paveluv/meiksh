## Context

Defines the overall structure and parsing of here-documents. See 2.7.4 Here-Document in utilities/V3_chap02.html.

**Behavior required:**
- A here-document is a single word that begins after the next NEWLINE token following the `<<` or `<<-` operator.
- It continues until a line is found containing ONLY the delimiter followed by a newline, with no blank characters between them.
- If multiple here-documents appear on the same line, each starts after the previous one ends.
- The end of a `command_string` operand (from `sh -c`) SHALL be treated as a newline for the purpose of locating the terminating delimiter line.
- The end of the `commands` string in `$(commands)` or `` `commands` `` MAY be treated as a newline for the same purpose.
- If end-of-input is reached without finding the terminating delimiter line, the shell SHOULD (but need not) treat this as a redirection error.

**Parser implications:**
- The parser must track pending here-documents: after tokenizing a `<<` or `<<-` and its delimiter word, the here-document body is read starting at the next newline.
- The delimiter comparison is exact: the line must match the delimiter with no leading/trailing blanks (unless `<<-` strips leading tabs, see SHALL-19-07-04-011).

**Key reference:** `sh` utility (utilities/sh.html) for `command_string` operand.
