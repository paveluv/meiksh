## Extracted Text

> The following rules and definitions apply to bracket expressions:: A non-matching list expression begins with a <circumflex> ('^'), and the matching behavior shall be the logical inverse of the corresponding matching list expression (the same bracket expression but without the leading <circumflex>). For example, since the RE "[abc]" only matches 'a', 'b', or 'c', it follows that "[^abc]" is an RE that matches any character except 'a', 'b', or 'c'. It is unspecified whether a non-matching list expression matches a multi-character collating element that is not matched by any of the expressions. The <circumflex> shall have this special meaning only when it occurs first in the list, immediately following the <left-square-bracket>.

## Context

**Behavior:** A non-matching list expression begins with `^` and its matching behavior shall be the logical inverse of the corresponding matching list expression. `[^abc]` matches any character except 'a', 'b', or 'c'. The `^` has this special meaning only when it occurs first in the list, immediately after `[`. Multi-character collating element matching in non-matching lists is unspecified.

**Shell relevance:** In shell patterns, both `[^abc]` and `[!abc]` denote non-matching lists (section 2.14 allows `!` as the negation character). The shell must implement the logical inverse: the bracket expression matches any single character NOT in the specified set. The `^` (or `!`) must be immediately after the opening `[` to have special meaning; elsewhere in the list it is literal.
