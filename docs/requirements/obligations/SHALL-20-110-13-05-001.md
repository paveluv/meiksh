## Extracted Text

> In command mode for the command line editing feature, decimal digits not beginning with 0 that precede a command letter shall be remembered. Some commands use these decimal digits as a count number that affects the operation.

## Context

**Behavior:** In vi command mode, decimal digits (1-9) that precede a command letter shall be accumulated and remembered as a numeric count. The digit `0` does NOT start a count (it is a separate command that moves to column 0). Some commands use this count to repeat the operation.

**Conditions:** Only applies when the shell is in vi command mode (entered via ESC from insert mode). The count accumulation resets after each command execution.

**Implementation:** Maintain a count accumulator in the vi editing state. When a digit 1-9 is received in command mode, append it to the current count string. When a non-digit command character arrives, parse the accumulated digits as the count and apply it to the command. If no digits preceded the command, the count is implicitly 1 (unless the command specifies otherwise).
