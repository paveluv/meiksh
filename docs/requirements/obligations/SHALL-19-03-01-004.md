## Extracted Text

> After a token has been categorized as type TOKEN (see 2.10.1 Shell Grammar Lexical Conventions), including (recursively) any token resulting from an alias substitution, the TOKEN shall be subject to alias substitution if all of the following conditions are true:: An alias with that name is in effect.

## Context

**Behavior:** Alias substitution condition 3: An alias with the TOKEN's name must currently be in effect (defined and not unset). If no alias with that name exists, alias substitution does not occur and the token is processed normally.

**Applies to:** All shell modes during alias processing.

**Implementation detail:** Look up the token text in the alias table (a string→string map maintained by the `alias` builtin and cleared by `unalias`). If found, the condition is satisfied. The alias table is part of the shell's execution environment.

**Spec reference:** Section 2.3.1, lines 324-325 in `utilities/V3_chap02.html`.
