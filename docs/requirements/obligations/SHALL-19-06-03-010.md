## Context

**Behavior:** Command substitution supports nesting — a command substitution can appear inside another command substitution.

For the `$(...)` form, nesting is natural and requires no special escaping: `$(echo $(date))`.

For the backquoted form, nesting requires escaping inner backquotes with backslashes. Each level of nesting requires an additional level of backslash escaping:
- Level 1: `` `commands` ``
- Level 2: `` `echo \`date\`` ``
- Level 3: `` `echo \`echo \\\`date\\\`\`` `` (backslashes double at each level)

The application (not the shell) is responsible for providing the correct escaping in the backquoted form.

**Spec reference:** See 2.6.3 Command Substitution in utilities/V3_chap02.html, lines 1140–1144.
