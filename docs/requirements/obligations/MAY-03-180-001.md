## Extracted Text

> In the context of the shell, jobs are created when a list (see XCU 2.9.3 Lists) is executed while job control is enabled, and may be created when an asynchronous AND-OR list is executed while job control is disabled.

## Context

**Definition — direct shell relevance.** Defines "job" in the shell context: jobs are created when a list (see XCU 2.9.3 Lists) is executed while job control is enabled. Jobs MAY also be created when an asynchronous AND-OR list is executed while job control is disabled.

**Shell impact:** This is the foundational definition for the shell's job control system. A "job" corresponds to a pipeline or list that the shell tracks. When job control is enabled (`set -m`, interactive shells), every list execution creates a job. The shell assigns job numbers, tracks process group IDs, and manages job state (running, stopped, done). The `jobs`, `fg`, `bg`, `wait`, `kill`, and `disown` builtins all operate on jobs.

**Design decision:** Whether `meiksh` creates jobs for async lists when job control is disabled (see also MAY-03-35-001).
