## Extracted Text

> If set to a non-empty string value, override the values of all the other internationalization variables.

## Context

**Behavior:** If LC_ALL is set to a non-empty string value, it shall override all other internationalization variables. This means:
- LC_COLLATE, LC_CTYPE, LC_MESSAGES, and any other LC_* category variables are ignored.
- LANG is also overridden.
- The locale specified by LC_ALL is used for all locale categories.

**Applies to:** All shell modes (interactive and non-interactive).

**Details:** The shell must check LC_ALL before consulting individual LC_* variables or LANG. If LC_ALL is set and non-empty, its value determines the locale for all categories: character classification (ctype), collation, and messages. If LC_ALL is unset or empty, the shell falls back to the individual LC_* variables, then to LANG.
