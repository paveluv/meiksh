## Extracted Text

> If the -p option is specified, the output shall consist of one line for each process ID:

## Context

**Behavior:** When `jobs -p` is used, the output format is simplified to one process ID per line:
```
"%d\n", <process ID>
```

For job-control background jobs, this is the process group leader PID. For non-job-control background jobs (if supported), this is the process ID associated with the job.

**No other fields:** Unlike the default format, `-p` does not output job number, current indicator, state, or command — only the PID.

**Spec reference:** POSIX.1-2024, `jobs` STDOUT, line 112 of `utilities/jobs.html`.
