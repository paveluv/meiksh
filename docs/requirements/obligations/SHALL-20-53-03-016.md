## Extracted Text

> The shell variables OPTIND and OPTARG shall not be exported by default. An error in setting any of these variables (such as if name has previously been marked readonly) shall be considered an error of getopts processing, and shall result in a return value greater than one.

## Context

**Behavior:** Two distinct requirements:

1. **OPTIND and OPTARG must not be auto-exported:** When the shell initializes or when getopts sets these variables, they must not have the export attribute unless the user explicitly exports them. They exist only in the current shell execution environment by default.

2. **Readonly variable errors are getopts processing errors:** If `getopts` cannot set `OPTIND`, `OPTARG`, or the user-specified `name` variable (e.g., because one has been marked `readonly`), this is a getopts processing error and the exit status must be **>1** (not 0 or 1).

**Distinction:** This is the only case where getopts returns >1. Unrecognized options and missing arguments return 0; end-of-options returns 1.

**Spec reference:** POSIX.1-2024, `getopts` DESCRIPTION, lines 86-87 of `utilities/getopts.html`.
