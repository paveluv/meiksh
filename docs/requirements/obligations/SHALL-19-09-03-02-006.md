## Extracted Text

> If the shell is interactive and the asynchronous AND-OR list did not become a background job, the process ID associated with the asynchronous AND-OR list shall be written to standard error in an unspecified format.

## Context

**Behavior:** When the shell is interactive and the async AND-OR list did NOT become a background job (it became a background command without a job number), the process ID MUST be written to stderr in an UNSPECIFIED format. The shell must still notify the user, but the format is implementation's choice.

**Applies to:** Interactive shells where `cmd &` did not get a job number (only possible when job control is disabled and the implementation chose not to assign a job number).

**Implementation detail:** meiksh may print just the PID, or use a format like `%d\n`. Since the format is unspecified, any reasonable output to stderr is acceptable. If meiksh always assigns job numbers, this case may never arise.

**Spec reference:** Section 2.9.3.1, line 2173 in `utilities/V3_chap02.html`.
