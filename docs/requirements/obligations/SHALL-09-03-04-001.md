## Extracted Text

> When not inside a bracket expression, a <period> ('.') is a BRE that shall match any character in the supported character set except NUL.

## Context

**Behavior:** When not inside a bracket expression, a period (`.`) in a BRE shall match any character in the supported character set except NUL.

**Shell relevance:** The shell equivalent is the `?` pattern character, which matches any single character except NUL. In filename expansion (globbing), `?` additionally does not match a leading `.` in a filename component unless the pattern explicitly starts with `.`. The BRE period (`.`) has no special meaning in shell patterns — it is a literal character.
