## Extracted Text

> If the -m option is not in effect, it is unspecified whether SIGTTIN, SIGTTOU, and SIGTSTP signals are ignored, set to the default action, or caught. If they are caught, the shell shall, in the signal-catching function, set the signal to the default action and raise the signal (after taking any appropriate steps, such as restoring terminal settings).

## Context

**Behavior:** When the shell is **interactive** and the `-m` option is **not** enabled, the handling of SIGTTIN, SIGTTOU, and SIGTSTP is **unspecified** — the implementation may ignore them, set them to default, or catch them. However, **if** the implementation catches them, the signal-catching function must:
1. Set the signal back to the default action.
2. Raise the signal (re-deliver it to self).
3. Take any appropriate steps before raising, such as restoring terminal settings.

**Implementation choice:** POSIX leaves the top-level policy unspecified when `-m` is not in effect. The only mandatory part is the catch-handler behavior if the implementation chooses to catch these signals.

**Applies to:** Interactive shell without `-m` option.
