## Extracted Text

> The following rules can be used to construct BREs matching multiple characters from BREs matching a single character:: When a BRE matching a single character, a subexpression, or a back-reference is followed by the special character <asterisk> ('*'), together with that <asterisk> it shall match what zero or more consecutive occurrences of the BRE would match. For example, "[ab]*" and "[ab][ab]" are equivalent when matching the string "ab".

## Context

**Behavior:** When a BRE matching a single character, subexpression, or back-reference is followed by `*`, together with the `*` it shall match zero or more consecutive occurrences of what the preceding BRE matches. Example: `[ab]*` and `[ab][ab]` are equivalent when matching "ab".

**Shell relevance:** In shell patterns, `*` matches zero or more of ANY characters (not zero or more of the preceding element). This is a fundamental difference between BRE `*` (repetition quantifier) and shell glob `*` (wildcard). This obligation is BRE-specific.
