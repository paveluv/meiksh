## Extracted Text

> When a variable assignment is done, the variable shall be created if it did not already exist. If value is not specified, the variable shall be given a null value.

## Context

Two key rules for variable assignment:

1. **Auto-creation**: If a variable does not exist when an assignment targets it, the shell must create it. There is no need to "declare" variables before assignment. For example, `NEWVAR=hello` creates `NEWVAR` if it doesn't exist.
2. **Null default**: If no value is specified (i.e., `VAR=` with nothing after the `=`), the variable is set to the null (empty) string `""`, not unset. This is distinct from an unset variable — `VAR=` means `VAR` exists with an empty value, while an unset `VAR` means it doesn't exist at all. This difference is observable via `${VAR+set}` and `${VAR-default}` parameter expansions.

The shell's variable store must distinguish between "set to null" and "unset" states for each variable.
