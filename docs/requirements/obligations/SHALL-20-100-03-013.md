## Extracted Text

> If more than one var operand is yet to be processed and one or more output fields are yet to be used, the variable named by the first unprocessed var operand shall be assigned the value of the first unused output field.

## Context

**Behavior required:** Same text as SHALL-20-100-03-011, extracted as a standalone obligation. When more than one var operand remains unprocessed and output fields are available, assign the next output field to the next var operand.

**Condition:** `remaining_vars > 1` AND `remaining_fields >= 1`.

**Implementation detail:** This is the primary assignment step in the loop. Iterate through var operands and output fields in parallel, assigning one field per var, until only one var remains (at which point check 2 / SHALL-20-100-03-012/014 takes over).
