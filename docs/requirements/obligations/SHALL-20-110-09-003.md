## Extracted Text

> If the shell is interactive:: SIGQUIT and SIGTERM signals shall be ignored.

## Context

**Behavior:** When the shell is **interactive**, SIGQUIT and SIGTERM signals must be **ignored** (set to SIG_IGN). This prevents the interactive shell from being accidentally killed by these signals — users must use `exit` or EOF to leave the shell.

**Implementation:** On startup of an interactive shell, set the signal disposition for SIGQUIT and SIGTERM to SIG_IGN. This disposition is inherited by child processes unless they override it.

**Applies to:** Interactive shell only.
