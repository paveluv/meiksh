## Extracted Text

> When the -V option is specified, standard output shall be formatted as:

## Context

**Behavior:** When `command -V` is used, stdout must be formatted as:
```
"%s\n", <unspecified>
```
The format string content is unspecified (implementation-defined), but it must include the command category identification and required information as specified in SHALL-20-22-04-008. Common formats include:
- `ls is /usr/bin/ls`
- `cd is a shell builtin`
- `if is a reserved word`
- `ll is aliased to 'ls -la'`

Each result is a single line terminated by a newline.

**Spec reference:** POSIX.1-2024, `utilities/command.html`, section STDOUT (tag_20_22_10), line 176.
