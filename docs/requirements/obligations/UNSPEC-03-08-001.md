## Extracted Text

> To cause the user's terminal to give some audible or visual indication that an error or some other event has occurred. When the standard output is directed to a terminal device, the method for alerting the terminal user is unspecified. When the standard output is not directed to a terminal device, the alert is accomplished by writing the alert to standard output (unless the utility description indicates that the use of standard output produces undefined results in this case).

## Context

**Definition — moderate shell relevance.** Defines "alert" as causing the terminal to give an audible or visual notification. When stdout goes to a terminal, the method is unspecified. When stdout is not a terminal, alert is accomplished by writing the BEL character (`\a`, 0x07) to stdout.

**Shell impact:** The shell must correctly handle `\a` in contexts like `echo` (when XSI extensions apply) and `$'\a'` in parameter expansion. The `echo` builtin and `printf` builtin should emit the alert character when `\a` is encountered. The shell does not need to implement the terminal's alert mechanism — it only writes the byte.
