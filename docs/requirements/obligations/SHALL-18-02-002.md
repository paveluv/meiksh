## Extracted Text

> All numeric limits defined by the System Interfaces volume of POSIX.1-2024, such as {PATH_MAX}, shall also apply to this volume of POSIX.1-2024. All the utilities defined by this volume of POSIX.1-2024 are implicitly limited by these values, unless otherwise noted in the utility descriptions.

## Context

All numeric limits defined by the System Interfaces volume (such as `{PATH_MAX}`, `{NAME_MAX}`, `{ARG_MAX}`, `{OPEN_MAX}`) **shall** also apply to shell utilities. The shell must not impose arbitrary limits below these system-defined values. For example, the shell must be able to handle pathnames up to `{PATH_MAX}` bytes, command arguments up to `{ARG_MAX}` bytes combined, and keep open up to `{OPEN_MAX}` file descriptors (within the constraints of its own descriptor usage). Unless a utility description explicitly specifies a different limit, these system limits are the implicit bounds. Reference: XCU 1.2.
