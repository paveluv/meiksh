## Extracted Text

> shall cause the file whose name is the expansion of word to be opened for both reading and writing on the file descriptor denoted by n, or standard input if n is not specified. If the file does not exist, it shall be created.

## Context

Defines the read-write file descriptor operator `[n]<>word`. See 2.7.7 Open File Descriptors for Reading and Writing in utilities/V3_chap02.html.

**Syntax:** `[n]<>word`

**Behavior required:**
- Open the file named by the expansion of `word` for BOTH reading AND writing on the fd denoted by `n` (or fd 0 if `n` is omitted).
- If the file does not exist, it SHALL be created.
- The file is NOT truncated (unlike `>`).

**System call:** Equivalent to `open(word_expanded, O_RDWR | O_CREAT, mode)` where mode is typically 0666 modified by umask.

**Use case:** This is used for in-place file modification or for opening a file that needs both read and write access (e.g., `/dev/tcp` on some systems, or updating a file in place).

**Default fd:** Unlike `>` (defaults to fd 1) and `<` (defaults to fd 0), `<>` defaults to fd 0 (standard input).
