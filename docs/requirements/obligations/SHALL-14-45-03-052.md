## Extracted Text

> The <signal.h> header shall also define the following symbolic constants:: SS_ONSTACK

## Context

The `<signal.h>` header shall define the symbolic constant `SS_ONSTACK`. This is an [XSI] constant used with `sigaltstack()` to indicate the state of the alternate signal stack. When returned in `ss_flags` of a `stack_t` structure via `sigaltstack(NULL, &oss)`, `SS_ONSTACK` indicates that the process is currently executing on the alternate signal stack. For `meiksh`, this constant is not directly used since the shell does not typically employ alternate signal stacks. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html line 543).
