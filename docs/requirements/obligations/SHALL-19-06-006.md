## Context

This establishes the single-field rule for word expansions and its exceptions.

**Required behavior:**
- **General rule:** Tilde expansions, parameter expansions, command substitutions, arithmetic expansions, and quote removals that occur within a single word must expand to a single field.
- **Exceptions — the shell may create multiple fields or no fields from a single word ONLY through:**
  1. Field splitting (step 2)
  2. Pathname expansion (step 3)
  3. The special cases described in SHALL-19-06-007 (`$@`/`$*`) and SHALL-19-06-008/009 (brace expansion)

**Conditions:** This is a fundamental invariant of the expansion pipeline. An expansion like `${var}` in a non-splitting context (e.g., assignment) must always produce exactly one field, even if the value contains IFS characters.

See 2.6 "Word Expansions" in `utilities/V3_chap02.html`.
