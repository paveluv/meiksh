## Extracted Text

> Execute the current command line. If the current command line is not empty, this line shall be entered into the command history (see fc).

## Context

This obligation defines the behavior of the **`<newline>` character in vi insert mode** for the interactive command line editor.

**Behavior required:** When the user presses `<newline>` (Enter/Return) while in vi insert mode:
1. The shell shall execute the current command line — i.e., submit the accumulated line buffer for parsing and execution by the shell's command language processor.
2. If the current command line is **not empty** (contains at least one character other than the newline itself), the line shall be appended to the command history list managed by `fc` (see `utilities/fc.html`).
3. If the command line **is empty**, it is still executed (as a no-op) but is NOT added to the command history.

**Conditions:** Only applies when the shell is interactive, vi editing mode is enabled (`set -o vi`), and the shell is in insert mode. The identical behavior also applies to `<newline>` in command mode (see `utilities/sh.html` line 398–400).

**After execution:** The shell returns to insert mode for the next command (per the rule that `sh` enters insert mode after termination of the previous command — see `utilities/sh.html` line 334–335).

**History interaction:** The history entry is managed per the `fc` builtin's semantics. The `HISTFILE` environment variable controls persistent storage; see `utilities/sh.html` tag_20_110_13_01 (Command History List).
