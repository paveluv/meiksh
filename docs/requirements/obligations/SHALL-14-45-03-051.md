## Extracted Text

> The <signal.h> header shall also define the following symbolic constants:: [CX] Causes signal not to be automatically blocked on entry to signal handler.

## Context

This is the [CX] extension defining the semantics of `SA_NODEFER`: when set in `sa_flags`, the signal that triggered the handler is not automatically blocked on entry to the signal handler. Without this flag, the delivered signal is added to the blocked set for the duration of the handler, preventing recursive signal delivery. With `SA_NODEFER`, the same signal can be delivered again while the handler is still executing. For `meiksh`, this is generally avoided because the shell's signal handlers modify shared state (job tables, flags) and recursive invocation could cause data corruption. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 540–541).
