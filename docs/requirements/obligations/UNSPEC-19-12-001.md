## Extracted Text

> If multiple signals are pending for the shell for which there are associated trap actions, the order of execution of trap actions is unspecified.

## Context

**Order of multiple pending trap actions is unspecified.**

When multiple signals with associated trap actions are pending simultaneously (e.g., both SIGTERM and SIGUSR1 arrive while the shell is executing a foreground command), the order in which their trap handlers execute is unspecified by POSIX.

The implementation may choose any order — e.g., signal number order, arrival order, or arbitrary. No conformance test can mandate a specific order. The implementation should document its chosen behavior but is free to pick the simplest approach (e.g., iterate pending signals in numeric order).
