## Extracted Text

> When the option requires an option-argument, the getopts utility shall place it in the shell variable OPTARG . If no option was found, or if the option that was found does not have an option-argument, OPTARG shall be unset.

## Context

The `getopts` builtin must manage the `OPTARG` variable as follows: (1) When an option that requires an option-argument (followed by `:` in `optstring`) is successfully parsed, `OPTARG` must be set to the option-argument value (just the argument characters, not the option character itself). (2) When no option is found (end of options), `OPTARG` must be unset. (3) When an option WITHOUT an option-argument is found, `OPTARG` must be unset. The unsetting must use the equivalent of `unset OPTARG` — the variable should not exist in the environment, not merely be empty. Reference: POSIX.1-2024 §getopts, DESCRIPTION (tag_20_53_03).
