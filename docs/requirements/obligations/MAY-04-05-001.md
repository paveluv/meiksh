## Extracted Text

## Context

**Definition — low shell relevance.** Describes the sticky bit (S_ISVTX) on directories: when a writable directory has the sticky bit set, a process may only remove/rename files if certain conditions are met (listed in -002 through -004). This is the `/tmp` protection mechanism.

**Shell impact:** The shell does not implement sticky bit checking — the filesystem enforces it. The shell may encounter EPERM when attempting to remove/rename files in sticky-bit directories via redirections (`> /tmp/otheruser_file`). The `umask` and `chmod` utilities interact with permission bits including S_ISVTX. The shell should report permission errors correctly.

**Spec reference:** basedefs/V1_chap04.html, section 4.5.
