## Extracted Text

> Interactive Shell: Shell language syntax error -- shall not exit

## Context

An interactive shell SHALL NOT exit when a shell language syntax error occurs. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required:**
- When the shell is interactive, a syntax error causes the shell to write a diagnostic to stderr but NOT exit.
- The shell discards the erroneous command and returns to the prompt for more input.
- A shell diagnostic message IS required for this error type.

**Implication:** The interactive shell must recover gracefully from syntax errors — reset the parser state, discard any partially parsed command, and re-prompt. See also SHALL-19-08-01-027 for the requirement that no further processing of the erroneous command occurs.
