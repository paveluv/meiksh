## Extracted Text

> An integer set to a value indicating the positioning of the positive_sign for a monetary quantity with a non-negative value. The following integer values shall be recognized for int_n_sign_posn, int_p_sign_posn, n_sign_posn, and p_sign_posn: 0 Parentheses enclose the quantity and the currency_symbol. 1 The sign string precedes the quantity and the currency_symbol. 2 The sign string succeeds the quantity and the currency_symbol. 3 The sign string precedes the currency_symbol. 4 The sign string succeeds the currency_symbol.

## Context

The `p_sign_posn` value (0-4) controls sign positioning for non-negative monetary quantities. These same values apply to all four sign position keywords (`p_sign_posn`, `n_sign_posn`, `int_p_sign_posn`, `int_n_sign_posn`): 0=parentheses enclose value+symbol, 1=sign before value+symbol, 2=sign after value+symbol, 3=sign before symbol, 4=sign after symbol. The shell does not directly format monetary values. The shell must correctly propagate the LC_MONETARY environment variable.

See: `basedefs/V1_chap07.html` §7.3.3 LC_MONETARY.
