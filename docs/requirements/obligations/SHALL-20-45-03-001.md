## Extracted Text

> If job control is enabled (see the description of set -m), the shell is interactive, and the current shell execution environment (see 2.13 Shell Execution Environment) is not a subshell environment, the fg utility shall move a background job in the current execution environment into the foreground, as described in 2.11 Job Control; it may also do so if the shell is non-interactive or the current shell execution environment is a subshell environment.

## Context

The `fg` builtin's core behavior depends on three conditions: (1) Job control must be enabled (`set -m`). (2) The shell must be interactive. (3) The current execution environment must not be a subshell. When ALL three conditions are met, `fg` MUST move the specified background job into the foreground per the Job Control mechanism (XCU 2.11). The implementation MAY also support `fg` when the shell is non-interactive or in a subshell environment, but this is optional. Moving to foreground means: the job's process group gets the terminal (via `tcsetpgrp`), SIGCONT is sent if the job was stopped, and the shell waits for the job to complete or stop again. `fg` is an intrinsic utility and must be implemented as a shell builtin. Reference: POSIX.1-2024 §fg, DESCRIPTION (tag_20_45_03); XCU 2.11 Job Control.
