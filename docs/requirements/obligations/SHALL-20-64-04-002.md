## Extracted Text

> The following options shall be supported:

## Context

**Behavior:** The `kill` utility **shall** support the following options:
- `-s signal_name`: specify the signal to send by name (e.g., `-s TERM`, `-s HUP`). The signal name is recognized in a case-independent fashion without the "SIG" prefix (though "SIG" prefix should also be accepted).
- `-l` [exit_status]: list signal names. Without argument, write all signal names supported by the implementation. With an exit_status argument (a decimal integer), write the signal name corresponding to that status value. If the exit_status is a value of the `$?` special parameter indicating a signal was received, the signal name for that signal shall be written.
- [XSI] `-signal_number`: signal specified by number
- [XSI] `-signal_name`: signal specified by name (without `-s`)

**Conditions:** The `-s` and `-l` options are base POSIX. The `-signal_number` and `-signal_name` forms are XSI extensions.

**Shell relevance:** The `kill` builtin must implement:
1. `kill -l` → list all signal names (one per line or space-separated)
2. `kill -l exit_status` → map exit status to signal name (e.g., `kill -l 137` → `KILL`)
3. `kill -s SIGNAL pid...` → send named signal
4. `kill -SIGNAL pid...` → XSI shorthand
5. `kill pid...` → send SIGTERM (default)
