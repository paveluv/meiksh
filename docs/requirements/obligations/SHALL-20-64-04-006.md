## Extracted Text

> The following options shall be supported:: Specify the signal to send, using one of the symbolic names defined in the <signal.h> header. Values of signal_name shall be recognized in a case-independent fashion, without the SIG prefix. In addition, the symbolic name 0 shall be recognized, representing the signal value zero. The corresponding signal shall be sent instead of SIGTERM.

## Context

**Behavior:** When `-s signal_name` is given:
1. The `signal_name` argument must be matched case-insensitively against signal names defined in `<signal.h>`, but **without** the `SIG` prefix. For example, `kill -s hup`, `kill -s HUP`, and `kill -s Hup` must all be accepted as SIGHUP.
2. The symbolic name `0` must be recognized, representing signal value zero (used to test process existence without sending a signal).
3. The matched signal replaces the default SIGTERM for the kill operation.

**Forbidden:** The `SIG` prefix must **not** be required or expected. The spec does not require accepting the prefix (e.g., `kill -s SIGHUP` is not required to work).

**Inputs:** A string representing the signal name.

**Error handling:** If the signal_name does not match any known signal, the implementation should produce a diagnostic and exit >0.
