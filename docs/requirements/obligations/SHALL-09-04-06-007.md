## Extracted Text

> The following rules shall be used to construct EREs matching multiple characters from EREs matching a single character:: Each of the duplication symbols ('+', '*', '?', and intervals) can be suffixed by the repetition modifier '?' (<question-mark>), in which case matching behavior for that repetition shall be changed from the leftmost longest possible match to the leftmost shortest possible match, including the null match (see A.9 Regular Expressions). For example, the ERE ".*c" matches up to and including the last character ('c') in the string "abc abc", whereas the ERE ".*?c" matches up to and including the first character 'c', the third character in the string. If the REG_MINIMAL flag, defined in the <regex.h> header, is used when compiling an ERE via regcomp(), the leftmost shortest possible match shall be the default for all duplication symbols, and the repetition modifier '?' can be used to select the leftmost longest possible match for the repetition it modifies.

## Context

The ERE engine must support the non-greedy (lazy) repetition modifier `?` suffix on all duplication operators (`+?`, `*?`, `??`, `{m,n}?`). When this modifier is present, matching must use leftmost shortest semantics instead of leftmost longest. Example: `.*c` on "abc abc" matches "abc abc" (greedy, longest), while `.*?c` matches "abc" (non-greedy, shortest). Additionally, when the `REG_MINIMAL` flag is set via `regcomp()`, the default for all repetitions becomes non-greedy, and the `?` modifier then switches to greedy behavior. For the shell's `[[ =~ ]]` construct and pattern matching, the default (greedy) behavior applies unless explicitly modified with `?`.
