## Extracted Text

> If the shell is interactive, SIGINT signals received during command line editing shall be handled as described in the EXTENDED DESCRIPTION, and SIGINT signals received at other times shall be caught but no action performed.

## Context

**Behavior:** When the shell is **interactive**:
- SIGINT signals received **during command line editing** must be handled as described in the EXTENDED DESCRIPTION (vi-mode editing: abort current edit line, reissue prompt, reset history — see SHALL-20-110-13-03-003).
- SIGINT signals received **at other times** (e.g., while waiting for a foreground job) must be caught but **no action performed** (the signal is absorbed without terminating the shell).

**Implementation:** The shell needs two SIGINT handlers depending on context:
1. During line editing: interrupt the current edit, discard the line, redisplay the prompt.
2. At other times (waiting for commands, between prompts): catch and ignore.

**Applies to:** Interactive shell only.

**Spec reference:** ASYNCHRONOUS EVENTS section in `utilities/sh.html`, lines 264-265.
