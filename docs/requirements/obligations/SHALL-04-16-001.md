## Extracted Text

> Pathname resolution is performed for a process to resolve a pathname to a particular directory entry for a file in the file hierarchy. There may be multiple pathnames that resolve to the same directory entry, and multiple directory entries for the same file. When a process resolves a pathname of an existing directory entry, the entire pathname shall be resolved as described below. When a process resolves a pathname of a directory entry that is to be created immediately after the pathname is resolved, pathname resolution terminates when all components of the path prefix of the last component have been resolved. It is then the responsibility of the process to create the final component.

## Context

The shell performs pathname resolution in numerous contexts: command search (`PATH` lookup), the `cd` builtin, the `.` (dot) builtin, redirection targets, pathname expansion (globbing), and the `test` builtin's file operands. This obligation defines the fundamental rule: when resolving a pathname to an existing file, the entire pathname must be resolved component by component. When creating a new file (e.g., redirection `> newfile`), only the path prefix (parent directory) components are resolved; the shell/OS then creates the final component.

The shell must ensure that when it opens files for redirection or sources scripts, it follows this resolution model. For command search, each directory in `PATH` is tried as a prefix, and the command name is appended as the final component.
