## Extracted Text

> The varname and value parts shall meet the requirements for a name and a word, respectively, except that they are delimited by the embedded unquoted <equals-sign>, in addition to other delimiters.

## Context

In a variable assignment `varname=value`:

- **`varname`**: Must be a valid shell Name as defined in basedefs §3.228 — starts with a letter or underscore, followed by zero or more letters, digits, or underscores. The shell's lexer/parser must validate this.
- **`value`**: Must be a valid shell Word as defined in basedefs §3.421 — any sequence of characters, potentially including quotes, expansions, etc.
- **Delimiter**: The unquoted `=` sign separates the name from the value. If the `=` is quoted (e.g., `'FOO=bar'` or `FOO\=bar`), it is not treated as an assignment delimiter — the entire token is a regular word.

The shell must correctly identify the first unquoted `=` in a word to split it into `varname` and `value` parts. The `varname` must not contain any characters outside the Name definition.

See: `basedefs/V1_chap04.html` §4.26 Variable Assignment; `basedefs/V1_chap03.html` §3.228 Name, §3.421 Word.
