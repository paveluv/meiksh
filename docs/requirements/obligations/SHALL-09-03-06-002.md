## Extracted Text

> The following rules can be used to construct BREs matching multiple characters from BREs matching a single character:: A subexpression can be defined within a BRE by enclosing it between the character pairs "\(" and "\)". Such a subexpression shall match whatever it would have matched without the "\(" and "\)", except that anchoring within subexpressions is optional behavior; see 9.3.8 BRE Expression Anchoring. Subexpressions can be arbitrarily nested.

## Context

**Behavior:** A BRE subexpression enclosed between `\(` and `\)` shall match whatever it would have matched without the grouping delimiters, except that anchoring within subexpressions is optional behavior. Subexpressions can be arbitrarily nested.

**Shell relevance:** Shell pattern matching does not have a subexpression/grouping mechanism in POSIX. This is BRE-specific. The shell does not support `\(` and `\)` in glob or `case` patterns. This obligation does not directly apply to shell pattern matching.

**Spec reference:** Section 9.3.6, lines 335-338 in `basedefs/V1_chap09.html`; see also 9.3.8 BRE Expression Anchoring.
