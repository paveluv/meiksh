## Extracted Text

> A sequence of characters starting with a <dollar-sign> immediately followed by a single-quote ($') shall preserve the literal value of all characters up to an unescaped terminating single-quote ('), with the exception of certain <backslash>-escape sequences, as follows:: \' yields an <apostrophe> (single-quote) character.

## Context

**Behavior:** Inside `$'...'`, the escape sequence `\'` yields a literal single-quote character (`'`). This is the ONLY way to include a single-quote inside a `$'...'` string, since an unescaped `'` terminates the construct.

**Applies to:** All shell modes during `$'...'` escape processing.

**Implementation detail:** When `\` is followed by `'`, emit `'` (0x27) and advance past both characters. Critically, this escaped single-quote must NOT terminate the `$'...'` sequence (see also SHALL-19-02-04-020).

**Spec reference:** Section 2.2.4, lines 172-173 in `utilities/V3_chap02.html`.
