## Extracted Text

> A function that may be safely invoked by an application while the asynchronous form of cancellation is enabled. No function is async-cancel-safe unless explicitly described as such.

## Context

**Definition only — no shell relevance.** Defines "async-cancel-safe function" as a function safe to call while asynchronous cancellation is enabled. This is a pthreads/C-level concept.

**Shell impact:** None. The shell does not use pthreads cancellation in its command language semantics. This is purely an internal implementation concern if `meiksh` uses threads internally.
