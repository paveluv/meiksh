## Extracted Text

> The capability to specify additional locales to those provided by an implementation is optional, denoted by the _POSIX2_LOCALEDEF symbol. If the option is not supported, only implementation-supplied locales are available. Such locales shall be documented using the format specified in this section.

## Context

**Behavior:** Custom locale creation (via `localedef`) is optional, gated by `_POSIX2_LOCALEDEF`. Whether or not custom locales are supported, all implementation-supplied locales must be documented in the format specified by Section 7.3.

**Shell relevance:** The shell does not create or parse locale definition files. However, the shell depends on the system's locales being correctly defined per these rules. The shell may query `_POSIX2_LOCALEDEF` via `getconf` but does not need to act on it directly.
