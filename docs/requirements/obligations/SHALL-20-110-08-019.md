## Extracted Text

> The following environment variables shall affect the execution of sh:: Determine the locale for the interpretation of sequences of bytes of text data as characters (for example, single-byte as opposed to multi-byte characters in arguments and input files), which characters are defined as letters (character class alpha), and the behavior of character classes within pattern matching.

## Context

**Behavior:** LC_CTYPE determines three aspects of the shell's character handling:

1. **Byte-to-character interpretation:** Whether byte sequences are interpreted as single-byte or multi-byte characters. This affects how the shell counts characters in `${#var}`, how it processes input, and how it handles pattern matching at character boundaries.
2. **Letter classification (alpha class):** Which characters are considered letters. This affects:
   - What constitutes a valid variable name (Name = letter or underscore, followed by letters, digits, or underscores).
   - The `[:alpha:]` character class in pattern matching.
3. **Character class behavior in pattern matching:** How `[:upper:]`, `[:lower:]`, `[:digit:]`, `[:alnum:]`, `[:blank:]`, `[:space:]`, etc. are interpreted in bracket expressions (e.g., `[[:alpha:]]`).

**Applies to:** All shell modes (interactive and non-interactive).

**Details:** If LC_CTYPE is unset or null, the value of LANG is used. If LC_ALL is set and non-empty, it overrides LC_CTYPE. The POSIX locale provides ASCII-based character classification.
