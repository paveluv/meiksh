## Extracted Text

> A character set mapping definition line shall exist for all symbolic names specified in Portable Character Set, and shall define the coded character value that corresponds to the character indicated in the table, or the coded character value that corresponds to the control character symbolic name. If the control characters commonly associated with the symbolic names in Non-Portable Control Characters are supported by the implementation, the symbolic name and the corresponding encoding value shall be included in the file. Additional unique symbolic names may be included. A coded character value can be represented by more than one symbolic name.

## Context

**Behavior:** Every character in the Portable Character Set must have a mapping line in the charmap file. Non-portable control characters, if supported, must also be mapped. Additional symbolic names may be added, and multiple symbolic names may map to the same coded value.

**Shell relevance:** The shell does not parse charmap files, but it depends on the guarantee that all Portable Character Set members (including shell syntax characters like `|`, `&`, `;`, `<`, `>`, `$`, `` ` ``, `\`, `"`, `'`, `(`, `)`, `{`, `}`, `#`, `!`, `~`, `*`, `?`, `[`, `]`, space, tab, newline) have well-defined, unique encodings. This is a foundational guarantee for correct tokenization.
