## Extracted Text

> it shall not affect the shell variables in the caller's environment.

## Context

**Behavior required:** When `read` is executed in a subshell or separate utility execution environment, the variable assignments made by `read` must NOT affect the caller's (parent's) shell variables.

**Condition:** When `read` runs in a subshell context, such as:
- `(read var)` — explicit subshell
- `echo foo | read var` — pipeline (each command in a pipeline may run in a subshell)
- `var=$(read var)` — command substitution

**Implementation detail:** Since subshells get a copy of the parent's environment, variable modifications in the subshell naturally don't propagate back. The shell must ensure that `read` in these contexts operates on the subshell's copy of the variable environment. No special handling is needed if the shell correctly implements subshell variable isolation per Section 2.13.
