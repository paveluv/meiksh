## Extracted Text

> The standard error shall be used for diagnostic messages and prompts for continued input.

## Context

**Behavior required:** The `read` builtin must use standard error (fd 2) for two purposes:
1. **Diagnostic messages:** Error messages (e.g., readonly variable, invalid option).
2. **Continuation prompts:** The PS2 prompt when line continuation occurs in an interactive shell.

No other output may be written to stderr. Normal output from `read` goes to shell variables, not to any file descriptor.

**Condition:** All invocations of `read`.

**Implementation detail:** Unlike most utilities, `read` produces no stdout output — its "output" is variable assignments. Stderr is used for both errors and the PS2 continuation prompt. This is distinct from `pwd` and other utilities that only use stderr for diagnostics.
