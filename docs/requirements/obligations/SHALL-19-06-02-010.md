## Extracted Text

> In the parameter expansions shown previously, use of the <colon> in the format shall result in a test for a parameter that is unset or null; omission of the <colon> shall result in a test for a parameter that is only unset. ## Context

**Behavior:** The colon in parameter expansion modifiers controls the test condition:
- **With colon** (`${param:-word}`, `${param:=word}`, `${param:?word}`, `${param:+word}`): tests whether `parameter` is **unset OR null** (empty string). Both unset and set-to-empty are treated the same.
- **Without colon** (`${param-word}`, `${param=word}`, `${param?word}`, `${param+word}`): tests **only** whether `parameter` is **unset**. A parameter set to the empty string is considered "set" and does NOT trigger the `word` action.

**Special case for `#`:** When `parameter` is `#` and the colon is omitted, `word` must be specified. This is because `${#-}` would be ambiguous with the string-length expansion `${#parameter}`. The application must write `${#-word}` to avoid this ambiguity.

**Summary table (from spec):**
| Form | Set & Non-Null | Set But Null | Unset |
|---|---|---|---|
| `${p:-w}` | substitute p | substitute w | substitute w |
| `${p-w}` | substitute p | substitute null | substitute w |
| `${p:=w}` | substitute p | assign w | assign w |
| `${p=w}` | substitute p | substitute null | assign w |
| `${p:?w}` | substitute p | error, exit | error, exit |
| `${p?w}` | substitute p | substitute null | error, exit |
| `${p:+w}` | substitute w | substitute null | substitute null |
| `${p+w}` | substitute w | substitute w | substitute null |
