## Extracted Text

> If the string contains at least three characters, begins with a <left-curly-bracket> ('{') and ends with a <right-curly-bracket> ('}'), and the delimiter character is one of '<' or '>', the token identifier IO_LOCATION may result; if the result is not IO_LOCATION, the token identifier TOKEN shall result.

## Context

When a token has at least 3 characters, begins with `{`, ends with `}`, and the delimiter character is `<` or `>`, the lexer **may** classify it as `IO_LOCATION`; if not, it **shall** classify it as `TOKEN`. IO_LOCATION is a POSIX.1-2024 feature for named file descriptor locations (e.g., `{fd}>file` which allocates a file descriptor and stores the number in variable `fd`).

The implementation has discretion on whether to support IO_LOCATION. If IO_LOCATION is not supported, the token must be classified as TOKEN. If IO_LOCATION is supported, the token must match the pattern `{NAME}` where NAME is a valid variable name (see XBD 3.216 Name in `basedefs/V1_chap03.html#tag_03_216`).

This is the third classification rule, checked after operator and IO_NUMBER. The `{...}` form must contain characters between the braces (minimum 3 chars total: `{`, at least one inner char, `}`).
