## Extracted Text

> If {MAX_CANON} is defined for this terminal device, it shall be a limit on the number of bytes in a line. The behavior of the system when this limit is exceeded is implementation-defined. If {MAX_CANON} is not defined, there shall be no such limit; see pathconf().

## Context

**Behavior:** In canonical mode, MAX_CANON defines the maximum line length in bytes (if defined for the terminal). If defined, exceeding it causes implementation-defined behavior. If not defined, there is no line length limit.

**Conditions:** Applies to canonical mode input. The value can be queried via pathconf().

**Shell relevance:** The shell should be aware of MAX_CANON when reading lines in canonical mode. Very long input lines might be truncated or cause errors if MAX_CANON is exceeded. The shell's line editing mode (non-canonical) is not subject to MAX_CANON.

**Spec reference:** XBD 11.1.6 "Canonical Mode Input Processing" (tag_11_01_06), `basedefs/V1_chap11.html`, lines 162–164.
