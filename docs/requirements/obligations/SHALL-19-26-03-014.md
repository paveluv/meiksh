## Extracted Text

> When the shell tries to expand, in a parameter expansion or an arithmetic expansion, an unset parameter other than the '@' and '*' special parameters, it shall write a message to standard error and the expansion shall fail with the consequences specified in 2.8.1 Consequences of Shell Errors.

## Context

**Behavior:** The `-u` (`nounset`) option: when the shell tries to expand an **unset** parameter in a parameter expansion or arithmetic expansion (other than `$@` and `$*`), the shell must:
1. Write a message to standard error.
2. The expansion must fail with the consequences specified in 2.8.1 Consequences of Shell Errors (non-interactive shell exits; interactive shell continues).

**Exception:** `$@` and `$*` are exempt — expanding them when no positional parameters are set does not trigger an error.

**Applies to:** All shell modes.

**Spec reference:** `set` DESCRIPTION in `utilities/V3_chap02.html`, lines 4896-4898. Cross-ref: 2.8.1 Consequences of Shell Errors.
