## Extracted Text

> The following declarations can precede the character definitions. Each shall consist of the symbol shown in the following list, starting in column 1, including the surrounding brackets, followed by one or more <blank> characters, followed by the value to be assigned to the symbol.

## Context

**Behavior:** In a charmap file, header declarations (such as `<code_set_name>`, `<mb_cur_max>`, `<mb_cur_min>`, `<escape_char>`, `<comment_char>`) must appear before the CHARMAP section. Each declaration starts in column 1 with the bracketed keyword, followed by blanks, followed by the value.

**Shell relevance:** The shell does not parse charmap files. This obligation constrains charmap file syntax, which is consumed by `localedef` and the system's locale infrastructure. The shell indirectly depends on correct charmap processing to get correct character encodings for the current locale.
