## Extracted Text

> The following option shall be supported:: Remove all alias definitions from the current shell execution environment.

## Context

**Behavior:** This is the description for the `-a` option of `unalias`. When `-a` is specified, `unalias` shall remove every alias definition from the current shell execution environment. After `unalias -a`, no aliases are defined and no alias substitution will occur until new aliases are created with `alias`.

**Scope:** The removal applies to the current shell execution environment (Section 2.13). In a subshell, `unalias -a` only removes aliases in that subshell's environment.

**Security use case:** `unalias -a` is useful in secure shell scripts to ensure no unexpected aliases are in effect (e.g., aliases set via ENV or implementation-defined predefined aliases).
