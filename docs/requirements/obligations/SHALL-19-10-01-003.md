## Extracted Text

> If the string consists solely of digits and the delimiter character is one of '<' or '>', the token identifier IO_NUMBER shall result.

## Context

When a token consists entirely of digits (`[0-9]+`) and the character immediately following it (the delimiter) is `<` or `>`, the lexer must classify it as `IO_NUMBER`. This token represents a file descriptor number in I/O redirection (e.g., `2>` where `2` is the IO_NUMBER). The IO_NUMBER must be recognized before the generic TOKEN classification—it is the second classification rule.

If the digit string is not followed by `<` or `>`, it falls through to subsequent classification rules (IO_LOCATION check, then TOKEN). The implementation should check the delimiter character at the point where the token is delimited, not after further tokenization.
