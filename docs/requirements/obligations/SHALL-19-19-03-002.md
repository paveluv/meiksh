## Extracted Text

> If file does not contain a <slash>, the shell shall use the search path specified by PATH to find the directory containing file. Unlike normal command search, however, the file searched for by the dot utility need not be executable. If no readable file is found, a non-interactive shell shall abort; an interactive shell shall write a diagnostic message to standard error.

## Context

**Behavior:** File search rules for the `.` (dot) built-in:

1. If `file` does **not** contain a `/`, the shell must search for the file using the `PATH` variable (each directory in PATH is checked for the file).
2. Unlike normal command search, the file found by `.` does **not** need to be executable — only readable.
3. If no readable file is found:
   - **Non-interactive shell:** must abort (exit with an error).
   - **Interactive shell:** must write a diagnostic message to standard error (but continue running).

**Applies to:** All shell modes, with different error handling for interactive vs non-interactive.
