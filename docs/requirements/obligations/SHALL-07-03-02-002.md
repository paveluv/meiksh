## Extracted Text

> A collation sequence definition shall define the relative order between collating elements (characters and multi-character collating elements) in the locale. This order is expressed in terms of collation values; that is, by assigning each element one or more collation values (also known as collation weights). This does not imply that implementations shall assign such values, but that ordering of strings using the resultant collation definition in the locale behaves as if such assignment is done and used in the collation process. At least the following capabilities are provided:: Multi-character collating elements. Specification of multi-character collating elements (that is, sequences of two or more characters to be collated as an entity).

## Context

**Behavior:** LC_COLLATE must support multi-character collating elements — sequences of two or more characters that are treated as a single unit for collation purposes (e.g., in some locales, "ch" or "ll" are treated as single collating elements).

**Shell relevance:** The shell must support multi-character collating elements in bracket expressions using the `[.xx.]` notation (collating element expressions). For example, `[[.ch.]]` would match the multi-character collating element "ch" if defined in the locale. The shell's pattern matching engine needs to check for multi-character collating elements during bracket expression evaluation.
