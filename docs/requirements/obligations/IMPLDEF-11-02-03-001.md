## Extracted Text

> The initial output control value after open() is implementation-defined.

## Context

**Behavior:** The initial value of the `c_oflag` (output control) field after `open()` on a terminal device is implementation-defined. POSIX does not mandate defaults for OPOST, ONLCR, or any other output mode flag.

**Relevance to meiksh:** meiksh must document its chosen initial output control values. When the shell starts interactively, it should explicitly configure the output modes it requires (typically OPOST and ONLCR for proper newline handling) rather than relying on inherited or default values.

**Spec reference:** POSIX.1-2024, §11.2.3 Output Modes (tag_11_02_03), `basedefs/V1_chap11.html` line 810.
