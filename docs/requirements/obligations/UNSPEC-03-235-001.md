## Extracted Text

> A regular file containing the output of a compiler, formatted as input to a linkage editor for linking with other object files into an executable form. The methods of linking are unspecified and may involve the dynamic linking of objects at runtime. The internal format of an object file is unspecified, but a conforming application cannot assume an object file is a text file.

## Context

**Definition only — no shell relevance.** Defines "object file" as a compiler output formatted for the linker. Internal format is unspecified; may involve dynamic linking at runtime. A conforming application cannot assume it is a text file.

**Shell impact:** None. The shell does not process object files. The shell may attempt to execute an object file and will get ENOEXEC, which should be handled appropriately. No action required.
