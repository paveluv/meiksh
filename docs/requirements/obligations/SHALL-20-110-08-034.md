## Extracted Text

> [UP] This variable, when and only when an interactive shell is invoked, shall be subjected to parameter expansion (see 2.6.2 Parameter Expansion) by the shell, and the resulting value shall be used as a pathname of a file containing shell commands to execute in the current environment. The file need not be executable. If the expanded value of ENV is not an absolute pathname, the results are unspecified. ENV shall be ignored if the real and effective user IDs or real and effective group IDs of the process are different. The file specified by ENV need not be processed if the file can be written by any user other than the user identified by the real (and effective) user ID of the shell process.

## Context

**Behavior:** This is a duplicate extraction of the ENV variable description (same source lines as SHALL-20-110-08-003). The full behavior specification is identical:

1. **Interactive-only:** ENV is processed when and only when an interactive shell is invoked.
2. **Parameter expansion:** The value of ENV shall be subjected to parameter expansion (see 2.6.2 in `utilities/V3_chap02.html#tag_19_06_02`).
3. **Execution:** The resulting expanded pathname is sourced (executed) in the current shell environment.
4. **Executable permission:** The file need not be executable.
5. **Non-absolute path:** If the expanded value is not an absolute pathname, results are unspecified.
6. **Security — setuid/setgid:** ENV shall be ignored if real != effective UID or real != effective GID.
7. **Security — writable file:** The ENV file need not be processed if writable by users other than the real (and effective) UID.
