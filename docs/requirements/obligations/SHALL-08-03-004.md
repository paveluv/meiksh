## Extracted Text

> The system shall initialize this variable at the time of login to be the user's login name. See <pwd.h>. For a value of LOGNAME to be portable across implementations of POSIX.1-2024, the value should be composed of characters from the portable filename character set.

## Context

**Behavior:** The system shall initialize `LOGNAME` at login time to be the user's login name (from the password database, see `<pwd.h>`). For portability, the value should be composed of characters from the portable filename character set (`[A-Za-z0-9._-]`).

**Shell relevance:** The shell must propagate `LOGNAME` to child processes. The shell should not modify `LOGNAME`. Some scripts use `LOGNAME` to determine the current user identity. The shell does not use `LOGNAME` directly for any built-in functionality, but it is part of the standard environment the shell must maintain.
