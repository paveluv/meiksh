## Extracted Text

> If the O_NONBLOCK flag is clear, then the read request shall be blocked until data is available or a signal has been received. If the O_NONBLOCK flag is set, then the read request shall be completed, without blocking, in one of three ways:

## Context

**Behavior:** Terminal read behavior depends on O_NONBLOCK:
- O_NONBLOCK clear → read blocks until data is available or a signal interrupts
- O_NONBLOCK set → read completes without blocking (three cases detailed in SHALL-11-01-05-002 through 004)

**Conditions:** Applies to read() on terminal device files. The shell typically reads with O_NONBLOCK clear (blocking reads for interactive input).

**Shell relevance:** Interactive shell reads are blocking. The shell must handle signal interruption of blocked reads (EINTR) and retry as appropriate. Non-blocking reads may be used for line editing or checking for pending input.
