## Extracted Text

> Whenever a process requests file access permission for read, write, or execute/search, if no additional mechanism denies access, access shall be determined as follows:: If a process has appropriate privileges: If read, write, or directory search permission is requested, access shall be granted. If execute permission is requested, access shall be granted if execute permission is granted to at least one user by the file permission bits or by an alternate access control mechanism; otherwise, access shall be denied.

## Context

**Requirement — moderate shell relevance.** For a process with appropriate privileges:
- Read, write, or directory search permission: access SHALL be granted.
- Execute permission: access SHALL be granted only if at least one user has execute permission (via permission bits or alternate ACL); otherwise, access SHALL be denied.

**Shell impact:** When the shell runs as root (EUID 0), it can read/write any file and search any directory. However, root CANNOT execute a file unless at least one execute bit is set (owner, group, or other). This affects `$PATH` command lookup for the root user — a file without any `x` bit cannot be executed even by root.

The `test -x` builtin should reflect this: for root, `-x` should return true only if at least one execute bit is set.
