> The umask utility shall conform to XBD 12.2 Utility Syntax Guidelines.

## Context

**Behavior:** The `umask` utility's option parsing shall conform to the POSIX Utility Syntax Guidelines defined in XBD Section 12.2. This means:
- Options shall be single characters preceded by a `-` character.
- Options without option-arguments can be grouped after a single `-`.
- The first operand not starting with `-` or the `--` argument terminates option processing.
- Option `-S` is the only defined option.

**Conditions:** This applies to all invocations of `umask`. The shell must correctly parse `-S` as an option and reject unknown options.

**Error handling:** Invalid options (e.g., `umask -Z`) shall produce a diagnostic message on stderr and a non-zero exit status.
