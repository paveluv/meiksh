## Extracted Text

> Non-Interactive Shell: Unrecoverable read error when reading commands -- shall exit4

## Context

A non-interactive shell SHALL exit when an unrecoverable read error occurs while reading commands. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required:**
- When the shell is non-interactive and encounters an unrecoverable error reading command input (e.g., I/O error on the script file, broken pipe on stdin), the shell SHALL exit.
- A shell diagnostic message IS required.
- See note 4 (SHALL-19-08-01-025) for additional rules about what happens after the read error and special handling for the `dot` built-in.

**"Unrecoverable":** This means the shell cannot continue reading input at all — not a transient error that might resolve. Examples: EIO on the input fd, the input file has been deleted mid-read.
