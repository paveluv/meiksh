## Extracted Text

> The following options shall be supported by the implementation:: Handle the operand dot-dot logically; symbolic link components shall not be resolved before dot-dot components are processed (see steps 8. and 9. in the DESCRIPTION).

## Context

With `-L` (logical mode), the `cd` builtin **shall** handle dot-dot (`..`) logically: symbolic link components are NOT resolved before dot-dot components are processed. This means `cd ..` from a symlinked directory goes to the logical parent (the directory containing the symlink), not the physical parent of the symlink target. The shell processes dot-dot by manipulating the logical path string (steps 8 and 9 of the DESCRIPTION algorithm) rather than resolving symlinks first. Reference: cd utility OPTIONS, cd utility DESCRIPTION steps 8-9.
