## Extracted Text

> The evaluation of arithmetic expressions shall be equivalent to that described in Section 6.5, Expressions, of the ISO C standard.

## Context

The evaluation of arithmetic expressions in shell arithmetic contexts (particularly `$(( ))` expansion) **shall** follow the rules described in ISO C Section 6.5 (Expressions). This means: subexpressions are evaluated according to C operator precedence and associativity; side effects from assignment and increment/decrement operators take effect at sequence points; short-circuit evaluation applies to `&&` and `||`; and the ternary `?:` operator only evaluates the selected branch. Division by zero and overflow produce undefined behavior per C semantics (the shell should treat these as errors). Reference: XCU 1.1.2.1, ISO C 6.5.
