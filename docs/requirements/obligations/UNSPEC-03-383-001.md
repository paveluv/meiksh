## Extracted Text

> A character that in the output stream indicates that printing or displaying should start at the next horizontal tabulation position on the current line. It is the character designated by '\t' in the C language. If the current position is at or past the last defined horizontal tabulation position, the behavior is unspecified. It is unspecified whether this character is the exact sequence transmitted to an output device by the system to accomplish the tabulation.

## Context

**Definition — moderate shell relevance.** Defines the tab character `<tab>` as `\t` (0x09). It advances to the next horizontal tabulation position. Behavior at or past the last tab stop is unspecified. Whether this byte is the exact terminal sequence is unspecified.

**Shell impact:** The shell must recognize `\t` in escape-sequence contexts: `printf '\t'`, `echo` with XSI interpretation, and `$'\t'` quoting. The tab character is also:
- Part of the default `IFS` value (space, tab, newline) — used for word splitting
- Significant in here-documents with `<<-` (leading tabs are stripped)
- Used in the interactive line editor for command/filename completion
