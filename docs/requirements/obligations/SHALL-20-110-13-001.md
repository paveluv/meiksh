## Extracted Text

> See 2. Shell Command Language. [UP] The functionality described in the rest of the EXTENDED DESCRIPTION section shall be provided on implementations that support the User Portability Utilities option (and the rest of this section is not further shaded for this option).

## Context

The `sh` utility's EXTENDED DESCRIPTION section defers the core shell language behavior to "2. Shell Command Language" (utilities/V3_chap02.html#tag_19), which defines all parsing, expansion, execution, and control flow semantics.

The remainder of the EXTENDED DESCRIPTION (command history, command line editing via `fc`, `vi`-mode and `emacs`-mode editing) is gated behind the **User Portability Utilities option [UP]**. An implementation that supports [UP] **shall** provide all of that functionality. The `meiksh` implementation targets [UP] support.

This obligation is primarily a structural/scoping statement: the shell must implement the full Shell Command Language from chapter 2, and (given [UP] support) must also provide command history and line editing as described in the rest of the sh EXTENDED DESCRIPTION.

See utilities/sh.html#tag_20_110_13 for the full EXTENDED DESCRIPTION.
