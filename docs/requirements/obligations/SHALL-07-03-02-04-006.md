## Extracted Text

> The symbol UNDEFINED shall be interpreted as including all coded character set values not specified explicitly or via the ellipsis symbol. Such characters shall be inserted in the character collation order at the point indicated by the symbol, and in ascending order according to their coded character set values. If no UNDEFINED symbol is specified, and the current coded character set contains characters not specified in this section, the utility shall issue a warning message and place such characters at the end of the character collation order.

## Context

The UNDEFINED symbol in the LC_COLLATE collation order section acts as a catch-all placeholder for any characters not explicitly listed or covered by an ellipsis. Characters matching UNDEFINED are inserted at the position where UNDEFINED appears in the collation order, sorted by their code point values. If UNDEFINED is omitted and there are unspecified characters, a warning must be issued and those characters are placed at the end of the collation order. The shell is affected because unspecified characters' collation positions determine whether they fall within bracket expression ranges in pattern matching. If a character is not in the collation order, its sorting behavior for `[a-z]`-style ranges becomes implementation-dependent.
