## Extracted Text

> Each field containing the results from an expansion shall be processed in order, intermixed with fields not containing the results of expansions, processed as described above, as if by using the following algorithm, examining bytes in the input field, from beginning to end:

## Context

**Behavior:** This introduces the main field splitting algorithm. Each input field that contains expansion results must be processed using the byte-by-byte algorithm detailed in SHALL-19-06-05-011 through SHALL-19-06-05-013. Fields without expansion results are passed through unchanged (per SHALL-19-06-05-005). The algorithm processes all fields in their original order, intermixing split and unsplit fields in the output.

**Algorithm overview:** The algorithm examines bytes in each input field from left to right, classifying each byte/sequence as:
1. Literal (non-expansion) bytes → append to candidate
2. Expansion bytes that are not IFS characters → append to candidate
3. Expansion bytes that are IFS white space → consume (coalesce)
4. Expansion bytes that are non-white-space IFS characters → consume, note as delimiter

See SHALL-19-06-05-012 for the detailed steps.

**Spec reference:** See 2.6.5 Field Splitting in utilities/V3_chap02.html, lines 1250–1252.
