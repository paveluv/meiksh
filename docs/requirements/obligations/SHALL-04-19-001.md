## Extracted Text

> How any changes to the value of seconds since the Epoch are made to align to a desired relationship with the current actual time is implementation-defined. As represented in seconds since the Epoch, each and every day shall be accounted for by exactly 86400 seconds.

## Context

The Epoch time model (each day = exactly 86400 seconds, no leap seconds) affects the shell when performing time arithmetic or comparisons. The `test` builtin's `-nt`/`-ot` operators compare file modification times in this time model. The shell's `SECONDS` variable and the `time` reserved word also operate within this framework. The shell must treat each day as exactly 86400 seconds when performing any time-related calculations. This is an OS-level guarantee that the shell relies on.
