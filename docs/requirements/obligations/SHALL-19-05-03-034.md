## Extracted Text

> The following variables shall affect the execution of the shell:: Set by the shell and by the cd utility. In the shell the value shall be initialized from the environment as follows. If a value for PWD is passed to the shell in the environment when it is executed, the value is an absolute pathname of the current working directory that is no longer than {PATH_MAX} bytes including the terminating null byte, and the value does not contain any components that are dot or dot-dot, then the shell shall set PWD to the value from the environment. Otherwise, if a value for PWD is passed to the shell in the environment when it is executed, the value is an absolute pathname of the current working directory, and the value does not contain any components that are dot or dot-dot, then it is unspecified whether the shell sets PWD to the value from the environment or sets PWD to the pathname that would be output by pwd -P. Otherwise, the sh utility sets PWD to the pathname that would be output by pwd -P. In cases where PWD is set to the value from the environment, the value can contain components that refer to files of type symbolic link. In cases where PWD is set to the pathname that would be output by pwd -P, if there is insufficient permission on the current working directory, or on any parent of that directory, to determine what that pathname would be, the value of PWD is unspecified. Assignments to this variable may be ignored. If an application sets or unsets the value of PWD , the behaviors of the cd and pwd utilities are unspecified.

## Context

This defines the full semantics of the `PWD` variable (present working directory).

**Required behavior — initialization:**
The shell must initialize `PWD` from the environment using a three-tier decision:
1. **Best case:** If `PWD` from the environment is an absolute pathname of the current working directory, is ≤ `{PATH_MAX}` bytes (including null terminator), and contains no `.` or `..` components → use it as-is.
2. **Second case:** If `PWD` from the environment is an absolute pathname of the current working directory and contains no `.` or `..` components (but may exceed PATH_MAX) → it is unspecified whether the shell uses this value or falls back to `pwd -P` output.
3. **Fallback:** Otherwise, set `PWD` to the pathname that would be output by `pwd -P` (physical path, no symlinks).

**Additional rules:**
- When `PWD` is taken from the environment, it may contain symlink components.
- When `PWD` is set via `pwd -P` fallback, if insufficient permissions prevent determining the physical path, the value is unspecified.
- User assignments to `PWD` may be ignored by the shell.
- If an application sets or unsets `PWD`, the behavior of `cd` and `pwd` builtins becomes unspecified.

**Conditions:** `PWD` is also updated by the `cd` builtin. The shell must maintain `PWD` consistency with the actual working directory.

See 2.5.3 "Shell Variables" (`PWD`), `utilities/cd.html`, `utilities/pwd.html`, and `utilities/sh.html`.
