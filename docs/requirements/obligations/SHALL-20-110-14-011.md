## Extracted Text

> An unrecoverable read error was detected while reading commands, except from the file operand of the dot special built-in.

## Context

**Behavior:** The `sh` utility shall return exit status 128 when an unrecoverable read error is detected while reading commands from the shell's primary input (command_file or stdin).

**Exception:** Read errors from the file operand of the `dot` (`.`) special built-in are excluded. If a read error occurs while sourcing a file via `.`, it does not trigger exit status 128 for the shell — instead, it is handled as a dot-builtin error (non-zero exit status per SHALL-19-19-14-001, or abort in non-interactive mode).

**Applies to:** The `sh` utility itself, not sourced files.

**Details:** An "unrecoverable read error" means the shell's input stream has encountered a fatal I/O error that prevents further reading. This is distinct from:
- EOF (normal end of input, not an error)
- Syntax errors (handled by 1-125 range)
- Temporary errors that could be retried

**Cross-reference:** See `utilities/dot.html` for the dot built-in specification.

**Spec reference:** Section "sh" EXIT STATUS, lines 692-693 in `utilities/sh.html`.
