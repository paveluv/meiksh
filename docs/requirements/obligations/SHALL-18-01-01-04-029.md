## Extracted Text

> When a file is to be read or written, the file shall be opened with an access mode corresponding to the operation to be performed. If file access permissions deny access, the requested operation shall fail.

## Context

When a file is to be read or written, the file shall be opened with an access mode corresponding to the operation to be performed. If file access permissions deny access, the requested operation shall fail. For `meiksh`, this means: `< file` must open for reading (`O_RDONLY`), `> file` must open for writing (`O_WRONLY`), and `<> file` must open for read-write (`O_RDWR`). If the user lacks the appropriate permission, the `open()` call fails with `EACCES` and the shell must report the error and not execute the associated command. See POSIX Utilities Chapter 1, Section 1.1.1.4 (utilities/V3_chap01.html line 691).
