## Extracted Text

> Non-Interactive Shell: Expansion error -- shall exit

## Context

A non-interactive shell SHALL exit when an expansion error occurs. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required:**
- When the shell is non-interactive and an expansion error occurs during word expansion (see 2.6 Word Expansions), the shell SHALL write a diagnostic to stderr and exit.
- A shell diagnostic message IS required.

**What constitutes an expansion error:** An error during the shell expansions defined in 2.6, such as:
- Invalid parameter expansion operators (e.g., `${x!y}` — `!` is not valid)
- `${var?message}` when `var` is unset or null
- Arithmetic expansion with invalid expressions
- See also MAY-19-08-01-002 which allows treating expansion errors as syntax errors if detectable during tokenization.

**Exit status:** Non-zero.
