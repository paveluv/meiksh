## Extracted Text

> The shell shall format the output, including the proper use of quoting, so that it is suitable for reinput to the shell as commands that achieve the same value and readonly attribute-setting results in a shell execution environment in which:: Variables with values at the time they were output do not have the readonly attribute set.

## Context

**Behavior:** Condition for `readonly -p` reinput guarantee: variables with values at the time they were output do **not** have the readonly attribute set in the target environment. This means the reinput commands can successfully assign values before setting readonly, as the variables aren't yet locked.

**Applies to:** All shell modes.
