## Extracted Text

> The if compound-list shall be executed; if its exit status is zero, the then compound-list shall be executed and the command shall complete. Otherwise, each elif compound-list shall be executed, in turn, and if its exit status is zero, the then compound-list shall be executed and the command shall complete. Otherwise, the else compound-list shall be executed.

## Context

**Behavior:** Execution order of `if`/`elif`/`else`:

1. Execute the `if` compound-list. If its exit status is **zero**, execute the corresponding `then` compound-list and the `if` command completes.
2. Otherwise, execute each `elif` compound-list in order. If any `elif` condition has a **zero** exit status, execute its `then` compound-list and the `if` command completes.
3. If no `if` or `elif` condition succeeded and an `else` clause is present, execute the `else` compound-list.

**Exit status:** The exit status of the `if` command is the exit status of the `then` or `else` compound-list that was executed. If no condition was true and there is no `else` clause, the exit status is zero.

**Interaction with `-e`:** The compound-lists following `if` and `elif` (condition parts) are executed with `-e` ignored — a non-zero exit status in a condition does not trigger `set -e` exit.

**Applies to:** All shell modes.
