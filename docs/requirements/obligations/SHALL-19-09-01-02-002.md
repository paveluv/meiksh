## Extracted Text

> Variable assignments shall be performed as follows:: If no command name results, variable assignments shall affect the current execution environment.

## Context

**Behavior:** When a simple command consists of only variable assignments (no command name after expansion), the assignments MUST affect the current execution environment. They become persistent shell variables. Example: `VAR=hello` with no command makes `VAR` available in subsequent commands.

**Applies to:** Simple commands with only variable assignments and optional redirections, where step 2 of 2.9.1.1 produces no command name.

**Implementation detail:** Set the variables in the current shell's variable table (not in a temporary/exported-only scope). These assignments persist for the remainder of the shell session or until overwritten/unset.
