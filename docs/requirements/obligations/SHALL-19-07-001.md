## Context

This obligation defines the syntax for the file descriptor number `n` in redirection expressions. See 2.7 Redirection in utilities/V3_chap02.html.

**Behavior required:**
- The optional file descriptor number `n` is one or more decimal digits.
- It must immediately precede the redirection operator (`<`, `>`, `>>`, `<<`, `<&`, `>&`, `<>`, `>|`, `<<-`) with no intervening blank characters.
- It must be delimited from any preceding text (i.e., separated from previous arguments/words).
- If `n` is quoted (e.g., `\2>file`), the number is NOT recognized as part of the redirection expression. In that case, `\2` becomes a literal argument and `>file` is parsed as a separate redirection (defaulting to fd 1 for output).

**Parser implications:** The tokenizer must detect an unquoted digit sequence immediately before a redirection operator token and treat it as the fd number rather than as part of a word. If any character in `n` is quoted, the entire number must be treated as literal text, not as a fd designator.

**Example:** `echo \2>a` writes the character `2` into file `a` (the `\` quotes the `2`, so it's not an fd number; `>a` redirects stdout).
