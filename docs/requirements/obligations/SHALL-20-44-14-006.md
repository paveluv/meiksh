## Extracted Text

> Otherwise, the exit status shall be that of the commands executed by fc.

## Context

When `fc` operates in edit-and-re-execute mode (not `-l`), and the operation succeeds (the editor exits 0, or `-s` is used), the exit status of `fc` must be the exit status of the re-executed command, NOT the exit status of `fc` itself or the editor. This means the shell must capture the exit status of the command that was re-executed from the history and use it as the return value of the `fc` invocation. For example, if `fc -s` re-executes `false`, `fc`'s exit status is 1. This applies to both `-s` (quick re-execute) and editor-based re-execution modes. Reference: POSIX.1-2024 §fc, EXIT STATUS (tag_20_44_14).
