## Extracted Text

> The following variables shall affect the execution of the shell:: [UP] The processing of the ENV shell variable shall be supported if the system supports the User Portability Utilities option. This variable, when and only when an interactive shell is invoked, shall be subjected to parameter expansion (see 2.6.2 Parameter Expansion) by the shell and the resulting value shall be used as a pathname of a file. Before any interactive commands are read, the shell shall tokenize (see 2.3 Token Recognition) the contents of the file, parse the tokens as a program (see 2.10 Shell Grammar), and execute the resulting commands in the current environment. (In other words, the contents of the ENV file are not parsed as a single compound_list. This distinction matters because it influences when aliases take effect.) The file need not be executable. If the expanded value of ENV is not an absolute pathname, the results are unspecified. ENV shall be ignored if the user's real and effective user IDs or real and effective group IDs are different.

## Context

This defines the `ENV` variable's behavior for interactive shell startup. Tagged `[UP]` (User Portability Utilities option).

**Required behavior:**
- Processing of `ENV` must be supported (meiksh targets UP compliance).
- `ENV` processing occurs **only when an interactive shell is invoked** — not for non-interactive shells (scripts, `-c` commands).
- When an interactive shell starts:
  1. The value of `ENV` is subjected to parameter expansion (2.6.2).
  2. The resulting string is used as a pathname of a file to read.
  3. Before any interactive commands are read, the shell tokenizes the file contents per 2.3 "Token Recognition", parses the tokens as a complete program per 2.10 "Shell Grammar", and executes the resulting commands in the current environment.
  4. The file is parsed as a program (multiple commands), NOT as a single `compound_list`. This distinction matters for alias processing timing.
  5. The file need not be executable (no `+x` permission required).
- If the expanded value of `ENV` is not an absolute pathname, the results are unspecified (implementation may choose to skip, search, or error).
- **Security constraint:** `ENV` must be ignored if the real and effective user IDs differ OR the real and effective group IDs differ (setuid/setgid execution).

**Conditions:** Interactive shell startup only. Must check uid/gid match before processing.

See 2.5.3 "Shell Variables" (`ENV`), 2.6.2 "Parameter Expansion", 2.3 "Token Recognition", and 2.10 "Shell Grammar" in `utilities/V3_chap02.html`.
