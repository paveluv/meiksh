## Extracted Text

> If an AND-OR list is terminated by the control operator <ampersand> ('&'), the shell shall execute the AND-OR list asynchronously in a subshell environment. This subshell shall execute in the background; that is, the shell shall not wait for the subshell to terminate before executing the next command (if any); if there are no further commands to execute, the shell shall not wait for the subshell to terminate before exiting.

## Context

**Behavior:** When an AND-OR list is terminated by `&`, the shell MUST execute it asynchronously in a subshell environment. This subshell runs in the background: the shell MUST NOT wait for the subshell to terminate before executing the next command. If there are no further commands, the shell MUST NOT wait before exiting.

**Applies to:** All `cmd &` constructs. The entire AND-OR list (not just the last pipeline) runs in the subshell.

**Implementation detail:** Fork a child process. The child executes the AND-OR list. The parent continues immediately to the next command (or exits). The child's PID becomes the value of `$!`. The parent must record this PID for later `wait` calls and job control.

**Spec reference:** Section 2.9.3.1, line 2138 in `utilities/V3_chap02.html`.
