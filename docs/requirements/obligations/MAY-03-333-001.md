## Extracted Text

> A program that interprets sequences of text input as commands. It may operate on an input stream or it may interactively prompt and read commands from a terminal.

## Context

**Definition — direct shell relevance.** Defines "shell" as a program that interprets text input as commands. It may operate on an input stream or interactively prompt and read commands from a terminal.

**Shell impact:** This is the core identity definition of `meiksh`. The shell must support two fundamental modes:
1. **Non-interactive (stream):** Read and execute commands from a file, pipe, or `-c` string argument
2. **Interactive:** Prompt the user, read commands from the terminal, and execute them

The distinction between these modes affects numerous behaviors: PS1/PS2 prompting, job control availability, signal handling defaults, error handling (interactive shells continue after errors; non-interactive shells may exit), and startup file processing.
