## Extracted Text

> Delete the character previous to the current cursor position and move the current cursor position back one character. In insert mode, characters shall be erased from both the screen and the buffer when backspacing.

## Context

This obligation defines the behavior of the **terminal `erase` character in vi insert mode** for the interactive command line editor.

**Behavior required:** When the user types the terminal's `erase` character (typically Backspace, as configured by `stty`) while in vi insert mode:
1. The character immediately preceding the current cursor position shall be deleted from the command line buffer.
2. The cursor position shall move back by one position (to the left).
3. The deletion shall be reflected on **both** the screen (terminal display) and the internal editing buffer simultaneously — i.e., the character must visually disappear from the displayed line, not merely be logically removed.

**Edge cases to consider:**
- If the cursor is already at the beginning of the command line, this sentence does not specify additional behavior beyond the stated deletion rule.
- The `erase` character is the one set by `stty erase` (see `utilities/sh.html` line 325–326).

**Conditions:** Only applies when the shell is interactive, vi editing mode is enabled (`set -o vi`), and the shell is in insert mode.

**Display requirement:** The phrase "erased from both the screen and the buffer" is significant — it distinguishes insert mode backspace from potential command-mode behavior where characters might be logically removed but the display update differs. In insert mode, the terminal display must be immediately updated to reflect the deletion.
