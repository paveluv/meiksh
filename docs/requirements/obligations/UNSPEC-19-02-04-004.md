## Extracted Text

> If the octal value specified by \ddd will not fit in a byte, the results are unspecified.

## Context

**Behavior:** If the octal value specified by `\ddd` inside `$'...'` does not fit in a byte (i.e., the value exceeds 255 / 0xFF, which means octal values >= `\400`), the results are unspecified.

**Applies to:** All shell modes during `$'...'` escape processing.

**Implementation choice:** Since up to 3 octal digits are consumed, the maximum possible value is `\777` = 511. Values from `\400` (256) through `\777` (511) exceed a byte. The implementation may: (a) truncate to the low 8 bits, (b) report an error, (c) use the full value as a multi-byte sequence, or (d) any other behavior. The choice should be documented.

**Note:** With the termination rule consuming at most 3 octal digits, values `\400`-`\777` are the only overflow range.

**Spec reference:** Section 2.2.4, line 230 in `utilities/V3_chap02.html`.
