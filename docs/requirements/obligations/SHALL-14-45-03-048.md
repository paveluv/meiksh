## Extracted Text

> The <signal.h> header shall also define the following symbolic constants:: SA_NOCLDWAIT

## Context

The `<signal.h>` header shall define the symbolic constant `SA_NOCLDWAIT`. This flag is used in the `sa_flags` member of `struct sigaction` when installing a handler for `SIGCHLD`. When set, the system does not create zombie processes on child termination and does not preserve status information for terminated children. For `meiksh`, this flag must generally NOT be set because the shell needs to call `waitpid()` to retrieve child exit status for job control and `$?`. Setting `SA_NOCLDWAIT` would cause `waitpid()` to fail or return unexpected results. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html line 535).
