## Extracted Text

> The pathname resolution algorithm, described by XBD 4.16 Pathname Resolution, shall be used by implementations conforming to this volume of POSIX.1-2024; see also XBD 4.8 File Hierarchy.

## Context

The shell implementation **shall** use the POSIX pathname resolution algorithm (XBD 4.16) when resolving any pathname. This means that all pathname operations — command lookup via `PATH`, `cd`, `.` (dot), redirection targets, file tests in `test`/`[`, etc. — must follow the rules defined in XBD 4.16 (Pathname Resolution) and respect the file hierarchy described in XBD 4.8 (File Hierarchy). Specifically: pathnames beginning with `/` are absolute; relative pathnames are resolved from the current working directory; symbolic links are followed during resolution (except where `-P` options override); components are resolved left-to-right; and `..` refers to the parent directory. Reference: XCU 1.1.1.8, XBD 4.16, XBD 4.8.
