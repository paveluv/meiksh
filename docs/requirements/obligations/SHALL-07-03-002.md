## Extracted Text

> Locales can be described with the file format presented in this section. The file format is that accepted by the localedef utility. For the purposes of this section, the file is referred to as the "locale definition file", but no locales shall be affected by this file unless it is processed by localedef or some similar mechanism. Any requirements in this section imposed upon the utility shall apply to localedef or to any other similar utility used to install locale information using the locale definition file format described here.

## Context

**Behavior:** Locale definition files describe locales in the format accepted by `localedef`. Locale definitions have no effect until processed by `localedef` or a similar utility. Requirements on "the utility" in this section apply to `localedef`.

**Shell relevance:** The shell does not process locale definition files. This is purely about the `localedef` utility's input format. The shell consumes the compiled locales that `localedef` produces. The refs to `localedef.html` are for an external utility — not shell-relevant for extraction.
