## Extracted Text

> The behavior of an unescaped <backslash> immediately followed by any other character, including <newline>, is unspecified.

## Context

**Behavior:** Inside `$'...'`, a backslash immediately followed by any character that is not a recognized escape sequence character (including a literal newline) has unspecified behavior. This means `\` followed by characters like `d`, `g`, `w`, newline, space, etc., may be handled in any implementation-defined way.

**Applies to:** All shell modes during `$'...'` escape processing.

**Implementation choice:** The implementation is free to choose any behavior. Options include: treating `\X` as literal `\X`, stripping the backslash, producing an error, etc. For `\<newline>` specifically, the implementation could either preserve both characters, treat it as a line continuation (remove both), or produce an error. The choice should be documented.

**Spec reference:** Section 2.2.4, lines 216-217 in `utilities/V3_chap02.html`.
