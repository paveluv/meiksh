## Extracted Text

> The following "special built-in" utilities shall be supported in the shell command language. The output of each command, if any, shall be written to standard output, subject to the normal redirection and piping possible with all commands.

## Context

**Special built-in utilities must be supported; output goes to stdout.**

The shell **shall** support all of the following special built-in utilities as part of the shell command language: `break`, `colon` (`:`), `continue`, `dot` (`.`), `eval`, `exec`, `exit`, `export`, `readonly`, `return`, `set`, `shift`, `times`, `trap`, `unset`.

The output of each command, if any, **shall** be written to standard output. This output is subject to normal redirection and piping — special built-ins can have their output redirected just like any other command (e.g., `export -p > vars.txt`).

These utilities are "special" because they have different error handling and variable assignment semantics compared to regular built-ins (see SHALL-19-15-002 through SHALL-19-15-005).

See 2.15 Special Built-In Utilities in utilities/V3_chap02.html.
