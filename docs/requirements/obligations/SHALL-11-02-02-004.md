## Extracted Text

> If IGNPAR is set, a byte with a framing or parity error (other than break) shall be ignored.

## Context

**Behavior:** When IGNPAR is set in `c_iflag`, bytes received with framing or parity errors (excluding break conditions) shall be silently **ignored** — they are not placed in the input queue and are not readable by any process.

**Conditions:** Applies to terminal input on serial connections where parity/framing errors can occur. IGNPAR is in `c_iflag`. Break conditions are handled separately by IGNBRK/BRKINT (see SHALL-11-02-02-003).

**Shell relevance:** Primarily relevant for serial terminal connections. The `stty` builtin must support the `ignpar`/`-ignpar` flag. For modern pseudo-terminal usage, parity errors are not typically encountered, but the shell must correctly expose this flag via `stty`.

**Spec reference:** POSIX.1-2024, §11.2.2 Input Modes (tag_11_02_02), `basedefs/V1_chap11.html` line 510.
