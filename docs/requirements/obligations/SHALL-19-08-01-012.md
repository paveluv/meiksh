## Extracted Text

> Non-Interactive Shell: Redirection error with function execution -- shall not exit

## Context

A non-interactive shell SHALL NOT exit when a redirection error occurs during function execution. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required:**
- When the shell is non-interactive and a redirection attached to a function invocation fails (e.g., `myfunc >readonly_file`), the shell SHALL write a diagnostic to stderr but SHALL NOT exit.
- A shell diagnostic message IS required.
- The function is not executed, but the script continues.

**Note:** This covers redirections on the function call, not redirections inside the function body (those are handled per the command type within the function).
