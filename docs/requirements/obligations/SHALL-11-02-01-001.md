## Extracted Text

> Routines that need to control certain terminal I/O characteristics shall do so by using the termios structure as defined in the <termios.h> header.

## Context

**Behavior:** Any code that needs to control terminal I/O characteristics **shall** use the `struct termios` structure as defined in `<termios.h>`. This is the only portable way to manipulate terminal settings.

**Conditions:** Applies to all terminal I/O control operations.

**Shell relevance:** meiksh must use the termios API (`tcgetattr()`, `tcsetattr()`, and the `struct termios` structure) for all terminal mode manipulation. This includes:
- Switching between canonical and non-canonical mode for line editing
- Configuring echo, signals (ISIG), flow control
- Setting special character values (VMIN, VTIME, etc.)
- The `stty` builtin implementation

The shell must `#include <termios.h>` and use the standard structure and functions. Do not use legacy `ioctl()` interfaces for terminal control where termios functions are available.

**Spec reference:** POSIX.1-2024, §11.2.1 The termios Structure (tag_11_02_01), `basedefs/V1_chap11.html` lines 290–291.
