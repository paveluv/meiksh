## Extracted Text

> If the O_NONBLOCK flag is clear, then the read request shall be blocked until data is available or a signal has been received. If the O_NONBLOCK flag is set, then the read request shall be completed, without blocking, in one of three ways:: If there is no data available, the read() shall return -1, with errno set to [EAGAIN].

## Context

**Behavior:** Non-blocking terminal read, case 3: If O_NONBLOCK is set and no data is available, read() shall return -1 with errno=EAGAIN.

**Conditions:** O_NONBLOCK set, input queue is empty.

**Error handling:** Return -1, set errno to EAGAIN. The shell should not treat EAGAIN as a fatal error — it means "try again later."

**Shell relevance:** When using non-blocking reads for line editing or input polling, the shell must handle EAGAIN gracefully by retrying.
