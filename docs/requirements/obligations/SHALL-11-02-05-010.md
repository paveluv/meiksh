## Extracted Text

> If NOFLSH is set, the normal flush of the input and output queues associated with the INTR, QUIT, and SUSP characters shall not be done.

## Context

**Behavior:** When NOFLSH is set in `c_lflag`:
1. The normal flush of input and output queues that occurs when INTR, QUIT, or SUSP characters are processed shall **not** be done.
2. Without NOFLSH (the default), receiving INTR/QUIT/SUSP causes both input and output queues to be flushed before the signal is delivered.
3. With NOFLSH, the signal is still generated, but queued input/output data is preserved.

**Conditions:** NOFLSH is a flag in `c_lflag`. It only affects queue flushing behavior when ISIG is also set (since INTR, QUIT, SUSP are only processed when ISIG is set).

**Shell relevance:** The shell may want to set NOFLSH during line editing to preserve partially typed input when the user presses Ctrl-C. Without NOFLSH, a SIGINT flushes the input queue, losing any typeahead. With NOFLSH, the shell's signal handler can decide what to do with the remaining input. The `stty` builtin must support `noflsh`/`-noflsh`.

**Spec reference:** POSIX.1-2024, §11.2.5 Local Modes (tag_11_02_05), `basedefs/V1_chap11.html` line 1195.
