## Extracted Text

> If the locale value begins with a <slash>, it shall be interpreted as the pathname of a file that was created in the output format used by the localedef utility; see OUTPUT FILES under localedef. Referencing such a pathname shall result in that locale being used for the indicated category.

## Context

**Behavior:** If the locale value begins with a `/` (slash), it shall be interpreted as the pathname of a file in the `localedef` output format. Referencing such a pathname shall result in that locale being used for the indicated category.

**Shell relevance:** The shell must accept locale values that are absolute pathnames (starting with `/`). When the shell encounters such a value in `LC_ALL`, `LC_*`, or `LANG`, it must attempt to use the locale definition file at that path. In practice this is handled by the C library's `setlocale()`, but the shell must not reject or special-case slash-prefixed locale names.

**Spec reference:** Section 8.2 "Internationalization Variables", lines 619-621 in `basedefs/V1_chap08.html`.
