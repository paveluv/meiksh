## Context

When listing multi-line commands with `fc -l`, the first line follows the standard format (`"%d\t%s\n"` or `"\t%s\n"` with `-n`), but all subsequent lines of the same command must use the format `"\t%s\n", <continued-command>`. Each continuation line is indented with a tab character to visually align with the command text of the first line. This ensures multi-line commands (e.g., here-documents, compound commands) are displayed readably. Reference: POSIX.1-2024 §fc, STDOUT (tag_20_44_10).
