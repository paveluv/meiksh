## Extracted Text

> The byte position in the file where the next I/O operation begins. Each open file description associated with a regular file, block special file, or directory has a file offset. A character special file that does not refer to a terminal device may have a file offset. There is no file offset specified for a pipe or FIFO.

## Context

**Definition — low shell relevance.** Defines "file offset" as the byte position where the next I/O operation begins. Regular files, block special files, and directories have file offsets. Character special files (non-terminal) may have one. Pipes and FIFOs have no file offset.

**Shell impact:** File offsets matter for shell redirections with append mode (`>>`), which positions the offset at EOF before writing. The shell uses `lseek()` indirectly for `<>` (read-write) redirections. For pipes (used in pipelines and process substitution), there is no offset — reads/writes are sequential. No special implementation beyond correct use of `O_APPEND` and standard I/O operations is required.
