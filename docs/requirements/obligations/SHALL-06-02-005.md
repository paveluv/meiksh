## Extracted Text

> The maximum allowable number of bytes in a character in the current locale shall be indicated by {MB_CUR_MAX}, defined in the <stdlib.h> header and by the <mb_cur_max> value in a character set description file; see 6.4 Character Set Description File. The implementation's maximum number of bytes in a character shall be defined by the C-language macro {MB_LEN_MAX}.

## Context

The shell must be aware of `MB_CUR_MAX` (maximum bytes per character in the current locale) for correct multibyte character handling. This affects:

1. **Buffer sizing**: When reading characters from input (e.g., `read` builtin), the shell must allocate enough buffer space for the largest possible multibyte character.
2. **Character iteration**: When iterating over characters in a string (for `${#var}`, pattern matching, etc.), the shell must advance by up to `MB_CUR_MAX` bytes per character.
3. **In the POSIX locale**: `MB_CUR_MAX` is 1 (single-byte characters only).
4. **In UTF-8 locale**: `MB_CUR_MAX` is typically 4 (or 6 on some older systems).

The shell should query `MB_CUR_MAX` when initializing locale support and after locale changes.
