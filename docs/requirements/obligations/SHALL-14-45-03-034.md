## Extracted Text

> The <signal.h> header shall define the following macros which shall expand to integer constant expressions that need not be usable in #if preprocessing directives:: [CX] The resulting set is the intersection of the current set and the complement of the signal set pointed to by the argument set.

## Context

This is the [CX] extension defining the semantics of `SIG_UNBLOCK`: when used as the `how` argument to `sigprocmask()`, the resulting signal mask is the intersection of the current set and the complement of the signal set pointed to by `set`. In other words, `sigprocmask(SIG_UNBLOCK, &set, NULL)` removes the signals in `set` from the currently blocked signals. For `meiksh`, this is used after critical sections to re-enable delivery of signals like `SIGCHLD` or `SIGINT`, and when preparing child processes for execution. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 505–507).
