## Extracted Text

> The values of locale categories shall be determined by a precedence order; the first condition met below determines the value:: Otherwise, the implementation-defined default locale shall be used.

## Context

**Behavior:** Locale precedence rule 4 (fallback): If `LC_ALL`, the specific `LC_*` variable, and `LANG` are all unset or null, the implementation-defined default locale shall be used.

**Shell relevance:** The shell must define a default locale to use when no locale environment variables are set. In practice, `meiksh` should default to the POSIX locale ("C") as its implementation-defined default. This ensures deterministic behavior for pattern matching, collation, and character classification when no locale variables are configured.
