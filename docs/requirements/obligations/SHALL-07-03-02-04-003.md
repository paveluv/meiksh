## Extracted Text

> A <collating-element> shall be used to specify multi-character collating elements, and indicates that the character sequence specified via the <collating-element> is to be collated as a unit and in the relative order specified by its place.

## Context

A collating-element in the LC_COLLATE collation order section specifies a multi-character sequence (e.g., `<ch>` for Spanish "ch") that is treated as a single collation unit. The position of this element in the collation order determines where it sorts relative to other elements. The shell is affected because pattern matching bracket expressions can reference collating elements (e.g., `[[.ch.]]`), and the collation order determines range behavior. The shell must support collating element syntax in bracket expressions per POSIX pattern matching rules.

See: `basedefs/V1_chap07.html` §7.3.2.4 Collation Order.
