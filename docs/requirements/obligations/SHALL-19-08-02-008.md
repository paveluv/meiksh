## Extracted Text

> Otherwise, if the command terminated due to the receipt of a signal, the shell shall assign it an exit status greater than 128. The exit status shall identify, in an implementation-defined manner, which signal terminated the command. Note that shell implementations are permitted to assign an exit status greater than 255 if a command terminates due to a signal.

## Context

**Behavior:** When a command is terminated by a signal, the shell MUST assign an exit status > 128 that identifies the signal in an implementation-defined manner. Values > 255 are explicitly permitted. This is a standalone restatement of SHALL-19-08-02-004.

**Applies to:** Any child process terminated by signal (WIFSIGNALED from waitpid). Applies in all modes.

**Implementation detail (IMPLDEF):** Same as SHALL-19-08-02-004. meiksh should use the conventional `128 + signum` encoding and document it.

**Spec reference:** Section 2.8.2, line 1703 in `utilities/V3_chap02.html`.
