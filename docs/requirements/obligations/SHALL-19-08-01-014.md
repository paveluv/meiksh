## Extracted Text

> Non-Interactive Shell: Redirection error with other utilities (not special built-ins) -- shall not exit

## Context

A non-interactive shell SHALL NOT exit when a redirection error occurs with a non-special-built-in utility. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required:**
- When the shell is non-interactive and a redirection attached to a regular built-in or external command fails (e.g., `ls >readonly_file`), the shell SHALL write a diagnostic to stderr but SHALL NOT exit.
- A shell diagnostic message IS required.
- The command is not executed, but the script continues.

**Contrast:** Only redirection errors with special built-ins cause a non-interactive shell to exit (SHALL-19-08-01-008).
