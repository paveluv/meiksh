## Extracted Text

> The following operand shall be supported:: Specify the job to be run as a foreground job. If no job_id operand is given, the job_id for the job that was most recently suspended, placed in the background, or run as a background job shall be used. The format of job_id is described in XBD 3.182 Job ID.

## Context

This is the full `job_id` operand description for `fg`. The implementation must: (1) Accept a `job_id` operand in the format defined by XBD 3.182 Job ID (e.g., `%1`, `%+`, `%-`, `%string`, `%?string`). (2) If no `job_id` is given, default to the "current job" — the job that was most recently suspended (via SIGTSTP/SIGSTOP), placed in the background (via `bg`), or started as a background job (via `&`). (3) The selected job must be moved from the background to the foreground. If the job_id doesn't match any known job, `fg` must report an error and exit with status >0. Reference: POSIX.1-2024 §fg, OPERANDS (tag_20_45_05); XBD 3.182.
