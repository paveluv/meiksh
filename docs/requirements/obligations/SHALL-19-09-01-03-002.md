## Extracted Text

> Additionally, if there is no command name but the command contains a command substitution, the command shall complete with the exit status of the command substitution whose exit status was the last to be obtained. Otherwise, the command shall complete with a zero exit status.

## Context

**Behavior:** For commands with no command name: if the command contains a command substitution, the exit status MUST be the exit status of the last command substitution whose exit status was obtained. If the command contains no command substitution, it MUST complete with exit status 0.

**Applies to:** Commands like `VAR=$(cmd1)$(cmd2)` (no command name, multiple command substitutions → exit status of cmd2) or `VAR=literal` (no command substitution → exit status 0).

**Implementation detail:** Track the exit status of each command substitution as it is expanded. After all expansions complete, `$?` should reflect the last command substitution's exit status, or 0 if none were present.

**Spec reference:** Section 2.9.1.3, line 1808 in `utilities/V3_chap02.html`.
