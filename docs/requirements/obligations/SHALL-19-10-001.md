## Extracted Text

> The following grammar defines the Shell Command Language. This formal syntax shall take precedence over the preceding text syntax description.

## Context

The formal YACC-style grammar in Section 2.10 is the **authoritative definition** of the Shell Command Language syntax. Where the prose descriptions in earlier sections (2.1–2.9) conflict with or are ambiguous relative to the formal grammar, the formal grammar takes precedence. The shell's parser must implement this grammar exactly.

The grammar defines productions for: complete commands, lists, AND-OR lists, pipelines, commands (simple, compound, function definitions), compound commands (brace groups, subshells, for/case/if/while/until loops), I/O redirections, here-documents, and the linebreak/separator rules.

The implementation's parser (whether hand-written recursive descent or table-driven) must accept exactly the language defined by this grammar. Any deviation from the grammar constitutes a conformance violation. See the full grammar in Section 2.10 of `utilities/V3_chap02.html`.
