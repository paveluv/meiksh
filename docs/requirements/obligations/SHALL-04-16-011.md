## Extracted Text

> The special filename dot shall refer to the directory specified by its predecessor. The special filename dot-dot shall refer to the parent directory of its predecessor directory. As a special case, in the root directory, dot-dot may refer to the root directory itself.

## Context

The special directory entries `.` (dot) and `..` (dot-dot) are fundamental to shell pathname handling:

- **`.` (dot)**: Refers to the current directory. Used in paths like `./script.sh` to execute from the current directory. Also used by the `.` builtin command (though that's a different construct).
- **`..` (dot-dot)**: Refers to the parent directory. Used in `cd ..`, relative paths, and the shell's logical `cd -L` path canonicalization (where the shell must handle `..` components logically by removing the preceding component from `PWD`).
- **Root directory special case**: `..` at the root directory resolves to the root directory itself, so `cd /..` stays at `/`.

The shell's `cd -L` implementation must handle `..` specially: when tracking the logical path, encountering `..` means removing the last component from the logical `PWD` string, not relying on the file system's `..` entry (which could differ if symlinks are involved).

See: `basedefs/V1_chap04.html` §4.16 Pathname Resolution.
