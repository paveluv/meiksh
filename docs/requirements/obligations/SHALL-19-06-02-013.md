## Extracted Text

> Remove Smallest Suffix Pattern. The word shall be expanded to produce a pattern. The parameter expansion shall then result in parameter, with the smallest portion of the suffix matched by the pattern deleted. If present, word shall not begin with an unquoted '%'.

## Context

**Behavior:** The `${parameter%word}` expansion (Remove Smallest Suffix Pattern):
1. Expand `word` to produce a pattern (using pattern matching notation per 2.14).
2. Match the pattern against the **suffix** (end) of the value of `parameter`.
3. Find the **smallest** (shortest) matching suffix.
4. Return the value of `parameter` with that smallest suffix removed.

**Constraint:** If `word` is present, it must not begin with an unquoted `%`. This avoids ambiguity with the `%%` (largest suffix) form. A quoted `%` at the start is permitted.

**Inputs/Outputs:**
- `x="/usr/local/bin"; echo ${x%/*}` → `/usr/local` (removes shortest suffix matching `/*`)
- `x="file.tar.gz"; echo ${x%.*}` → `file.tar` (removes `.gz`, the shortest `.`-suffix)
- `x="hello"; echo ${x%}` → `hello` (empty pattern matches nothing)

**Spec reference:** See 2.6.2 Parameter Expansion in utilities/V3_chap02.html, lines 975–978.
