## Extracted Text

> If the redirection operator is "<<-", all leading <tab> characters shall be stripped from input lines after <backslash><newline> line continuation (when it applies) has been performed, and from the line containing the trailing delimiter. Stripping of leading <tab> characters shall occur as the here-document is read from the shell input (and consequently does not affect any <tab> characters that result from expansions).

## Context

Defines the tab-stripping behavior of the `<<-` (here-document with tab stripping) operator. See 2.7.4 Here-Document in utilities/V3_chap02.html.

**Behavior required:**
- When the redirection operator is `<<-` (as opposed to `<<`), ALL leading tab characters (`\t`, ASCII 0x09) SHALL be stripped from:
  1. Each input line of the here-document body
  2. The line containing the trailing delimiter
- Tab stripping occurs AFTER `\<newline>` line continuation has been performed (when applicable, i.e., for unquoted delimiters).
- Tab stripping happens as the here-document is read from shell input, which means it does NOT affect tab characters that result from expansions (parameter expansion, command substitution, etc.).

**Key distinctions:**
- Only tab characters are stripped, NOT spaces or other whitespace.
- Only LEADING tabs are stripped (tabs at the beginning of the line, before any non-tab character).
- The delimiter line is also subject to tab stripping, so `<<-EOF` with a body where the `EOF` line is indented with tabs will still match.

**Use case:** Allows here-documents to be indented in shell scripts for readability without the indentation appearing in the output.
