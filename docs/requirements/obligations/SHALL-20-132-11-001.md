## Extracted Text

> The standard error shall be used only for diagnostic messages.

## Context

**Behavior:** The `umask` utility shall write to stderr only for diagnostic messages (errors, warnings). No regular output shall be sent to stderr.

**Conditions:** Diagnostic messages include errors for invalid mask operands, unrecognized options, or other error conditions. Informational output (the current mask value) goes to stdout.

**Spec reference:** Section STDERR (tag_20_132_11) in `utilities/umask.html`, line 134.

GPT-5.4 review: The extracted quote matches `docs/posix/utilities/umask.html#tag_20_132_11`. The enrichment is correct. The neighboring stdout rule in `docs/posix/utilities/umask.html#tag_20_132_10` is also useful context for review: `umask` with no operand writes the current mask to stdout, while `umask` with a `mask` operand writes nothing to stdout. That makes it especially important that stderr remain empty on successful output and successful state-change paths.
