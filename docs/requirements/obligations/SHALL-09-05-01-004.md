## Extracted Text

> Represents a numeric constant. It shall be an integer in the range 0 <=DUP_COUNT <={RE_DUP_MAX}. This token is only recognized when the context of the grammar requires it. At all other times, digits not preceded by a <backslash> character are treated as ORD_CHAR.

## Context

This is a duplicate extraction of the DUP_COUNT rule from SHALL-09-05-01-001. DUP_COUNT is a context-sensitive token: it is recognized as a numeric constant (0 to RE_DUP_MAX) only inside interval expressions. Outside that context, bare digits are ordinary characters. The lexer must switch between treating digit sequences as DUP_COUNT or ORD_CHAR based on the parser state.
