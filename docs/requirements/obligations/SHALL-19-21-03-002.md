## Extracted Text

> If exec is specified with a utility operand, the shell shall execute a non-built-in utility as described in 2.9.1.6 Non-built-in Utility Execution with utility as the command name and the argument operands (if any) as the command arguments.

## Context

**Behavior:** When `exec` is specified with a **utility operand** (a command name), the shell must execute that utility as a non-built-in command (per 2.9.1.6 Non-built-in Utility Execution), **replacing** the current shell process. The command name is `utility` and any additional arguments become the command's arguments. The shell process image is replaced by the new program — `exec` does not return on success.

**Implementation:** Use the `execve()` system call (or equivalent) to replace the shell process. Any redirections on the `exec` command are applied before the exec.

**Applies to:** All shell modes.

**Spec reference:** `exec` DESCRIPTION in `utilities/V3_chap02.html`, lines 3876-3878. Cross-ref: 2.9.1.6 Non-built-in Utility Execution.
