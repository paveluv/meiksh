## Extracted Text

> Use the editor named by editor to edit the commands. The editor string is a utility name, subject to search via the PATH variable (see XBD 8. Environment Variables). The value in the FCEDIT variable shall be used as a default when -e is not specified. If FCEDIT is null or unset, ed shall be used as the editor.

## Context

This obligation (duplicate extraction of the `-e` option description) specifies the editor resolution chain for `fc` edit mode. The implementation must: (1) If `-e editor` is given, look up `editor` via `PATH` search and invoke it. (2) If `-e` is not given, expand `FCEDIT` and use its value as the editor name (also resolved via `PATH`). (3) If `FCEDIT` is null (empty string) or unset, default to `ed`. The editor is invoked with a temporary file as its argument containing the selected command(s). Reference: POSIX.1-2024 §fc, OPTIONS (tag_20_44_04).
