## Extracted Text

> Interactive Shell: Special built-in utility error -- shall not exit

## Context

An interactive shell SHALL NOT exit when a special built-in utility error occurs. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required:**
- When the shell is interactive and a special built-in utility encounters an error, the shell SHALL NOT exit.
- The shell continues to the next prompt after the error.
- A shell diagnostic message is NOT required — the utility itself writes its diagnostic.

**Implication:** Interactive shells are resilient to special built-in errors. For example, `export -Z` (invalid option) in an interactive shell produces an error message from `export` but the shell continues.
