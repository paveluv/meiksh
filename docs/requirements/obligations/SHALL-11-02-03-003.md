## Extracted Text

> [XSI] If ONLCR is set, the NL character shall be transmitted as the CR-NL character pair. If OCRNL is set, the CR character shall be transmitted as the NL character. If ONOCR is set, no CR character shall be transmitted when at column 0 (first position). If ONLRET is set, the NL character is assumed to do the carriage-return function; the column pointer shall be set to 0 and the delays specified for CR shall be used. Otherwise, the NL character is assumed to do just the line-feed function; the column pointer remains unchanged. The column pointer shall also be set to 0 if the CR character is actually transmitted.

## Context

**Behavior (XSI):** When OPOST is set, the following NL/CR output translations apply:
1. If ONLCR is set, the NL character shall be transmitted as the CR-NL character pair.
2. If OCRNL is set, the CR character shall be transmitted as the NL character.
3. If ONOCR is set, no CR character shall be transmitted when the cursor is at column 0 (first position).
4. If ONLRET is set, the NL character is assumed to do the carriage-return function: the column pointer shall be set to 0 and the delays specified for CR shall be used. Otherwise, NL only does line-feed and the column pointer remains unchanged.
5. The column pointer shall also be set to 0 if the CR character is actually transmitted.

**Conditions:** These flags are XSI extensions. They only take effect when OPOST is also set.

**Relevance to meiksh:** The `stty` builtin must support setting/clearing ONLCR, OCRNL, ONOCR, and ONLRET. The column pointer tracking is internal to the terminal driver.
