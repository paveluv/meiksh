## Extracted Text

> Specify the name of an existing locale which shall be used as the definition of this category. If this keyword is specified, no other keyword shall be specified. Note: This is a localedef utility keyword, unavailable through localeconv().

## Context

This is the `copy` keyword description for LC_NUMERIC. When `copy` is used, the specified locale's numeric category is inherited verbatim, and no other LC_NUMERIC keywords may appear. This is a `localedef` utility mechanism. The shell does not parse locale definitions but must correctly propagate LC_NUMERIC so the system resolves the copied definition. The shell's `printf` builtin depends on the resolved numeric locale data (decimal point, grouping).

See: `basedefs/V1_chap07.html` §7.3.4 LC_NUMERIC.
