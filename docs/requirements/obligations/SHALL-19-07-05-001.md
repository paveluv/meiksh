## Extracted Text

> shall duplicate one input file descriptor from another, or shall close one. If word evaluates to one or more digits, the file descriptor denoted by n, or standard input if n is not specified, shall be made to be a copy of the file descriptor denoted by word; if the digits in word do not represent an already open file descriptor, a redirection error shall result (see 2.8.1 Consequences of Shell Errors); if the file descriptor denoted by word represents an open file descriptor that is not open for input, a redirection error may result. If word evaluates to '-', file descriptor n, or standard input if n is not specified, shall be closed. Attempts to close a file descriptor that is not open shall not constitute an error. If word evaluates to something else, the behavior is unspecified.

## Context

Defines the behavior of the input file descriptor duplication operator `[n]<&word`. See 2.7.5 Duplicating an Input File Descriptor in utilities/V3_chap02.html.

**Syntax:** `[n]<&word`

**Behavior required — word evaluates to digits:**
- The fd denoted by `n` (or fd 0 if `n` is omitted) SHALL be made a copy of the fd denoted by `word` (using `dup2(word_fd, n_fd)` or equivalent).
- If `word` does not represent an already-open fd, a redirection error SHALL result (see 2.8.1 Consequences of Shell Errors, tag_19_08_01).
- If `word` represents an open fd that is not open for input (i.e., open for write-only), a redirection error MAY result.

**Behavior required — word evaluates to `-`:**
- The fd denoted by `n` (or fd 0 if `n` is omitted) SHALL be closed.
- Attempting to close an fd that is not open SHALL NOT constitute an error.

**Behavior — word evaluates to something else:**
- The behavior is unspecified. The implementation may treat it as an error.

**Error handling:** Redirection errors from `<&` follow the rules in 2.8.1 based on the command context (special built-in, compound command, etc.).
