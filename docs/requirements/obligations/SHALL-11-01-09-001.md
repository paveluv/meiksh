## Extracted Text

> Certain characters have special functions on input or output or both. These functions are summarized as follows:: Special character on input, which is recognized if the ISIG flag is set. Generates a SIGINT signal which is sent to all processes in the foreground process group for which the terminal is the controlling terminal. If ISIG is set, the INTR character shall be discarded when processed.

## Context

**Behavior:** The INTR special character (typically ^C):
1. Recognized on input only when ISIG is set in `c_lflag`.
2. When recognized, generates a SIGINT signal sent to **all** processes in the foreground process group for which this terminal is the controlling terminal.
3. The INTR character itself shall be **discarded** when processed (not placed in the input queue).

**Conditions:** ISIG must be set in `c_lflag`. The terminal must be the controlling terminal for the affected process group.

**Shell relevance:** This is the mechanism behind Ctrl-C interrupting foreground commands. The shell relies on the terminal driver generating SIGINT to the foreground process group. When the shell is in the foreground (e.g., at the prompt), it receives SIGINT and must handle it gracefully (e.g., discard current input, redisplay prompt). When a child process is in the foreground, the signal goes to the child. The shell's line editor typically keeps ISIG set so this mechanism works.

**Spec reference:** POSIX.1-2024, §11.1.9 Special Characters (tag_11_01_09), `basedefs/V1_chap11.html` line 220.
