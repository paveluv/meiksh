## Extracted Text

> The Basic Regular Expression (BRE) notation and construction rules in 9.3 Basic Regular Expressions shall apply to most utilities supporting regular expressions. Some utilities, instead, support the Extended Regular Expressions (ERE) described in 9.4 Extended Regular Expressions; any exceptions for both cases are noted in the descriptions of the specific utilities using regular expressions. Both BREs and EREs are supported by the Regular Expression Matching interface in the System Interfaces volume of POSIX.1-2024 under regcomp(), regexec(), and related functions.

## Context

**Behavior:** Basic Regular Expression (BRE) notation shall apply to most utilities supporting regular expressions. Some utilities support Extended Regular Expressions (ERE) instead; exceptions are noted per-utility. Both BRE and ERE are supported by `regcomp()`/`regexec()`.

**Shell relevance:** The shell itself does not directly use BRE or ERE syntax — shell pattern matching (section 2.14) uses a different notation (glob patterns with `?`, `*`, `[...]`). However, the shell's bracket expression syntax (section 9.3.5) is shared between REs and shell patterns. Built-in utilities or utilities commonly invoked by the shell (like `grep`, `sed`, `expr`) use BRE/ERE. The shell must correctly pass RE-containing arguments without misinterpreting special characters.
