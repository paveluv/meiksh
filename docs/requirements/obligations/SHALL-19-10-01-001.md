## Extracted Text

> The input language to the shell shall be first recognized at the character level. The resulting tokens shall be classified by their immediate context according to the following rules (applied in order). These rules shall be used to determine what a "token" is that is subject to parsing at the token level. The rules for token recognition in 2.3 Token Recognition shall apply.

## Context

The lexer operates in two phases: (1) character-level recognition that produces raw tokens, and (2) context-dependent classification of those tokens. This obligation establishes that the lexer must first break input into tokens at the character level, then classify each token based on its immediate syntactic context.

The token recognition rules from Section 2.3 "Token Recognition" (see `utilities/V3_chap02.html#tag_19_03`) define how characters are grouped into tokens. These rules must be applied in the specified order. The classification rules below (in the grammar's lexical conventions) then determine whether a token is an operator, IO_NUMBER, IO_LOCATION, or a generic TOKEN that will be further refined by grammar rules.

The implementation's lexer must: (a) consume input character by character following the Section 2.3 rules, (b) produce token boundaries, and (c) assign initial token types based on the classification rules that follow in this section.
