## Extracted Text

> [CX] The <signal.h> header shall define the mcontext_t type through typedef.

## Context

The `<signal.h>` header shall define the `mcontext_t` type through `typedef` ([CX] extension). This type holds a machine-specific representation of saved context (registers, program counter, etc.) and is embedded within `ucontext_t`. For `meiksh`, this type is not directly used by shell code — it is an OS-level type that underlies signal context handling. The shell relies on the platform header providing this type. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 556–557).
