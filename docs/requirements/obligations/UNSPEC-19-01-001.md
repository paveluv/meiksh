## Extracted Text

> The shell reads its input from a file (see sh), from the -c option or from the system() and popen() functions defined in the System Interfaces volume of POSIX.1-2024. If the first line of a file of shell commands starts with the characters "#!", the results are unspecified.

## Context

The shell receives its input from one of three sources: (1) a file named on the command line (e.g., `sh script.sh`), (2) the `-c` option string (e.g., `sh -c 'echo hello'`), or (3) the `system()` / `popen()` C library functions (which invoke the shell internally). The implementation must support all three input modes.

**Shebang (`#!`) behavior is explicitly unspecified.** If a file of shell commands begins with the two characters `#!` on its first line, POSIX defines the results as unspecified. The shell implementation is free to either ignore the `#!` line (treating it as a comment, since `#` starts a comment), or to take other action—but it must not rely on any particular shebang behavior for conformance. In practice, the kernel handles shebang interpretation before the shell sees the file, so the shell typically just sees `#!` as a comment.

The `system()` and `popen()` references are to POSIX C library functions (see `functions/system.html`, `functions/popen.html`); the shell itself does not implement these, but must be invocable by them, meaning the shell must be locatable via the standard `sh` path and must correctly process `-c` command strings.

See Section 2.1 "Shell Introduction" in `utilities/V3_chap02.html` and the `sh` utility page (`utilities/sh.html`) for the `-c` option and file operand details.
