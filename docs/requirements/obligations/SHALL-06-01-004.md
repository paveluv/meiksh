## Extracted Text

> POSIX.1-2024 places only the following requirements on the encoded values of the characters in the portable character set:: The encoded values associated with <period>, <slash>, <newline>, and <carriage-return> shall be invariant across all locales supported by the implementation.

## Context

Four characters must have the same encoded value in every locale:

1. **Period `.`**: Used in filenames, pathname expansion (hidden file convention), and the `.` builtin command.
2. **Slash `/`**: The pathname component separator. The shell's pathname resolution, `cd`, command search, and all file operations depend on `/` being recognized reliably.
3. **Newline `\n`**: The shell's primary line delimiter. Command parsing, here-documents, `read` builtin, IFS processing, and output all depend on newline recognition.
4. **Carriage return `\r`**: Must be invariant for consistent text processing across locales.

For the shell, this guarantee is essential: the shell can always recognize `/` as a path separator and `\n` as a line terminator regardless of the current locale. This is particularly important for multibyte encodings (like UTF-8) where these characters must not appear as part of a multibyte sequence.
