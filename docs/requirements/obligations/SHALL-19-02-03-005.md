## Extracted Text

> Enclosing characters in double-quotes ("") shall preserve the literal value of all characters within the double-quotes, with the exception of the characters backquote, <dollar-sign>, and <backslash>, as follows:: The backquote shall retain its special meaning introducing the other form of command substitution (see 2.6.3 Command Substitution). The portion of the quoted string from the initial backquote and the characters up to the next backquote that is not preceded by a <backslash>, having escape characters removed, defines that command whose output replaces "`...`" when the word is expanded. Either of the following cases produces undefined results: A quoted (single-quoted, double-quoted, or dollar-single-quoted) string that begins, but does not end, within the "`...`" sequence A "`...`" sequence that begins, but does not end, within the same double-quoted string

## Context

**Behavior:** Specifies how backquote command substitution works inside double-quotes:

1. **Delimiting the command:** The command is defined by characters from the opening `` ` `` up to the next `` ` `` that is not preceded by `\`. Escape characters are removed before the command is executed.

2. **Output replacement:** The output of the command replaces the entire `` `...` `` sequence when the word is expanded.

3. **Undefined results (two cases):**
   - A quoted string (single-quoted, double-quoted, or dollar-single-quoted) that begins but does not end within the `` `...` `` sequence produces undefined results.
   - A `` `...` `` sequence that begins but does not end within the same double-quoted string produces undefined results.

**Applies to:** All shell modes during expansion.

**Implementation detail:** The shell should track quote state carefully when scanning for the closing backquote. Inside the backquote substitution, `\` followed by `$`, `` ` ``, or `\` is an escape; other `\` sequences are literal. The implementation may choose any behavior (including error) for the two undefined-result cases, but should ideally issue a diagnostic.

**Spec reference:** Section 2.2.3, lines 133-143 in `utilities/V3_chap02.html`. See also 2.6.3 Command Substitution.
