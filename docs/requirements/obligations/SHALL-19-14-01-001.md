## Extracted Text

> The following patterns shall match a single character: ordinary characters, special pattern characters, and pattern bracket expressions. The pattern bracket expression also shall match a single collating element.

## Context

**Three types of single-character patterns.**

The following pattern types each match exactly one character:

1. **Ordinary characters**: Any character that is not a special pattern character and is not a shell special character (when unquoted). An ordinary character pattern matches itself.

2. **Special pattern characters** (`?`, `*`, `[`): When unquoted/unescaped and not inside a bracket expression, these have special meaning. `?` matches any single character (see SHALL-19-14-01-009/014).

3. **Pattern bracket expressions** (`[...]`): Match a single character (or collating element) from the set described by the bracket expression. The bracket expression syntax follows XBD 9.3.5 RE Bracket Expression rules, but uses `!` instead of `^` for negation.

The pattern bracket expression also **shall** match a single collating element, which in multi-byte locales may consist of more than one byte but is still one logical character.

See 2.14.1 Patterns Matching a Single Character in utilities/V3_chap02.html.
