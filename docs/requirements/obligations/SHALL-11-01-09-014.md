## Extracted Text

> Special character on input, which is recognized if the ICANON flag is set. When received, all the bytes waiting to be read are immediately passed to the process without waiting for a <newline>, and the EOF is discarded. Thus, if there are no bytes waiting (that is, the EOF occurred at the beginning of a line), a byte count of zero shall be returned from the read(), representing an end-of-file indication. If ICANON is set, the EOF character shall be discarded when processed.

## Context

**Behavior:** The EOF special character (typically ^D) — duplicate/alternate extraction of the same spec text as SHALL-11-01-09-005:
1. Recognized on input only when ICANON is set in `c_lflag`.
2. All waiting bytes are immediately passed to the process without waiting for NL; EOF itself is discarded.
3. If no bytes are waiting (EOF at beginning of line) → `read()` returns 0 (end-of-file indication).
4. The EOF character shall be **discarded** when processed.

**Conditions:** ICANON must be set in `c_lflag`.

**Shell relevance:** See SHALL-11-01-09-005 for full details. ^D at line start → shell sees EOF → exits or prompts. ^D mid-line → partial line delivered. VEOF may alias VMIN.

**Spec reference:** POSIX.1-2024, §11.1.9 Special Characters (tag_11_01_09), `basedefs/V1_chap11.html` lines 235–238.
