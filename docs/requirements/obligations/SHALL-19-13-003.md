## Extracted Text

> Variables with the export attribute, along with those explicitly exported for the duration of the command, shall be passed to the utility environment variables

## Context

**Exported variables are passed as environment variables to child utilities.**

When the shell forks/execs a child process for a utility, the child's environment **shall** include:
1. All shell variables that have the `export` attribute (set via `export VAR` or `export VAR=value`)
2. Variables explicitly exported for the duration of a single command via prefix assignments (e.g., `VAR=value command`)

The `export` built-in (see 2.23 in utilities/V3_chap02.html) marks variables for export. Prefix assignments on a command line create temporary exports that last only for that command execution (for non-special built-ins).

Implementation: When building the `envp` array for `execve()`, collect all variables with the export flag plus any command-prefix assignments.
