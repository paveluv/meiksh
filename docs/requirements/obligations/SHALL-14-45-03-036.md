## Extracted Text

> The <signal.h> header shall define the following macros which shall expand to integer constant expressions that need not be usable in #if preprocessing directives:: [CX] The resulting set is the signal set pointed to by the argument set.

## Context

This is the [CX] extension defining the semantics of `SIG_SETMASK`: when used as the `how` argument to `sigprocmask()`, the resulting signal mask is exactly the signal set pointed to by the `set` argument, completely replacing the current mask. `sigprocmask(SIG_SETMASK, &set, NULL)` sets the blocked signal set to exactly what `set` contains. For `meiksh`, this is the typical method for restoring a saved signal mask (e.g., after `sigprocmask(SIG_BLOCK, ..., &oldmask)` the old mask is restored with `sigprocmask(SIG_SETMASK, &oldmask, NULL)`). See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 509–510).
