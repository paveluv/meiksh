## Extracted Text

> Certain combinations of the *_sign_posn, positive_sign, and negative_sign values are invalid and shall not be accepted by localedef:

## Context

The `localedef` utility must reject certain invalid combinations of sign positioning and sign string values in the LC_MONETARY category. This is a `localedef` validation requirement, not a shell runtime requirement. The shell does not parse or validate locale definitions. However, the shell benefits from this validation because it ensures that locale data consumed at runtime is consistent and well-formed.

See: `basedefs/V1_chap07.html` §7.3.3 LC_MONETARY.
