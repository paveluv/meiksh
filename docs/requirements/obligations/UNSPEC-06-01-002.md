## Extracted Text

> If the encoded values associated with each member of the portable character set are not invariant across all locales supported by the implementation, if an application uses any pair of locales where the character encodings differ, or accesses data from an application using a locale which has different encodings from the locales used by the application, the results are unspecified.

## Context

This is a duplicate/companion of UNSPEC-06-01-001. If the portable character set encoding varies across locales and the shell processes data across locale boundaries, the results are unspecified. The shell should avoid assuming that character encodings are invariant beyond the four guaranteed characters (`.`, `/`, newline, carriage return) and the digits 0-9. In practice, most modern systems use UTF-8 which is a superset of ASCII, making this a non-issue, but the shell should be prepared for locale changes.

See: `basedefs/V1_chap06.html` §6.1 Portable Character Set.
