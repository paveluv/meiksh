## Extracted Text

> The following exit values shall be returned:

## Context

**Behavior:** This is the introductory statement for the `sh` utility's EXIT STATUS section. The `sh` utility shall return specific exit values depending on the circumstances of termination. The defined exit values are:
- **0:** Script consisted solely of blank lines, comments, or both.
- **1-125:** Non-interactive shell detected a non-fatal error (syntax, redirection, variable assignment, etc.).
- **126:** command_file could not be executed due to ENOEXEC error.
- **127:** command_file could not be found by a non-interactive shell.
- **128:** Unrecoverable read error while reading commands.
- **Otherwise:** Terminate as for `exit` with no operands (last command's exit status).

**Applies to:** The `sh` utility process exit status as seen by its parent process.
