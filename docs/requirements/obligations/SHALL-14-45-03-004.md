> The <signal.h> header shall define the following data types:

## Context

The `<signal.h>` header must define three data types: `sig_atomic_t`, `sigset_t`, and `pid_t`. These are fundamental to the shell's signal handling infrastructure. `sig_atomic_t` is used for signal-safe flag variables, `sigset_t` for signal mask manipulation (`sigprocmask`, `sigpending`), and `pid_t` for process identification in signal delivery (`kill`, `waitpid`). The shell relies on the platform's `<signal.h>` providing these types.
