## Extracted Text

> The exit status of a sequential AND-OR list shall be the exit status of the last pipeline in the AND-OR list that is executed.

## Context

When an AND-OR list is executed **synchronously** (i.e., terminated with `;` or newline, not `&`), the exit status of the entire AND-OR list must be the exit status of **the last pipeline actually executed** within that list.

In an AND-OR list like `cmd1 && cmd2 || cmd3`, short-circuit evaluation applies:
- If `cmd1` succeeds (exit 0), `cmd2` runs; if `cmd2` fails, `cmd3` runs. The exit status is from whichever of `cmd2` or `cmd3` was last executed.
- If `cmd1` fails (exit non-zero), `cmd2` is skipped, `cmd3` runs. The exit status is from `cmd3`.

The key point is "last pipeline that is executed" — not the last pipeline in the textual list, but the last one the shell actually ran due to `&&`/`||` short-circuit logic.

See 2.9.3.3 Sequential AND-OR Lists in utilities/V3_chap02.html#tag_19_09_03_04.
