## Extracted Text

> For a parameter that is not enclosed in braces:: Otherwise, the parameter is a single-character symbol, and behavior is unspecified if that character is neither a digit nor one of the special parameters (see 2.5.2 Special Parameters).

## Context

**Behavior:** For unbraced parameter expansions where the character after `$` does not start a valid name (per XBD 3.216 Name), the shell treats it as a single-character parameter. Behavior is **unspecified** if that character is not a digit (`0`–`9`) and not one of the special parameter characters (`@`, `*`, `#`, `?`, `-`, `$`, `!`).

**Conditions:** Applies to unbraced `$X` where `X` is a single character that is neither a name-start character, nor a digit, nor a special parameter symbol. Examples: `$,`, `$=`, `$[`.

**Implementation guidance:** The shell may choose to treat such sequences as literal text (leaving `$X` unexpanded), expand to empty, or raise an error. Since this is unspecified, meiksh should document its chosen behavior. A common choice is to treat `$` followed by an unrecognized single character as a literal `$` followed by that character.

**Spec reference:** See 2.6.2 Parameter Expansion in utilities/V3_chap02.html, lines 796–797. For the list of special parameters, see 2.5.2 Special Parameters (same file, #tag_19_05_02).
