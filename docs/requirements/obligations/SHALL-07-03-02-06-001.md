## Extracted Text

> The minimum collation sequence definition of the POSIX locale follows; the code listing depicts the localedef input. All characters not explicitly listed here shall be inserted in the character collation order after the listed characters and shall be assigned unique primary weights. If the listed characters have ASCII encoding, the other characters shall be in ascending order according to their coded character set values; otherwise, the order of the other characters is unspecified. The collation sequence shall not include any multi-character collating elements.

## Context

The POSIX locale's LC_COLLATE definition has specific requirements: (1) All characters not explicitly listed are inserted after the listed characters with unique primary weights. (2) If the locale uses ASCII encoding, unlisted characters are ordered by ascending code point value. (3) The POSIX locale must not define any multi-character collating elements. This is critical for the shell because in the POSIX/C locale, bracket expression ranges like `[a-z]` must match based on the code-point ordering of characters — the collation order matches the character encoding order. This makes the POSIX locale the most predictable locale for shell pattern matching. The shell must ensure that when `LC_COLLATE=POSIX` or `LC_COLLATE=C`, collation-based operations use this simple code-point ordering.
