## Extracted Text

> When a foreground job terminates, or becomes a suspended job (see below), if the shell has a controlling terminal it shall set the foreground process group ID associated with the terminal to the process group ID of the shell.

## Context

**Reclaim terminal after foreground job ends or suspends:** When a foreground job terminates (all processes exit) or becomes a suspended job (a process is stopped by a signal), if the shell has a controlling terminal, it shall set the terminal's foreground process group ID back to the shell's own process group ID via `tcsetpgrp(terminal_fd, shell_pgid)`.

This is essential: after a foreground job finishes or is suspended (Ctrl+Z), the shell must reclaim the terminal so it can read further input. Without this, the shell would receive SIGTTIN/SIGTTOU when trying to read from or write to the terminal.
