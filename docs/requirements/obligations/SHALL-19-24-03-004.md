## Extracted Text

> The shell shall format the output, including the proper use of quoting, so that it is suitable for reinput to the shell as commands that achieve the same value and readonly attribute-setting results in a shell execution environment in which:

## Context

**Behavior:** When `readonly -p` is specified, `readonly` must write to standard output the names and values of all readonly variables. The output must be formatted with proper quoting so it is suitable for reinput to achieve the same value and readonly attribute-setting results, given the conditions listed in SHALL-19-24-03-005 and -006.

**Format:** `readonly name="value"` for set variables, `readonly name` for unset readonly variables.

**Applies to:** All shell modes.

**Spec reference:** `readonly` DESCRIPTION in `utilities/V3_chap02.html`, lines 4452-4453.
