## Extracted Text

> Expands to the shortest representation of the decimal exit status (see 2.8.2 Exit Status for Commands) of the pipeline (see 2.9.2 Pipelines) executed from the current shell execution environment (not a subshell environment) that most recently either terminated or, optionally but only if the shell is interactive and job control is enabled, was stopped by a signal. If this pipeline terminated, the status value shall be its exit status; otherwise, the status value shall be the same as the exit status that would have resulted if the pipeline had been terminated by a signal with the same number as the signal that stopped it. The value of the special parameter '?' shall be set to 0 during initialization of the shell. When a subshell environment is created, the value of the special parameter '?' from the invoking shell environment shall be preserved in the subshell. Note: In var=$(some_command); echo $? the output is the exit status of some_command, which is executed in a subshell environment, but this is because its exit status becomes the exit status of the assignment command var=$(some_command) (see 2.9.1 Simple Commands) and this assignment command is the most recently completed pipeline. Likewise for any pipeline consisting entirely of a simple command that has no command word, but contains one or more command substitutions. (See 2.9.1 Simple Commands.)

## Context

This defines the expansion behavior of the special parameter `$?`.

**Required behavior:**
- `$?` must expand to the shortest decimal representation of the exit status of the most recently completed pipeline in the current shell execution environment (not a subshell).
- "Most recently completed" means the pipeline that most recently either terminated or (optionally, only in interactive shells with job control enabled) was stopped by a signal.
- If the pipeline terminated normally, `$?` is its exit status (0–255).
- If the pipeline was stopped by a signal, `$?` is the value that would result if the pipeline had been *terminated* by that same signal (typically 128 + signal_number).
- `$?` must be initialized to `0` during shell startup.
- When a subshell environment is created (e.g., `( ... )`), the value of `$?` from the parent shell is preserved in the subshell.

**Important subtlety (command substitution in assignments):**
In `var=$(some_command); echo $?`, the `$?` reflects the exit status of `some_command` because the assignment `var=$(some_command)` is the most recently completed pipeline, and the exit status of a simple command with no command word but command substitutions is the exit status of the last command substitution. See 2.9.1 "Simple Commands".

**Conditions:** Current shell execution environment only. Subshell pipelines update `$?` in the subshell, not the parent (except as the subshell's own exit status propagates).

See 2.5.2 "Special Parameters" (`?`), 2.8.2 "Exit Status for Commands", 2.9.1 "Simple Commands", and 2.9.2 "Pipelines" in `utilities/V3_chap02.html`.
