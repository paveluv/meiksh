## Extracted Text

> Listed below are the special parameters and the values to which they shall expand. Only the values of the special parameters are listed; see 2.6 Word Expansions for a detailed summary of all the stages involved in expanding words.

## Context

This is the introductory obligation for section 2.5.2 "Special Parameters". It establishes that the special parameters (`@`, `*`, `#`, `?`, `-`, `$`, `!`, `0`) have defined expansion values listed in the subsections that follow.

**Required behavior:**
- The shell must recognize each of the special parameters listed in 2.5.2 and expand them to the values specified by the individual paragraphs (covered in subsequent obligations SHALL-19-05-02-002 through SHALL-19-05-02-007 and others).
- The expansion values described in section 2.5.2 are only the raw values; the full expansion pipeline (tilde expansion, parameter expansion, command substitution, arithmetic expansion, field splitting, pathname expansion, quote removal) described in 2.6 "Word Expansions" applies on top.

**Conditions:** Applies universally. Special parameters cannot be assigned to by the user; they are read-only and set implicitly by the shell.

See 2.5.2 "Special Parameters" and 2.6 "Word Expansions" in `utilities/V3_chap02.html`.
