## Extracted Text

> The backquote shall retain its special meaning introducing the other form of command substitution (see 2.6.3 Command Substitution). The portion of the quoted string from the initial backquote and the characters up to the next backquote that is not preceded by a <backslash>, having escape characters removed, defines that command whose output replaces "`...`" when the word is expanded. Either of the following cases produces undefined results: A quoted (single-quoted, double-quoted, or dollar-single-quoted) string that begins, but does not end, within the "`...`" sequence A "`...`" sequence that begins, but does not end, within the same double-quoted string

## Context

**Behavior:** Duplicate extraction of the backquote rules inside double-quotes (same spec lines 133-143 as SHALL-19-02-03-005). Inside double-quotes, backquote retains its special meaning for legacy command substitution:

1. **Command delimiting:** The command runs from the opening `` ` `` to the next unescaped `` ` `` (one not preceded by `\`). Escape characters are removed before execution.
2. **Output replacement:** Command output replaces the `` `...` `` sequence during word expansion.
3. **Undefined results:** (a) A quoted string that starts but doesn't end within the `` `...` `` sequence, and (b) a `` `...` `` sequence that starts but doesn't end within the same double-quoted string.

**Applies to:** All shell modes during expansion.

**Implementation detail:** When scanning for the closing backquote inside a double-quoted string, backslash only escapes `$`, `` ` ``, and `\`. Other `\X` pairs are passed through literally to the subcommand.

**Spec reference:** Section 2.2.3, lines 133-143 in `utilities/V3_chap02.html`. See also 2.6.3 Command Substitution.
