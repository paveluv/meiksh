## Extracted Text

> The result of applying rules that map a numeric code value to each element of a character set. An element of a character set may be related to more than one numeric code value but the reverse is not true. However, for state-dependent encodings the relationship between numeric code values and elements of a character set may be further controlled by state information. The character set may contain fewer elements than the total number of possible numeric code values; that is, some code values may be unassigned.

## Context

**Definition — low shell relevance.** Defines "codeset" as a mapping from numeric code values to character set elements. A character may map to multiple code values, but not vice versa. State-dependent encodings may add further complexity. Some code values may be unassigned.

**Shell impact:** The shell operates on bytes/characters according to the current locale's codeset (set via `LC_CTYPE`). This affects pattern matching, character class interpretation in `[[:alpha:]]` etc., and multi-byte character handling. The shell must be aware of the active codeset when performing string operations but does not define codesets itself.
