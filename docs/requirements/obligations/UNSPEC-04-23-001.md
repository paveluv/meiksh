## Extracted Text

> On implementations that support the IEC 60559 Floating-Point option, whether or when functions in the <math.h> header raise an undeserved underflow floating-point exception is unspecified. Otherwise, as implied by XSH feraiseexcept(), the <math.h> functions do not raise spurious floating-point exceptions (detectable by the user), other than the inexact floating-point exception.

## Context

This is a C math library requirement about floating-point exception handling. It is not relevant to a POSIX shell implementation — the shell performs integer arithmetic only (via `$(( ))` arithmetic expansion). The shell does not use `<math.h>` functions or handle floating-point exceptions. No shell action is required.

See: `basedefs/V1_chap04.html` §4.23 Treatment of Error Conditions for Mathematical Functions.
