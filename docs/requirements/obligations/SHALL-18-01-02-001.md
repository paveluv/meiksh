## Extracted Text

> Some of the standard utilities perform complex data manipulation using their own procedure and arithmetic languages, as defined in their EXTENDED DESCRIPTION or OPERANDS sections. Unless otherwise noted, the arithmetic and semantic concepts (precision, type conversion, control flow, and so on) shall be equivalent to those defined in the ISO C standard, as described in the following sections. Note that there is no requirement that the standard utilities be implemented in any particular programming language.

## Context

When the shell performs arithmetic operations (e.g., in `$(( ))` arithmetic expansion, `let`, or arithmetic `for` loops), the arithmetic and semantic concepts — including precision, type conversion, and control flow — **shall** be equivalent to those defined in the ISO C standard, unless the specific utility description states otherwise. There is no requirement to implement the shell in C, but the arithmetic behavior must match C semantics. This primarily affects the shell's arithmetic expansion (`$(( ))`), the `expr` utility behavior, and any arithmetic contexts in shell builtins. Reference: XCU 1.1.2.
