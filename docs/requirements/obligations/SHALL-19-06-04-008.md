## Extracted Text

> All changes to variables in an arithmetic expression shall be in effect after the arithmetic expansion, as in the parameter expansion "${x=value}".

## Context

**Behavior:** Any variable assignments that occur during arithmetic expression evaluation (via the `=` operator or compound assignment operators like `+=`, `-=`, `*=`, etc.) must persist in the current shell environment after the arithmetic expansion completes. The assignment is NOT limited to a subshell — it affects the current execution environment, just like `${x=value}`.

**Inputs/Outputs:**
- `echo $((x = 5))` → outputs `5`, and `$x` is now `5` in the current shell
- `echo $((x += 3))` → if `x` was `5`, outputs `8`, and `$x` is now `8`
- `echo $((a = 1, b = 2, a + b))` → outputs `3`, and both `$a` and `$b` are set

**Conditions:** This applies to all arithmetic expansion contexts: standalone `$((...))`, within double-quotes, etc. The variable changes persist even if the expansion is part of a larger word that ultimately isn't used.

**Spec reference:** See 2.6.4 Arithmetic Expansion in utilities/V3_chap02.html, lines 1178–1179.
