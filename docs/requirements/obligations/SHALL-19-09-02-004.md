## Extracted Text

> If the pipeline is not in the background (see 2.9.3.1 Asynchronous AND-OR Lists and 2.11 Job Control), the shell shall wait for the last command specified in the pipeline to complete, and may also wait for all commands to complete.

## Context

**Behavior:** If a pipeline is NOT in the background (not terminated with `&`), the shell MUST wait for the last command in the pipeline to complete. The shell MAY also wait for all commands in the pipeline to complete (not just the last one).

**Applies to:** Foreground pipelines. Background pipelines (see 2.9.3.1) and job-controlled pipelines (see 2.11) have different waiting semantics.

**Implementation detail:** The minimum requirement is to wait for the last pipeline member. Most shells wait for all members (especially when job control is enabled). meiksh should track all pipeline PIDs and wait for all of them, reporting the exit status of the last one. The exit status of the pipeline is the exit status of the last command (or the logical negation if `!` prefixes the pipeline).

**Spec reference:** Section 2.9.2, lines 2026-2027 in `utilities/V3_chap02.html`. Cross-refs: 2.9.3.1 Asynchronous AND-OR Lists, 2.11 Job Control.
