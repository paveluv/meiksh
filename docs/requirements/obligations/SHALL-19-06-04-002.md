## Extracted Text

> The expression shall be treated as if it were in double-quotes, except that a double-quote inside the expression is not treated specially. The shell shall expand all tokens in the expression for parameter expansion, command substitution, and quote removal.

## Context

**Behavior:** Before evaluating the arithmetic expression, the shell must perform these pre-processing steps:

1. **Double-quote-like context:** The expression inside `$((...))` is treated as if it were in double-quotes for expansion purposes. This means:
   - Parameter expansion occurs (`$var` is expanded)
   - Command substitution occurs (`$(cmd)` is expanded)
   - Backslash escaping works as in double-quotes
   - Single-quotes are NOT recognized as quoting mechanisms inside arithmetic
   - However, a literal `"` (double-quote) inside the expression is NOT treated specially — it doesn't start or end a quoted string within the arithmetic expression
2. **Expansions performed:** Parameter expansion, command substitution, and quote removal are performed on the expression tokens BEFORE arithmetic evaluation.
3. **Expansions NOT performed:** Tilde expansion, field splitting, and pathname expansion are NOT performed within arithmetic expressions.

**Inputs/Outputs:**
- `x=5; echo $((x + 1))` → parameter `x` expanded to `5`, then `5 + 1` evaluated → `6`
- `echo $(( $(echo 3) + 2 ))` → command substitution yields `3`, then `3 + 2` → `5`
