## Extracted Text

> If the command name is a standard utility implemented as a function (see XBD 4.25 Utility), the effect of variable assignments shall be as if the utility was not implemented as a function.

## Context

**Behavior:** Standalone restatement of SHALL-19-09-01-02-004. If a standard utility is implemented as a function, prefix variable assignments MUST behave as if it were NOT a function (i.e., exported temporarily, not persistent).

**Applies to:** Standard utilities implemented as shell functions.

**Implementation detail:** See SHALL-19-09-01-02-004.

**Spec reference:** Section 2.9.1.2, lines 1773-1774 in `utilities/V3_chap02.html`. Cross-ref: XBD 4.25 in `basedefs/V1_chap04.html`.
