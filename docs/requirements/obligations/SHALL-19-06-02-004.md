## Extracted Text

> If the parameter is a name, the expansion shall use the longest valid name (see XBD 3.216 Name), whether or not the variable denoted by that name exists.

## Context

**Behavior:** This is a duplicate extraction of the same rule as SHALL-19-06-02-003. For unbraced parameter expansion `$name`, the shell must greedily consume the longest valid name (letters, digits, underscores, starting with a letter or underscore per XBD 3.216). The existence of the variable is irrelevant to parsing — the longest valid name is always used.

**Conditions:** Same as SHALL-19-06-02-003 — unbraced parameter expansions where the first character after `$` begins a valid name.

**Inputs/Outputs:**
- Example: `$foo_bar!` → parameter name is `foo_bar` (stops at `!` which is not a valid name character), regardless of whether `foo_bar` is set
- Example: `$x9` → parameter name is `x9`

**Spec reference:** See 2.6.2 Parameter Expansion in utilities/V3_chap02.html, lines 794–795. For "Name" definition, see basedefs/V1_chap03.html#tag_03_216.
