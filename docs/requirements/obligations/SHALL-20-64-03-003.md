## Extracted Text

> For each pid operand, the kill utility shall perform actions equivalent to the kill() function defined in the System Interfaces volume of POSIX.1-2024 called with the following arguments:: The value of the pid operand shall be used as the pid argument.

## Context

**Behavior:** The `pid` operand from the command line **shall** be used as the `pid` argument to the `kill()` system call. The value is interpreted as a signed integer:
- Positive: target a specific process
- 0: target the sender's process group
- Negative: target the process group with ID equal to the absolute value
- -1: target all processes the caller can signal

**Conditions:** The pid operand must be parsed as a decimal integer. The shell must also support job ID syntax (%job) as an extension, resolving it to the appropriate process group ID.

**Shell relevance:** The `kill` builtin must parse pid operands correctly, handling both numeric PIDs and `%job` job specifications. For `%job`, the shell resolves to `-pgid` (negative process group ID) to signal the entire job. The shell must handle invalid pid values gracefully with an error message.
