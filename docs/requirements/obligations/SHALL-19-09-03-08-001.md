## Context

**Behavior:** The shell parser must recognize the `||` control operator as introducing an OR list. The grammar format is:

```
command1 || command2
```

Multiple `||` and `&&` operators can be chained in a single AND-OR list. The `||` operator has equal precedence with `&&` and they are left-associative (evaluated left-to-right).

**Parsing:** The tokenizer must recognize `||` as a two-character control operator (not two individual `|` characters — a single `|` is the pipe operator). The parser must construct an OR-list AST node.

**Applies to:** All shell modes (interactive, non-interactive, subshell).

**Spec reference:** Section 2.9.3.4 OR Lists in `utilities/V3_chap02.html`, line 2202. See also the grammar rule `and_or` in section 2.10.2.
