## Extracted Text

> The shell shall exit only if the special built-in utility is executed directly. If it is executed via the command utility, the shell shall not exit.

## Context

The `command` utility suppresses the special built-in exit-on-error behavior. See 2.8.1 Consequences of Shell Errors, note 1, in utilities/V3_chap02.html. Also references the `command` utility (utilities/command.html).

**Behavior required:**
- When a special built-in utility is executed directly and encounters an error in a non-interactive shell, the shell SHALL exit (per SHALL-19-08-01-004).
- However, if the special built-in is executed via the `command` utility (e.g., `command set --invalid`), the shell SHALL NOT exit even in a non-interactive shell.
- The `command` utility effectively "demotes" a special built-in to behave like a regular built-in for error-handling purposes.

**Implementation note:** When dispatching built-in commands, check whether the invocation is via `command`. If so, do not propagate the "special built-in error" exit behavior to the shell.

**Key reference:** `command` utility specification (utilities/command.html).
