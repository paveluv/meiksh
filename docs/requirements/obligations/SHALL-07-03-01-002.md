## Extracted Text

> The following keywords shall be recognized. In the descriptions, the term "automatically included" means that it shall not be an error either to include or omit any of the referenced characters; the implementation provides them if missing (even if the entire keyword is missing) and accepts them silently if present. When the implementation automatically includes a missing character, it shall have an encoded value dependent on the charmap file in effect (see the description of the localedef -f option); otherwise, it shall have a value derived from an implementation-defined character mapping.

## Context

**Behavior:** LC_CTYPE keyword semantics:
- The keywords `upper`, `lower`, `alpha`, `digit`, `alnum`, `space`, `cntrl`, `punct`, `graph`, `print`, `xdigit`, `blank`, `charclass`, `toupper`, `tolower` must be recognized.
- "Automatically included" characters are provided by the implementation whether or not they appear in the locale definition. This is not an error.
- Automatically-included characters get their encoding from the charmap file if present, or from an implementation-defined mapping.

**Shell relevance:** The shell uses these character classes via POSIX character class expressions in bracket expressions (e.g., `[[:upper:]]`). The "automatically included" guarantee means the shell can always rely on the portable character set members being in the expected classes (e.g., A-Z in `upper`, a-z in `lower`, 0-9 in `digit`).

The ref to `localedef.html` is for an external utility — not shell-relevant for extraction.
