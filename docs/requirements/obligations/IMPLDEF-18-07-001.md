## Extracted Text

> Whether any additional utility is considered an intrinsic utility is implementation-defined. Because applications are unable to override an intrinsic utility with a utility from PATH , implementations should not make any utility an intrinsic utility beyond the utilities in Intrinsic Utilities.

## Context

Whether any utility beyond those listed in XCU Table "Intrinsic Utilities" is treated as an intrinsic utility is **implementation-defined**. Because applications cannot override an intrinsic utility via `PATH` (intrinsics are found before `PATH` search), implementations should be conservative about making additional utilities intrinsic. For `meiksh`, the implementation must document which additional utilities (if any) are treated as intrinsic. The POSIX recommendation is to not make any utility intrinsic beyond the listed set, to avoid breaking applications that expect to override a utility via `PATH`. Reference: XCU 1.7.
