## Extracted Text

> If the shell is interactive and the asynchronous AND-OR list became a background job: a message indicating completion of the corresponding job is written to standard error. If set -b is enabled, it is unspecified whether the process ID is removed from the list of known process IDs when the message is written or immediately prior to when the shell writes the next prompt for input.

## Context

**Behavior:** When the shell is interactive and an async AND-OR list became a background job, a completion message MUST be written to stderr when the job completes. If `set -b` (notify) is enabled, it is UNSPECIFIED whether the PID is removed from the known-process list when the completion message is written, or immediately before the shell writes the next prompt.

**Applies to:** Interactive shells with background jobs. The completion message is the `[N]+ Done` style notification.

**Implementation detail:** meiksh must check for completed background jobs and print completion messages. With `set -b`, the message is printed asynchronously (as soon as the shell detects completion). Without `set -b`, the message is printed just before the next prompt. The timing of PID removal from the known-process list is meiksh's choice.

**Spec reference:** Section 2.9.3.1, lines 2158-2159 in `utilities/V3_chap02.html`. Cross-ref: `set -b`.
