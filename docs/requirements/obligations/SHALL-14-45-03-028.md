## Context

The `sigaction` structure must be declared in `<signal.h>` (CX extension) with at least these members: `void (*sa_handler)(int)` (signal handler function), `sigset_t sa_mask` (signals to block during handler execution), `int sa_flags` (signal action flags), and `void (*sa_sigaction)(int, siginfo_t *, void *)` (extended signal handler). For `meiksh`, the `sigaction` structure is essential—the shell must use `sigaction()` (not `signal()`) for reliable signal handling. The `sa_mask` field controls which signals are blocked while a handler runs, `sa_flags` controls behavior like `SA_RESTART` and `SA_SIGINFO`, and the handler function defines the shell's response to each signal.
