## Extracted Text

> The following variables shall affect the execution of the shell:: A string treated as a list of characters that is used for field splitting, expansion of the '*' special parameter, and to split lines into fields with the read utility. If the value of IFS includes any bytes that do not form part of a valid character, the results of field splitting, expansion of '*', and use of the read utility are unspecified. If IFS is not set, it shall behave as normal for an unset variable, except that field splitting by the shell and line splitting by the read utility shall be performed as if the value of IFS is <space><tab><newline>; see 2.6.5 Field Splitting. The shell shall set IFS to <space><tab><newline> when it is invoked.

## Context

This defines the detailed semantics of the `IFS` (Internal Field Separator) variable.

**Required behavior:**
- `IFS` is a string treated as a list of characters used for:
  1. Field splitting (2.6.5) of unquoted expansion results.
  2. Expansion of the `*` special parameter (joining positional parameters in `"$*"`).
  3. Splitting lines into fields with the `read` utility.
- If `IFS` contains bytes that don't form valid characters in the current locale, the results of field splitting, `*` expansion, and `read` are unspecified.
- **Unset IFS behavior:** If `IFS` is unset, it behaves as a normal unset variable in all respects EXCEPT that field splitting and `read` line splitting must behave as if IFS were set to `<space><tab><newline>` (the default value).
- **Initialization:** The shell must set `IFS` to `<space><tab><newline>` when it is invoked. This happens regardless of any `IFS` value in the inherited environment.

**Key implementation details:**
- IFS whitespace characters (`<space>`, `<tab>`, `<newline>`) have special splitting semantics: leading/trailing IFS whitespace is trimmed, and sequences of IFS whitespace are collapsed.
- Non-whitespace IFS characters delimit fields without trimming — two adjacent non-whitespace IFS characters produce an empty field between them.

**Conditions:** Applies to all field splitting and `read` operations throughout shell execution.

See 2.5.3 "Shell Variables" (`IFS`), 2.6.5 "Field Splitting" in `utilities/V3_chap02.html`, and the `read` utility at `utilities/read.html`.
