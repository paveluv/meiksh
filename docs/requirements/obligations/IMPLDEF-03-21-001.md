## Extracted Text

> An implementation-defined means of associating privileges with a process with regard to the function calls, function call options, and the commands that need special privileges. There may be zero or more such means. These means (or lack thereof) are described in the conformance document.

## Context

**Definition — moderate shell relevance.** Defines "appropriate privileges" as an implementation-defined means of associating privileges with a process. The concept is used throughout POSIX wherever an operation requires elevated permissions (e.g., changing file ownership, sending signals to other users' processes, binding to privileged ports).

**Shell impact:** The shell references "appropriate privileges" in several contexts: the `kill` builtin sending signals to arbitrary processes, `cd` to directories without search permission, `umask` operations, etc. The shell itself does not grant or check privileges — it defers to the OS. However, shell error messages for permission failures should be aware of this concept. `meiksh` should propagate errno-based errors (EPERM, EACCES) correctly.
