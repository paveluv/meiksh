## Extracted Text

> The NL and CR characters cannot be changed. It is implementation-defined whether the START and STOP characters can be changed. The values for INTR, QUIT, ERASE, KILL, EOF, EOL, and SUSP shall be changeable to suit individual tastes. Special character functions associated with changeable special control characters can be disabled individually.

## Context

**Behavior:**
1. NL and CR characters are **fixed** — they cannot be changed via the `c_cc` array.
2. Whether START and STOP can be changed is **implementation-defined**.
3. INTR, QUIT, ERASE, KILL, EOF, EOL, and SUSP **shall** be changeable (via `c_cc` array entries and `tcsetattr()`).
4. Changeable special character functions can be individually **disabled** (by setting the `c_cc` entry to `_POSIX_VDISABLE`).

**Conditions:** Applies to all terminal devices.

**Shell relevance:** The `stty` builtin must allow users to change INTR, QUIT, ERASE, KILL, EOF, EOL, and SUSP characters. It must allow disabling them individually via `_POSIX_VDISABLE`. meiksh must document whether it supports changing START and STOP characters (implementation-defined choice). The shell should save and restore these values when modifying terminal state.

**Spec reference:** POSIX.1-2024, §11.1.9 Special Characters (tag_11_01_09), `basedefs/V1_chap11.html` line 262.
