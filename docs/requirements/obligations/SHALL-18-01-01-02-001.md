## Context

The `fork()` function's functionality shall be available on all conforming systems. Specifically, two capabilities are required: (1) independent processes shall execute independently without either terminating, and (2) a process shall be able to create a new process with all standard process attributes via `fork()` followed by `exec()`. For `meiksh`, this is fundamental: the shell uses `fork()` to create child processes for every external command, pipeline component, subshell, and command substitution. The shell must be able to `fork()` and have both parent and child continue execution independently. See POSIX Utilities Chapter 1, Section 1.1.1.2 (utilities/V3_chap01.html lines 71–78).
