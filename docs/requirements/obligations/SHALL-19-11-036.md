## Extracted Text

> When a background job completes or is terminated by a signal, an interactive shell shall write a message to standard error, formatted as described by the jobs utility (without the -l option) for a job that completed or was terminated by a signal, respectively, at the following time:: If set -b is disabled, the message shall be written immediately prior to writing the next prompt for input.

## Context

**Background job done/terminated notification — `set -b` disabled (parent obligation):** When `set -b` is disabled (the default) and a background job completes or is terminated by a signal, the message shall be written **immediately prior to writing the next prompt for input**. This is the standard deferred notification—the user sees the status when the shell next prompts for input.

The implementation queues completion events and flushes them in the prompt display loop, just before outputting `PS1`.
