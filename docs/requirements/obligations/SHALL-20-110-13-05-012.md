## Extracted Text

> Display the possible shell word expansions (see 2.6 Word Expansions ) of the bigword at the current command line position. Note: This does not modify the content of the current line, and therefore does not cause the current line to become the edit line. These expansions shall be displayed on subsequent terminal lines. If the bigword contains none of the characters '?', '*', or '[', an <asterisk> ('*') shall be implicitly assumed at the end. If any directories are matched, these expansions shall have a '/' character appended. After the expansion, the line shall be redrawn, the cursor repositioned at the current cursor position, and sh shall be placed in command mode.

## Context

**Behavior:** In vi command mode, `=` shall display the possible shell word expansions (see 2.6 Word Expansions in `utilities/V3_chap02.html`) of the bigword at the current cursor position. This does NOT modify the current line content (and therefore does not cause a history line to become the edit line per SHALL-20-110-13-05-002).

**Details:**
- If the bigword contains none of `?`, `*`, or `[`, an `*` is implicitly appended for glob matching.
- Matched directories shall have `/` appended.
- Expansions are displayed on subsequent terminal lines.
- After display, the command line shall be redrawn, cursor repositioned at the current position, and the shell remains in command mode.

**Spec reference:** vi Line Editing Command Mode in `utilities/sh.html`, line 407–418. See `utilities/V3_chap02.html#tag_19_06` for word expansions.
