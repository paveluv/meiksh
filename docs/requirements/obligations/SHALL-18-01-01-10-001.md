## Extracted Text

> The functionality of the setlocale() function defined in the System Interfaces volume of POSIX.1-2024 shall be available on all systems conforming to this volume of POSIX.1-2024; that is, utilities that require the capability of establishing an international operating environment shall be permitted to set the specified category of the international environment.

## Context

The shell **shall** support establishing an international operating environment equivalent to calling `setlocale()` with the appropriate category. In practice, this means the shell must honor locale-related environment variables (`LANG`, `LC_ALL`, `LC_CTYPE`, `LC_COLLATE`, `LC_MESSAGES`, `LC_MONETARY`, `LC_NUMERIC`, `LC_TIME`) and make locale-sensitive operations (character classification, collation, message formatting, etc.) behave according to the active locale. The shell must initialize its locale settings at startup based on these environment variables, and update them when these variables change within the execution environment. Reference: XCU 1.1.1.10.
