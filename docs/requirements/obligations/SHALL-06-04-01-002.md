## Extracted Text

> If the implementation supports a character encoding of this type, any of the standard utilities that describe character (versus byte) or text-file manipulation shall have the following characteristics:

## Context

**Behavior:** When locking-shift (state-dependent) character encodings are supported, any standard utility that performs character-level or text-file manipulation (as opposed to pure byte-level processing) must comply with specific requirements (detailed in SHALL-06-04-01-003 through SHALL-06-04-01-006).

**Shell relevance:** The shell performs character-level operations in pattern matching, parameter expansion (`${#var}` for character length, `${var:offset:length}`), and the `read` builtin. If a state-dependent encoding is active, these operations must handle shift states correctly. This is the umbrella requirement; the specifics are in the sibling obligations.
