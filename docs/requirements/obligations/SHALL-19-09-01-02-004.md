## Extracted Text

> Variable assignments shall be performed as follows:: If the command name is a standard utility implemented as a function (see XBD 4.25 Utility), the effect of variable assignments shall be as if the utility was not implemented as a function.

## Context

**Behavior:** If the command name is a standard utility that the implementation has chosen to implement as a shell function (per XBD 4.25 Utility), variable assignments MUST behave as if the utility were NOT implemented as a function. This means the assignments are exported for the command but do not persist in the current environment — same behavior as for external commands (SHALL-19-09-01-02-003).

**Applies to:** Standard POSIX utilities that meiksh implements as shell functions rather than builtins or external commands. This ensures that `VAR=x utility_as_function` does not accidentally persist VAR.

**Implementation detail:** When the command search resolves to a function that wraps a standard utility, use the external-command assignment semantics (temporary export only), not the function assignment semantics.

**Spec reference:** Section 2.9.1.2, lines 1773-1774 in `utilities/V3_chap02.html`. Cross-ref: XBD 4.25 Utility in `basedefs/V1_chap04.html`.
