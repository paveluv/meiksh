## Extracted Text

> If IXOFF is set, start/stop input control shall be enabled. The system shall transmit STOP characters, which are intended to cause the terminal device to stop transmitting data, as needed to prevent the input queue from overflowing and causing implementation-defined behavior, and shall transmit START characters, which are intended to cause the terminal device to resume transmitting data, as soon as the device can continue transmitting data without risk of overflowing the input queue. The precise conditions under which STOP and START characters are transmitted are implementation-defined.

## Context

**Behavior:** When IXOFF is set in `c_iflag`, start/stop input control is enabled. The system shall transmit STOP characters to the terminal device to prevent input queue overflow, and shall transmit START characters to resume data transmission when the input queue can accept more data. The precise conditions under which STOP and START characters are transmitted are implementation-defined.

**Relevance to meiksh:** The `stty` builtin must correctly set/clear the IXOFF flag. The actual flow control transmission is handled by the OS terminal driver. meiksh must document its implementation-defined behavior for the precise conditions of STOP/START transmission.

**Spec reference:** POSIX.1-2024, §11.2.2 Input Modes (tag_11_02_02), `basedefs/V1_chap11.html` line 530.
