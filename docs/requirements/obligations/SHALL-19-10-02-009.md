## Context

**Grammar Rule 9 (Body of function definition):** When parsing the body of a function definition (the compound command that forms the function body), word expansion and assignment shall **never** occur, even when other rules would normally require them. Each TOKEN that might otherwise be expanded or have assignment applied must instead be returned as a plain WORD consisting of exactly the characters from token recognition (Section 2.3, `utilities/V3_chap02.html#tag_19_03`).

This means the function body is parsed purely syntactically—no expansions, no assignment detection. The tokens are stored verbatim in the AST. Expansion and assignment happen later when the function is actually invoked.

This is critical for correct deferred evaluation: a function body like `f() { echo $x; }` must store `$x` literally, not expand it at definition time.
