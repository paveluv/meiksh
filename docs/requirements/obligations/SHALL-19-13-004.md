## Extracted Text

> The environment of the shell process shall not be changed by the utility unless explicitly specified by the utility description (for example, cd and umask).

## Context

**Child utilities shall not change the parent shell's environment.**

The environment of the shell process **shall not** be changed by a utility invoked in a child process, unless the utility's specification explicitly says it does so. The two notable exceptions called out are:
- `cd` — changes the current working directory of the shell (it's a special built-in, runs in the current environment)
- `umask` — changes the file creation mask of the shell (also a special built-in)

This means external commands and regular built-ins that run in a subshell/child cannot alter the parent shell's variables, working directory, umask, traps, etc. Only special built-ins (and utilities explicitly documented to run in the current environment) can modify the shell's own state.

Implementation: external commands run via fork+exec; their modifications to environment, cwd, etc. die with the child process. Special built-ins (cd, umask, export, etc.) execute in the current shell process.
