## Extracted Text

> A character that in the output stream indicates that printing should start at the beginning of the same physical line in which the carriage-return occurred. It is the character designated by '\r' in the C language. It is unspecified whether this character is the exact sequence transmitted to an output device by the system to accomplish the movement to the beginning of the line.

## Context

**Definition — moderate shell relevance.** Defines the carriage-return character `<carriage-return>` as `\r` (0x0D). It indicates printing should restart at the beginning of the same physical line. It is unspecified whether this byte is the exact sequence transmitted to the output device.

**Shell impact:** The shell must recognize `\r` in escape-sequence contexts: `printf '\r'`, `echo` with XSI interpretation, and `$'\r'` quoting. The shell writes byte 0x0D. The interactive line editor may need to handle `\r` in raw terminal input (it may appear instead of or alongside `\n` depending on terminal settings).
