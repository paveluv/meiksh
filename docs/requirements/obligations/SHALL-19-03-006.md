## Extracted Text

> If the previous character was used as part of an operator and the current character cannot be used with the previous characters to form an operator, the operator containing the previous character shall be delimited.

## Context

**Behavior:** Token recognition Rule 3: If the previous character was part of an operator token being built, and the current character CANNOT extend it to form a longer operator, then the operator built so far is delimited (completed and emitted to the parser). The current character is NOT consumed—it becomes the start of new tokenization (processed by subsequent rules).

**Applies to:** All shell modes during tokenization.

**Implementation detail:** This rule fires when rule 2 does not apply. The operator token accumulated so far is complete. The tokenizer resets to "start of token" state and re-evaluates the current character from rule 1 onward. For example, in `<<X`, after consuming `<<`, if `X` cannot extend `<<` to a valid operator, then `<<` is delimited as an operator token and `X` starts a new token (likely a word).

**Spec reference:** Section 2.3, rule 3, lines 263-264 in `utilities/V3_chap02.html`.
