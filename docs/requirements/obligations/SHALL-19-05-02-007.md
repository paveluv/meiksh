## Extracted Text

> Expands to the shortest representation of the decimal process ID of the invoked shell. In a subshell (see 2.13 Shell Execution Environment), '$' shall expand to the same value as that of the current shell.

## Context

This defines the expansion behavior of the special parameter `$$` (dollar-sign).

**Required behavior:**
- `$$` must expand to the shortest decimal representation of the process ID (PID) of the invoked shell.
- In a subshell (see 2.13 "Shell Execution Environment"), `$$` must expand to the same value as in the parent shell — i.e., the PID of the original (top-level) shell, NOT the PID of the subshell process.

**Conditions:** Applies universally. This is critical for subshells created by `( ... )`, command substitution `$(...)`, and pipe segments.

**Implementation note:** The shell must capture its own PID at initialization (via `getpid()`) and store it. When creating subshell environments, this stored value must be inherited unchanged. This contrasts with `$PPID` and `$!`, which have different subshell semantics.

See 2.5.2 "Special Parameters" (`$`) and 2.13 "Shell Execution Environment" in `utilities/V3_chap02.html`.
