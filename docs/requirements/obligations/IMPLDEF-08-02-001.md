## Extracted Text

> it refers to an implementation-provided locale, where settings of language, territory, and codeset are implementation-defined.

## Context

**Behavior:** When a locale value does not start with `/` and is not `"C"` or `"POSIX"`, it refers to an implementation-provided locale. The settings of language, territory, and codeset for such a locale are implementation-defined.

**Shell relevance:** The shell (via the C library) must support looking up implementation-provided locale names like `en_US.UTF-8`, `fr_FR.ISO-8859-1`, etc. The exact set of supported locale names and their mappings to language/territory/codeset are implementation-defined. `meiksh` should document which locales it supports (typically whatever the host OS provides via `setlocale()`).
