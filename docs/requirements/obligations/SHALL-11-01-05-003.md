## Extracted Text

> If the O_NONBLOCK flag is clear, then the read request shall be blocked until data is available or a signal has been received. If the O_NONBLOCK flag is set, then the read request shall be completed, without blocking, in one of three ways:: If there is not enough data available to satisfy the entire request, the read() shall complete successfully, having read as much data as possible, and shall return the number of bytes it was able to read.

## Context

**Behavior:** Non-blocking terminal read, case 2: If O_NONBLOCK is set and some data is available but not enough to fill the request, read() shall succeed with a partial read, returning the count of bytes actually read.

**Conditions:** O_NONBLOCK set, 0 < available bytes < requested bytes.

**Shell relevance:** The shell must handle short reads when using non-blocking I/O — it may need to issue additional reads to get a complete line.
