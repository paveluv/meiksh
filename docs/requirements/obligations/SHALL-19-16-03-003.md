> A loop shall enclose a break or continue command if the loop lexically encloses the command. ## Context

**Behavior:** First condition for lexical enclosure: the `break`/`continue` command must be executing in the **same execution environment** as the compound-list of the loop's do-group. This means a `break` inside a subshell `( )` within a loop does NOT break the loop — the subshell is a separate execution environment.

**Example:** `for i in 1 2 3; do (break); echo $i; done` — the `break` inside the subshell does not break the `for` loop; all three `echo` commands execute.

**Applies to:** All shell modes.

**Spec reference:** `break` DESCRIPTION in `utilities/V3_chap02.html`, lines 3140-3141. Cross-ref: 2.13 Shell Execution Environment, 2.10.2 Shell Grammar Rules.
