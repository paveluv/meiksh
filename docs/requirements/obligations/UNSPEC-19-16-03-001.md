## Extracted Text

> If n is greater than the number of lexically enclosing loops and there is a non-lexically enclosing loop in progress in the same execution environment as the break or continue command, it is unspecified whether that loop encloses the command.

## Context

**Behavior (unspecified):** If `n` is greater than the number of **lexically** enclosing loops, and there is a non-lexically-enclosing loop in progress in the same execution environment as the `break`/`continue` command, it is **unspecified** whether that loop encloses the command.

**Example:** This can occur with `eval` — `for i in 1 2 3; do eval 'break 2'; done` where the `break` is executed via `eval` and there may be ambiguity about whether the calling context's loop is considered enclosing.

**Implementation choice:** meiksh should document its behavior for this edge case. Common implementations either treat eval'd `break` as enclosed or ignore the non-lexical context.

**Applies to:** All shell modes.
