## Extracted Text

> The following rules and definitions apply to bracket expressions:: A character class expression shall represent the union of two sets: The set of single characters that belong to the character class, as defined in the LC_CTYPE category in the current locale. An unspecified set of multi-character collating elements. All character classes specified in the current locale shall be recognized. A character class expression is expressed as a character class name enclosed within bracket-<colon> ("[:" and ":]") delimiters. The following character class expressions shall be supported in all locales: [:alnum:] [:cntrl:] [:lower:] [:space:] [:alpha:] [:digit:] [:print:] [:upper:] [:blank:] [:graph:] [:punct:] [:xdigit:] In addition, character class expressions of the form: [:name:] are recognized in those locales where the name keyword has been given a charclass definition in the LC_CTYPE category.

## Context

**Behavior:** A character class expression represents the union of: (1) the set of single characters belonging to the character class as defined by `LC_CTYPE`, and (2) an unspecified set of multi-character collating elements. All character classes in the current locale shall be recognized. A character class is expressed as `[:name:]` inside a bracket expression. The following 12 classes shall be supported in all locales: `[:alnum:]`, `[:alpha:]`, `[:blank:]`, `[:cntrl:]`, `[:digit:]`, `[:graph:]`, `[:lower:]`, `[:print:]`, `[:punct:]`, `[:space:]`, `[:upper:]`, `[:xdigit:]`. Additional classes are recognized if defined in the locale's `LC_CTYPE`.

**Shell relevance:** The shell must support all 12 standard character classes in bracket expressions. For example, `[[:digit:]]` in a glob pattern matches any digit character. The membership of each class depends on `LC_CTYPE`. In the POSIX locale: `[:alpha:]` = `[A-Za-z]`, `[:digit:]` = `[0-9]`, `[:space:]` = space/tab/newline/etc. The shell must also accept locale-specific additional classes if the locale defines them.
