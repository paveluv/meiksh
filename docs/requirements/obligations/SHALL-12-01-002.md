## Extracted Text

> Frequently, names of parameters that require substitution by actual values are shown with embedded <underscore> characters. Alternatively, parameters are shown as follows: <parameter name> The angle brackets are used for the symbolic grouping of a phrase representing a single parameter and conforming applications shall not include them in data submitted to the utility.

## Context

This is a documentation convention requirement. The angle brackets `<` and `>` used in SYNOPSIS descriptions are symbolic notation for parameter placeholders and must not be included literally in data submitted to utilities. This is a constraint on conforming applications (users), not directly on the shell implementation. However, the shell's built-in help/usage messages should use this notation consistently, and the shell must not require users to include angle brackets when providing arguments.
