## Extracted Text

> When an io_here token has been recognized by the grammar (see 2.10 Shell Grammar), one or more of the subsequent lines immediately following the next NEWLINE token form the body of a here-document and shall be parsed according to the rules of 2.7.4 Here-Document. Any non-NEWLINE tokens (including more io_here tokens) that are recognized while searching for the next NEWLINE token shall be saved for processing after the here-document has been parsed. If a saved token is an io_here token, the corresponding here-document shall start on the line immediately following the line containing the trailing delimiter of the previous here-document. If any saved token includes a <newline> character, the behavior is unspecified.

## Context

**Behavior:** When the grammar recognizes an `io_here` token (`<<` or `<<-`), the here-document body starts on the line(s) immediately after the next NEWLINE token, and is parsed per 2.7.4 Here-Document rules. Several complex interactions are specified:

1. **Saved tokens:** Any non-NEWLINE tokens recognized while searching for the next NEWLINE are saved for processing AFTER the here-document is fully parsed. This means in `cat <<EOF; echo hello`, the `; echo hello` tokens are saved and executed after the here-document.

2. **Chained here-documents:** If a saved token is itself an `io_here`, its here-document starts on the line immediately after the trailing delimiter of the previous here-document. Example: `cmd <<A <<B` results in two consecutive here-document bodies.

3. **Unspecified behavior:** If any saved token includes a literal newline character, the behavior is unspecified. (This could happen via `$'\n'` in a token position.)

**Applies to:** All shell modes during tokenization/parsing.

**Implementation detail:** The parser needs a queue of pending here-document delimiters. After recognizing a NEWLINE, it processes here-documents in order before continuing with saved tokens. The here-document parser reads raw lines until the delimiter line is found.

**Spec reference:** Section 2.3, lines 240-244 in `utilities/V3_chap02.html`. Cross-refs: 2.10 Shell Grammar, 2.7.4 Here-Document.
