## Extracted Text

> If INLCR is set, a received NL character shall be translated into a CR character. If IGNCR is set, a received CR character shall be ignored (not read). If IGNCR is not set and ICRNL is set, a received CR character shall be translated into an NL character.

## Context

**Behavior:** Three `c_iflag` flags control NL/CR translation on input:
1. If INLCR is set, a received NL character shall be translated into a CR character.
2. If IGNCR is set, a received CR character shall be ignored (discarded, not delivered to the reader).
3. If IGNCR is not set and ICRNL is set, a received CR character shall be translated into an NL character.

**Conditions:** These translations are applied by the terminal driver in the input path. The flags interact: IGNCR takes precedence over ICRNL (CR is ignored rather than translated). INLCR is independent.

**Relevance to meiksh:** The `stty` builtin must correctly set/clear INLCR, IGNCR, and ICRNL. The shell relies on the OS terminal driver for actual translation. Proper configuration matters for interactive line editing and script input from terminals.

**Spec reference:** POSIX.1-2024, §11.2.2 Input Modes (tag_11_02_02), `basedefs/V1_chap11.html` line 523.
