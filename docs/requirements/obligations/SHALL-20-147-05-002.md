## Extracted Text

> The following operand shall be supported:: pid

## Context

**Behavior:** The `wait` utility shall accept `pid` operands. Each operand can be one of two forms:
1. An unsigned decimal integer representing a child process ID.
2. A job ID (e.g., `%1`, `%%`) that identifies a background job's process group (for job-control jobs) or process ID (for non-job-control background jobs).

**Conditions:** Zero or more pid operands can be specified. When none are given, `wait` waits for all known PIDs.
