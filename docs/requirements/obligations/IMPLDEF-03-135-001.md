## Extracted Text

> Implementation-defined security controls allowed by the file access permission and appropriate privileges (see also 3.21 Appropriate Privileges) mechanisms, through which an implementation can support different security policies from those described in POSIX.1-2024.

## Context

**Definition only — low shell relevance.** Defines "extended security controls" as implementation-defined security controls allowed by the file access permission and appropriate privileges mechanisms. They allow implementations to support different security policies (e.g., SELinux, AppArmor, macOS sandboxing).

**Shell impact:** The shell does not implement security controls but may be affected by them. Extended security controls could prevent the shell from executing commands, opening files, or forking processes. The shell should handle such failures gracefully with appropriate error messages. No direct implementation action needed.

**Spec reference:** basedefs/V1_chap03.html, section 3.135; see also 3.21 Appropriate Privileges.
