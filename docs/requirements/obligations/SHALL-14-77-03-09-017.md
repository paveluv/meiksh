## Extracted Text

> The <termios.h> header shall define the pid_t type as described in <sys/types.h>.

## Context

**Behavior:** `<termios.h>` **shall** make the `pid_t` type available (as described in `<sys/types.h>`). This is needed because the `tcgetpgrp()` and `tcsetpgrp()` functions use `pid_t` for process group IDs.

**Conditions:** Including `<termios.h>` must make `pid_t` available without requiring a separate `#include <sys/types.h>`.

**Shell relevance:** The shell uses `tcgetpgrp()` and `tcsetpgrp()` extensively for job control — getting and setting the foreground process group of the controlling terminal. Having `pid_t` available from `<termios.h>` simplifies includes. The shell uses these functions to:
- Move jobs between foreground and background
- Restore its own process group to the foreground after a child exits
- Check which process group currently owns the terminal
