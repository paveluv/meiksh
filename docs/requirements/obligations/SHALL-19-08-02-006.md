## Extracted Text

> If the command is not found, the exit status shall be 127.

## Context

**Behavior:** If the command is not found after PATH search or direct path lookup, the exit status MUST be 127. This is a standalone restatement of the rule in SHALL-19-08-02-002.

**Applies to:** Simple commands where the command name does not match any builtin, function, or executable on PATH, and slash-containing paths where the file does not exist.

**Error handling:** The shell must also write an error diagnostic to stderr (e.g., "command: not found").

**Implementation detail:** Same rule as SHALL-19-08-02-002. The implementation should have a single code path that returns 127 when command lookup fails.

**Spec reference:** Section 2.8.2, line 1701 in `utilities/V3_chap02.html`.
