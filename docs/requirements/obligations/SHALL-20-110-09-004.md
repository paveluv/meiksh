## Extracted Text

> If the shell is interactive:: If the -m option is in effect, SIGTTIN, SIGTTOU, and SIGTSTP signals shall be ignored.

## Context

**Behavior:** When the shell is **interactive** and the `-m` (monitor/job control) option is **enabled**, SIGTTIN, SIGTTOU, and SIGTSTP signals must be **ignored** (set to SIG_IGN). This is necessary for the shell to manage job control properly — the shell itself should not be stopped by terminal signals while managing foreground/background jobs.

**Implementation:** When job control mode is active, set SIGTTIN, SIGTTOU, and SIGTSTP to SIG_IGN in the shell process. Child processes in their own process groups will have these signals reset to default.

**Applies to:** Interactive shell with `-m` option enabled.

**Spec reference:** ASYNCHRONOUS EVENTS section in `utilities/sh.html`, lines 271-272.
