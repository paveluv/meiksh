## Extracted Text

> The following environment variables shall affect the execution of read:: Determine the locale for the interpretation of sequences of bytes of text data as characters (for example, single-byte as opposed to multi-byte characters in arguments).

## Context

**Behavior required:** This is the descriptive text for LC_CTYPE: it determines how byte sequences are interpreted as characters for the `read` builtin.

**Condition:** During character-level processing in `read`.

**Implementation detail:** LC_CTYPE controls whether the shell interprets input bytes as single-byte characters (C/POSIX locale) or multi-byte characters (e.g., UTF-8). This affects:
- How IFS characters are identified in the input (a multi-byte IFS character must be matched as a complete character, not individual bytes)
- How escaped bytes are interpreted
- The distinction between single-byte and multi-byte characters in arguments to `-d delim`
