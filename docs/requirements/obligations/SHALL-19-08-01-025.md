## Extracted Text

> If an unrecoverable read error occurs when reading commands, other than from the file operand of the dot special built-in, the shell shall execute no further commands (including any already successfully read but not yet executed) other than any specified in a previously defined EXIT trap action. An unrecoverable read error while reading from the file operand of the dot special built-in shall be treated as a special built-in utility error.

## Context

Defines the consequences of an unrecoverable read error in detail (note 4 from the table). See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required — read error on main command input:**
- If an unrecoverable read error occurs when reading commands (from script file, stdin, `sh -c`, etc.) — excluding the `dot` built-in's file operand — the shell SHALL execute NO further commands.
- This includes commands that were already successfully read but not yet executed. They are discarded.
- The ONLY exception: any action specified in a previously defined EXIT `trap` SHALL still be executed.

**Behavior required — read error on `dot` built-in's file:**
- An unrecoverable read error while reading from the file operand of the `dot` (`.`) special built-in SHALL be treated as a special built-in utility error (which means: exit in non-interactive, don't exit in interactive — see SHALL-19-08-01-004 and SHALL-19-08-01-005).

**Key references:** `dot` special built-in, `trap` special built-in.
