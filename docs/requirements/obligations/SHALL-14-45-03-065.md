## Context

In addition to the `si_code` constants, the `<signal.h>` header ([CX] extension) shall make available signal-specific information fields in `siginfo_t`. The table specifies which members are meaningful for each signal: `SIGILL`/`SIGFPE` provide `si_addr` (address of faulting instruction); `SIGSEGV` provides `si_addr` (address of faulting memory reference); `SIGBUS` provides `si_addr` (access address); `SIGCHLD` provides `si_pid` (child process ID), `si_status` (exit value or signal), and `si_uid` (real user ID of process that sent signal); `SIGPOLL` provides `si_band` (band event for POLL_IN, etc.). For `meiksh`, the `SIGCHLD`-specific fields (`si_pid`, `si_status`, `si_uid`) are directly relevant for job control and exit status tracking. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 1023–1119).
