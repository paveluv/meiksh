## Extracted Text

> A process shall be able to create a new process with all of the attributes referenced in 1.1.1.1 Process Attributes, determined according to the semantics of a call to the fork() function defined in the System Interfaces volume of POSIX.1-2024 followed by a call in the child process to one of the exec functions defined in the System Interfaces volume of POSIX.1-2024.

## Context

This is a duplicate extraction of the requirement that a process shall be able to create a new process with all standard process attributes via `fork()` followed by `exec()`. The child process inherits all attributes listed in 1.1.1.1 (file descriptors, environment, working directory, umask, user/group IDs, etc.) according to `fork()` semantics, then the child calls `exec()` to replace its image with the target program. For `meiksh`, this is the core command execution model. See POSIX Utilities Chapter 1, Section 1.1.1.2 (utilities/V3_chap01.html lines 75–78).
