## Extracted Text

> When a multi-character collating element in a bracket expression (see 9.3.5 RE Bracket Expression) is involved, the longest sequence shall be measured in characters consumed from the string to be matched; that is, the collating element counts not as one element, but as the number of characters it matches.

## Context

**Behavior:** When a multi-character collating element in a bracket expression is involved, the "longest sequence" is measured in characters consumed from the input string, not in collating elements. A multi-character collating element counts as the number of characters it matches, not as one element.

**Shell relevance:** If the locale defines multi-character collating elements (e.g., `ch` as a single collating element in some Spanish locales), the shell's bracket expression matching in glob patterns must consume the correct number of input characters. For example, `[[.ch.]]` in a bracket expression matches two characters (`c` and `h` together) from the input.
