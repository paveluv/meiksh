## Context

The LC_MONETARY locale definition section must recognize a set of mandatory keywords for monetary formatting. Operand values follow the grammar in §7.4. Integer keywords set to -1 or omitted keywords indicate the value is unavailable. Empty string values indicate either an available empty value or an unavailable value. The shell is not directly responsible for parsing locale definitions, but must correctly set and propagate the LC_MONETARY environment variable. The shell's `locale` builtin (if implemented) may need to query these values.

See: `basedefs/V1_chap07.html` §7.3.3 LC_MONETARY, §7.4 Locale Definition Grammar.
