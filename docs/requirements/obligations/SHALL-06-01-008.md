## Extracted Text

> Conforming implementations shall support certain character and character set attributes, as defined in 7.2 POSIX Locale.

## Context

The shell must support the character attributes defined by the POSIX locale (basedefs §7.2). This means the shell must correctly handle character classification (upper, lower, digit, space, etc.) as defined by the POSIX locale, which affects:

1. **Pattern matching**: Character classes like `[[:alpha:]]`, `[[:digit:]]`, `[[:space:]]` in pathname expansion and `case` patterns.
2. **IFS processing**: Space and tab recognition for field splitting.
3. **Variable name validation**: Identifying letters and digits for Name validation.
4. **`test` string comparisons**: Collation order for string comparison operators.

The reference to basedefs §7.2 (POSIX Locale) is already extracted as `basedefs/V1_chap07.html`.

See: `basedefs/V1_chap06.html` §6.1; `basedefs/V1_chap07.html` §7.2 POSIX Locale.
