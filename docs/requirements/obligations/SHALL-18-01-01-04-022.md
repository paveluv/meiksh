## Extracted Text

> The following codes are used in Actions when Creating a File that Already Exists:: Regular file. When attempting to create a regular file, and the existing file is a regular file: The user ID, group ID, and permission bits of the file shall not be changed. The file shall be truncated to zero length. The last data modification and last file status change timestamps shall be marked for update.

## Context

Action code "RF" (Regular File): when attempting to create a regular file and the existing file is already a regular file: (1) the user ID, group ID, and permission bits shall not be changed, (2) the file shall be truncated to zero length, and (3) the last data modification and last file status change timestamps shall be marked for update. For `meiksh`, this is the standard behavior for `> existing_file` — the shell opens the file with `O_WRONLY | O_CREAT | O_TRUNC`, which truncates the existing file to zero length while preserving ownership and permissions. The `noclobber` option (`set -C`) prevents this truncation unless `>|` is used. See POSIX Utilities Chapter 1, Section 1.1.1.4 action code RF (utilities/V3_chap01.html lines 672–676).
