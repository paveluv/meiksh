## Extracted Text

> If the current character is a '#', it and all subsequent characters up to, but excluding, the next <newline> shall be discarded as a comment. The <newline> that ends the line is not considered part of the comment.

## Context

**Behavior:** Token recognition Rule 9: If the current character is `#` (and it was not preceded by a word character—i.e., we're at the start of a token), it and all subsequent characters up to (but excluding) the next newline are discarded as a comment. The newline itself is NOT part of the comment and is processed normally (as a NEWLINE operator token).

**Applies to:** All shell modes during tokenization. The `#` must be in a position where a new token would start (after a blank, operator, or at start of input). If `#` appears in the middle of a word (rule 8 fires first), it is NOT treated as a comment.

**Implementation detail:** When `#` is encountered at a token-start position (no current word being built), enter comment mode: consume all characters until newline or EOF. The newline is not consumed—it remains for normal processing. This means `echo hello # comment` treats `#` as a comment start, but `echo he#llo` includes `#` in the word.

**Key nuance:** The `#` must be unquoted AND in a token-start position. Inside quotes, `#` is literal. After a word character without intervening blank/operator, `#` is part of the word (rule 8 has higher priority).

**Spec reference:** Section 2.3, rule 9, lines 301-302 in `utilities/V3_chap02.html`.
