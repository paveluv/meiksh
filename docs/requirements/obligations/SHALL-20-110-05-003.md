## Extracted Text

> The following operands shall be supported:: A single <hyphen-minus> shall be treated as the first operand and then ignored. If both '-' and "--" are given as arguments, or if other operands precede the single <hyphen-minus>, the results are undefined.

## Context

When `sh` receives a single `-` (hyphen-minus) as an operand, the shell must:

1. **Treat it as the first operand** in the operand list.
2. **Ignore it** — it has no effect on shell behavior (it does not enable login-shell mode or any other special mode in POSIX; login-shell behavior from `-` is a common extension but not mandated here).

**Undefined behavior combinations**: If both `-` and `--` are given as arguments, or if other operands precede the single `-`, the results are **undefined**. The implementation may handle these however it chooses (error, ignore, etc.).

This is distinct from `-` appearing as part of an option (e.g., `-c`). The `-` operand must appear in the operand position (after options and `--`).
