## Extracted Text

> This variable shall determine the locale category for processing affirmative and negative responses and the language and cultural conventions in which messages should be written. It also affects the behavior of the catopen() function in determining the message catalog. Additional semantics of this variable, if any, are implementation-defined. The language and cultural conventions of diagnostic and informative messages whose format is unspecified by POSIX.1-2024 should be affected by the setting of LC_MESSAGES .

## Context

**Behavior:** The environment variable `LC_MESSAGES` shall determine the locale category used for (1) processing affirmative and negative responses (e.g., yes/no patterns from `LC_MESSAGES` `yesexpr`/`noexpr`), and (2) the language and cultural conventions for diagnostic and informative messages written by utilities.

**Shell relevance:** When the shell or its built-in utilities emit diagnostic messages (e.g., error messages from `cd`, `export`, syntax errors), the language and format of those messages should be influenced by `LC_MESSAGES`. The shell must pass `LC_MESSAGES` through to child processes and honor it when formatting its own messages.

**Conditions:** This applies whenever a utility or the shell itself writes messages to stderr or stdout that are locale-sensitive. The variable follows the standard locale precedence: `LC_ALL` overrides `LC_MESSAGES`, which overrides `LANG`.

**Additional semantics:** Any additional semantics beyond message formatting are implementation-defined. The shell should document any extra behavior it ties to `LC_MESSAGES`.
