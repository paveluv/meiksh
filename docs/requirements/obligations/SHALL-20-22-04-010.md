## Extracted Text

> Write a string to standard output that indicates how the name given in the command_name operand will be interpreted by the shell, in the current shell execution environment (see 2.13 Shell Execution Environment), but do not invoke command_name. Although the format of this string is unspecified, it shall indicate in which of the following categories command_name falls and shall include the information stated: Executable utilities, regular built-in utilities, and any implementation-provided functions that are found using the PATH variable (as described in 2.9.1.4 Command Search and Execution), shall be identified as such and include the absolute pathname in the string. Other shell functions shall be identified as functions. Aliases shall be identified as aliases and their definitions included in the string. Special built-in utilities shall be identified as special built-in utilities. Regular built-in utilities not associated with a PATH search shall be identified as regular built-in utilities. (The term "regular" need not be used.) Shell reserved words shall be identified as reserved words.

## Context

**Behavior:** Duplicate extraction of the `-V` option description (same as SHALL-20-22-04-008). The `command -V` output must identify the category of each command_name and include the required information:

- Executables/PATH-found built-ins: identified as such + absolute pathname.
- Shell functions: identified as functions.
- Aliases: identified as aliases + definition included.
- Special built-ins: identified as special built-in utilities.
- Regular built-ins (not PATH-associated): identified as (regular) built-in utilities.
- Reserved words: identified as reserved words.

The exact output format is unspecified (implementation-defined), but the required information must be present.

**Spec reference:** POSIX.1-2024, `utilities/command.html`, section OPTIONS (tag_20_22_04), lines 88-112.
