## Extracted Text

> A character can be represented as a decimal constant. A decimal constant shall be specified as the escape character followed by a 'd' followed by two or three decimal digits. Each constant represents a byte value. Multi-byte values can be represented by concatenated constants specified in byte order with the last constant specifying the least significant byte of the character. For example: \d99;\d231;\d99\d104 "\d77\d97\d121"

## Context

**Behavior:** Duplicate of decimal constant rules from SHALL-07-03-017. Decimal constants in locale definition files use escape character + `d` + 2-3 decimal digits, each representing a byte. Multi-byte characters use concatenated constants.

**Shell relevance:** The shell does not parse locale definition files. This is `localedef` input encoding syntax. No shell implementation action needed.
