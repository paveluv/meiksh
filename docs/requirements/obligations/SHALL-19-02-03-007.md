## Extracted Text

> Enclosing characters in double-quotes ("") shall preserve the literal value of all characters within the double-quotes, with the exception of the characters backquote, <dollar-sign>, and <backslash>, as follows:: Outside of "$(...)" and "${...}" the <backslash> shall retain its special meaning as an escape character (see 2.2.1 Escape Character (Backslash)) only when immediately followed by one of the following characters: $ ` \ <newline> or by a double-quote character that would otherwise be considered special (see 2.6.4 Arithmetic Expansion and 2.7.4 Here-Document).

## Context

**Behavior:** Outside of `$(...)` and `${...}` constructs, backslash inside double-quotes retains its escape meaning ONLY when immediately followed by one of these characters:
- `$` (dollar-sign)
- `` ` `` (backquote)
- `\` (backslash)
- `<newline>` (line continuation—the backslash-newline pair is removed)
- `"` (double-quote) — but only a double-quote that would otherwise be considered special (specifically: the closing `"` of an arithmetic expansion `$(("..."))`, or a `"` within a here-document delimiter).

Before any other character (e.g., `\n`, `\t`, `\a`), the backslash is literal and both the `\` and the following character are preserved in the result.

**Applies to:** All shell modes, specifically in the double-quote context but NOT inside nested `$(...)` or `${...}` (which have their own backslash rules—inside `$(...)` normal unquoted rules apply; inside `${...}` for substring expansions, all characters are active).

**Examples:**
- `"hello\nworld"` → the `\` is literal, result is `hello\nworld` (7 + backslash + n + 5 characters)
- `"hello\\world"` → `\` escapes `\`, result is `hello\world`
- `"hello\$x"` → `\` escapes `$`, result is `hello$x` (no expansion)
- `"hello\"world"` — the `\"` case only applies to special double-quotes per 2.6.4/2.7.4

**Spec reference:** Section 2.2.3, lines 148-155 in `utilities/V3_chap02.html`. Cross-refs: 2.2.1, 2.6.4, 2.7.4.
