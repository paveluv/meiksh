## Extracted Text

> If fg does not move a job into the foreground, the following exit value shall be returned:

## Context

When `fg` fails to move a job to the foreground (e.g., invalid job_id, no such job, job control disabled), it must return an exit status >0. Note that when `fg` SUCCEEDS, it does NOT return its own exit status — instead, the shell waits for the foregrounded job and uses that job's exit status. The >0 error exit only applies when `fg` itself fails before moving the job. See SHALL-20-45-14-002 and SHALL-20-45-14-003 for the specific values. Reference: POSIX.1-2024 §fg, EXIT STATUS (tag_20_45_14).
