## Extracted Text

> An implementation may provide implementation-defined extended security controls (see 3.135 Extended Security Controls). These permit an implementation to provide security mechanisms to implement different security policies than those described in POSIX.1-2024. These mechanisms shall not alter or override the defined semantics of any of the interfaces in POSIX.1-2024.

## Context

**Requirement — low shell relevance.** An implementation MAY provide extended security controls (see 3.135). These SHALL NOT alter or override the defined semantics of any POSIX interface.

**Shell impact:** This guarantees that extended security mechanisms (SELinux, AppArmor, etc.) cannot change the semantics of shell interfaces — they can only restrict access. The shell can rely on POSIX interface semantics being preserved. If an operation fails due to extended security, the shell should report the error but doesn't need to understand the specific security mechanism.

**Spec reference:** basedefs/V1_chap04.html, section 4.6; see also `basedefs/V1_chap03.html#tag_03_135`.
