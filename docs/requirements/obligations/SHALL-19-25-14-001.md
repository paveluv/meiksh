## Extracted Text

> The exit status shall be n, if specified, except that the behavior is unspecified if n is not an unsigned decimal integer or is greater than 255. If n is not specified, the result shall be as if n were specified with the current value of the special parameter '?' (see 2.5.2 Special Parameters), except that if the return command would cause the end of execution of a trap action, the value for the special parameter '?' that is considered "current" shall be the value it had immediately preceding the trap action.

## Context

**Behavior:** Exit status rules for the `return` special built-in:

1. **With explicit n:** The exit status shall be `n` (the value of the operand).
   - If `n` is not an unsigned decimal integer or is greater than 255, behavior is unspecified — the implementation may treat it as an error or silently truncate.
2. **Without operand:** The exit status shall be the current value of `$?` (the exit status of the most recently executed command).
3. **Special case — trap action:** If `return` is executed inside a trap action and would cause the trap to end, the "current" value of `$?` is the value it had **immediately before** the trap action began executing — NOT the value set by commands within the trap action itself.

**Applies to:** All shell modes. `return` is valid inside functions and dot scripts, and within trap actions.

**Implementation detail:** The trap-action special case requires saving `$?` before entering a trap handler and restoring it as the "current" value when `return` (without operand) ends the trap. Example:
```
trap 'false; return' EXIT  # return without n uses $? from before the trap, not from 'false'
```
