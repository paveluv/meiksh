## Extracted Text

> If a mask operand is specified, there shall be no output written to standard output.

## Context

**Behavior:** When `umask` is invoked with a mask operand (i.e., in "set" mode, not "query" mode), the utility shall not write anything to stdout. It silently sets the mask.

**Conditions:** This applies regardless of whether `-S` is also specified. If a mask operand is present, no stdout output is produced.

**Error handling:** Diagnostic messages for invalid mask operands should still go to stderr (not stdout).
