## Extracted Text

> The command utility shall be treated as a declaration utility if the first argument passed to the utility is recognized as a declaration utility. In this case, subsequent words of the form name=word shall be expanded in an assignment context. See 2.9.1.1 Order of Processing.

## Context

**Behavior:** `command` must act as a declaration utility when its first argument is itself a declaration utility (e.g., `export`, `readonly`, `local`). When this condition is met:

1. `command` is classified as a declaration utility for the purpose of word expansion.
2. Words of the form `name=word` that follow the declaration utility are expanded in an assignment context (tilde expansion occurs after `=`, field splitting is suppressed for the value, etc.).

Example: `command export FOO=~/bar` — here `command` sees `export` as a declaration utility, so `FOO=~/bar` gets assignment-context expansion (tilde is expanded).

**Spec reference:** POSIX.1-2024, `utilities/command.html`, section DESCRIPTION (tag_20_22_03), lines 51-53. See also 2.9.1.1 Order of Processing.
