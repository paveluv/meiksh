## Context

**Behavior:** The POSIX locale's LC_CTYPE character classifications are defined with these constraints:
- The minimum character set is specified (ASCII letters, digits, punctuation, control characters).
- Implementations may add characters ONLY to `cntrl` and `punct` classes.
- No additions to `upper`, `lower`, `alpha`, `digit`, `alnum`, `space`, `graph`, `print`, `xdigit`, or `blank` are permitted in the POSIX locale.

**Shell relevance:** This is critical for predictable POSIX locale behavior:
- `[[:alpha:]]` in POSIX locale matches EXACTLY A-Za-z (no extensions).
- `[[:digit:]]` matches EXACTLY 0-9.
- `[[:space:]]` matches EXACTLY space, tab, newline, carriage-return, form-feed, vertical-tab.
- Only `[[:cntrl:]]` and `[[:punct:]]` may have implementation-specific extras.
- The shell's test suite should verify these exact POSIX locale classifications.
