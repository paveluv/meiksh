## Extracted Text

> When a directory that is the root directory or current working directory of any process is removed, the effect is implementation-defined. If file access permissions deny access, the requested operation shall fail. Otherwise, when a file is removed:: The last data modification and last file status change timestamps of the containing directory shall be marked for update.

## Context

When a file is removed, the last data modification and last file status change timestamps of the containing (parent) directory shall be marked for update (step 6 of file removal). This is because removing a directory entry modifies the directory's contents. For `meiksh`, this is OS-level behavior handled by `unlink()` / `rmdir()` system calls and not directly relevant to shell implementation logic. See POSIX Utilities Chapter 1, Section 1.1.1.5 item 6 (utilities/V3_chap01.html line 724).
