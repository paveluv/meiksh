## Extracted Text

> In a locale definition, the following mandatory keywords shall be recognized:: Define the appropriate representation of the ante-meridiem and post-meridiem strings, corresponding to the %p conversion specification. The operand shall consist of two strings, separated by a <semicolon>, each surrounded by double-quotes; the first string shall represent the ante-meridiem designation, the last string the post-meridiem designation. If and only if the 12-hour format is not supported in the locale, both strings shall be empty.

## Context

The `am_pm` keyword defines the ante-meridiem and post-meridiem strings corresponding to the `%p` conversion specification. The operand shall consist of two strings separated by a semicolon, each surrounded by double-quotes. The first string is the AM designation, the second is the PM designation. If and only if the 12-hour format is not supported in the locale, both strings shall be empty.

**Constraints:** Exactly 2 strings. Empty iff no 12-hour support. POSIX locale: "AM";"PM".

**Shell relevance:** Used by `date +%p`. See XBD 7.3.5.1.
