## Extracted Text

> If the file is a regular file, the permission bits of the file shall be set to: S_IROTH | S_IWOTH | S_IRGRP | S_IWGRP | S_IRUSR | S_IWUSR (see the description of File Modes in XBD 14. Headers , <sys/stat.h>) except that the bits specified by the file mode creation mask of the process shall be cleared. If the file is a directory, the permission bits shall be set to: S_IRWXU | S_IRWXG | S_IRWXO except that the bits specified by the file mode creation mask of the process shall be cleared.

## Context

Duplicate extraction of SHALL-18-01-01-04-005: for new regular files, permission bits are set to 0666 minus umask; for new directories, 0777 minus umask. For `meiksh`, when creating files via redirection, the shell passes mode 0666 to `open()` and the kernel applies the umask. The `umask` builtin controls the file mode creation mask. See POSIX Utilities Chapter 1, Section 1.1.1.4 item 3 (utilities/V3_chap01.html lines 92–98).
