## Extracted Text

> shall be implemented, regardless of whether they are also implemented as regular built-ins, in a manner so that they can be accessed via the exec family of functions as defined in the System Interfaces volume of POSIX.1-2024 and can be invoked directly by those standard utilities that require it (env, find, nice, nohup, time, xargs).

## Context

Even if a standard utility is implemented as a regular builtin, it **shall** also be accessible as an external executable via the `exec` family of functions. This means each standard utility must exist as a file in the filesystem (typically in `/usr/bin` or similar) so that programs like `env`, `find -exec`, `nice`, `nohup`, `time`, and `xargs` can invoke it directly without going through the shell. For `meiksh`, this means the shell cannot ONLY provide builtins — the system must also have corresponding external executables. The shell itself does not need to install these, but must not rely on builtin-only availability for utilities that other programs need to `exec`. Reference: XCU 1.6.
