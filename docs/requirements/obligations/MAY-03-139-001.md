## Extracted Text

> An object that can be written to, or read from, or both. A file has certain attributes, including access permissions and type. File types include regular file, character special file, block special file, FIFO special file, symbolic link, socket, and directory. Other types of files may be supported by the implementation.

## Context

**Definition — moderate shell relevance.** Defines "file" as an object that can be read/written with attributes (permissions, type). Standard file types: regular file, character special, block special, FIFO, symbolic link, socket, directory. Implementations may support additional types.

**Shell impact:** The shell interacts with all file types: regular files for scripts and redirections, directories for `cd` and globbing, FIFOs for process substitution and named pipes, sockets (less common). The `test`/`[` builtin tests file types (`-f`, `-d`, `-L`, `-p`, `-S`, `-b`, `-c`). The shell must handle all standard file types correctly when performing redirections and command execution.
