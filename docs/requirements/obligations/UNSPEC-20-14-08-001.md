## Extracted Text

> A pathname of the previous working directory, used when the operand is '-'. If an application sets or unsets the value of OLDPWD , the behavior of cd with a '-' operand is unspecified.

## Context

**Behavior (unspecified):** When an application (user script) sets or unsets the OLDPWD variable directly, and then invokes `cd -`, the behavior is unspecified by POSIX. The implementation is free to:
- Use the application-set value of OLDPWD as the target directory
- Produce an error
- Ignore OLDPWD and do something else

**Conditions:** This only applies when OLDPWD has been explicitly modified by the application (not by the shell's own cd mechanism). The shell normally sets OLDPWD automatically during cd execution.

**Implementation choice:** The implementation should document its chosen behavior. A reasonable approach is to use whatever value OLDPWD currently holds, regardless of who set it.

**Spec reference:** POSIX.1-2024, `utilities/cd.html`, section ENVIRONMENT VARIABLES (tag_20_14_08), lines 201-202.
