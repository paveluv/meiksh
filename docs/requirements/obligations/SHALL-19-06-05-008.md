## Extracted Text

> If the results of the algorithm are that no fields are delimited; that is, if the input field is wholly empty or consists entirely of IFS white space, the result shall be zero fields (rather than an empty field).

## Context

**Behavior:** When processing an input field through the field splitting algorithm, if no field is delimited (i.e., the entire input was consumed by IFS white space without any non-IFS content being accumulated), the result is **zero fields**, not one empty field.

This means:
- An expansion that produces an empty string (`$empty_var`) results in zero fields (the empty field is removed)
- An expansion that produces only IFS white space characters (e.g., `x="   "; $x`) results in zero fields
- This is what causes `echo $empty` to produce just a newline (no arguments) rather than an empty argument

**Contrast with quoted expansion:** `"$empty_var"` always produces exactly one field (the empty string), because quoting protects the field from splitting and removal.

**Spec reference:** See 2.6.5 Field Splitting in utilities/V3_chap02.html, lines 1245–1246.
