## Extracted Text

> A BRE special character has special properties in certain contexts. Outside those contexts, or when preceded by an unescaped <backslash>, such a character is a BRE that matches the special character itself. The BRE special characters and the contexts in which they have their special meaning are as follows:: The <dollar-sign> shall be special when used as an anchor.

## Context

**Behavior:** In BRE context, the dollar-sign (`$`) shall be special when used as an anchor (matches end-of-string).

**Shell relevance:** The `$` character is not a pattern matching operator in shell patterns. In shell context, `$` is used for parameter expansion, not as an anchor. Shell patterns match the entire string (in `case`) or are applied to each filename component (in globbing), so there is no concept of start/end anchors in shell pattern matching. This obligation is BRE-specific.
