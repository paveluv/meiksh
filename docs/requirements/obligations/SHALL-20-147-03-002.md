## Extracted Text

> If the wait utility is invoked with no operands, it shall wait until all process IDs known to the invoking shell have terminated and exit with a zero exit status.

## Context

**Behavior:** When `wait` is called with no arguments, it shall block until every process ID that is known to the current shell has terminated. Once all have terminated, `wait` shall exit with status 0.

**Conditions:** This includes all background jobs started by the current shell that have not yet been waited for. If there are no known PIDs (all have already terminated or none were started), `wait` returns immediately with exit status 0.

**Exit status:** Always 0 when no operands are given (regardless of the exit statuses of the individual waited-for processes).

**Side effect:** A successful `wait` with no operands discards the exit status information for all waited-for processes (they are removed from the known PID list).
