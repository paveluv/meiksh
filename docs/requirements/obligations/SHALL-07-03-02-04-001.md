## Context

After the `order_start` keyword in an LC_COLLATE locale definition, the collation order section must contain collating identifier entries that define the ordering of characters and collating elements. Each entry specifies a collating element and its optional weights. This defines the collation sequence that the shell uses indirectly through system collation functions (`strcoll()`, `wcscoll()`). The shell is affected because LC_COLLATE determines bracket expression range ordering in pattern matching (e.g., `[a-z]`), equivalence classes, and collating elements in patterns.

See: `basedefs/V1_chap07.html` §7.3.2.4 Collation Order.
