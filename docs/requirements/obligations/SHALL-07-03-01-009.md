## Extracted Text

> Define characters to be classified as white-space characters. In the POSIX locale, exactly <space>, <form-feed>, <newline>, <carriage-return>, <tab>, and <vertical-tab> shall be included. In a locale definition file, no character specified for the keywords upper, lower, alpha, digit, graph, or xdigit shall be specified. The <space>, <form-feed>, <newline>, <carriage-return>, <tab>, and <vertical-tab> of the portable character set, and any characters included in the class blank are automatically included in this class.

## Context

**Behavior:** The `space` keyword defines the set of white-space characters:
- In the POSIX locale: exactly space, form-feed, newline, carriage-return, tab, and vertical-tab.
- In other locales: these 6 characters plus `blank` class members are automatically included, plus locale-specific whitespace.
- Constraint: No character in `space` may also be in `upper`, `lower`, `alpha`, `digit`, `graph`, or `xdigit`.

**Shell relevance:** The shell uses the `space` class for:
- `[[:space:]]` bracket expression in pattern matching.
- `IFS` whitespace splitting — the default `IFS` is space+tab+newline, but locale whitespace characters may interact with field splitting.
- The `isspace()` function may be used in various shell processing contexts.
