## Extracted Text

> If utility is specified and is executed, exec shall not return to the shell; rather, the exit status of the current shell execution environment shall be the exit status of utility. If utility is specified and an attempt to execute it as a non-built-in utility fails, the exit status shall be as described in 2.9.1.6 Non-built-in Utility Execution. If a redirection error occurs (see 2.8.1 Consequences of Shell Errors), the exit status shall be a value in the range 1-125. Otherwise, exec shall return a zero exit status.

## Context

**Behavior:** Exit status rules for the `exec` special built-in:

1. **Utility specified and successfully executed:** `exec` does not return to the shell. The process image is replaced. The exit status of the shell process (as seen by its parent) is the exit status of the executed utility.
2. **Utility specified but execution fails:** The exit status shall follow 2.9.1.6 Non-built-in Utility Execution rules:
   - **126:** The utility was found but could not be executed (e.g., not executable, ENOEXEC).
   - **127:** The utility was not found.
3. **Redirection error (no utility, or before exec):** If a redirection error occurs, the exit status shall be in the range 1-125. This is critical because `exec` with only redirections modifies the current shell's file descriptors — a redirection failure here is a shell error per 2.8.1.
4. **No utility, no error:** If `exec` is used with only redirections and they all succeed, `exec` shall return exit status 0.

**Applies to:** All shell modes (interactive and non-interactive).
