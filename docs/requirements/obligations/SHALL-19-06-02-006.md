## Extracted Text

> Use Default Values. If parameter is unset or null, the expansion of word (or an empty string if word is omitted) shall be substituted; otherwise, the value of parameter shall be substituted.

## Context

**Behavior:** The `${parameter:-word}` expansion (Use Default Values):
- If `parameter` is **unset or null** (empty string): expand `word` and substitute the result. If `word` is omitted, substitute an empty string.
- If `parameter` is **set and non-null**: substitute the value of `parameter` as-is; `word` is not expanded.

The no-colon variant `${parameter-word}` tests only for unset (not null) — see SHALL-19-06-02-010 for the colon semantics.

**Conditions:** Applies during parameter expansion whenever the `:-` (or `-`) operator is encountered inside `${...}`.

**Inputs/Outputs:**
- `${HOME:-/tmp}` → if HOME is unset or empty, yields `/tmp`; otherwise yields `$HOME`
- `${unset_var:-}` → if unset_var is unset or empty, yields empty string
- `${unset_var-default}` (no colon) → only yields `default` if unset_var is truly unset; if set to empty, yields empty

**Error handling:** No error conditions; this is a safe fallback mechanism.

**Spec reference:** See 2.6.2 Parameter Expansion in utilities/V3_chap02.html, lines 807–809.
