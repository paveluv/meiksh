## Extracted Text

> Any command that is preceded by count shall take a count (the numeric value of any preceding decimal digits). Unless otherwise noted, this count shall cause the specified operation to repeat by the number of times specified by the count. Also unless otherwise noted, a count that is out of range is considered an error condition and shall alert the terminal, but neither the cursor position, nor the command line, shall change.

## Context

**Behavior:** Any vi command mode command preceded by a count (accumulated decimal digits) shall take that count as the numeric value of the digits. Unless otherwise noted per command, the count causes the operation to repeat that many times. Also unless otherwise noted, a count that is out of range is an error: the terminal shall be alerted (bell), and neither the cursor position nor the command line shall change.

**Conditions:** Applies to all vi command mode commands that accept a count. Individual command descriptions may override the out-of-range behavior (e.g., some commands clamp to the line boundary instead of alerting).

**Implementation:** After parsing the count, pass it to the command handler. Each handler decides how to interpret the count. The default behavior for out-of-range is: ring bell, no-op. Commands that specify "shall not be considered an error" override this.
