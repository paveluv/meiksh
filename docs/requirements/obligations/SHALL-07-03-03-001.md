## Extracted Text

> The LC_MONETARY category shall define the rules and symbols that are used to format monetary numeric information.

## Context

The LC_MONETARY locale category defines symbols and formatting rules for monetary values (currency symbol, decimal point, thousands separator, sign positioning, etc.). The shell is affected because the `printf` builtin's `%'d` and `%'f` formatting may use locale-specific grouping, and locale environment variables (LC_MONETARY, LC_ALL) are part of the shell's environment. However, the shell itself does not directly format monetary values — this is primarily consumed by C library functions like `strfmon()` and `localeconv()`. The shell must correctly propagate the LC_MONETARY environment variable to child processes.

See: `basedefs/V1_chap07.html` §7.3.3 LC_MONETARY.
