## Extracted Text

> The subscript values are unique, except that the VMIN and VTIME subscripts may have the same values as the VEOF and VEOL subscripts, respectively.

## Context

**Behavior:** The `c_cc` array subscript constants (VINTR, VQUIT, VERASE, VKILL, VEOF, VEOL, VSUSP, VSTART, VSTOP, VMIN, VTIME) are all unique, with one exception: VMIN **may** have the same value as VEOF, and VTIME **may** have the same value as VEOL. This is because canonical mode uses VEOF/VEOL while non-canonical mode uses VMIN/VTIME, so they can share array slots.

**Conditions:** This aliasing is platform-dependent. On systems where VMIN==VEOF and VTIME==VEOL, setting VMIN also overwrites VEOF and vice versa.

**Shell relevance:** When meiksh switches between canonical mode (for basic input) and non-canonical mode (for line editing), it must be aware that changing VMIN/VTIME may destroy the VEOF/VEOL values and vice versa. The shell must save the full termios structure before switching modes and restore it when switching back. This is critical: if VEOF (^D) is overwritten when setting VMIN=1 for raw mode, canonical mode will no longer recognize EOF correctly until restored.

**Spec reference:** POSIX.1-2024, §11.2.6 Special Control Characters (tag_11_02_06), `basedefs/V1_chap11.html` line 1351.
