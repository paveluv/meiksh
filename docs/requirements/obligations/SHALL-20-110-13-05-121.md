## Extracted Text

> Perform pathname expansion (see 2.6.6 Pathname Expansion) on the current bigword, up to the largest set of characters that can be matched uniquely. If the bigword contains none of the characters '?', '*', or '[', an <asterisk> ('*') shall be implicitly assumed at the end. This maximal expansion then shall replace the original bigword in the command line, and the cursor shall be placed after this expansion. If the resulting bigword completely and uniquely matches a directory, a '/' character shall be inserted directly after the bigword. If some other file is completely matched, a single <space> shall be inserted after the bigword. After this operation, sh shall be placed in insert mode.

## Context

**Behavior:** The `\` command shall perform pathname expansion on the current bigword up to the largest uniquely matchable prefix. If no glob chars, `*` is appended. The expansion replaces the bigword, cursor placed after expansion. Unique directory match appends `/`, unique file match appends space. Shell enters insert mode.

**Details:** Same normative content as SHALL-20-110-13-05-014. See `utilities/V3_chap02.html#tag_19_06_06` for pathname expansion.
