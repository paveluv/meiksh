## Extracted Text

> A sequence of characters starting with a <dollar-sign> immediately followed by a single-quote ($') shall preserve the literal value of all characters up to an unescaped terminating single-quote ('), with the exception of certain <backslash>-escape sequences, as follows:: \cX yields the control character listed in the Value column of Values for cpio c_mode Field in the OPERANDS section of the stty utility when X is one of the characters listed in the ^c column of the same table, except that \c\\ yields the <FS> control character since the <backslash> character has to be escaped.

## Context

**Behavior:** Inside `$'...'`, the escape sequence `\cX` yields a control character. The mapping is defined by the "Values for cpio c_mode Field" table in the OPERANDS section of the `stty` utility page. The `X` must be one of the characters listed in the `^c` column of that table.

**Special case:** `\c\\` (backslash-c-backslash-backslash) yields the FS (file separator, 0x1C) control character, because the backslash after `\c` must itself be escaped.

**Standard control character mappings (from the stty table):**
- `\c@` → NUL (0x00), `\cA`/`\ca` → SOH (0x01), `\cB`/`\cb` → STX (0x02), ... through `\cZ`/`\cz` → SUB (0x1A)
- `\c[` → ESC (0x1B), `\c\\` → FS (0x1C), `\c]` → GS (0x1D), `\c^` → RS (0x1E), `\c_` → US (0x1F), `\c?` → DEL (0x7F)

**Applies to:** All shell modes during `$'...'` escape processing.

**Implementation detail:** After seeing `\c`, read the next character. If it's `\`, read one more (consuming the escaped backslash) and map to FS. Otherwise map X to the control character value. If the resulting character has no encoding in the current locale, the result is implementation-defined (see SHALL-19-02-04-019).

**Spec reference:** Section 2.2.4, lines 202-206 in `utilities/V3_chap02.html`. Cross-ref: `utilities/stty.html` (control character table).
