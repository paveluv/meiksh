## Extracted Text

> Certain characters have special functions on input or output or both. These functions are summarized as follows:: Special character on input, which is recognized if the ICANON flag is set; it is the <carriage-return> character. When ICANON and ICRNL are set and IGNCR is not set, this character shall be translated into an NL, and shall have the same effect as an NL character. It cannot be changed.

## Context

**Behavior:** The CR (carriage-return, '\r') special character:
1. Recognized on input when ICANON is set in `c_lflag`.
2. When ICANON and ICRNL are set (in `c_iflag`) and IGNCR is **not** set, CR shall be translated into NL and shall have the same effect as NL (i.e., it acts as a line delimiter).
3. The CR character **cannot be changed** — it is always '\r' (0x0D). It is not configurable via the `c_cc` array.

**Conditions:** ICANON must be set. The translation only occurs when ICRNL is set and IGNCR is not set. If IGNCR is set, CR is discarded entirely. If ICRNL is not set, CR is passed through as-is.

**Shell relevance:** This translation is important for terminal input: the Enter key often sends CR, and the ICRNL translation converts it to NL so the shell sees a newline (line terminator). The shell should ensure ICRNL is set for normal operation. When the shell's line editor is in non-canonical mode, ICANON is not set, so CR processing described here does not apply — but ICRNL input flag translation (which is independent of ICANON) still occurs.

**Spec reference:** POSIX.1-2024, §11.1.9 Special Characters (tag_11_01_09), `basedefs/V1_chap11.html` line 258.
