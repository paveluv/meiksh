## Extracted Text

> The following codes are used in Actions when Creating a File that Already Exists:: Open FIFO. When attempting to create a regular file, and the existing file is a FIFO special file: If the FIFO is not already open for reading, the attempt shall block until the FIFO is opened for reading. Once the FIFO is open for reading, the utility shall open the FIFO for writing and continue with its operation.

## Context

Action code "O" (Open FIFO): when attempting to create a regular file but the existing file is a FIFO special file: (1) if the FIFO is not already open for reading, the attempt shall block until the FIFO is opened for reading by another process, and (2) once the FIFO is open for reading, the utility shall open the FIFO for writing and continue. For `meiksh`, when `> fifo` is used in redirection and `fifo` is a named pipe, the shell's `open()` call will block until a reader opens the other end of the FIFO. This is normal FIFO behavior and the shell must handle it (including potential deadlock if the same shell is both reader and writer). See POSIX Utilities Chapter 1, Section 1.1.1.4 action code O (utilities/V3_chap01.html lines 662–665).
