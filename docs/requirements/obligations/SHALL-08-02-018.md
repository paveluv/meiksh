## Extracted Text

> If the LC_ALL environment variable is defined and is not null, the value of LC_ALL shall be used.

## Context

**Behavior:** Duplicate of the core rule in SHALL-08-02-014: If `LC_ALL` is defined and non-null, its value shall be used for all locale categories. This is the highest-priority override in the locale precedence chain.

**Shell relevance:** Same as SHALL-08-02-014. The shell must check `LC_ALL` first when determining any locale category value.

**Spec reference:** Section 8.2 "Internationalization Variables", line 607 in `basedefs/V1_chap08.html`.
