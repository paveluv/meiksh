## Extracted Text

> The collating order entries shall be terminated with an order_end keyword.

## Context

The `order_end` keyword in the LC_COLLATE locale definition marks the end of the collation order entries that began with `order_start`. This is a syntactic requirement for locale definition files. The shell is affected indirectly because it consumes locale data produced from these definitions. The shell itself does not parse locale definition files, but the system's locale infrastructure must enforce this syntax to produce correct collation data that the shell uses for pattern matching and string comparison.

See: `basedefs/V1_chap07.html` §7.3.2.5 The order_end Keyword.
