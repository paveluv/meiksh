## Extracted Text

> If OPOST is set, output data shall be post-processed as described below, so that lines of text are modified to appear appropriately on the terminal device; otherwise, characters shall be transmitted without change.

## Context

**Behavior:** When OPOST is set in `c_oflag`, output data shall be post-processed by the terminal driver — characters like NL, CR, TAB, BS, FF, and VT may be modified according to other output mode flags (ONLCR, OCRNL, etc.) before being sent to the terminal. When OPOST is not set, characters shall be transmitted without any modification.

**Relevance to meiksh:** The `stty` builtin must correctly set/clear the OPOST flag. OPOST is the master switch for all output processing; other output mode flags only take effect when OPOST is set.

**Spec reference:** POSIX.1-2024, §11.2.3 Output Modes (tag_11_02_03), `basedefs/V1_chap11.html` line 786.
