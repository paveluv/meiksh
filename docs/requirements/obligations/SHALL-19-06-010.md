## Extracted Text

> When the expansions in this section are performed other than in the context of preparing a command for execution, they shall be carried out in the current shell execution environment.

## Context

This defines the execution environment rule for non-command-execution expansions.

**Required behavior:**
- When word expansions are performed in a context OTHER than preparing a command for execution (e.g., in variable assignments like `var=$((1+2))`, in `case` words, in `[[ ]]` expressions, etc.), the expansions must be carried out in the **current shell execution environment** — not in a subshell.
- This means side effects of expansions (e.g., `${var:=default}` which assigns a default value) persist in the current shell.

**Conditions:** Applies to all expansion contexts that are not direct command argument/name preparation.

See 2.6 "Word Expansions" in `utilities/V3_chap02.html`.
