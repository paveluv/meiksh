## Extracted Text

> Typing an escape character shall switch sh into command mode (see vi Line Editing Command Mode). In command mode, an entered character shall either invoke a defined operation, be used as part of a multi-character operation, or be treated as an error. A character that is not recognized as part of an editing command shall terminate any specific editing command and shall alert the terminal. If sh receives a SIGINT signal in command mode (whether generated by typing the interrupt character or by other means), it shall terminate command line editing on the current command line, reissue the prompt on the next line of the terminal, and reset the command history (see fc) so that the most recently executed command is the previous command (that is, the command that was being edited when it was interrupted is not re-entered into the history).

## Context

**Behavior:** Vi-mode command mode behavior:

1. **Entering command mode:** Typing ESC switches from insert mode to command mode.
2. **Command mode input:** Each entered character either invokes a defined editing operation, is part of a multi-character operation, or is an error.
3. **Unrecognized characters:** A character not recognized as part of an editing command must terminate any in-progress specific editing command and **alert the terminal** (ring the bell / send BEL).
4. **SIGINT handling in command mode:** If the shell receives SIGINT while in command mode (whether from the interrupt character or other means):
   - Terminate command line editing on the current line.
   - Reissue the prompt on the next line of the terminal.
   - Reset the command history so the most recently executed command is the previous command (the command being edited when interrupted is **not** re-entered into history).

**Applies to:** Interactive shell in vi editing mode.
