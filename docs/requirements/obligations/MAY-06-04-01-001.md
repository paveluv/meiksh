## Extracted Text

> This section addresses the use of state-dependent character encodings (that is, those in which the encoding of a character is dependent on one or more shift codes that may precede it).

## Context

**Behavior (optional):** State-dependent character encodings (where the interpretation of a byte sequence depends on preceding shift codes) may be supported. This is an optional capability — systems are not required to support locking-shift or single-shift encodings.

**Shell relevance:** If the shell's platform supports state-dependent encodings, the shell must handle them correctly in text processing. In practice, modern systems use stateless encodings (UTF-8), so this is unlikely to affect most implementations. However, if supporting such encodings, the shell must treat shift-code+character sequences as multi-byte characters.
