## Extracted Text

> For C-language programs, the POSIX locale shall be the default locale when the setlocale() function is not called.

## Context

**Behavior:** When a C-language program does not call `setlocale()`, the default locale is the POSIX locale (C locale). This means all locale-dependent functions behave as if in the POSIX locale.

**Shell relevance:** If the shell does not call `setlocale(LC_ALL, "")` at startup, it will operate in the POSIX locale by default, ignoring the user's `LANG`/`LC_*` environment variables. The shell MUST call `setlocale(LC_ALL, "")` at startup to honor the user's locale settings. The ref to `functions/setlocale.html` is a C API page — not a shell-relevant file to extract.
