## Extracted Text

> The unalias utility shall remove the definition for each alias name specified. See 2.3.1 Alias Substitution. The aliases shall be removed from the current shell execution environment; see 2.13 Shell Execution Environment.

## Context

**Behavior:** The `unalias` builtin shall remove the definition for each alias name given as an operand. After removal, those alias names will no longer be subject to alias substitution during token recognition (Section 2.3.1).

**Scope:** Alias removal affects the current shell execution environment (Section 2.13). This means:
- Aliases removed in a subshell do not affect the parent shell.
- Aliases removed in the current shell remain removed for the lifetime of the shell unless re-defined with `alias`.

**Multiple operands:** When multiple alias-name operands are given, each shall be removed independently. If any operand does not match a currently defined alias, an error is reported for that specific name but others are still processed.

**Spec reference:** Section DESCRIPTION (tag_20_133_03) in `utilities/unalias.html`, lines 39-41. See also Section 2.3.1 Alias Substitution and Section 2.13 Shell Execution Environment.
