## Extracted Text

> A subexpression repeated by an <asterisk> ('*') or an interval expression shall not match a null expression unless this is the only match for the repetition or it is necessary to satisfy the exact or minimum number of occurrences for the interval expression.

## Context

**Behavior:** A subexpression repeated by `*` or an interval expression shall not match a null expression unless it is the only match for the repetition or is necessary to satisfy the exact/minimum count of the interval expression. This prevents greedy matching from producing unnecessary null matches.

**Shell relevance:** This is BRE-specific (repetition quantifiers). Shell pattern matching does not have subexpression repetition. However, the principle of avoiding unnecessary null matches is relevant to how shell `*` wildcard works — `*` should greedily consume characters and only match the empty string when no characters remain or when necessary.
