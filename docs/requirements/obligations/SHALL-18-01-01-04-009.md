## Extracted Text

> When a file that does not exist is created, the following features defined in the System Interfaces volume of POSIX.1-2024 shall apply unless the utility or function description states otherwise:: Unless otherwise specified, the file created shall be a regular file.

## Context

Unless otherwise specified by the utility description, the file created shall be a regular file. For `meiksh`, this means that shell output redirections (`> file`, `>> file`) create regular files by default. The shell uses `open()` with `O_CREAT` which creates regular files. Special file types (FIFOs, devices, etc.) are not created by the shell's redirection operators. See POSIX Utilities Chapter 1, Section 1.1.1.4 item 7 (utilities/V3_chap01.html line 105).
