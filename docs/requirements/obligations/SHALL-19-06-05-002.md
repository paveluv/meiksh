## Extracted Text

> For the remainder of this section, any reference to the results of an expansion, or results of expansions, shall be interpreted to mean the results from one or more unquoted variable or arithmetic expansions, or unquoted command substitutions.

## Context

**Behavior:** This is a definitional statement for the field splitting section. When the spec refers to "results of an expansion" in the context of field splitting, it specifically means bytes that came from:
- Unquoted parameter/variable expansion (`$var`, `${var}`)
- Unquoted arithmetic expansion (`$((expr))`)
- Unquoted command substitution (`$(cmd)`, `` `cmd` ``)

Bytes that did NOT result from an expansion (literal text in the word) are treated differently by the field splitting algorithm — they are never treated as delimiters and always pass through unchanged.

**Implementation significance:** The field splitting algorithm must track which bytes in a field came from expansions vs. which are literal text. Only expansion-result bytes are candidates for IFS delimiter matching.

**Spec reference:** See 2.6.5 Field Splitting in utilities/V3_chap02.html, lines 1209–1210.
