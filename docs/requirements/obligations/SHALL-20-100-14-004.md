## Extracted Text

> The following exit values shall be returned:: 1

## Context

**Behavior required:** The `read` builtin must return exit status 1 specifically when end-of-file is detected.

**Condition:** When EOF is reached on stdin before a terminating line delimiter is encountered.

**Implementation detail:** Return exactly 1 (not just any non-zero) when:
- EOF is detected before a newline (or `-d delim` character) is read
- Variables are still set from whatever partial input was available (per SHALL-20-100-03-018)

This is commonly used in `while read` loops: `while read line; do ...; done < file` — the loop terminates when `read` returns 1 (EOF).
