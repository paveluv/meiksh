## Extracted Text

> shall duplicate one output file descriptor from another, or shall close one. If word evaluates to one or more digits, the file descriptor denoted by n, or standard output if n is not specified, shall be made to be a copy of the file descriptor denoted by word; if the digits in word do not represent an already open file descriptor, a redirection error shall result (see 2.8.1 Consequences of Shell Errors); if the file descriptor denoted by word represents an open file descriptor that is not open for output, a redirection error may result. If word evaluates to '-', file descriptor n, or standard output if n is not specified, is closed. Attempts to close a file descriptor that is not open shall not constitute an error. If word evaluates to something else, the behavior is unspecified.

## Context

Defines the behavior of the output file descriptor duplication operator `[n]>&word`. See 2.7.6 Duplicating an Output File Descriptor in utilities/V3_chap02.html.

**Syntax:** `[n]>&word`

**Behavior required — word evaluates to digits:**
- The fd denoted by `n` (or fd 1 if `n` is omitted) SHALL be made a copy of the fd denoted by `word` (using `dup2(word_fd, n_fd)` or equivalent).
- If `word` does not represent an already-open fd, a redirection error SHALL result (see 2.8.1 tag_19_08_01).
- If `word` represents an open fd that is not open for output (i.e., open for read-only), a redirection error MAY result.

**Behavior required — word evaluates to `-`:**
- The fd denoted by `n` (or fd 1 if `n` is omitted) SHALL be closed.
- Attempting to close an fd that is not open SHALL NOT constitute an error.

**Behavior — word evaluates to something else:**
- The behavior is unspecified.

**Common idiom:** `2>&1` duplicates stdout to stderr. `>&-` closes stdout.

**Note:** The default fd is 1 (stdout) for `>&`, unlike `<&` which defaults to 0 (stdin).
