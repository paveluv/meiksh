## Extracted Text

> With both the backquoted and $(commands) forms, the commands string shall be tokenized (see 2.3 Token Recognition) and parsed (see 2.10 Shell Grammar). It is unspecified whether the commands string is parsed and executed incrementally as a program (as for a shell script), or is parsed as a single compound_list that is executed after the string has been completely parsed. In addition, it is unspecified whether the terminating ')' of the $(commands) form can result from alias substitution. With the $(commands) form any syntactically correct program can be used for commands, except that:: If the commands string is parsed as a single compound_list, before any commands are executed, alias and unalias commands in commands have no effect during parsing (see 2.3.1 Alias Substitution). Strictly conforming applications shall ensure that the commands string does not depend on alias changes taking effect incrementally as would be the case if parsed and executed as a program.

## Context

**Behavior:** When a command substitution's body is parsed as a single `compound_list` (parsed completely before execution begins), any `alias` or `unalias` commands within the body have no effect on the parsing of that same body. This is because alias substitution happens during tokenization, which is already complete before execution starts.

Strictly conforming applications must not rely on alias definitions within a command substitution affecting the parsing of later commands in the same substitution. This is a portability constraint — some shells parse incrementally (where aliases take effect), others parse all-at-once (where they don't).

**Implementation guidance:** If meiksh parses `$(...)` contents all at once (common approach), aliases defined/removed within the substitution body won't affect parsing of subsequent commands in that same body. This is acceptable behavior.
