## Extracted Text

> A collation sequence definition shall define the relative order between collating elements (characters and multi-character collating elements) in the locale. This order is expressed in terms of collation values; that is, by assigning each element one or more collation values (also known as collation weights). This does not imply that implementations shall assign such values, but that ordering of strings using the resultant collation definition in the locale behaves as if such assignment is done and used in the collation process. ## Context

**Behavior:** The LC_COLLATE category defines a collation sequence — the relative ordering of characters and multi-character collating elements. The ordering is expressed through collation values/weights. Implementations must behave "as if" weights are assigned, even if the internal representation differs.

**Shell relevance:** The shell uses collation for:
- Pattern matching range expressions: `[a-z]` in bracket expressions uses the collation sequence to determine which characters fall in the range.
- The `case` statement pattern matching also uses collation-based ranges.
- `sort` order within shell scripts (though `sort` is an external utility).
- `strcoll()` / `wcscoll()` functions used for string comparison in locale-aware contexts.

In the POSIX locale, collation order equals encoding order (byte values). In other locales, collation can differ significantly from encoding order.
