## Extracted Text

> The following operand shall be supported:: job_id

## Context

The `fg` builtin must support an optional `job_id` operand that specifies which background job to bring to the foreground. The `job_id` format is defined in XBD 3.182 Job ID and includes forms like `%n` (job number), `%string` (prefix match), `%?string` (substring match), `%%` or `%+` (current job), and `%-` (previous job). If no `job_id` is given, the default is the current job. Reference: POSIX.1-2024 §fg, OPERANDS (tag_20_45_05).
