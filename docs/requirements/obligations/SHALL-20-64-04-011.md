## Extracted Text

> (The letter ell.) Write all values of signal_name supported by the implementation, if no operand is given. If an exit_status operand is given and it is a value of the '?' shell special parameter (see 2.5.2 Special Parameters and wait) corresponding to a process that was terminated or stopped by a signal, the signal_name corresponding to the signal that terminated or stopped the process shall be written. If an exit_status operand is given and it is the unsigned decimal integer value of a signal number, the signal_name (the symbolic constant name without the SIG prefix defined in the Base Definitions volume of POSIX.1-2024) corresponding to that signal shall be written. Otherwise, the results are unspecified.

## Context

**Behavior:** Duplicate extraction of the `-l` option description (same as SHALL-20-64-04-004). The `-l` option behavior:
1. **No operand:** List all supported signal names to stdout.
2. **exit_status matching `$?` for a signaled process:** Write the signal name that caused termination/stop. Since shells typically encode signal exits as `128 + signum`, `kill -l` with such a value must reverse this to produce the signal name.
3. **exit_status is a raw signal number:** Write the signal name corresponding to that number.
4. **Other values:** Unspecified results.

**Output format:** Signal names are uppercase, without the `SIG` prefix (e.g., `HUP`, `INT`, `KILL`).
