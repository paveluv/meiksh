## Extracted Text

> POSIX.1-2024 places only the following requirements on the encoded values of the characters in the portable character set:: The encoded values associated with the digits 0 to 9 shall be such that the value of each character after 0 shall be one greater than the value of the previous character.

## Context

The digit characters '0' through '9' must have contiguous, ascending encoded values. For the shell, this is critical for:

1. **Arithmetic expansion `$(( ))`**: The shell converts digit characters to numeric values. The contiguous encoding guarantee means `'5' - '0' == 5` always works.
2. **String-to-integer conversion**: Used in `$?` exit status interpretation, `$!` PID handling, parameter expansion indices, `test` numeric comparisons (`-eq`, `-lt`, etc.), and the `read` builtin.
3. **Character class `[:digit:]`**: Pattern matching with `[0-9]` or `[[:digit:]]` can rely on contiguous digit encoding.

This is guaranteed by all common character sets (ASCII, UTF-8, EBCDIC). The shell can safely use `c - '0'` arithmetic for digit-to-value conversion.
