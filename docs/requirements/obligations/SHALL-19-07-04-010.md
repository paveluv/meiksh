## Extracted Text

> Any <backslash> characters in the input shall behave as the <backslash> inside double-quotes (see 2.2.3 Double-Quotes). However, the double-quote character ('"') shall not be treated specially within a here-document, except when the double-quote appears within "$()", "``", or "${}".

## Context

This is a duplicate extraction of the backslash/double-quote rule from SHALL-19-07-04-006. See 2.7.4 Here-Document in utilities/V3_chap02.html and 2.2.3 Double-Quotes (tag_19_02_03).

**Behavior required:**
- In an unquoted here-document, backslash behaves as inside double-quotes (special only before `$`, `` ` ``, `"`, `\`, `<newline>`).
- The double-quote character is NOT special in a here-document body, except within `$()`, `` ` ` ``, or `${}` constructs.

See SHALL-19-07-04-006 for full context. This obligation and SHALL-19-07-04-006 cover the same requirement from the same source lines.
