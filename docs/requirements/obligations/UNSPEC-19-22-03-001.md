## Extracted Text

> If n is specified and is not an unsigned decimal integer, or has a value of 256, or has a value greater than 256 but not corresponding to an exit status the shell assigns to commands terminated by a valid signal, the wait status of the shell or subshell is unspecified.

## Context

**Behavior (unspecified):** If `n` is specified to `exit` and is:
- Not an unsigned decimal integer, OR
- Has a value of exactly 256, OR
- Has a value greater than 256 but does not correspond to a signal-based exit status the shell assigns,

then the resulting wait status of the shell or subshell is **unspecified**. The implementation may truncate, wrap, or handle the value however it chooses.

**Implementation choice:** Common practice is to use `n & 0xFF` (mask to 8 bits), but this is not required.

**Applies to:** All shell modes.
