## Extracted Text

> The <signal.h> header shall define the following symbolic constants for the values of sigev_notify:: A queued signal, with an application-defined value, is generated when the event of interest occurs.

## Context

This describes the semantics of `SIGEV_SIGNAL`: a queued signal is generated with an application-defined value when the event occurs. The signal number comes from `sigev_signo` and the value from `sigev_value`. For `meiksh`, this mechanism allows the shell to receive signal-based notifications for timer expiration or asynchronous I/O completion, carrying additional context via the signal value. This is a system header semantics requirement.
