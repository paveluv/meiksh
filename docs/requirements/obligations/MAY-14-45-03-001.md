## Extracted Text

> Child has terminated abnormally and additional actions may have been taken.

## Context

This describes the `CLD_DUMPED` `si_code` value for `SIGCHLD` in the `siginfo_t` structure: when `si_code` equals `CLD_DUMPED`, it means the child process terminated abnormally and additional actions may have been taken (such as creating a core dump). The word "may" here refers to the platform possibly taking additional actions (like core dumping), not to a conformance optionality. For `meiksh`, when handling `SIGCHLD` with `SA_SIGINFO`, the shell should recognize `CLD_DUMPED` as indicating abnormal termination with possible core dump, which affects job status reporting (similar to `CLD_KILLED` but with core dump indication). See POSIX `<signal.h>` DESCRIPTION si_code table (basedefs/signal.h.html lines 900–903).
