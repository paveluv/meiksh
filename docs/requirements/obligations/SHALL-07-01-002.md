## Extracted Text

> The standard utilities in the Shell and Utilities volume of POSIX.1-2024 shall base their behavior on the current locale, as defined in the ENVIRONMENT VARIABLES section for each utility. The behavior of some of the C-language functions defined in the System Interfaces volume of POSIX.1-2024 shall also be modified based on a locale selection. The locale to be used by these functions can be selected in the following ways:: For functions such as isalnum_l() that take a locale object as an argument, a locale object can be obtained from newlocale() or duplocale() and passed to the function.

## Context

**Behavior:** C-language functions that take a locale object argument (like `isalnum_l()`) can use locale objects created by `newlocale()` or `duplocale()`. This is one mechanism for locale selection.

**Shell relevance:** The shell implementation may use `_l`-suffixed locale functions (e.g., `isalpha_l()`, `isupper_l()`) for thread-safe locale-aware character classification. This is an implementation option, not a requirement. The refs to `functions/` pages are C API documentation — not shell-relevant files to extract.
