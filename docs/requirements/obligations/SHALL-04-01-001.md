## Extracted Text

> When a standard utility or function that uses regular expressions or pattern matching specifies that matching shall be case insensitive, then if a string would match the regular expression or pattern when doing a case-sensitive match, the same string with any of its characters replaced with their case counterparts, as defined by the toupper and tolower character mappings (see 7.3.1 LC_CTYPE), shall also match when doing a case-insensitive match.

## Context

**Requirement — direct shell relevance.** When a utility or function specifies case-insensitive matching for regular expressions or patterns, replacing any character with its `toupper` or `tolower` counterpart (as defined by LC_CTYPE) SHALL also produce a match.

**Shell impact:** Case-insensitive matching applies to the shell in these contexts:
- The `case` statement does NOT do case-insensitive matching by default (it uses pattern matching, which is case-sensitive)
- External utilities invoked by the shell (e.g., `grep -i`) implement their own case-insensitive matching
- If `meiksh` provides any extension for case-insensitive globbing or matching, it must follow this rule
- The `toupper`/`tolower` mappings come from the LC_CTYPE locale category

The shell must ensure its locale-aware character comparison functions use the correct mappings from `basedefs/V1_chap07.html#tag_07_03_01`.

**Spec reference:** basedefs/V1_chap04.html, section 4.1; see also `basedefs/V1_chap07.html#tag_07_03_01` (already extracted).
