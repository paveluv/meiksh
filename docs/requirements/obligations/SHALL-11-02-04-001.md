## Extracted Text

> The c_cflag field describes the hardware control of the terminal, and is composed of the bitwise-inclusive OR of the masks shown, which shall be bitwise-distinct. The mask name symbols in this table are defined in <termios.h>; not all values specified are required to be supported by the underlying hardware (if any). If the terminal is a pseudo-terminal, it is unspecified whether non-default values are unsupported, or are supported and emulated in software, or are handled by tcsetattr(), tcgetattr(), and the stty utility as if they are supported but have no effect on the behavior of the terminal interface.

## Context

**Behavior:** The `c_cflag` field describes hardware control of the terminal, composed of bitwise-inclusive OR of masks that shall be bitwise-distinct. Mask names are defined in `<termios.h>`. Not all values are required to be supported by underlying hardware. For pseudo-terminals, it is unspecified whether non-default `c_cflag` values are unsupported, emulated in software, or handled by `tcsetattr()`/`tcgetattr()`/`stty` as if supported but with no effect.

**Relevance to meiksh:** The `stty` builtin must correctly read/set `c_cflag` bits. Since meiksh typically operates on pseudo-terminals, the shell must handle the case where control mode settings are accepted but may have no hardware effect. The shell must not error on valid `c_cflag` values even when running on a pty.

**Spec reference:** POSIX.1-2024, §11.2.4 Control Modes (tag_11_02_04), `basedefs/V1_chap11.html` lines 813–818.
