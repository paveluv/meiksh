## Extracted Text

> The following exit values shall be returned:: A specified command_file could not be executed due to an [ENOEXEC] error (see 2.9.1.4 Command Search and Execution, item 2).

## Context

**Behavior:** The `sh` utility shall return exit status 126 when a specified command_file could not be executed because the system returned an ENOEXEC error. Per 2.9.1.4 Command Search and Execution item 2:
- The file was found.
- An attempt was made to execute it (e.g., via `execve()`).
- The system returned ENOEXEC, meaning the file is not in a recognized executable format.

**Applies to:** Non-interactive `sh` invocations with a command_file operand (e.g., `sh myscript.sh` where myscript.sh triggers ENOEXEC).

**Details:** In practice, this situation can arise when:
- A script has a `#!` line pointing to a non-existent interpreter.
- A binary file is for a different architecture.
- The file has execute permission but is not a valid script or binary.

Note: Per 2.9.1.4 item 2, when ENOEXEC occurs during normal command execution, the shell re-invokes itself to interpret the file as a shell script. However, when `sh` itself is invoked with the file as command_file, this re-invocation logic may not apply — the 126 exit status indicates the initial exec failure.
