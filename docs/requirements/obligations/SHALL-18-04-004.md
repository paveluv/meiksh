## Extracted Text

> The STDIN section describes the standard input of the utility. This section is frequently merely a reference to the following section, as many utilities treat standard input and input files in the same manner. Unless otherwise stated, all restrictions described in the INPUT FILES section shall apply to this section as well. Use of a terminal for standard input can cause any of the standard utilities that read standard input to stop when used in the background. For this reason, applications should not use interactive features in scripts to be placed in the background. The specified standard input format of the standard utilities shall not depend on the existence or value of the environment variables defined in this volume of POSIX.1-2024, except as provided by this volume of POSIX.1-2024. Default Behavior: When this section is listed as "Not used.", it means that the standard input shall not be read when the utility is used as described by this volume of POSIX.1-2024.

## Context

For STDIN handling in shell builtins: (1) All restrictions from the INPUT FILES section also apply to stdin. (2) Using a terminal for stdin can cause utilities that read stdin to stop (SIGTTOU/SIGTTIN) when run in the background — applications should not use interactive features in background scripts. (3) The stdin format **shall not** depend on environment variable values except as explicitly specified. (4) When STDIN is listed as "Not used.", stdin **shall not** be read. The shell must ensure that builtins like `read`, `.` (dot), and the shell itself properly handle stdin according to these rules. Reference: XCU 1.4.
