## Extracted Text

> If IXON is set, start/stop output control shall be enabled. A received STOP character shall suspend output and a received START character shall restart output. When IXON is set, START and STOP characters are not read, but merely perform flow control functions. When IXON is not set, the START and STOP characters shall be read.

## Context

**Behavior:** When IXON is set in `c_iflag`, start/stop output control is enabled. A received STOP character (typically Ctrl-S) shall suspend output, and a received START character (typically Ctrl-Q) shall restart output. When IXON is set, START and STOP characters are consumed by the flow control mechanism and are not delivered to the reading process. When IXON is not set, the START and STOP characters shall be read as normal input data.

**Relevance to meiksh:** The `stty` builtin must correctly set/clear the IXON flag. This is especially relevant for interactive shells where Ctrl-S/Ctrl-Q flow control can affect terminal behavior. The shell's line editor must be aware that these characters may not be received when IXON is enabled.

**Spec reference:** POSIX.1-2024, §11.2.2 Input Modes (tag_11_02_02), `basedefs/V1_chap11.html` line 527.
