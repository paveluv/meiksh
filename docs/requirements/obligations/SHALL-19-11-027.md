## Extracted Text

> When a suspended job is created as a result of a foreground job being stopped, it shall be assigned a job number, and an interactive shell shall write, and a non-interactive shell may write, a message to standard error, formatted as described by the jobs utility (without the -l option) for a suspended job. The message may indicate that the commands comprising the job include commands that have already completed; in this case the completed commands shall not be repeated if execution of the job is subsequently continued. If the shell is interactive, it shall save the terminal settings before changing them to the settings it needs to read further commands.

## Context

**Suspended job notification and terminal save:** When a suspended job is created from a stopped foreground job:

1. **Job number assignment:** The suspended job shall be assigned a job number (from the job table).
2. **Status message:** An interactive shell shall (and a non-interactive shell may) write a message to stderr in the format described by the `jobs` utility (without `-l`), e.g., `[1]+  Stopped                 sleep 100`. See `utilities/jobs.html` for the format.
3. **Completed commands:** The message may mention commands that have already completed; these shall not be re-executed if the job is resumed.
4. **Terminal settings save:** If the shell is interactive, it shall save the current terminal settings (`tcgetattr()`) before changing them to the settings needed for reading commands. This saved state is restored by `fg` when the job is resumed (see SHALL-19-11-033).

The implementation must integrate with the `jobs` output format and manage terminal attribute save/restore.
