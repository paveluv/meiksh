## Extracted Text

> Next, the shell shall treat this as an arithmetic expression and substitute the value of the expression. The arithmetic expression shall be processed according to the rules given in 1.1.2.1 Arithmetic Precision and Operations, with the following exceptions:: Only signed long integer arithmetic is required.

## Context

**Behavior:** Shell arithmetic is only required to support **signed long integer** arithmetic. The implementation must use at least a C `signed long` (minimum 32 bits, typically 64 bits on modern platforms). The shell is NOT required to support:
- Unsigned arithmetic
- Short or char arithmetic types
- Floating-point arithmetic (though it may as an extension per SHALL-19-06-04-010)

The minimum range is at least −2,147,483,647 to +2,147,483,647 (32-bit signed long), but modern implementations typically use 64-bit.

**Overflow behavior:** Overflow of signed integer arithmetic is technically undefined behavior in C; the shell should handle this gracefully (wrapping or producing an error).
