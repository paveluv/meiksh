## Extracted Text

> If a modem disconnect is detected by the terminal interface for a controlling terminal, and if CLOCAL is not set in the c_cflag field for the terminal (see 11.2.4 Control Modes), the SIGHUP signal shall be sent to the controlling process for which the terminal is the controlling terminal. Unless other arrangements have been made, this shall cause the controlling process to terminate (see exit()). Any subsequent read from the terminal device shall return the value of zero, indicating end-of-file; see read(). Thus, processes that read a terminal file and test for end-of-file can terminate appropriately after a disconnect. If the EIO condition as specified in read() also exists, it is unspecified whether on EOF condition or [EIO] is returned. Any subsequent write() to the terminal device shall return -1, with errno set to [EIO], until the device is closed.

## Context

**Behavior:** When a modem disconnect (hangup) is detected on a controlling terminal and CLOCAL is not set:
1. SIGHUP shall be sent to the **controlling process** (typically the shell).
2. Unless the controlling process has arranged to handle or ignore SIGHUP, this causes it to terminate.
3. Any subsequent `read()` on the terminal device shall return 0 (EOF).
4. Any subsequent `write()` on the terminal device shall return -1 with `errno = EIO` until the device is closed.
5. If both EIO and EOF conditions exist, which is returned by `read()` is **unspecified**.

**Conditions:** CLOCAL must not be set in `c_cflag`. The terminal must be a controlling terminal with an active modem connection.

**Shell relevance:** When the terminal connection drops (user closes terminal emulator, SSH disconnects, etc.), the shell receives SIGHUP. The shell must handle SIGHUP: save history, send SIGHUP to its job list (if the jobs have not been disowned), and exit. The `disown` builtin prevents SIGHUP from being sent to specific jobs. The shell must also handle the case where reads return EOF after hangup during command processing.

**Spec reference:** POSIX.1-2024, §11.1.10 Modem Disconnect (tag_11_01_10), `basedefs/V1_chap11.html` lines 275–283. See also §11.2.4 Control Modes.
