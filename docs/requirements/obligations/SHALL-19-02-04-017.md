## Extracted Text

> In cases where a variable number of characters can be used to specify an escape sequence (\xXX and \ddd), the escape sequence shall be terminated by the first character that is not of the expected type or, for \ddd sequences, when the maximum number of characters specified has been found, whichever occurs first.

## Context

**Behavior:** For variable-length escape sequences inside `$'...'` (specifically `\xXX` and `\ddd`), the escape sequence terminates at whichever comes first:
1. The first character that is NOT of the expected type (not a hex digit for `\x`, not an octal digit for `\ddd`), OR
2. The maximum number of characters has been consumed (2 hex digits for `\x`, 3 octal digits for `\ddd`).

**Applies to:** All shell modes during `$'...'` escape processing.

**Implementation detail:** For `\x`: consume 1-2 hex digits, stop at non-hex or after 2. For `\ddd`: consume 1-3 octal digits, stop at non-octal or after 3. Remaining characters after the escape are processed normally (either as literal text or as the start of the next escape sequence).

**Examples:**
- `$'\x41B'` → consumes `\x41` (= `A`), then `B` is literal → `AB`
- `$'\101C'` → consumes `\101` (= `A`), then `C` is literal → `AC`
- `$'\1z'` → consumes `\1` (= 0x01), then `z` is literal
