## Extracted Text

> The <signal.h> header shall also define the following symbolic constants:: SA_NODEFER

## Context

The `<signal.h>` header shall define the symbolic constant `SA_NODEFER`. This flag is used in the `sa_flags` member of `struct sigaction`. When set, the signal being handled is not automatically added to the process's signal mask during execution of the signal handler. By default (without `SA_NODEFER`), the signal is blocked while its handler runs to prevent recursive invocation. For `meiksh`, this flag is generally not needed since the shell typically wants signals to be blocked during handler execution to avoid reentrancy issues. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html line 539).
