## Extracted Text

> In all other cases, the value of OPTIND is unspecified, but shall encode the information needed for the next invocation of getopts to resume parsing options after the option just parsed.

## Context

This obligation (duplicate extraction of the OPTIND-unspecified-encoding rule) covers the case where `getopts` parses an option that does NOT take an option-argument, or encounters an error (unknown option). In these cases, `OPTIND`'s exact value is unspecified — it may remain the same index (if more options are grouped in the current parameter, e.g., parsing `-a` from `-abc`) or advance. The only requirement is that the value encodes enough state for the next `getopts` call to correctly resume. Implementations commonly use a sub-index scheme (e.g., storing the character position within a grouped options parameter). Reference: POSIX.1-2024 §getopts, DESCRIPTION (tag_20_53_03).
