## Extracted Text

> When it is not processing an io_here, the shell shall break its input into tokens by applying the first applicable rule below to each character in turn in its input. At the start of input or after a previous token has just been delimited, the first or next token, respectively, shall start with the first character that has not already been included in a token and is not discarded according to the rules below. Once a token has started, zero or more characters from the input shall be appended to the token until the end of the token is delimited according to one of the rules below. When both the start and end of a token have been delimited, the characters forming the token shall be exactly those in the input between the two delimiters, including any quoting characters. If a rule below indicates that a token is delimited, and no characters have been included in the token, that empty token shall be discarded.

## Context

**Behavior:** This is the master rule for how the tokenizer breaks input into tokens. It establishes the fundamental tokenization algorithm:

1. **Character-by-character processing:** The first applicable rule (from the ordered list of rules 1-10) is applied to each character.
2. **Token start:** At the start of input or after a previous token was delimited, the next token starts with the first non-discarded character.
3. **Token growth:** Once started, characters are appended until a rule delimits the token's end.
4. **Token content:** The token contains exactly the characters between its start and end delimiters, including any quoting characters (quotes are preserved in the token at this stage—they are removed later during quote removal).
5. **Empty token discard:** If a delimiting rule fires but no characters have been accumulated, the empty token is discarded (not passed to the parser).

**Applies to:** All shell modes during tokenization (when not in here-document mode).

**Implementation detail:** The tokenizer is a state machine. Key states: start-of-token, in-word, in-operator, in-quoted-string (with sub-states for single-quote, double-quote, dollar-single-quote, backslash-escape). Rules are tried in priority order (1 = highest).

**Rule ordering (numbered 1-10):**
1. End of input → delimit (SHALL-19-03-004)
2. Continue operator (SHALL-19-03-005)
3. Delimit operator (SHALL-19-03-006)
4. Quoting (SHALL-19-03-007)
5. Expansion candidates (SHALL-19-03-008)
6. Start new operator (SHALL-19-03-009)
7. Blank → discard (SHALL-19-03-010)
8. Continue word (SHALL-19-03-011)
9. Comment (SHALL-19-03-012)
10. Start new word
