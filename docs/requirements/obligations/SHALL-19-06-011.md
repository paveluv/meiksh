## Extracted Text

> When expanding words for a command about to be executed, and the word will be the command name or an argument to the command, the expansions shall be carried out in the current shell execution environment. (The environment for the command to be executed is unknown until the command word is known.)

## Context

This defines the execution environment rule for command-execution word expansions.

**Required behavior:**
- When expanding words for a command about to be executed, and the word will become the command name or an argument to the command, the expansions must be carried out in the **current shell execution environment**.
- The rationale is that the target command's environment is unknown until the command word itself is determined (after expansion). Therefore, expansions that produce the command name and arguments must happen in the current shell.
- Side effects of these expansions (e.g., `${var:=value}`) affect the current shell.

**Conditions:** Applies specifically to command word and argument expansion during command preparation. Contrasts with UNSPEC-19-06-001 which covers the unspecified case for redirections and variable assignments in commands.

See 2.6 "Word Expansions" in `utilities/V3_chap02.html`.
