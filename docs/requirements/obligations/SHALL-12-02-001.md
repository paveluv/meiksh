## Extracted Text

> Each option name should be a single alphanumeric character (the alnum character classification) from the portable character set. The -W (capital-W) option shall be reserved for vendor options. Multi-digit options should not be allowed.

## Context

Guideline 3 of the Utility Syntax Guidelines: option names must be single alphanumeric characters from the portable character set. The `-W` option is reserved for vendor-specific extensions and must not be used for standard functionality. The shell's builtins must follow this: each option is a single letter/digit, `-W` is reserved, and multi-digit options (like `-10`) must not be defined for standard builtins. If `meiksh` defines vendor-specific extensions, they should use `-W` as the prefix.
