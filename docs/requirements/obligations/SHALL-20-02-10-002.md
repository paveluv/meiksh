## Extracted Text

> The value string shall be written with appropriate quoting so that it is suitable for reinput to the shell. See the description of shell quoting in 2.2 Quoting.

## Context

The value string in `alias` output **shall** be written with appropriate quoting so that it is suitable for reinput to the shell. This means the output must be valid shell syntax — if the alias value contains special characters (spaces, `$`, backticks, quotes, etc.), they must be properly quoted (e.g., with single quotes, or escaping). The output of `alias` should be directly usable as input to recreate the aliases (e.g., `eval "$(alias)"`). See XCU 2.2 (Quoting) for quoting rules. Reference: alias utility STDOUT, XCU 2.2.
