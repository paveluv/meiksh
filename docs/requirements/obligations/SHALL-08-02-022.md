## Extracted Text

> If the locale value is "C" or "POSIX", the POSIX locale shall be used and the standard utilities behave in accordance with the rules in 7.2 POSIX Locale for the associated category.

## Context

**Behavior:** If the effective locale value (after applying the precedence rules) is the string `"C"` or `"POSIX"`, the POSIX locale shall be used and all standard utilities shall behave according to the rules in section 7.2 (POSIX Locale) for the associated category.

**Shell relevance:** When `LC_ALL=C` (or `POSIX`), the shell must use POSIX locale rules for: (1) `LC_CTYPE` — only single-byte ASCII characters, `[:alpha:]` is `[A-Za-z]`, etc.; (2) `LC_COLLATE` — collation order is by byte value, so `[a-z]` matches only lowercase ASCII; (3) `LC_MESSAGES` — messages in English; (4) pattern matching uses byte-value collation. This is the baseline portable behavior.

**Spec reference:** Section 8.2 "Internationalization Variables", lines 616-617 in `basedefs/V1_chap08.html`; section 7.2 for POSIX locale rules.
