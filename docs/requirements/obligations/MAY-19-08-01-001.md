## Extracted Text

> Non-Interactive Shell: Command not found -- may exit

## Context

A non-interactive shell MAY exit when a command is not found. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior (optional):**
- When the shell is non-interactive and a command is not found (not in PATH, not a function, not a built-in), the shell MAY exit.
- A shell diagnostic message IS required (the "command not found" message).
- The exit status for the failed command itself is 127 (see 2.8.2).

**Decision for meiksh:** Choose whether a non-interactive shell exits on command-not-found or continues. Common practice (bash, dash, etc.) is to NOT exit — just set `$?` to 127 and continue. Exiting would make scripts fragile. Recommendation: do not exit.

**Contrast:** An interactive shell SHALL NOT exit on command-not-found (see SHALL-19-08-01-020).
