## Extracted Text

> Next, the shell shall treat this as an arithmetic expression and substitute the value of the expression. The arithmetic expression shall be processed according to the rules given in 1.1.2.1 Arithmetic Precision and Operations, with the following exceptions:: Selection, iteration, and jump statements are not supported.

## Context

**Behavior:** Shell arithmetic expressions must NOT support C-language control flow statements:
- **Selection statements:** `if`, `switch` (note: the C ternary operator `?:` IS supported as it is an expression, not a statement)
- **Iteration statements:** `for`, `while`, `do`
- **Jump statements:** `break`, `continue`, `return`, `goto`

These are C statement-level constructs that have no meaning in shell arithmetic expressions, which are purely expression-based.

**What IS supported:** All C arithmetic expression operators including assignment (`=`, `+=`, etc.), comma (`,`), ternary (`?:`), logical (`&&`, `||`), comparison, bitwise, shift, and basic arithmetic operators.

**Spec reference:** See 2.6.4 Arithmetic Expansion in utilities/V3_chap02.html, line 1176.
