## Context

**Behavior:** All standard utilities (including the shell and its builtins) must base their behavior on the current locale. The locale affects character classification (e.g., what is an `alpha` character), collation order (affects pattern matching with `[a-z]`), numeric formatting, date/time formatting, and message translation.

**Shell relevance:** The shell must honor locale settings for:
- Character classification in pattern matching (`[[:alpha:]]`, `[[:digit:]]`, etc.)
- Collation order in bracket expressions (`[a-z]`)
- `LC_MESSAGES` for shell diagnostic messages
- `LC_NUMERIC` for arithmetic (though POSIX shell arithmetic uses C locale rules)
- `LC_CTYPE` for multi-byte character handling throughout tokenization and expansion

Each builtin's ENVIRONMENT VARIABLES section specifies which locale categories it uses.
