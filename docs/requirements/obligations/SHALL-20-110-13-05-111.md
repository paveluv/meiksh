## Extracted Text

> The following commands shall be recognized in command mode:: Move backwards through the command history, searching for the specified pattern, beginning with the previous command line. Patterns use the pattern matching notation described in 2.14 Pattern Matching Notation, except that the '^' character shall have special meaning when it appears as the first character of pattern. In this case, the '^' is discarded and the characters after the '^' shall be matched only at the beginning of a line. Commands in the command history shall be treated as strings, not as filenames. If the pattern is not found, the current command line shall be unchanged and the terminal shall be alerted. If it is found in a previous line, the current command line shall be set to that line and the cursor shall be set to the first character of the new command line. If pattern is empty, the last non-empty pattern provided to / or ? shall be used. If there is no previous non-empty pattern, the terminal shall be alerted and the current command line shall remain unchanged.

## Context

**Behavior:** In vi command mode, `/pattern<newline>` searches BACKWARD (toward older entries) through the command history for a line matching `pattern`, starting from the previous command line. Patterns use the pattern matching notation from 2.14 Pattern Matching Notation (see `utilities/V3_chap02.html#tag_19_14`), with the special exception that `^` at the start of the pattern anchors the match to the beginning of the line (the `^` is discarded).

**Details:**
- History commands are treated as strings, not filenames.
- If not found: alert terminal, command line unchanged.
- If found: set current command line to that history entry, cursor on first character.
- If pattern is empty: reuse the last non-empty pattern from `/` or `?`. If there is no previous pattern, alert terminal, no change.

**Spec reference:** vi Line Editing Command Mode in `utilities/sh.html`, line 643–651. See `utilities/V3_chap02.html#tag_19_14` for pattern matching notation.
