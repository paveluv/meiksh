## Extracted Text

> The exit status of a grouping command shall be the exit status of compound-list.

## Context

A grouping command is either `{ compound-list; }` (brace group, executes in the current shell environment) or `( compound-list )` (subshell group, executes in a subshell environment). In both cases, the exit status of the grouping command must be the exit status of the `compound-list` that was executed within the group.

The `compound-list` is a sequence of AND-OR lists. Its exit status is determined by the last AND-OR list executed within it.

Examples:
- `{ true; false; }` — exit status is 1 (from `false`)
- `( exit 42 )` — exit status is 42
- `{ true; }` — exit status is 0

See 2.9.4.1 Grouping Commands in utilities/V3_chap02.html#tag_19_09_04_01.
