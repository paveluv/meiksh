## Extracted Text

> The name of an existing or nonexisting shell variable. If a var operand names the variable IFS , the behavior is unspecified. If a var operand names one of the variables LANG , LC_CTYPE , or LC_ALL and the new value assigned to the variable would change how the bytes in IFS form characters, or which characters in IFS are considered to be IFS white space (see 2.6.5 Field Splitting), it is unspecified what effects, if any, the change has on how read performs field splitting.

## Context

**Behavior:** The following are explicitly unspecified:
1. **`read IFS`**: Reading into the `IFS` variable itself. The implementation may set IFS to the read value, may ignore it, or may produce an error. Any behavior is conforming.
2. **Reading into `LANG`, `LC_CTYPE`, or `LC_ALL`**: If the new value would change how IFS bytes form characters or which characters are IFS whitespace, the effects on the current `read` field splitting are unspecified.

**Implementation freedom:** The implementation can:
- Allow `read IFS` and apply the new IFS value mid-processing, or defer it
- Allow `read LANG` and either use the old locale or the new one for remaining splitting
- Reject these as errors (also conforming)
