## Extracted Text

> Changes made to the subshell environment shall not affect the shell environment. Command substitution, commands that are grouped with parentheses, and asynchronous AND-OR lists shall be executed in a subshell environment. Additionally, each command of a multi-command pipeline is in a subshell environment; as an extension, however, any or all commands in a pipeline may be executed in the current environment. Except where otherwise stated, all other commands shall be executed in the current shell environment.

## Context

**Subshell isolation and which constructs execute in subshells vs. current environment.**

Key rules:
1. **Subshell changes do not propagate**: Changes made to variables, traps, working directory, etc. in a subshell **shall not** affect the parent shell environment.

2. **Constructs that SHALL execute in a subshell**:
   - Command substitution: `$(cmd)` or `` `cmd` ``
   - Parenthesized groups: `( cmd1; cmd2 )`
   - Asynchronous AND-OR lists: `cmd &`

3. **Pipeline subshells**: Each command in a multi-command pipeline runs in a subshell environment. However, as an extension, the implementation **may** execute any or all pipeline commands in the current environment (this is what allows the common optimization of running the last pipeline command in the current shell).

4. **Current environment**: All other commands (simple commands of special built-ins, compound commands not in parentheses, function calls, etc.) **shall** execute in the current shell environment, except where otherwise stated.

Implementation: fork() for subshell constructs; the child inherits a copy of the shell state. The parent's state remains unchanged after the child exits.
