## Extracted Text

> The following options shall be supported:: Use the editor named by editor to edit the commands. The editor string is a utility name, subject to search via the PATH variable (see XBD 8. Environment Variables). The value in the FCEDIT variable shall be used as a default when -e is not specified. If FCEDIT is null or unset, ed shall be used as the editor.

## Context

This describes the full behavior of the `-e editor` option for `fc`. The implementation must: (1) Accept a utility name as the editor argument. (2) Resolve the editor via `PATH` search (per XBD 8 Environment Variables). (3) When `-e` is omitted, use the value of `FCEDIT` (after shell expansion) as the default editor. (4) When `FCEDIT` is null or unset, fall back to `ed`. The editor is invoked with a temporary file containing the selected command(s); after the editor exits, the file contents are re-executed. The editor lookup order is: `-e` argument > `FCEDIT` > `ed`. Reference: POSIX.1-2024 §fc, OPTIONS (tag_20_44_04).
