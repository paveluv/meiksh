## Extracted Text

> All modifications to a particular atomic object M occur in some particular total order, called the modification order of M. If A and B are modifications of an atomic object M, and A happens before B, then A shall precede B in the modification order of M, which is defined below.

## Context

This is a C-level memory model requirement about atomic object modification ordering. It applies to the implementation language of the shell (if using C atomics for internal concurrency), not to shell language semantics visible to users. For a Rust-based shell like meiksh, Rust's memory model provides equivalent guarantees through its atomic types. No shell-language-level action is required — this constrains the internal implementation if it uses shared mutable state across threads.

See: `basedefs/V1_chap04.html` §4.15.1.1 Data Races.
