## Extracted Text

> When a TOKEN is subject to alias substitution, the value of the alias shall be processed as if it had been read from the input instead of the TOKEN, with token recognition (see 2.3 Token Recognition) resuming at the start of the alias value. When the end of the alias value is reached, the shell may behave as if an additional <space> character had been read from the input after the TOKEN that was replaced. If it does not add this <space>, it is unspecified whether the current token is delimited before token recognition is applied to the character (if any) that followed the TOKEN in the input.

## Context

**Behavior:** When alias substitution occurs, the alias VALUE replaces the TOKEN in the input, and token recognition resumes from the start of the alias value. Several sub-rules apply:

1. **Replacement:** The alias value is processed as if it were read from input instead of the TOKEN. Token recognition (2.3) restarts at the beginning of the alias value.

2. **Optional trailing space:** When the end of the alias value is reached, the shell MAY behave as if an additional space were read after the replaced TOKEN. If it doesn't add this space, it is unspecified whether the current token is delimited before processing the character that followed the original TOKEN.

3. **Blank-ending alias chaining:** If the alias value ends in an unquoted blank (and optionally a quoted blank), the shell must check the NEXT token for alias substitution too, even if that token would not normally be in command-name position. This chain continues until a token is not a valid alias or an alias value doesn't end in a blank.

**Applies to:** All shell modes during alias processing.

**Implementation detail:** When performing alias substitution:
- Save the remaining input after the replaced TOKEN.
- Prepend the alias value to the input stream.
- Resume tokenization from the alias value's first character.
- Track whether the alias value ended with a blank to enable chaining.
- The optional trailing space behavior should be documented as an implementation choice.

**Example:** `alias noglob='set -f '` (note trailing space). When `noglob ls` is entered, `noglob` expands to `set -f ` (with trailing space), and because of the trailing space, `ls` is also checked for alias expansion.
