## Extracted Text

> A character that in the output stream indicates that printing should start on the next page of an output device. It is the character designated by '\f' in the C language. If the form-feed is not the first character of an output line, the result is unspecified. It is unspecified whether this character is the exact sequence transmitted to an output device by the system to accomplish the movement to the next page.

## Context

**Definition — moderate shell relevance.** Defines the form-feed character `<form-feed>` as `\f` (0x0C). It indicates printing should start on the next page. If form-feed is not the first character of an output line, behavior is unspecified. Whether this byte is the exact terminal sequence is unspecified.

**Shell impact:** The shell must recognize `\f` in escape-sequence contexts: `printf '\f'`, `echo` with XSI interpretation, and `$'\f'` quoting. The shell writes byte 0x0C. No terminal page-advance logic is needed in the shell itself.
