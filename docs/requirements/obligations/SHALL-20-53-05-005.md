## Extracted Text

> The following operands shall be supported:: The name of a shell variable that shall be set by the getopts utility to the option character that was found.

## Context

**Behavior:** The `name` operand specifies a shell variable name. On each successful invocation where an option is found, `getopts` shall set this variable to the single option character that was parsed. For error conditions, it is set to `?` or `:` depending on the error type and `optstring` mode.

**Values assigned to `name`:**
- Valid option found: the option character (e.g., `a`, `b`, `c`)
- Unknown option found (verbose mode): `?`
- Unknown option found (silent mode): `?`
- Missing argument (verbose mode): `?`
- Missing argument (silent mode): `:`
- End of options: `?`

**Error handling:** If `name` has been marked readonly, getopts cannot set it and must return >1 (see SHALL-20-53-03-016).

**Spec reference:** POSIX.1-2024, `getopts` OPERANDS, line 119 of `utilities/getopts.html`.
