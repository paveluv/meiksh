## Extracted Text

> The standard error shall be used only for diagnostic messages.

## Context

**Behavior:** The `kill` built-in must write only diagnostic (error) messages to stderr. No normal output should go to stderr. Normal output (signal listings) goes to stdout only when `-l` is specified.

**Conditions:** Diagnostic messages include errors like invalid signal names, invalid PIDs, permission denied, and no such process.

**Spec reference:** Section STDERR (tag_20_64_11) in `utilities/kill.html`, line 202.

GPT-5.4 review: The extracted quote matches `docs/posix/utilities/kill.html#tag_20_64_11`. The enrichment is accurate: successful `kill` operations, including successful `-l` output defined by `docs/posix/utilities/kill.html#tag_20_64_10`, should not use stderr. This SHALL is narrow and only constrains stderr usage; it does not itself require a diagnostic in every error case, only that stderr be reserved for diagnostics.
