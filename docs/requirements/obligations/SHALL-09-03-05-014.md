## Extracted Text

> A character class expression shall represent the union of two sets: The set of single characters that belong to the character class, as defined in the LC_CTYPE category in the current locale. An unspecified set of multi-character collating elements. All character classes specified in the current locale shall be recognized. A character class expression is expressed as a character class name enclosed within bracket-<colon> ("[:" and ":]") delimiters. The following character class expressions shall be supported in all locales: [:alnum:] [:cntrl:] [:lower:] [:space:] [:alpha:] [:digit:] [:print:] [:upper:] [:blank:] [:graph:] [:punct:] [:xdigit:] In addition, character class expressions of the form: [:name:] are recognized in those locales where the name keyword has been given a charclass definition in the LC_CTYPE category.

## Context

**Behavior:** Duplicate of SHALL-09-03-05-007. Character class expressions use `[:` and `:]` delimiters. All 12 standard classes must be supported in all locales. Additional classes are recognized if defined in the locale's `LC_CTYPE`.

**Shell relevance:** Same as SHALL-09-03-05-007. The shell must support `[[:alnum:]]`, `[[:alpha:]]`, `[[:blank:]]`, `[[:cntrl:]]`, `[[:digit:]]`, `[[:graph:]]`, `[[:lower:]]`, `[[:print:]]`, `[[:punct:]]`, `[[:space:]]`, `[[:upper:]]`, `[[:xdigit:]]` in bracket expressions.
