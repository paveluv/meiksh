## Extracted Text

> The WORD tokens shall have the word expansion rules applied to them immediately before the associated command is executed, not at the time the command is parsed.

## Context

Word expansion (tilde expansion, parameter expansion, command substitution, arithmetic expansion, field splitting, pathname expansion, quote removal—see Sections 2.6–2.6.7 in `utilities/V3_chap02.html`) must **not** be performed at parse time. Expansion must be deferred until just before the associated command is actually executed.

This is essential for correct behavior: a variable's value at execution time (not parse time) determines the expansion result. For example, in `x=hello; echo $x`, the parser stores `$x` as a literal token; expansion to `hello` happens only when the `echo` command is about to execute. This applies to all WORD tokens in the grammar. The AST must preserve the unexpanded token text.
