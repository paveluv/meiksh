## Extracted Text

> A <backslash> character that is not inside a bracket expression shall preserve the literal value of the following character, unless the following character is in a part of the pattern where shell quoting can be used and is a shell quoting character, in which case the behavior is unspecified.

## Context

**Duplicate of SHALL-19-14-01-003: backslash outside bracket expressions in non-shell-quoting contexts escapes the next character.**

A `\` not inside a bracket expression **shall** preserve the literal value of the following character in pattern contexts where shell quoting cannot be used. If the following character is a shell quoting character in a part where shell quoting is available, the behavior is unspecified.

See SHALL-19-14-01-003 for full context.
