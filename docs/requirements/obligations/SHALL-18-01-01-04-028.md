## Extracted Text

> When a file is to be appended, the file shall be opened in a manner equivalent to using the O_APPEND flag, without the O_TRUNC flag, in the open() function defined in the System Interfaces volume of POSIX.1-2024.

## Context

When a file is to be appended, the file shall be opened in a manner equivalent to using the `O_APPEND` flag, without the `O_TRUNC` flag, in the `open()` function. This means every write to the file is atomically appended to the end of the file, regardless of the current file offset. For `meiksh`, this directly governs the `>>` redirection operator: the shell must open the target file with `O_WRONLY | O_CREAT | O_APPEND` (and explicitly NOT `O_TRUNC`). This ensures that multiple concurrent writers to the same file append correctly without overwriting each other's data. See POSIX Utilities Chapter 1, Section 1.1.1.4 (utilities/V3_chap01.html lines 688–689).
