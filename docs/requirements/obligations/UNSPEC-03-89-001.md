## Extracted Text

> An unspecified object of unspecified format that may be generated when a process terminates abnormally.

## Context

**Definition — low shell relevance.** Defines "core image" (core dump) as an unspecified object of unspecified format that may be generated when a process terminates abnormally.

**Shell impact:** When a child process terminates due to a signal that produces a core dump (e.g., SIGSEGV, SIGABRT, SIGQUIT), the shell may report this in its exit status or diagnostic messages. The shell's `$?` value encodes signal information via `WIFSIGNALED`/`WTERMSIG`. Some shells append "(core dumped)" to the notification. `meiksh` should detect `WCOREDUMP()` (if available) and report it, but core dump generation itself is an OS concern.
