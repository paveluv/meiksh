## Extracted Text

> Special character on input, which is recognized if the ICANON flag is set. Deletes the entire line, as delimited by an NL, EOF, or EOL character. If ICANON is set, the KILL character shall be discarded when processed.

## Context

**Behavior:** The KILL special character (typically ^U) — duplicate/alternate extraction of the same spec text as SHALL-11-01-09-004:
1. Recognized on input only when ICANON is set in `c_lflag`.
2. Deletes the entire current line (all un-delimited data).
3. The KILL character shall be **discarded** when processed.

**Conditions:** ICANON must be set in `c_lflag`.

**Shell relevance:** See SHALL-11-01-09-004 for full details. Terminal driver provides KILL in canonical mode; the shell's line editor implements its own line-kill in non-canonical mode.

**Spec reference:** POSIX.1-2024, §11.1.9 Special Characters (tag_11_01_09), `basedefs/V1_chap11.html` line 232.
