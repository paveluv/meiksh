## Extracted Text

> If any part of word is quoted, not counting double-quotes outside a command substitution if the here-document is inside one, the delimiter shall be formed by performing quote removal on word, and the here-document lines shall not be expanded. Otherwise:

## Context

Defines how quoting in the delimiter word affects here-document expansion. See 2.7.4 Here-Document in utilities/V3_chap02.html.

**Behavior required — quoted delimiter (any part of `word` is quoted):**
- The delimiter is formed by performing quote removal on `word`. For example, `<<'EOF'` uses delimiter `EOF`; `<<\EOF` uses delimiter `EOF`; `<<"E"OF` uses delimiter `EOF`.
- The here-document body lines SHALL NOT be expanded (no parameter expansion, command substitution, or arithmetic expansion). The content is treated as literal text.
- Important nuance: double-quotes OUTSIDE a command substitution do NOT count as quoting when the here-document is inside a command substitution. For example, in `"$(cat <<EOF ... EOF)"`, the outer double-quotes do not quote the `EOF` delimiter.

**This obligation covers the "if quoted" branch.** The "otherwise" branch (unquoted delimiter) is covered by SHALL-19-07-04-003 through SHALL-19-07-04-010.
