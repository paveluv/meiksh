## Extracted Text

> Integer variables and constants, including the values of operands and option-arguments, used by the standard utilities listed in this volume of POSIX.1-2024 shall be implemented as equivalent to the ISO C standard signed long data type; floating point shall be implemented as equivalent to the ISO C standard double type. Conversions between types shall be as described in the ISO C standard. All variables shall be initialized to zero if they are not otherwise assigned by the input to the application.

## Context

Integer variables and constants in shell arithmetic contexts **shall** be equivalent to the C `signed long` type. This means on most modern 64-bit systems, arithmetic values must support at least the range of a 64-bit signed integer (-2^63 to 2^63-1). Floating-point values (if used by utilities like `awk` or `bc`) **shall** be equivalent to C `double`. Type conversions between integer and floating-point **shall** follow ISO C conversion rules. All variables in arithmetic contexts **shall** be initialized to zero unless otherwise assigned by the application input. For the shell's `$(( ))` expansion, this means all operands are signed long integers, uninitialized variables evaluate to 0, and overflow behavior follows C signed long semantics. Reference: XCU 1.1.2.1.
