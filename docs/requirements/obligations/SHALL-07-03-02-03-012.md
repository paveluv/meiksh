## Extracted Text

> If no operands are specified, a single forward operand shall be assumed.

## Context

When the `order_start` keyword in LC_COLLATE is used without any operands specifying collation directives (forward, backward, position), the default behavior is a single weight level with the `forward` direction. This means collation comparison proceeds from the start of the string toward the end, with a single pass. The shell is affected because LC_COLLATE governs pattern matching bracket expression ranges and locale-aware string comparisons. The shell delegates to system collation routines which apply this default.

See: `basedefs/V1_chap07.html` §7.3.2.3 The order_start Keyword.
