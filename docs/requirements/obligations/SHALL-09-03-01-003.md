## Extracted Text

> When not inside a bracket expression, the following shall match a single character:: a BRE special character or ']' preceded by an unescaped <backslash>

## Context

**Behavior:** A BRE special character (or `]`) preceded by an unescaped backslash, when outside a bracket expression, shall match the literal special character itself.

**Shell relevance:** In shell pattern matching, backslash-escaping of special pattern characters (`?`, `*`, `[`) causes them to match literally. For example, `\?` in a shell pattern matches a literal `?` character, not any single character. This is analogous to the BRE rule but uses shell quoting rules (section 2.2) rather than BRE backslash escaping.
