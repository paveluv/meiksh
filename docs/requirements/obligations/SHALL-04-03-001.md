## Extracted Text

> For an object of aggregate type with an explicit initializer, the initialization shall occur in initializer list order, each initializer provided for a particular subobject overriding any previously listed initializer for the same subobject; all subobjects that are not initialized explicitly shall be initialized implicitly according to the rules for default initialization.

## Context

**Definition only — no shell relevance.** Describes C aggregate type initialization rules: explicit initializers override previous ones for the same subobject; uninitialized subobjects get default initialization. This is a C language semantics rule.

**Shell impact:** None. The shell command language does not have aggregate types or initializer lists. No action required.
