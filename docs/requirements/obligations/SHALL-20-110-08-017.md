## Extracted Text

> The following environment variables shall affect the execution of sh:: Determine the behavior of range expressions, equivalence classes, and multi-character collating elements within pattern matching.

## Context

**Behavior:** LC_COLLATE determines the behavior of three features within shell pattern matching:

1. **Range expressions** (e.g., `[a-z]`): The collation order determines which characters fall within the range. In the POSIX locale, `[a-z]` matches lowercase ASCII letters; in other locales, the set of matching characters may differ.
2. **Equivalence classes** (e.g., `[[=a=]]`): Characters that collate equivalently are treated as matching. For example, in a French locale, `[[=e=]]` might match é, è, ê, ë.
3. **Multi-character collating elements** (e.g., `[[.ch.]]`): Some locales define multi-character sequences that are treated as single collating elements.

**Applies to:** All shell modes during pattern matching — this includes `case` statements, filename globbing (pathname expansion), and the `[[ ]]` / `test` pattern operations.

**Details:** If LC_COLLATE is unset or null, the value of LANG is used. If LC_ALL is set and non-empty, it overrides LC_COLLATE.
