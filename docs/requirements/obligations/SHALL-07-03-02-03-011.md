## Extracted Text

> Specifies that comparison operations for the weight level shall consider the relative position of elements in the strings not subject to IGNORE. The string containing an element not subject to IGNORE after the fewest collating elements subject to IGNORE from the start of the compare shall collate first. If both strings contain a character not subject to IGNORE in the same relative position, the collating values assigned to the elements shall determine the ordering. In case of equality, subsequent characters not subject to IGNORE shall be considered in the same manner.

## Context

The `position` directive in the LC_COLLATE `order_start` keyword modifies how IGNOREd collating elements are treated at a given weight level. Normally, IGNORE causes an element to be stripped before comparison. With `position`, the relative position of non-IGNORE elements matters: the string with a non-IGNORE element appearing after fewer IGNORE elements collates first. If non-IGNORE elements appear at the same relative position, their collation weights determine ordering; ties are broken by examining subsequent non-IGNORE elements. The shell is affected because LC_COLLATE controls bracket expression ranges in pattern matching and locale-aware string comparisons. The shell delegates to system collation routines (`strcoll()`) which implement the `position` directive.
