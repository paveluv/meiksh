## Extracted Text

> A positional parameter is a parameter denoted by a decimal representation of a positive integer. The digits denoting the positional parameters shall always be interpreted as a decimal value, even if there is a leading zero. When a positional parameter with more than one digit is specified, the application shall enclose the digits in braces (see 2.6.2 Parameter Expansion).

## Context

Positional parameters (`$1`, `$2`, ..., `$N`) are set by the `set` builtin, by shell invocation arguments, and by function calls. They are denoted by decimal integer representations.

**Required behavior:**
- Positional parameter numbers must always be interpreted as base-10 decimal, even if they have leading zeros. For example, `${09}` refers to the 9th positional parameter, not an octal value.
- Single-digit positional parameters (`$1` through `$9`) can be referenced without braces: `$1`.
- Multi-digit positional parameters (`$10` and above) **must** be enclosed in braces: `${10}`, `${11}`, etc. Without braces, `$10` is parsed as `$1` followed by the literal character `0`.
- The application is responsible for using braces for multi-digit access, but the shell must correctly parse braced multi-digit references.

**Conditions:** Applies in all execution contexts (current shell, subshell, functions).

**Error handling:** If a positional parameter is not set, it expands to the empty string (unless `set -u` / `nounset` is active, in which case an error is raised).

See 2.5.1 "Positional Parameters" and 2.6.2 "Parameter Expansion" in `utilities/V3_chap02.html`.
