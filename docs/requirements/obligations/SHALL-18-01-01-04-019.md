## Extracted Text

> The following codes are used in Actions when Creating a File that Already Exists:: Follow link. Unless otherwise specified, the symbolic link shall be followed as specified for pathname resolution, and the operation performed shall be as if the target of the symbolic link (after all resolution) had been named. If the target of the symbolic link does not exist, it shall be as if that nonexistent target had been named directly.

## Context

Action code "FL" (Follow Link): when the existing file is a symbolic link, unless otherwise specified, the symlink shall be followed per pathname resolution and the operation shall proceed as if the target of the symlink had been named. If the symlink target does not exist, it shall be treated as if that nonexistent path had been named directly (i.e., a new file will be created at the target path). For `meiksh`, when `> symlink` is used in redirection, the shell follows the symlink and operates on the target file. This is the default behavior of `open()` which follows symlinks. See POSIX Utilities Chapter 1, Section 1.1.1.4 action code FL (utilities/V3_chap01.html line 658).
