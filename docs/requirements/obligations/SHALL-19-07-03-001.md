## Extracted Text

> Appended output redirection shall cause the file whose name results from the expansion of word to be opened for output on the designated file descriptor. The file shall be opened as if the open() function as defined in the System Interfaces volume of POSIX.1-2024 was called with the O_APPEND flag set. If the file does not exist, it shall be created.

## Context

Defines appending output redirection using the `>>` operator. See 2.7.3 Appending Redirected Output in utilities/V3_chap02.html.

**Syntax:** `[n]>>word`

**Behavior required:**
- Open the file named by the expansion of `word` for writing on the designated fd (or fd 1 if `n` is omitted).
- The file must be opened with the `O_APPEND` flag set, so all writes go to the end of the file regardless of the current file position.
- If the file does NOT exist, it shall be created (equivalent to `open()` with `O_WRONLY | O_CREAT | O_APPEND`, mode typically 0666 modified by umask).
- If the file DOES exist, it is opened for appending (no truncation).

**Key difference from `>`:** The `>>` operator never truncates the file and always appends. The noclobber (`set -C`) option does NOT affect `>>` — it only affects `>`.

**Error conditions:** If the open/create fails, this is a redirection error (see SHALL-19-07-005).
