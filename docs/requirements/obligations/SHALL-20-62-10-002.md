## Context

**Behavior:** The default output format (when neither `-p` nor `-l` is specified) is:
```
"[%d] %c %s %s\n", <job-number>, <current>, <state>, <command>
```

Each background job produces one line with four fields:
- `<job-number>`: integer job number (used with `%n` in job_id)
- `<current>`: `+` (current job), `-` (previous job), or space
- `<state>`: Running, Done, Done(code), Stopped, etc.
- `<command>`: the command string as given to the shell

**Spec reference:** POSIX.1-2024, `jobs` STDOUT, line 116 of `utilities/jobs.html`.
