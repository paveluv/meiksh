## Extracted Text

> The utilities in the Shell and Utilities volume of POSIX.1-2024 that claim conformance to these guidelines shall conform completely to these guidelines as if these guidelines contained the term "shall" instead of "should". On some implementations, the utilities accept usage in violation of these guidelines for backwards-compatibility as well as accepting the required form.

## Context

For all shell builtins and utilities that claim conformance to the Utility Syntax Guidelines (Section 12.2), every "should" in those guidelines becomes a "shall" (mandatory). This means ALL 14 guidelines must be fully implemented: single-character options, `-` prefix, option grouping, separate option-arguments, `--` end-of-options delimiter, etc. The implementation may additionally accept non-conforming usage for backward compatibility, but the conforming forms must always work. Builtins documented as conforming to these guidelines (most standard builtins) must strictly follow all 14 rules.
