## Extracted Text

> The exit status of a command shall be determined as follows:: Otherwise, the exit status shall be the value obtained by the equivalent of the WEXITSTATUS macro applied to the status obtained by the wait() function (as defined in the System Interfaces volume of POSIX.1-2024). Note that for C programs, this value is equal to the result of performing a modulo 256 operation on the value passed to _Exit(), _exit(), or exit() or returned from main().

## Context

**Behavior:** When a command terminates normally (not by signal, not "not found", not "not executable"), the exit status MUST be the value from `WEXITSTATUS()` applied to the status returned by `wait()`. For C programs, this is equivalent to `value % 256` where `value` is the argument passed to `_Exit()`, `_exit()`, or `exit()`, or the return value from `main()`.

**Applies to:** The normal-termination case — the final rule in the exit-status priority chain. This is the most common path for successfully executed external commands.

**Implementation detail:** After `waitpid()` returns and `WIFEXITED(status)` is true, use `WEXITSTATUS(status)` to extract the low 8 bits. Note that the shell itself cannot distinguish values > 255 passed to exit() — they are truncated by the OS to `value & 0xFF`. The refs to wait/exit/etc. are libc functions used by the shell implementation, not shell-level concerns.

**Spec reference:** Section 2.8.2, lines 1706-1710 in `utilities/V3_chap02.html`.
