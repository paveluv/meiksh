## Extracted Text

> The <signal.h> header shall define the following macros which shall expand to integer constant expressions that need not be usable in #if preprocessing directives:: SIG_UNBLOCK

## Context

The `<signal.h>` header shall define the macro `SIG_UNBLOCK` as an integer constant expression. This macro is used as the `how` argument to `sigprocmask()` (and `pthread_sigmask()`). When `SIG_UNBLOCK` is specified, the resulting process signal mask is the intersection of the current mask and the complement of the signal set pointed to by the `set` argument — i.e., signals in `set` are removed from the blocked set. The constant need not be usable in `#if` preprocessing directives. For `meiksh`, this is used when the shell needs to unblock signals after a critical section, or when restoring signal masks for child processes before exec. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 504–507).
