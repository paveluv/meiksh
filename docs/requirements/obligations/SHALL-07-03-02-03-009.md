## Extracted Text

> Specifies that comparison operations for the weight level shall proceed from start of string towards the end of string.

## Context

In the LC_COLLATE `order_start` keyword, the `forward` directive specifies the comparison direction for a given weight level. When `forward` is specified, collation comparison for that weight level proceeds from the beginning of the string toward the end. This is the default direction. The shell is affected because LC_COLLATE governs how `sort`-order-dependent operations work (e.g., bracket expression ranges in pattern matching like `[a-z]`, and the `sort` ordering used by locale-aware comparisons). The shell itself delegates to the system's collation routines (e.g., `strcoll()`), but must ensure the locale is correctly set so these routines honor the forward directive.
