## Extracted Text

> The following options shall be supported:: -s signal_name

## Context

**Behavior:** The `kill` built-in must support the `-s signal_name` option. This option specifies which signal to send by symbolic name rather than the default SIGTERM.

**Conditions:** The `-s` option takes one argument: a signal name. See SHALL-20-64-04-006 and SHALL-20-64-04-012 for the detailed requirements on how signal names are recognized (case-independent, without `SIG` prefix, plus the special name `0`).
