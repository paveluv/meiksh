## Extracted Text

> [CX] The <signal.h> header shall define the symbolic constants in the Code column of the following table for use as values of si_code that are signal-specific or non-signal-specific reasons why the signal was generated.

## Context

The `<signal.h>` header shall define symbolic constants for use as `si_code` values in `siginfo_t` ([CX] extension). These constants indicate signal-specific or non-signal-specific reasons for signal generation. Key constants include: for `SIGILL` — `ILL_ILLOPC`, `ILL_ILLOPN`, `ILL_ILLADR`, `ILL_ILLTRP`, `ILL_PRVOPC`, `ILL_PRVREG`, `ILL_COPROC`, `ILL_BADSTK`; for `SIGFPE` — `FPE_INTDIV`, `FPE_INTOVF`, `FPE_FLTDIV`, `FPE_FLTOVF`, `FPE_FLTUND`, `FPE_FLTRES`, `FPE_FLTINV`, `FPE_FLTSUB`; for `SIGSEGV` — `SEGV_MAPERR`, `SEGV_ACCERR`; for `SIGBUS` — `BUS_ADRALN`, `BUS_ADRERR`, `BUS_OBJERR`; for `SIGTRAP` — `TRAP_BRKPT`, `TRAP_TRACE`; for `SIGCHLD` — `CLD_EXITED`, `CLD_KILLED`, `CLD_DUMPED`, `CLD_TRAPPED`, `CLD_STOPPED`, `CLD_CONTINUED`; for any signal — `SI_USER`, `SI_QUEUE`, `SI_TIMER`, `SI_ASYNCIO`, `SI_MESGQ`. For `meiksh`, the `SIGCHLD` codes (`CLD_EXITED`, `CLD_KILLED`, `CLD_STOPPED`, `CLD_CONTINUED`) are relevant for job status tracking. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 599–1016).
