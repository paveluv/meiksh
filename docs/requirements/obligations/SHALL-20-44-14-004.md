## Extracted Text

> The following exit values shall be returned:: >0

## Context

The `fc` builtin must return exit status >0 when an error occurs. In listing mode (`-l`), this means a write failure, invalid arguments, or inability to access the history. In edit/re-execute mode, an error in `fc` itself (e.g., editor launch failure, temp file creation failure) should also result in >0. If the editor returns non-zero, the command is not re-executed and `fc`'s own exit status reflects the error. Reference: POSIX.1-2024 §fc, EXIT STATUS (tag_20_44_14).
