## Extracted Text

> This variable shall determine the locale category for numeric formatting (for example, thousands separator and radix character) information in various utilities as well as the formatted I/O operations in printf() and scanf() and the string conversion functions in strtod(). Additional semantics of this variable, if any, are implementation-defined.

## Context

**Behavior:** The environment variable `LC_NUMERIC` shall determine the locale category for numeric formatting, including the radix character (decimal point) and thousands separator, used by various utilities and formatted I/O operations.

**Shell relevance:** The shell's `printf` built-in uses `LC_NUMERIC` when formatting `%f`, `%e`, `%g` conversion specifiers — the radix character is locale-dependent. Arithmetic expansion (`$(( ))`) uses C locale internally, but `printf` output formatting is affected. The shell must propagate `LC_NUMERIC` to child processes.

**Additional semantics:** Any further behavior beyond numeric formatting is implementation-defined.
