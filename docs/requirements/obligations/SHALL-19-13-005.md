## Context

**Subshell environment creation: duplicate of parent with exceptions.**

A subshell environment **shall** be created as a duplicate of the shell environment. This means the subshell starts with copies of all:
- Open file descriptors
- Working directory
- File creation mask (umask)
- Shell variables (including unexported ones)
- Shell functions
- Shell options (set flags)
- Aliases

However, specific exceptions apply (detailed in SHALL-19-13-006 through SHALL-19-13-009):
- Non-ignored traps are reset to default (SHALL-19-13-006/008)
- Interactive subshells behave as non-interactive with minor exceptions (SHALL-19-13-007/009)

This is the foundational rule for subshell semantics. The "duplicate" means a conceptual copy-on-write: modifications in the subshell do not propagate back (SHALL-19-13-010).
