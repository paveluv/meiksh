## Extracted Text

> Otherwise, if the command name is found, but it is not an executable utility, the exit status shall be 126.

## Context

**Behavior:** If the command name resolves to a file that exists but is not an executable utility, the exit status MUST be 126. This is a standalone restatement of the rule in SHALL-19-08-02-003.

**Applies to:** Commands where the file exists but cannot be executed (no execute permission, is a directory, etc.).

**Error handling:** Write diagnostic to stderr (e.g., "command: Permission denied").

**Implementation detail:** Same rule as SHALL-19-08-02-003. The implementation should have a single code path that returns 126 when exec fails with EACCES or the target is not a regular executable file.

**Spec reference:** Section 2.8.2, line 1702 in `utilities/V3_chap02.html`.
