## Context

**Behavior:** The commands string inside a command substitution (both `` `...` `` and `$(...)`) must be:
1. **Tokenized** according to the standard token recognition rules (2.3).
2. **Parsed** according to the shell grammar (2.10).

**Unspecified aspects:**
- Whether the commands string is parsed and executed **incrementally** (like a shell script, line by line) or parsed as a **single compound_list** and then executed. This affects whether alias/unalias commands within the substitution take effect for later commands in the same substitution.
- Whether the closing `)` of `$(...)` can come from alias expansion.

**`$(...)` form flexibility:** Any syntactically valid shell program can appear inside `$(...)`, with the exceptions listed in SHALL-19-06-03-005 through SHALL-19-06-03-007.

**Spec reference:** See 2.6.3 Command Substitution in utilities/V3_chap02.html, lines 1120–1127. See 2.3 Token Recognition (#tag_19_03) and 2.10 Shell Grammar (#tag_19_10).
