## Extracted Text

> The international currency symbol. The operand shall be a four-character string, with the first three characters containing the alphabetic international currency symbol. The international currency symbol should be chosen in accordance with those specified in the ISO 4217 standard. The fourth character shall be the character used to separate the international currency symbol from the monetary quantity.

## Context

The `int_curr_symbol` value must be exactly four characters: three alphabetic characters forming the ISO 4217 currency code (e.g., "USD", "EUR", "GBP") followed by a separator character (typically a space). The shell does not validate or directly use this value. The shell must correctly propagate the LC_MONETARY environment variable so programs can access this through `localeconv()`.

See: `basedefs/V1_chap07.html` §7.3.3 LC_MONETARY.
