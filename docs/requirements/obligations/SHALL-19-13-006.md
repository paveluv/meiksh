## Extracted Text

> A subshell environment shall be created as a duplicate of the shell environment, except that:: Unless specified otherwise (see trap), traps that are not being ignored shall be set to the default action.

## Context

**Subshell exception: non-ignored traps reset to default.**

When creating a subshell environment, traps that are currently *caught* (have a handler set via `trap 'action' SIGNAL`) **shall** be reset to the default action (SIG_DFL). Traps that are *ignored* (`trap '' SIGNAL`) **shall** remain ignored in the subshell.

Exception: the `trap` built-in specification (see 2.29 in utilities/V3_chap02.html) may specify additional cases — in particular, `trap` with no arguments in a subshell may reveal the parent's traps under certain conditions.

Implementation: when forking for a subshell (`(...)`, command substitution, async list), iterate the trap table. For each signal with a non-empty, non-ignore action, reset it to SIG_DFL. Leave SIG_IGN entries untouched.
