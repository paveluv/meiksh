## Extracted Text

> The following variables shall affect the execution of the shell:: [UP] The processing of the LINENO shell variable shall be supported if the system supports the User Portability Utilities option. Set by the shell to a decimal number representing the current sequential line number (numbered starting with 1) within a script or function before it executes each command. If the user unsets or resets LINENO , the variable may lose its special meaning for the life of the shell. If the shell is not currently executing a script or function, the value of LINENO is unspecified.

## Context

This defines the semantics of the `LINENO` variable. Tagged `[UP]` (User Portability Utilities option).

**Required behavior:**
- Processing of `LINENO` must be supported (meiksh targets UP compliance).
- The shell must set `LINENO` to a decimal number representing the current sequential line number (starting from 1) within a script or function, before each command is executed.
- Line numbering is per-script or per-function: each function and each sourced/invoked script has its own line counter starting at 1.
- If the user unsets or resets `LINENO` (e.g., `unset LINENO` or `LINENO=5`), the variable may lose its special auto-updating meaning for the life of the shell.
- If the shell is not currently executing a script or function (e.g., at the interactive prompt), the value of `LINENO` is unspecified.

**Conditions:** Script and function execution only. Interactive mode value is implementation-defined.

**Implementation note:** The shell must track source line numbers during parsing and update `LINENO` before executing each command. This interacts with the `set -x` trace output (which may include `LINENO` in `PS4`).

See 2.5.3 "Shell Variables" (`LINENO`) in `utilities/V3_chap02.html`.
