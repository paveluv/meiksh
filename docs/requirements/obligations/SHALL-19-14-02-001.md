## Extracted Text

> The following rules are used to construct patterns matching multiple characters from patterns matching a single character:: The <asterisk> ('*') is a pattern that shall match any string, including the null string.

## Context

**`*` matches any string including the empty string.**

The `*` (asterisk) pattern **shall** match any string, including the null (empty) string. This is the most fundamental multi-character matching rule.

Examples:
- `*` matches `""`, `"a"`, `"abc"`, `"anything"`
- `a*` matches `"a"`, `"ab"`, `"abc"`
- `*z` matches `"z"`, `"az"`, `"xyz"`

In filename expansion context, `*` is further constrained: it does not match `/` characters and does not match a leading `.` (see 2.14.3).

See 2.14.2 Patterns Matching Multiple Characters in utilities/V3_chap02.html.
