## Extracted Text

> All characters between <space> and 'a' shall have the same primary equivalence class and individual secondary weights based on their ordinal encoded values.

## Context

In an LC_COLLATE example, characters in the code-point range between `<space>` and `'a'` are assigned the same primary weight (making them an equivalence class at the primary level) and individual secondary weights equal to their ordinal encoded values. This means at the primary comparison level these characters are considered equal, but at the secondary level they are ordered by code point. The shell is affected because this weight assignment pattern directly influences bracket expression matching: characters in the same primary equivalence class can be matched with `[=x=]` syntax, and range expressions at the primary level may include all such characters.

See: `basedefs/V1_chap07.html` §7.3.2.4 Collation Order.
