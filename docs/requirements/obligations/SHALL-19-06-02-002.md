## Extracted Text

> The value, if any, of parameter shall be substituted.

## Context

**Behavior:** In the simplest form of parameter expansion `${parameter}` (or `$parameter`), the shell must substitute the current value of the named parameter. If the parameter has a value (including the empty string), that value replaces the expansion. If the parameter is unset, the expansion yields an empty string (unless `set -u` is in effect, which causes an error).

**Conditions:** Applies to the basic `${parameter}` form with no modifier operators (`:−`, `:=`, `:?`, `:+`, `#`, `%`, `##`, `%%`). The parameter can be a variable name, a positional parameter (`$1`, `$2`, etc.), or a special parameter (`$?`, `$#`, `$@`, `$*`, `$$`, `$!`, `$0`, `$-`).

**Inputs/Outputs:**
- Input: `${parameter}` or `$parameter`
- Output: the string value currently assigned to the parameter, or empty string if unset (and `set -u` is not active)

**Error handling:** If `set -u` (`nounset`) is enabled and the parameter is unset, the expansion must fail with an error.

**Spec reference:** See 2.6.2 Parameter Expansion in utilities/V3_chap02.html, line 788.
