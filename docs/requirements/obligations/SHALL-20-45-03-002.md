## Extracted Text

> Using fg to place a job into the foreground shall remove its process ID from the list of those "known in the current shell execution environment"; see 2.9.3.1 Asynchronous AND-OR Lists.

## Context

When `fg` successfully moves a job to the foreground, the process IDs of that job must be removed from the shell's "known PIDs" list. This list tracks PIDs of asynchronous commands (background jobs started with `&`) whose exit statuses can be retrieved by `wait`. After `fg` brings a job to the foreground, `wait` should no longer be able to retrieve the exit status of that job's PIDs — they are no longer "known." This is important for correct `$!` and `wait` semantics. Reference: POSIX.1-2024 §fg, DESCRIPTION (tag_20_45_03); XCU 2.9.3.1.
