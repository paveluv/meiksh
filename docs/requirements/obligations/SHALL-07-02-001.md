## Extracted Text

> Conforming systems shall provide a POSIX locale, also known as the C locale. In POSIX.1 the requirements for the POSIX locale are more extensive than the requirements for the C locale as specified in the ISO C standard. However, in a conforming POSIX implementation, the POSIX locale and the C locale are identical. The behavior of standard utilities and functions in the POSIX locale shall be as if the locale was defined via the localedef utility with input data from the POSIX locale tables in 7.3 Locale Definition.

## Context

**Behavior:** Every conforming system must provide a POSIX locale (identical to the C locale). The POSIX locale's behavior matches the locale tables defined in Section 7.3. The POSIX locale is the baseline — all character classifications, collation, and other locale-dependent behaviors have well-defined values in this locale.

**Shell relevance:** The shell must:
- Support `LANG=C`, `LANG=POSIX`, `LC_ALL=C`, `LC_ALL=POSIX` to select this locale.
- In the POSIX locale, character classes contain only ASCII characters (e.g., `[[:alpha:]]` matches only `A-Z` and `a-z`).
- Collation order in the POSIX locale is by character encoding value (byte value).
- Pattern ranges like `[a-z]` in the POSIX locale match only lowercase ASCII letters.
- The shell may use the POSIX locale as its default when no locale variables are set.
