## Extracted Text

> If one or more pid operands are specified that represent known process IDs, the wait utility shall wait until all of them have terminated. If one or more pid operands are specified that represent unknown process IDs, wait shall treat them as if they were known process IDs that exited with exit status 127. The exit status returned by the wait utility shall be the exit status of the process requested by the last pid operand.

## Context

**Behavior:** When `wait` is given one or more PID operands:

1. **Known PIDs:** `wait` shall block until all specified known PIDs have terminated.
2. **Unknown PIDs:** Any operand that does not match a known PID shall be treated as if it were a known process that already exited with status 127. `wait` does not block for unknown PIDs.
3. **Exit status:** The exit status of `wait` is determined by the **last** pid operand only. If the last operand was a known PID, `wait` returns that process's exit status. If the last operand was unknown, `wait` returns 127.

**Example:** `wait $pid1 $pid2` — waits for both, but returns the exit status of the process identified by `$pid2`.

**Example:** `wait 99999` where 99999 is unknown — returns 127 immediately.
