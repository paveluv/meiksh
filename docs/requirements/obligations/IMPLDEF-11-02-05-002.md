## Extracted Text

> The initial local control value after open() is implementation-defined.

## Context

**Behavior:** When a terminal device file is opened via `open()`, the initial value of the `c_lflag` field (local modes) in the termios structure is implementation-defined. This means the flags ECHO, ECHOE, ECHOK, ECHONL, ICANON, IEXTEN, ISIG, NOFLSH, and TOSTOP may be in any combination after the initial open.

**Conditions:** Applies only to the initial state of a terminal device after `open()`. Once the terminal attributes are set via `tcsetattr()`, the values are under application control.

**Shell relevance:** When meiksh opens a terminal (e.g., at startup or when allocating a new PTY), it must not assume any particular local mode flags are set. The shell should explicitly call `tcgetattr()` to read the current state and then use `tcsetattr()` to configure the desired modes (e.g., setting ICANON and ECHO for canonical input, or clearing ICANON for line-editing mode). The shell should document its chosen initial local mode values as its implementation-defined behavior.

**Spec reference:** POSIX.1-2024, §11.2.5 Local Modes (tag_11_02_05), `basedefs/V1_chap11.html` line 1202.
