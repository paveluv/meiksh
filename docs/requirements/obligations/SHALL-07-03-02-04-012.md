## Extracted Text

> The special keyword IGNORE as a weight shall indicate that when strings are compared using the weights at the level where IGNORE is specified, the collating element shall be ignored; that is, as if the string did not contain the collating element. In regular expressions and pattern matching, all characters that are subject to IGNORE in their primary weight form an equivalence class.

## Context

The IGNORE keyword as a collation weight causes the collating element to be skipped during comparison at that weight level — the element is treated as if it doesn't exist in the string for that level's comparison. Additionally, all characters whose primary weight is IGNORE form a single equivalence class for purposes of regular expressions and pattern matching. The shell is affected directly: characters with IGNORE as their primary weight can be referenced as an equivalence class in bracket expressions (e.g., `[=x=]` where x has IGNORE primary weight would match all IGNORE-primary characters). This also affects how collation-based sorting and comparison behave for these characters.

See: `basedefs/V1_chap07.html` §7.3.2.4 Collation Order.
