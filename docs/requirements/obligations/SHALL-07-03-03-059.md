## Extracted Text

> Certain combinations of the *_sign_posn, positive_sign, and negative_sign values are invalid and shall not be accepted by localedef:: If p_sign_posn and n_sign_posn are both 0, this combination is invalid because it requires parentheses to be used but does not provide the means to distinguish negative from positive values using parentheses.

## Context

A `localedef` validation rule: if both `p_sign_posn` and `n_sign_posn` are 0 (meaning parentheses indicate sign), the combination is invalid because parentheses around both positive and negative values provides no way to distinguish them. This is a `localedef` utility constraint, not a shell runtime requirement.

See: `basedefs/V1_chap07.html` §7.3.3 LC_MONETARY.
