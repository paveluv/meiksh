## Extracted Text

> The shell shall read commands but does not execute them; this can be used to check for shell script syntax errors. Interactive shells and subshells of interactive shells, recursively, may ignore this option.

## Context

**Behavior:** The `-n` (`noexec`) option: the shell reads and parses commands but does **not** execute them. This is useful for checking shell script syntax errors without running the script.

**Exception:** Interactive shells and subshells of interactive shells (recursively) **may** ignore this option. This means setting `-n` in an interactive shell might have no effect.

**Applies to:** Primarily non-interactive shells (scripts).
