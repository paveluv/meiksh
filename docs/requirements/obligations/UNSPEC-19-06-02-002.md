## Extracted Text

> Otherwise, the parameter is a single-character symbol, and behavior is unspecified if that character is neither a digit nor one of the special parameters (see 2.5.2 Special Parameters).

## Context

**Behavior:** This is a duplicate extraction of UNSPEC-19-06-02-001. For unbraced parameter expansion where the character after `$` is a single non-name-start character, behavior is unspecified if that character is not a digit and not a special parameter character (`@`, `*`, `#`, `?`, `-`, `$`, `!`).

**Implementation guidance:** Same as UNSPEC-19-06-02-001. The shell's behavior for `$` followed by an unrecognized single character should be documented as an implementation choice.

**Spec reference:** See 2.6.2 Parameter Expansion in utilities/V3_chap02.html, lines 796–797. See 2.5.2 Special Parameters (#tag_19_05_02) for the complete list.
