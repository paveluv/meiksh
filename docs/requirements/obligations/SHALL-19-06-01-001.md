## Extracted Text

> If the tilde-prefix consists of only the <tilde> character, it shall be replaced by the value of the variable HOME . If HOME is unset, the results are unspecified.

## Context

**Behavior:** When performing tilde expansion and the tilde-prefix is a bare `~` (with no login name following it), the shell must replace it with the value of the `HOME` shell variable.

**Conditions:** This applies during word expansion whenever a tilde-prefix is recognized — at the beginning of a word, or following an unquoted `=` or `:` in an assignment context. The tilde-prefix `~` is the simplest form (no username appended).

**Inputs/Outputs:**
- Input: a word beginning with `~` where the tilde-prefix is just `~` (i.e., followed by `/`, end-of-word, or `:` in assignment context)
- Output: the tilde-prefix is replaced by the string value of `$HOME`

**Error handling:** If `HOME` is **unset** (not merely empty), the result is unspecified — the implementation may choose to leave `~` unexpanded, expand to empty string, or produce an error. If `HOME` is set to the empty string, the replacement is the empty string (producing an empty field per SHALL-19-06-01-004).

**Spec reference:** See 2.6.1 Tilde Expansion in utilities/V3_chap02.html, lines 735–769.
