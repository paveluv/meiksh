## Extracted Text

> If exactly one var operand is yet to be processed and there was some remaining unsplit input returned from the modified field splitting algorithm, the variable named by the unprocessed var operand shall be assigned the unsplit input.

## Context

**Behavior required:** Same text as SHALL-20-100-03-012, extracted as a standalone obligation. When exactly one var operand remains and there is unsplit remainder from the field splitting algorithm, assign the entire unsplit remainder to that last variable.

**Condition:** `remaining_vars == 1` AND there is unsplit input remaining.

**Implementation detail:** The last variable gets everything that wasn't consumed by the field splitting loop, with leading/trailing IFS whitespace already removed but internal content preserved. This is why the last variable in `read a b c` gets "the rest" of the line.
