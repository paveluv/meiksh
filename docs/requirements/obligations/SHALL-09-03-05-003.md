## Extracted Text

> The following rules and definitions apply to bracket expressions:: A matching list expression specifies a list that shall match any single character that is matched by one of the expressions represented in the list. The first character in the list cannot be the <circumflex>. An ordinary character in the list shall only match that character; for example, "[abc]" is an RE that only matches one of the characters 'a', 'b', or 'c'. It is unspecified whether a matching list expression matches a multi-character collating element that is matched by one of the expressions.

## Context

**Behavior:** A matching list expression specifies a list that shall match any single character matched by one of the expressions in the list. The first character cannot be `^`. An ordinary character in the list shall only match that character (e.g., `[abc]` matches only 'a', 'b', or 'c'). It is unspecified whether a matching list expression matches a multi-character collating element.

**Shell relevance:** In shell patterns, `[abc]` matches exactly one character: 'a', 'b', or 'c'. The implementation must check each character in the list against the input character. Multi-character collating element matching is unspecified, so the shell need not support it but may.
