## Extracted Text

> The following operands shall be supported:: The pathname of a file containing commands. If the pathname contains one or more <slash> characters, the implementation attempts to read that file; the file need not be executable. If the pathname does not contain a <slash> character: The implementation shall attempt to read that file from the current working directory; the file need not be executable. If the file is not in the current working directory, the implementation may perform a search for an executable file using the value of PATH , as described in 2.9.1.4 Command Search and Execution. Special parameter 0 (see 2.5.2 Special Parameters) shall be set to the value of command_file. If sh is called using a synopsis form that omits command_file, special parameter 0 shall be set to the value of the first argument passed to sh from its parent (for example, argv[0] for a C program), which is normally a pathname used to execute the sh utility.

## Context

When `sh` is invoked with a `command_file` operand (a pathname of a file containing commands), the shell must resolve and open it according to these rules:

**Path resolution**:
- If the pathname **contains one or more `/`** characters: attempt to read that file directly. The file does not need to be executable (no `+x` permission required).
- If the pathname **does not contain `/`**:
  1. First, attempt to read the file from the **current working directory**. The file does not need to be executable.
  2. If the file is not found in the current working directory, the implementation **may** (optional) search for an executable file using `PATH`, as described in 2.9.1.4 Command Search and Execution (utilities/V3_chap02.html#tag_19_09_01_04).

**Special parameter $0**:
- `$0` must be set to the value of `command_file`.
- If `sh` is called **without** a `command_file` (e.g., `-s` mode or `-c` mode without `command_name`), `$0` must be set to the value of the first argument passed to `sh` from its parent (typically `argv[0]`, which is normally the pathname used to invoke `sh`).

See utilities/sh.html#tag_20_110_05 and 2.5.2 Special Parameters in utilities/V3_chap02.html#tag_19_05_02.
