## Extracted Text

> String Length. The shortest decimal representation of the length in characters of the value of parameter shall be substituted. If parameter is '*' or '@', the result of the expansion is unspecified. If parameter is unset and set -u is in effect, the expansion shall fail.

## Context

**Behavior:** The `${#parameter}` expansion (String Length):
- Substitutes the **length in characters** (not bytes) of the value of `parameter`, expressed as the shortest decimal integer string (no leading zeros, except `0` itself).
- The length is measured in **characters**, which matters for multi-byte locales.

**Restrictions:**
- If `parameter` is `*` or `@`, the result is **unspecified** — implementations should avoid treating `${#*}` or `${#@}` as meaningful.
- If `parameter` is **unset** and `set -u` (nounset) is in effect, the expansion must **fail** (produce an error and, for non-interactive shells, exit).

**Inputs/Outputs:**
- `x="hello"; echo ${#x}` → `5`
- `x=""; echo ${#x}` → `0`
- `unset x; set -u; echo ${#x}` → error

**Spec reference:** See 2.6.2 Parameter Expansion in utilities/V3_chap02.html, lines 961–964.
