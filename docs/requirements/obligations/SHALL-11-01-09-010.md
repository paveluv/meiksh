## Extracted Text

> Special character on input, which is recognized if the ISIG flag is set. Generates a SIGINT signal which is sent to all processes in the foreground process group for which the terminal is the controlling terminal. If ISIG is set, the INTR character shall be discarded when processed.

## Context

**Behavior:** The INTR special character (typically ^C) — duplicate/alternate extraction of the same spec text as SHALL-11-01-09-001:
1. Recognized on input only when ISIG is set in `c_lflag`.
2. Generates SIGINT sent to all processes in the foreground process group of the controlling terminal.
3. The INTR character shall be **discarded** when processed.

**Conditions:** ISIG must be set in `c_lflag`. The terminal must be the controlling terminal.

**Shell relevance:** See SHALL-11-01-09-001 for full details. The shell relies on the terminal driver generating SIGINT via the INTR character for interrupt handling during interactive use. The shell must handle SIGINT in its own signal handler (discard partial input, redisplay prompt) and allow foreground children to receive it.

**Spec reference:** POSIX.1-2024, §11.1.9 Special Characters (tag_11_01_09), `basedefs/V1_chap11.html` line 220.
