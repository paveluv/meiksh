## Extracted Text

> If ECHO is set, input characters shall be echoed back to the terminal. If ECHO is clear, input characters shall not be echoed.

## Context

**Behavior:** When ECHO is set in `c_lflag`, input characters shall be echoed back to the terminal. When ECHO is clear, input characters shall not be echoed.

**Relevance to meiksh:** The `stty` builtin must support `echo`/`-echo`. This is critical for interactive shells: ECHO is normally set for command-line input, but may be cleared for password prompts (e.g., via `read -s` or `stty -echo`). The shell's line editor may need to manage ECHO state when switching between raw and cooked modes.

**Spec reference:** POSIX.1-2024, §11.2.5 Local Modes (tag_11_02_05), `basedefs/V1_chap11.html` line 1174.
