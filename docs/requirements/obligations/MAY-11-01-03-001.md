## Extracted Text

> When a controlling process terminates, the controlling terminal is dissociated from the current session, allowing it to be acquired by a new session leader. Subsequent access to the terminal by other processes in the earlier session may be denied, with attempts to access the terminal treated as if a modem disconnect had been sensed.

## Context

**Behavior (optional):** When the controlling process (session leader) terminates, the controlling terminal is dissociated from the session. A new session leader may then acquire it. Access by remaining processes in the old session may be denied — treated as a modem disconnect (SIGHUP delivery, read returns 0, write returns EIO).

**Conditions:** Applies when a session leader (typically the login shell) exits. Other processes in the session may lose terminal access.

**Shell relevance:** When the shell (as session leader) exits, the terminal is dissociated. Child processes still running may receive SIGHUP and lose terminal access. The shell should handle SIGHUP appropriately for its child processes.
