## Extracted Text

> A non-negative integer, which can be contained in an object of type gid_t, that is used to identify a group of system users. Each system user is a member of at least one group. When the identity of a group is associated with a process, a group ID value is referred to as a real group ID, an effective group ID, one of the supplementary group IDs, or a saved set-group-ID. The value (gid_t)-1 shall not be a valid group ID, but does have a defined use in some interfaces defined in this standard.

## Context

**Definition — low shell relevance.** Defines "group ID" as a non-negative integer of type `gid_t` identifying a group of system users. Each user belongs to at least one group. Group IDs appear as real, effective, supplementary, and saved set-group-ID. The value `(gid_t)-1` SHALL NOT be a valid group ID.

**Shell impact:** The shell interacts with group IDs through file permission checks, the `id` utility (external), and the `newgrp` utility. The `umask` builtin affects group permissions on newly created files. Group IDs are part of the process environment inherited by child processes. The shell does not manipulate group IDs directly but inherits them.
