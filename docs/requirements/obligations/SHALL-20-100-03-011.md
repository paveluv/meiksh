## Extracted Text

> The specified var operands shall be processed in the order they appear on the command line, and the output fields generated by the field splitting algorithm shall be used in the order they were generated, by repeating the following checks until neither is true:: If more than one var operand is yet to be processed and one or more output fields are yet to be used, the variable named by the first unprocessed var operand shall be assigned the value of the first unused output field.

## Context

**Behavior required:** Assignment check 1: If MORE THAN ONE var operand has not yet been assigned, AND there are still unused output fields from field splitting, then:
- Assign the first unused output field to the first unassigned var.
- Advance both the var pointer and the field pointer.

**Condition:** During the variable assignment loop when `remaining_vars > 1` AND `remaining_fields >= 1`.

**Example:** `read a b c` with fields `["one", "two", "three"]`:
- Iteration 1: vars remaining = 3 (>1), fields remaining = 3 (≥1) → `a="one"`
- Iteration 2: vars remaining = 2 (>1), fields remaining = 2 (≥1) → `b="two"`
- Iteration 3: vars remaining = 1 (not >1) → check 1 fails, proceed to check 2
