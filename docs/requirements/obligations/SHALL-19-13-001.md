## Extracted Text

> Utilities other than the special built-ins (see 2.15 Special Built-In Utilities) shall be invoked in a separate environment that consists of the following. The initial value of these objects shall be the same as that for the parent shell, except as noted below.

## Context

**Non-special-built-in utilities execute in a separate (child) environment.**

When executing any utility that is NOT a special built-in (see 2.15 Special Built-In Utilities in utilities/V3_chap02.html), the shell **shall** invoke it in a separate environment. This separate environment is initialized as a copy of the parent shell's environment, with specific modifications listed in subsequent obligations (SHALL-19-13-002 through SHALL-19-13-004).

The separate environment includes:
- Open files (inherited + exec-modified + redirections for this command)
- Current working directory
- File creation mask (umask)
- Trap dispositions (reset per SHALL-19-13-002)
- Exported variables (per SHALL-19-13-003)

This means regular built-ins and external commands cannot directly modify the parent shell's variables, traps, working directory, etc. Only special built-ins and commands explicitly documented to affect the environment (like `cd`, `umask`) can do so.
