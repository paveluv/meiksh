## Extracted Text

> The maximum number of bytes in a multi-byte character. This shall default to 1.

## Context

**Behavior:** The `<mb_cur_max>` declaration in a charmap file specifies the maximum number of bytes in a multi-byte character for this character set. If omitted, it defaults to 1 (single-byte encoding).

**Shell relevance:** The shell must handle multi-byte characters correctly in tokenization, pattern matching, parameter expansion, and display width calculations. The value of `MB_CUR_MAX` (derived from this charmap declaration for the active locale) determines whether multi-byte processing is needed. When `MB_CUR_MAX` is 1, the shell can use single-byte optimizations. The shell queries this via the C library (`MB_CUR_MAX` macro), not by reading the charmap file directly.
