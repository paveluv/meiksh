## Extracted Text

> The behavior is unspecified if the terminating ')' is not present in the token containing the command substitution; that is, if the ')' is expected to result from alias substitution.

## Context

**Behavior:** This is a duplicate extraction of the same rule documented in SHALL-19-06-03-007. The closing `)` of a `$(commands)` substitution must be present in the literal source token. If it would only appear through alias expansion, behavior is **unspecified**.

**Implementation guidance:** The parser should locate the closing `)` during initial tokenization without relying on alias expansion. This simplifies parsing and is the portable expectation.

**Spec reference:** See 2.6.3 Command Substitution in utilities/V3_chap02.html, lines 1135–1136.
