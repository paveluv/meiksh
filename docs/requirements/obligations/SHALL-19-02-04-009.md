## Extracted Text

> A sequence of characters starting with a <dollar-sign> immediately followed by a single-quote ($') shall preserve the literal value of all characters up to an unescaped terminating single-quote ('), with the exception of certain <backslash>-escape sequences, as follows:: \n yields a <newline> character.

## Context

**Behavior:** Inside `$'...'`, the escape sequence `\n` yields a newline character.

**Applies to:** All shell modes during `$'...'` escape processing.

**Implementation detail:** Emit byte 0x0A (LF/newline). This is the same character that C's `'\n'` produces. Note that `\<newline>` (backslash followed by an actual newline) inside `$'...'` has unspecified behavior (see UNSPEC-19-02-04-002)—only `\n` (backslash-n) reliably produces a newline.

**Spec reference:** Section 2.2.4, lines 190-191 in `utilities/V3_chap02.html`.
