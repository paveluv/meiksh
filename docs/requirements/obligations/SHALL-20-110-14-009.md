## Extracted Text

> The following exit values shall be returned:: A specified command_file could not be found by a non-interactive shell.

## Context

**Behavior:** The `sh` utility shall return exit status 127 when a specified command_file could not be found by a non-interactive shell.

**Applies to:** Non-interactive shell invocations with a command_file operand (e.g., `sh nonexistent.sh`).

**Details:** This covers the case where:
- `sh` is invoked with a command_file operand.
- The file does not exist at the given path.
- The shell is non-interactive.

The shell should write a diagnostic message to stderr before exiting with status 127 (e.g., "sh: nonexistent.sh: No such file or directory").

**Note:** This applies to the command_file operand of `sh` itself, not to commands within scripts. Command-not-found within scripts is handled by 2.9.1.4 Command Search and Execution.
