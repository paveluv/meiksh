## Extracted Text

> The specified var operands shall be processed in the order they appear on the command line, and the output fields generated by the field splitting algorithm shall be used in the order they were generated, by repeating the following checks until neither is true:: If exactly one var operand is yet to be processed and there was some remaining unsplit input returned from the modified field splitting algorithm, the variable named by the unprocessed var operand shall be assigned the unsplit input.

## Context

**Behavior required:** Assignment check 2: If EXACTLY ONE var operand has not yet been assigned, AND there is remaining unsplit input from the field splitting algorithm (per SHALL-20-100-03-007/009), then:
- Assign the unsplit remainder to the last var.

**Condition:** During the variable assignment loop when `remaining_vars == 1` AND there is unsplit remainder input.

**Example:** `read a b` with input `"one two three"`:
- Check 1: vars=2 (>1), assign `a="one"` from first field
- Check 2: vars=1 (exactly 1), unsplit remainder = `"two three"` → `b="two three"`

**Key distinction:** The unsplit remainder has already had leading/trailing IFS whitespace removed (per the field splitting modification), but internal separators are preserved as-is.
