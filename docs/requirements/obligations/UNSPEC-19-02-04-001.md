## Extracted Text

> \xXX yields the byte whose value is the hexadecimal value XX (one or more hexadecimal digits). If more than two hexadecimal digits follow \x, the results are unspecified.

## Context

**Behavior:** If more than two hexadecimal digits follow `\x` inside `$'...'`, the results are unspecified. The spec only guarantees behavior for `\x` followed by one or two hex digits.

**Applies to:** All shell modes during `$'...'` escape processing.

**Implementation choice:** The safest approach is to consume at most 2 hex digits after `\x` (per SHALL-19-02-04-017, the escape terminates at the first non-hex character or after the maximum count). This naturally avoids the unspecified-behavior case since remaining hex digits become literal. If the implementation chooses to consume more than 2 hex digits, it enters unspecified territory.

**Spec reference:** Section 2.2.4, lines 209-211 in `utilities/V3_chap02.html`.
