## Extracted Text

> Specify the name of an existing locale which shall be used as the definition of this category. If this keyword is specified, no other keyword shall be specified. Note: This is a localedef utility keyword, unavailable through localeconv().

## Context

This is the `copy` keyword description for LC_MONETARY. When `copy` is used, the specified locale's monetary category is inherited verbatim, and no other LC_MONETARY keywords may appear in the definition. This is a `localedef` utility mechanism — the shell does not parse locale definitions. The shell's responsibility is to correctly handle and propagate the LC_MONETARY environment variable so the system locale infrastructure can resolve copied definitions.

See: `basedefs/V1_chap07.html` §7.3.3 LC_MONETARY.
