## Extracted Text

> The resolution of timestamps of files in a file system is implementation-defined, but shall be no coarser than one-second resolution. The three timestamps shall always have values that are supported by the file system. Whenever any of a file's timestamps are to be set to a value V according to the rules of the preceding paragraphs of this section, the implementation shall immediately set the timestamp to the greatest value supported by the file system that is not greater than V.

## Context

File timestamp resolution is at least one-second granularity (may be finer, e.g., nanosecond). For the shell, this affects the `test` builtin's `-nt` (newer than) and `-ot` (older than) operators, and any place the shell compares file modification times. The shell must be aware that two files modified within the same second may have identical timestamps on file systems with only one-second resolution. No special shell code is needed — this is an OS-level guarantee. The shell relies on the OS providing timestamps with at least one-second resolution.

See: `basedefs/V1_chap04.html` §4.12 File Times Update.
