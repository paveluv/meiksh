## Extracted Text

> The following rules and definitions apply to bracket expressions:: An equivalence class expression shall represent the set of collating elements belonging to an equivalence class, as described in 7.3.2.4 Collation Order. Only primary equivalence classes shall be recognized. The class shall be expressed by enclosing any one of the collating elements in the equivalence class within bracket-equal ("[=" and "=]") delimiters. For example, if 'a', 'à', and 'â' belong to the same equivalence class, then "[[=a=]b]", "[[=à=]b]", and "[[=â=]b]" are each equivalent to "[aàâb]". If the collating element does not belong to an equivalence class, the equivalence class expression shall be treated as a collating symbol.

## Context

**Behavior:** An equivalence class expression represents the set of collating elements belonging to an equivalence class. Only primary equivalence classes shall be recognized. The class is expressed by enclosing any one of its collating elements within `[=` and `=]` delimiters. For example, if 'a', 'à', and 'â' are in the same equivalence class, `[[=a=]b]` is equivalent to `[aàâb]`. If the element does not belong to an equivalence class, the expression shall be treated as a collating symbol.

**Shell relevance:** The shell must support equivalence class expressions in bracket expressions within glob and `case` patterns. The parser must recognize `[=` and `=]` delimiters. The equivalence class membership is determined by the locale's `LC_COLLATE` settings. In the POSIX locale, each character is its own equivalence class, so `[[=a=]]` matches only `a`.
