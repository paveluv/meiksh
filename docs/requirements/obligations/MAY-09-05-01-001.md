## Extracted Text

> The following tokens are processed (in addition to those string constants shown in the grammar):: Applicable only to basic regular expressions. The character '^' when it appears either as the first character of a basic regular expression or, if the implementation does not match the escape sequence "\|" to the literal character '|', when used immediately following a "\|" escape sequence that is not inside a subexpression, and when not QUOTED_CHAR. The '^' may be recognized as an anchor elsewhere; see 9.3.8 BRE Expression Anchoring.

## Context

This applies only to BRE mode. The `L_ANCHOR` token (`^`) is recognized as such when `^` appears as the first character of a BRE, or immediately after a `\|` (if the implementation supports `\|` as alternation rather than literal `|`), provided it is not inside a subexpression and is not a QUOTED_CHAR. The implementation MAY also recognize `^` as an anchor in other positions within a BRE (Section 9.3.8). This is optional behavior—the implementation may choose to treat `^` as an anchor only in the specified positions or extend anchor recognition to additional positions. For `meiksh`, deciding whether to extend anchor recognition beyond the required positions is an implementation choice.
