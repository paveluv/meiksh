## Extracted Text

> When a given simple command is required to be executed (that is, when any conditional construct such as an AND-OR list or a case statement has not bypassed the simple command), the following expansions, assignments, and redirections shall all be performed from the beginning of the command text to the end:: Redirections shall be performed as described in 2.7 Redirection.

## Context

**Behavior:** Step 3 of simple command processing: Redirections saved in step 1 MUST be performed according to the rules in section 2.7 Redirection. This includes opening/closing file descriptors, here-documents, here-strings, duplicating descriptors, etc.

**Applies to:** All simple commands with redirections. Redirections are processed after word expansion (step 2) but potentially before or after variable assignments (step 4) — see MAY-19-09-01-01-001 for the ordering flexibility.

**Implementation detail:** The redirect words must be expanded (parameter expansion, command substitution, etc. in the filename) before the redirect is performed. Redirections are processed left-to-right as they appear in the command text. If any redirection fails, the command fails with a nonzero exit status and subsequent redirections are not performed.

**Spec reference:** Section 2.9.1.1, line 1751 in `utilities/V3_chap02.html`. Cross-ref: 2.7 Redirection.
