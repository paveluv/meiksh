## Extracted Text

> If the current character is an unquoted <backslash>, single-quote, or double-quote or is the first character of an unquoted <dollar-sign> single-quote sequence, it shall affect quoting for subsequent characters up to the end of the quoted text. The rules for quoting are as described in 2.2 Quoting. During token recognition no substitutions shall be actually performed, and the result token shall contain exactly the characters that appear in the input unmodified, including any embedded or enclosing quotes or substitution operators, between the start and the end of the quoted text. The token shall not be delimited by the end of the quoted field.

## Context

**Behavior:** Token recognition Rule 4: If the current character is an unquoted backslash, single-quote, or double-quote, or is the first character of an unquoted `$'` sequence, it triggers quoting. Key requirements:

1. **Quoting affects subsequent characters** up to the end of the quoted text (per the rules in 2.2 Quoting: `\` escapes one character, `'...'` quotes until closing `'`, `"..."` quotes until closing `"`, `$'...'` quotes until closing unescaped `'`).

2. **No substitutions during tokenization:** During token recognition, no parameter expansion, command substitution, or arithmetic expansion is actually performed. The raw characters (including quote characters, `$`, `{`, `}`, etc.) are preserved verbatim in the token.

3. **Token preservation:** The result token contains exactly the characters from the input, unmodified, including any embedded or enclosing quotes or substitution operators.

4. **No delimiting:** The end of a quoted region does NOT delimit the token—the token continues and may accumulate more characters after the closing quote.

**Applies to:** All shell modes during tokenization.

**Implementation detail:** The tokenizer must maintain a quoting state stack/flag. When entering a quoted region, characters are accumulated into the current token without being checked against operator or blank rules (they are "quoted"). When the quoted region ends, tokenization continues in the normal state, with the accumulated characters as part of the same token.

**Example:** In `he"ll"o`, the tokenizer sees: `h` (word), `e` (word), `"` (enter double-quote), `l` (quoted), `l` (quoted), `"` (exit double-quote), `o` (word). The result is a single token `he"ll"o` containing exactly those 7 characters.

**Spec reference:** Section 2.3, rule 4, lines 267-270 in `utilities/V3_chap02.html`. Cross-ref: 2.2 Quoting.
