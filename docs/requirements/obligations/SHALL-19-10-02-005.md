## Extracted Text

> [NAME in for] When the TOKEN meets the requirements for a name (see XBD 3.216 Name), the token identifier NAME shall result. Otherwise, the token WORD shall be returned.

## Context

**Grammar Rule 5 (NAME in `for` loop):** In the grammar production `for_clause : For name ...`, the token after `for` must be a NAME. If the TOKEN meets the requirements for a Name (see XBD 3.216 Name in `basedefs/V1_chap03.html#tag_03_216`—a word consisting solely of underscores, digits, and letters from the portable character set, not beginning with a digit), the token identifier NAME shall result. Otherwise, the token WORD shall be returned (which would then cause a syntax error since the grammar requires a NAME here).

A valid Name matches the regex `[_a-zA-Z][_a-zA-Z0-9]*`. The implementation must validate the token against this pattern when Rule 5 applies.
