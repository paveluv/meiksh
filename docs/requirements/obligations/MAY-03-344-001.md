## Extracted Text

> A resource limitation established for each process that the process may set to any value less than or equal to the hard limit.

## Context

**Definition — moderate shell relevance.** Defines "soft limit" as a per-process resource limitation that may be set to any value ≤ the hard limit.

**Shell impact:** The `ulimit` builtin (XSI) manages soft limits by default (without `-H`). Soft limits are the effective resource constraints for the process. `ulimit -S` explicitly targets the soft limit. The shell must use `getrlimit()`/`setrlimit()` to query and modify soft limits. Setting a soft limit above the hard limit fails.
