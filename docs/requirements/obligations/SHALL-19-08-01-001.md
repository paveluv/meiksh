## Extracted Text

> Certain errors shall cause the shell to write a diagnostic message to standard error and exit as shown in the following table:

## Context

Introduces the shell error consequences table, requiring a diagnostic message and potential exit for certain error categories. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required:**
- For the error categories listed in the table (syntax errors, special built-in errors, other utility errors, redirection errors, variable assignment errors, expansion errors, command not found, unrecoverable read errors), the shell SHALL write a diagnostic message to standard error (fd 2).
- Whether the shell exits depends on the error category and whether the shell is interactive or non-interactive, as specified in the table.
- The "Shell Diagnostic Message Required" column in the table specifies whether the shell itself must write a diagnostic (as opposed to the utility writing one).

**This is the overarching obligation.** The specific exit/no-exit behaviors for each error category are detailed in SHALL-19-08-01-002 through SHALL-19-08-01-022. The diagnostic message should identify the error and be written to stderr.
