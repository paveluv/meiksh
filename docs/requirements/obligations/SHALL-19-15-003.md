## Extracted Text

> The term "built-in" implies that there is no need to execute a separate executable file because the utility is implemented in the shell itself. An implementation may choose to make any utility a built-in; however, the special built-in utilities described here differ from regular built-in utilities in two respects:: As described in 2.9.1 Simple Commands, variable assignments preceding the invocation of a special built-in utility affect the current execution environment; this shall not be the case with a regular built-in or other utility.

## Context

**Variable assignments before special built-ins persist; before regular built-ins they do not.**

As described in 2.9.1 Simple Commands (utilities/V3_chap02.html), variable assignments that precede a command (e.g., `VAR=value command`) have different effects depending on the command type:

- **Special built-in**: `VAR=value export FOO` — the assignment `VAR=value` **shall** affect the current execution environment (it persists after the command completes).
- **Regular built-in or external command**: `VAR=value echo hello` — the assignment **shall not** persist. It is only in effect for the duration of that command.

Implementation: when processing simple commands, if the resolved command is a special built-in, apply prefix variable assignments to the current shell environment before executing. For regular built-ins and external commands, apply them temporarily and restore after execution (or pass them only to the child environment).
