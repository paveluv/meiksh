## Extracted Text

> the value of the corresponding environment variable is used. If the environment variable is unset or is set to the empty string, the implementation shall set the appropriate environment as defined in 8. Environment Variables.

## Context

**Behavior:** When a locale environment variable (`LANG`, `LC_*`) is unset or set to the empty string, the implementation must set the appropriate locale environment as defined in Chapter 8 (Environment Variables). This means the system falls back to its default locale behavior.

**Shell relevance:** The shell must follow the locale environment variable precedence rules:
1. `LC_ALL` overrides all individual `LC_*` variables.
2. Individual `LC_*` variables (e.g., `LC_CTYPE`, `LC_COLLATE`) override `LANG` for their category.
3. `LANG` provides the default for any unset `LC_*` variable.
4. If all are unset/empty, the implementation's default locale applies.

The shell should call `setlocale(LC_ALL, "")` at startup to activate locale based on environment variables, and re-call it when locale variables are modified during execution.
