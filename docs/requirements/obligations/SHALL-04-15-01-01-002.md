## Extracted Text

> Certain system interfaces synchronize with other system interfaces performed by another thread. In particular, an atomic operation A that performs a release operation on an object M shall synchronize with an atomic operation B that performs an acquire operation on M and reads a value written by any side effect in the release sequence headed by A.

## Context

This is a C-level memory model requirement about release/acquire synchronization semantics for atomic operations. It is not relevant to shell language semantics. For a Rust-based shell, Rust's `Ordering::Release` and `Ordering::Acquire` provide equivalent guarantees. No shell-language-level action is required.

See: `basedefs/V1_chap04.html` §4.15.1.1 Data Races.
