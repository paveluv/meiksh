## Context

**Behavior:** After pre-processing (parameter expansion, command substitution, quote removal), the resulting string is evaluated as an arithmetic expression. The evaluation follows the rules from "1.1.2.1 Arithmetic Precision and Operations" in the Shell & Utilities volume, which defines:
- C-language-style arithmetic operators and precedence
- Integer arithmetic semantics
- Assignment operators within expressions
- Conditional (ternary) operator

The result of the evaluation is substituted as a decimal string.

The following obligations (SHALL-19-06-04-004 through SHALL-19-06-04-007) list the exceptions/restrictions compared to full C arithmetic.

**Spec reference:** See 2.6.4 Arithmetic Expansion in utilities/V3_chap02.html, lines 1166–1169. For the base arithmetic rules, see utilities/V3_chap01.html#tag_18_01_02_01 (1.1.2.1 Arithmetic Precision and Operations).
