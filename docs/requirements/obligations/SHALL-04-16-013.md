## Extracted Text

> Pathname resolution for a given pathname shall yield the same results when used by any interface in POSIX.1-2024 as long as there are no changes to any files evaluated during pathname resolution for the given pathname between resolutions.

## Context

This is a consistency guarantee: the same pathname must resolve to the same file across all POSIX interfaces (shell builtins, utilities, syscalls) as long as the file system doesn't change between resolutions. For the shell, this means that if the shell resolves a pathname for `test -f` and then immediately opens it for redirection, it will access the same file (assuming no concurrent modification). The shell can rely on this property for correctness — there's no need to worry about different resolution algorithms for different operations.

This is an OS-level guarantee. No special shell code is needed.

See: `basedefs/V1_chap04.html` §4.16 Pathname Resolution.
