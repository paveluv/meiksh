## Extracted Text

> Each field containing the results from an expansion shall be processed in order, intermixed with fields not containing the results of expansions, processed as described above, as if by using the following algorithm, examining bytes in the input field, from beginning to end:: Once the input is empty, the candidate shall become an output field if and only if it is not empty.

## Context

**Behavior:** This is the termination step of the field splitting algorithm. When all input bytes have been consumed (the loop has finished because the input is empty):
- If the **candidate field is not empty** (it accumulated at least one byte), it becomes the final output field.
- If the **candidate field is empty** (all remaining input was consumed by IFS delimiters, or the input was empty from the start), no additional output field is produced.

This prevents trailing IFS delimiters from producing a spurious empty field at the end.

**Inputs/Outputs:**
- `IFS=:; x="a:b:"; echo $x` → `a` and `b` (no trailing empty field because candidate is empty at termination)
- `IFS=:; x="a:b:c"; echo $x` → `a`, `b`, `c` (candidate `c` is non-empty at termination, becomes output)

**Spec reference:** See 2.6.5 Field Splitting in utilities/V3_chap02.html, line 1285.
