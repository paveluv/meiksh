## Extracted Text

> While in insert mode, any character typed shall be inserted in the current command line, unless it is from the following set.

## Context

This obligation defines the **default behavior of vi-mode insert mode** for the interactive shell's command line editor.

**Behavior required:** When `set -o vi` is active and the shell is in insert mode (which is the initial mode upon entering `sh` and after each command terminates), every character typed by the user shall be inserted into the current command line buffer at the current cursor position. The cursor advances one position to the right after each insertion.

**Exceptions:** This default insertion behavior does NOT apply to a specific set of special characters enumerated in the vi Line Editing Insert Mode section (see `utilities/sh.html`, tag_20_110_13_04). The special characters that override normal insertion are:
- `<newline>` — executes the command line (see SHALL-20-110-13-04-002)
- `erase` (terminal erase character, typically Backspace) — deletes previous character (see SHALL-20-110-13-04-003)
- `interrupt` (terminal interrupt character, typically Ctrl-C) — terminates editing via SIGINT (see SHALL-20-110-13-04-004)
- `kill` (terminal kill character, typically Ctrl-U) — clears the entire input line
- `<control>-V` — literal next: inserts the next character verbatim, even if it would otherwise be special
- `<control>-W` — word delete: deletes backward to the preceding word boundary
- `end-of-file` (terminal EOF character, typically Ctrl-D) — interpreted as end of input (see SHALL-20-110-13-04-005)
- `<ESC>` — switches from insert mode to command mode

**Conditions:** Only applies when the shell is interactive, vi editing mode is enabled (`set -o vi`), and the shell is currently in insert mode (not command mode).

**Terminal characters:** The `erase`, `interrupt`, `kill`, and `end-of-file` characters are those configured by `stty` (see `utilities/sh.html` line 325–326).
