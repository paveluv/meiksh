## Extracted Text

> The kill utility shall send a signal to the process or processes specified by each pid operand.

## Context

**Behavior:** The `kill` utility **shall** send a signal to the process(es) specified by each `pid` operand. This is the fundamental purpose of the `kill` command.

**Conditions:** The pid operand can be:
- A positive integer: signal is sent to that specific process
- 0: signal is sent to all processes in the current process group
- -1: signal is sent to all processes the user has permission to signal
- A negative integer (other than -1): signal is sent to all processes in the process group whose ID is the absolute value

**Shell relevance:** `kill` is a required shell builtin (per POSIX shell spec). The shell must implement `kill` as a builtin because:
1. It must be able to signal jobs using job IDs (`kill %1`)
2. It must work even when external command execution is impaired (e.g., when too many processes exist)
The builtin uses the `kill()` system call to deliver signals.
