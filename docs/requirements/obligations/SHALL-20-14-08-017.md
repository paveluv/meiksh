## Extracted Text

> The following environment variables shall affect the execution of cd:: A pathname of the previous working directory, used when the operand is '-'. If an application sets or unsets the value of OLDPWD , the behavior of cd with a '-' operand is unspecified.

## Context

**Behavior:** OLDPWD semantics for cd:

1. **Purpose:** OLDPWD holds a pathname of the previous working directory. It is used when the cd operand is `-`.
2. **Normal use:** When the user runs `cd -`, the shell reads OLDPWD and behaves as if that value were the directory operand.
3. **Application modification:** If an application (user script) explicitly sets or unsets OLDPWD, the behavior of `cd -` is unspecified — the implementation may do anything.
4. **Shell responsibility:** The shell itself sets OLDPWD to the old cwd whenever it successfully changes directories and sets PWD. The shell should not assume OLDPWD is valid if it was externally modified.

**Spec reference:** POSIX.1-2024, `utilities/cd.html`, section ENVIRONMENT VARIABLES (tag_20_14_08), lines 201-202.
