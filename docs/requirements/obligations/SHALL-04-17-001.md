## Extracted Text

> A process group ID shall not be reused by the system until the process group lifetime ends.

## Context

The shell uses process group IDs extensively for job control. Each pipeline started by the shell in job control mode is placed in its own process group. The shell tracks jobs by their process group IDs for purposes of `fg`, `bg`, `jobs`, `kill`, and `wait` builtins. This guarantee ensures that a process group ID the shell is tracking will not be reassigned to a different process group while the shell still considers the job active. The shell can safely use stored process group IDs for sending signals (e.g., `kill -SIGTERM -pgid`) without risk of targeting an unrelated process group.

This is an OS-level guarantee. No special shell code is needed, but the shell's job table management relies on this property.

See: `basedefs/V1_chap04.html` §4.17 Process ID Reuse.
