## Extracted Text

> The following exit values shall be returned:: >1

## Context

**Behavior required:** The `read` builtin must return an exit status greater than 1 when an error occurs (distinct from EOF which is exactly 1).

**Condition:** When an error occurs during `read` processing, such as:
- Attempting to assign to a readonly variable
- Invalid option specified
- Other processing errors

**Implementation detail:** Return a value >1 (e.g., 2) on error. This is distinct from:
- 0: success (delimiter found)
- 1: EOF (no delimiter, but not an error)
- >1: actual error

Common convention is to return 2 for errors, but any value >1 is conforming.
