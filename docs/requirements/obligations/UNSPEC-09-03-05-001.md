## Extracted Text

> The following rules and definitions apply to bracket expressions:: If a bracket expression contains at least three list elements, where the first and last list elements are the same single-character element of <period>, <equals-sign>, or <colon>, then it is unspecified whether the bracket expression will be treated as a collating symbol, equivalence class, or character class, respectively; treated as a matching list expression; or treated as an invalid bracket expression.

## Context

**Behavior:** If a bracket expression contains at least three list elements where the first and last elements are the same single character of `.`, `=`, or `:`, it is unspecified whether the expression is treated as a collating symbol/equivalence class/character class, a matching list expression, or an invalid expression. Example: `[.a.b.]` could be interpreted as collating symbol `a.b`, a matching list of `.`, `a`, `.`, `b`, `.`, or an error.

**Shell relevance:** The shell should avoid ambiguous bracket expression parsing in these edge cases. Since behavior is unspecified, the shell may choose any interpretation. A reasonable approach is to prioritize collating symbol/equivalence class/character class interpretation when the delimiters match, and fall back to matching list for other cases.
