## Extracted Text

> A collation sequence definition shall define the relative order between collating elements (characters and multi-character collating elements) in the locale. This order is expressed in terms of collation values; that is, by assigning each element one or more collation values (also known as collation weights). This does not imply that implementations shall assign such values, but that ordering of strings using the resultant collation definition in the locale behaves as if such assignment is done and used in the collation process. At least the following capabilities are provided:: Ordering by weights. When two strings are compared to determine their relative order, the two strings are first broken up into a series of collating elements; the elements in each successive pair of elements are then compared according to the relative primary weights for the elements. If equal, and more than one weight has been assigned, then the pairs of collating elements are re-compared according to the relative subsequent weights, until either a pair of collating elements compare unequal or the weights are exhausted.

## Context

**Behavior:** Multi-level string comparison algorithm:
1. Break strings into collating elements.
2. Compare elements pairwise using primary weights.
3. If equal at primary level and more weights exist, re-compare using secondary weights.
4. Continue through all weight levels until unequal or exhausted.

This is the `strcoll()` / `wcscoll()` algorithm.

**Shell relevance:** This multi-level comparison affects the shell when it uses `strcoll()` for locale-aware string comparison. The shell's `[[ str1 < str2 ]]` comparison operator may use `strcoll()`. Pattern matching range expressions (`[a-z]`) use the collation sequence. The shell typically delegates this to the C library.
