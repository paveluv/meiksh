## Extracted Text

> The following environment variables shall affect the execution of sh:: Determine the pathname of the user's home directory. The contents of HOME are used in tilde expansion as described in 2.6.1 Tilde Expansion.

## Context

**Behavior:** HOME determines the pathname of the user's home directory. Its contents are used in tilde expansion:
1. An unquoted `~` at the beginning of a word expands to the value of HOME.
2. `~username` expands to the home directory of the specified user (from the user database, not HOME).

**Applies to:** All shell modes (interactive and non-interactive).

**Details:** The shell uses HOME for:
- Tilde expansion (`~` → `$HOME`), see 2.6.1 Tilde Expansion in `utilities/V3_chap02.html#tag_19_06_01`.
- The `cd` built-in with no arguments (changes to `$HOME`).
- As the fallback location for `.sh_history` when HISTFILE is unset.

**Error handling:** If HOME is unset, tilde expansion of bare `~` produces unspecified results. The `cd` built-in with no arguments shall report an error if HOME is unset.
