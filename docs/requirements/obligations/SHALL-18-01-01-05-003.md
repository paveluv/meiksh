## Extracted Text

> When a directory that is the root directory or current working directory of any process is removed, the effect is implementation-defined. If file access permissions deny access, the requested operation shall fail. Otherwise, when a file is removed:: The link count of the file shall be decremented.

## Context

When a file is removed, the link count of the file shall be decremented (step 2 of file removal). If the file has multiple hard links, the file data remains accessible via other links. For `meiksh`, this is relevant because the shell may have open file descriptors to files being removed — the file data remains accessible through the open descriptor even after the last directory entry is removed. See POSIX Utilities Chapter 1, Section 1.1.1.5 item 2 (utilities/V3_chap01.html line 699).
