## Extracted Text

> it shall not affect the file mode creation mask of the caller's environment.

## Context

**Behavior:** When `umask` is invoked in a subshell or a separate utility execution environment (e.g., `(umask 002)`, `nohup umask ...`, `find . -exec umask ... \;`), the file mode creation mask change shall NOT propagate to the caller's (parent) shell environment.

**Conditions:** This applies whenever `umask` runs in a context that creates a new execution environment separate from the current shell, including: subshells created by `(...)`, command substitution `$(...)`, and utilities that create their own execution environments (like `nohup`, `find -exec`).

**Implementation detail:** Since subshells inherit a copy of the parent environment, changes to the umask in a child process naturally do not affect the parent. The shell must ensure it does not provide any mechanism for subshell umask changes to propagate upward.
