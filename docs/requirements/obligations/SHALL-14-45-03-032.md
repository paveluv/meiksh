## Extracted Text

> The <signal.h> header shall define the following macros which shall expand to integer constant expressions that need not be usable in #if preprocessing directives:: [CX] The resulting set is the union of the current set and the signal set pointed to by the argument set.

## Context

This is the [CX] extension defining the semantics of `SIG_BLOCK`: when used as the `how` argument to `sigprocmask()`, the resulting signal mask is the union of the current set and the signal set pointed to by the `set` argument. In other words, `sigprocmask(SIG_BLOCK, &set, NULL)` adds all signals in `set` to the currently blocked signals without removing any. For `meiksh`, this is used to temporarily block signals like `SIGCHLD`, `SIGINT`, or `SIGTSTP` during critical sections such as updating job tables or modifying shared state. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 501–502).
