## Extracted Text

> The -a, -b, -C, -e, -f, -h, -m, -n, -o option, -u, -v, and -x options are described as part of the set utility in 2.15 Special Built-In Utilities. The option letters derived from the set special built-in shall also be accepted with a leading <plus-sign> ('+') instead of a leading <hyphen-minus> (meaning the reverse case of the option as described in this volume of POSIX.1-2024). If the -o or +o option is specified without an option-argument, the behavior is unspecified.

## Context

The `sh` command line must accept the following options, which are defined by the `set` special builtin (see 2.26 set in utilities/V3_chap02.html#tag_19_26 and 2.15 Special Built-In Utilities in utilities/V3_chap02.html#tag_19_15):

- `-a` (allexport), `-b` (notify), `-C` (noclobber), `-e` (errexit), `-f` (noglob), `-h` (hashall), `-m` (monitor), `-n` (noexec), `-o option` (named options), `-u` (nounset), `-v` (verbose), `-x` (xtrace)

Each of these option letters must also be accepted with a leading `+` instead of `-`, which reverses the meaning (disables the option). For example, `sh +e` means "do not enable errexit", while `sh -e` means "enable errexit".

**Unspecified behavior**: If `-o` or `+o` is specified **without** an option-argument, the behavior is unspecified.

See utilities/sh.html#tag_20_110_04 and utilities/V3_chap02.html#set for `set` option definitions.
