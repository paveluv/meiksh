## Extracted Text

> All implementations shall define a locale as the default locale, to be invoked when no environment variables are set, or set to the empty string. This default locale can be the POSIX locale or any other implementation-defined locale. Some implementations may provide facilities for local installation administrators to set the default locale, customizing it for each location. POSIX.1-2024 does not require such a facility.

## Context

**Behavior:** Every implementation must define a default locale used when no locale environment variables are set (or all are empty). This default may be the POSIX locale or an implementation-defined locale. Administrators may be able to configure the system default.

**Shell relevance:** When the shell starts with no `LANG`, `LC_ALL`, or individual `LC_*` variables set, `setlocale(LC_ALL, "")` will activate the system's default locale. The shell must handle this gracefully — the default locale might be UTF-8 or any other encoding. The shell should not assume the default locale is POSIX/C.
