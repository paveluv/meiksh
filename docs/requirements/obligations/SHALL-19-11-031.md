## Extracted Text

> If set -b is enabled, the message shall be written either immediately after the job became suspended or immediately prior to writing the next prompt for input.

## Context

**Background job stopped notification — `set -b` enabled (detail):** Same content as SHALL-19-11-029. When `set -b` is enabled, the suspended-job message for a stopped background job shall be written either immediately when the suspension is detected or immediately prior to the next prompt. The implementation has flexibility in choosing which of these two timings to use.
