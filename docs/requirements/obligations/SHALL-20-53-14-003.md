## Extracted Text

> The following exit values shall be returned:: An option, specified or unspecified by optstring, was found.

## Context

**Behavior:** Exit status 0 means an option was found during parsing. "Specified or unspecified by optstring" means this applies whether the option was recognized (in `optstring`) or not (unknown option). Both cases return 0 because getopts is still processing options; it hasn't reached the end.

**Key point:** Even when the option character is not in `optstring`, exit status is 0 (not 1). The error is reported via setting `name` to `?` and optionally writing to stderr, but the loop continues.

**Spec reference:** POSIX.1-2024, `getopts` EXIT STATUS, line 205 of `utilities/getopts.html`.
