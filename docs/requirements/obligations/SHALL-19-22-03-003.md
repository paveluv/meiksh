## Extracted Text

> If n is not specified, the result shall be as if n were specified with the current value of the special parameter '?' (see 2.5.2 Special Parameters), except that if the exit command would cause the end of execution of a trap action, the value for the special parameter '?' that is considered "current" shall be the value it had immediately preceding the trap action.

## Context

**Behavior:** When `exit` is called without `n`, the exit status must be the current value of `$?`. However, if the `exit` command would cause the end of execution of a trap action, the "current" value of `$?` is the value it had **immediately preceding** the trap action (not the value within the trap).

**Example:** If `$?` was 42 when a trap fired, and within the trap `$?` became 0 due to successful commands, `exit` at the end of the trap uses 42 (the pre-trap value).

**Applies to:** All shell modes.

**Spec reference:** `exit` DESCRIPTION in `utilities/V3_chap02.html`, lines 4053-4057. Cross-ref: 2.5.2 Special Parameters, `trap` built-in.
