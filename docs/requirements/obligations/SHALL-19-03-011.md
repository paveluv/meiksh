## Extracted Text

> If the previous character was part of a word, the current character shall be appended to that word.

## Context

**Behavior:** Token recognition Rule 8: If the previous character was part of a word (not an operator, not a blank), the current character is appended to that word token.

**Applies to:** All shell modes during tokenization. This rule only fires if none of the higher-priority rules (1-7) applied.

**Implementation detail:** This is the "default continue" rule for word tokens. If we're currently building a word and the current character isn't an operator-starter, blank, quote character, `$`, `` ` ``, `#`, or EOF, it simply gets appended to the current word. This handles ordinary characters like letters, digits, `-`, `.`, `/`, `=`, etc.

**Spec reference:** Section 2.3, rule 8, lines 298-299 in `utilities/V3_chap02.html`.
