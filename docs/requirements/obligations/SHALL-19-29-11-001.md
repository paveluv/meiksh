## Extracted Text

> The standard error shall be used only for diagnostic messages and warning messages about invalid signal names [XSI] or numbers.

## Context

The `trap` special builtin (section 2.29 in utilities/V3_chap02.html#tag_19_29) must use standard error **only** for diagnostic messages and **warning messages about invalid signal names** [XSI] **or numbers**.

This is slightly broader than most builtins' stderr usage:
- **Diagnostic messages**: errors like syntax errors, invalid usage
- **Warnings about invalid signal names/numbers** [XSI extension]: when an unrecognized signal name or out-of-range signal number is provided, the shell writes a warning to stderr. The [XSI] tag means this specific behavior (warning about invalid signal numbers) is part of the X/Open System Interfaces extension.

Note that `trap -p` output goes to stdout, not stderr.

See 2.29 trap in utilities/V3_chap02.html#tag_19_29.
