## Extracted Text

> Remove Largest Suffix Pattern. The word shall be expanded to produce a pattern. The parameter expansion shall then result in parameter, with the largest portion of the suffix matched by the pattern deleted.

## Context

**Behavior:** The `${parameter%%word}` expansion (Remove Largest Suffix Pattern):
1. Expand `word` to produce a pattern (using pattern matching notation per 2.14).
2. Match the pattern against the **suffix** (end) of the value of `parameter`.
3. Find the **largest** (longest) matching suffix.
4. Return the value of `parameter` with that largest suffix removed.

**Inputs/Outputs:**
- `x="/usr/local/bin"; echo ${x%%/*}` → (empty string — longest match of `/*` from the suffix is the entire string starting from the first `/`)
- `x="file.tar.gz"; echo ${x%%.*}` → `file` (removes `.tar.gz`, the longest `.`-suffix)

**Spec reference:** See 2.6.2 Parameter Expansion in utilities/V3_chap02.html, lines 979–981.
