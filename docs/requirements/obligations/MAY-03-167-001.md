## Extracted Text

> A system resource limitation that may be reset to a lesser or greater limit by a privileged process. A non-privileged process is restricted to only lowering its hard limit.

## Context

**Definition — moderate shell relevance.** Defines "hard limit" as a system resource limitation that a privileged process may raise or lower, while a non-privileged process can only lower it.

**Shell impact:** The `ulimit` builtin (XSI) allows users to query and set both hard and soft limits. The `-H` flag of `ulimit` targets the hard limit. Lowering a hard limit is irreversible for non-privileged processes. `meiksh` must implement `ulimit -H` correctly using `setrlimit()`/`getrlimit()`.
