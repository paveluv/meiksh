**Behavior:** For each pid operand, `kill` **shall** perform actions equivalent to calling the `kill()` system function with the arguments specified immediately below in the same POSIX section.

**Conditions:** Applies to each pid operand provided on the command line. The equivalence to `kill()` means:
- Permission checking follows the rules of `kill()` (EPERM if insufficient privileges)
- Signal delivery semantics match `kill()`
- Error handling matches `kill()` (return values, errno)

**Shell relevance:** The shell's `kill` builtin must match the externally observable semantics of `kill()` for each `pid` operand. When job IDs are used, the shell must first resolve them to the appropriate process or process-group target before applying those semantics.
