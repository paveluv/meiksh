## Context

**Behavior:** For each pid operand, `kill` **shall** perform actions equivalent to calling the `kill()` system function. This means the shell builtin must use `kill(pid, sig)` (or equivalent) — it cannot use an alternative signaling mechanism.

**Conditions:** Applies to each pid operand provided on the command line. The equivalence to `kill()` means:
- Permission checking follows the rules of `kill()` (EPERM if insufficient privileges)
- Signal delivery semantics match `kill()`
- Error handling matches `kill()` (return values, errno)

**Shell relevance:** The shell's `kill` builtin must call the `kill()` system call for each pid. When job IDs are used (e.g., `kill %1`), the shell must first resolve the job ID to a process group ID and then call `kill(-pgid, sig)` to signal the entire job's process group.

**Spec reference:** POSIX.1-2024, `kill` DESCRIPTION (tag_20_64_03), `utilities/kill.html` lines 45–46. See also `kill()` in System Interfaces.
