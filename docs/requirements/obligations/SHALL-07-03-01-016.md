## Extracted Text

> Define one or more locale-specific character class names as strings separated by <semicolon> characters. Each named character class can then be defined subsequently in the LC_CTYPE definition. A character class name shall consist of at least one and at most {CHARCLASS_NAME_MAX} bytes of alphanumeric characters from the portable filename character set. The first character of a character class name shall not be a digit. The name shall not match any of the LC_CTYPE keywords defined in this volume of POSIX.1-2024. Future versions of this standard will not specify any LC_CTYPE keywords containing uppercase letters.

## Context

**Behavior:** The `charclass` keyword allows defining custom locale-specific character classes:
- Names are alphanumeric from the portable filename character set, 1 to `{CHARCLASS_NAME_MAX}` bytes.
- First character must not be a digit.
- Names must not conflict with standard LC_CTYPE keywords.
- POSIX guarantees future keywords will not contain uppercase, so custom classes using uppercase names are safe.

**Shell relevance:** Custom character classes defined via `charclass` can be used in bracket expressions: `[[:classname:]]`. The shell must support user-defined character classes in pattern matching if the locale provides them. This is accessed via `wctype()` / `iswctype()` C library functions.
