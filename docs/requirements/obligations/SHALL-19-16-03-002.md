> A loop shall enclose a break or continue command if the loop lexically encloses the command. A loop lexically encloses a break or continue command if the command is:

## Context

**Behavior:** A loop "encloses" a `break` or `continue` command if the loop **lexically encloses** the command. "Lexically encloses" is defined by three conditions (detailed in SHALL-19-16-03-003 through -005). This means enclosure is determined by the source text structure, not by dynamic call stack — a loop in one function does not enclose a `break` in a called function.

**Applies to:** All shell modes.

**Spec reference:** `break` DESCRIPTION in `utilities/V3_chap02.html`, lines 3137-3138.
