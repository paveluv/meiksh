## Extracted Text

> The following rules are used to construct patterns matching multiple characters from patterns matching a single character:: The concatenation of patterns matching a single character is a valid pattern that shall match the concatenation of the single characters or collating elements matched by each of the concatenated patterns.

## Context

**Concatenation of single-character patterns matches concatenation of characters.**

The concatenation of patterns that each match a single character forms a valid pattern that **shall** match the concatenation of the single characters (or collating elements) matched by each component.

Example: if `a` matches `a`, `?` matches any character, and `c` matches `c`, then the pattern `a?c` matches any 3-character string starting with `a` and ending with `c`.

This is the basic pattern composition rule: single-character patterns combine left-to-right to match fixed-length strings where each position satisfies its corresponding sub-pattern.

See 2.14.2 in utilities/V3_chap02.html.
