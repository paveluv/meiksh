## Context

**Behavior:** The default output format of `umask` (when invoked without `-S` and without a mask operand) is implementation-defined. However, the output shall be a string that, if passed as a `mask` operand to a subsequent `umask` invocation on the same system, restores the same file mode creation mask.

**Round-trip requirement:** The output of `umask` (without `-S`) must be a valid mask operand. For example, if `umask` outputs `0022`, then `umask 0022` must set the mask to the same value. This is a round-trip fidelity requirement.

**Typical implementation:** Most implementations output a four-digit octal number (e.g., `0022`), which is recognized as an octal mask operand.
