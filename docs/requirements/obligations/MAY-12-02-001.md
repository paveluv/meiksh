## Extracted Text

> The order of operands may matter and position-related interpretations should be determined on a utility-specific basis.

## Context

Guideline 12: operand ordering semantics are utility-specific. Some builtins may interpret operands positionally (e.g., `set` treats operands as positional parameters in order). The shell implementation must respect each builtin's documented operand ordering semantics. This is a permissive guideline—operand order MAY matter, and when it does, the specific builtin's documentation governs the interpretation.
