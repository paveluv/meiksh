## Context

**Job control initialization — controlling process case:** When the shell starts, if it has a controlling terminal AND is the controlling process (session leader) for that terminal session, it shall set the terminal's foreground process group ID to its own process group ID using `tcsetpgrp()`.

**Job control initialization — non-controlling process case:** If the shell has a controlling terminal but is NOT the session leader, it shall (if interactive) or may (if non-interactive) perform additional initialization steps (detailed in SHALL-19-11-002 through SHALL-19-11-005). These steps involve checking whether the shell's process group is the foreground group and either becoming the foreground group or stopping itself.

This obligation governs the initial terminal ownership setup. The shell must call `tcgetpgrp()` and `tcsetpgrp()` appropriately at startup. See also `basedefs/V1_chap11.html` for terminal interface definitions.
