## Extracted Text

> The standard utilities in the Shell and Utilities volume of POSIX.1-2024 shall base their behavior on the current locale, as defined in the ENVIRONMENT VARIABLES section for each utility. The behavior of some of the C-language functions defined in the System Interfaces volume of POSIX.1-2024 shall also be modified based on a locale selection. The locale to be used by these functions can be selected in the following ways:: Some functions, such as catopen() and those related to text domains, may reference various environment variables and a locale category of a specific locale to access files they need to use.

## Context

**Behavior:** Some functions (like `catopen()` for message catalogs) may reference environment variables and specific locale categories to locate files. This is a permissive ("may") statement within a "shall" context.

**Shell relevance:** If the shell uses message catalogs for internationalized error messages (via `catopen()`/`catgets()` or `gettext()`), it would use `LC_MESSAGES` to locate the appropriate catalog. The ref to `functions/catopen.html` is a C API page — not a shell-relevant file to extract.
