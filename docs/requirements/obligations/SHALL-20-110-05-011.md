## Extracted Text

> The following operands shall be supported:: A string that shall be interpreted by the shell as one or more commands, as if the string were the argument to the system() function defined in the System Interfaces volume of POSIX.1-2024. If the command_string operand is an empty string, sh shall exit with a zero exit status.

## Context

When `-c` is used, the `command_string` operand is a string that the shell must interpret as one or more commands. The string is parsed and executed as if it were the argument to `system()` — meaning it goes through the full shell language processing (tokenization, expansion, execution).

**Empty string special case**: If `command_string` is an empty string (`""`), the shell must exit with a **zero** exit status immediately, without executing any commands.

Examples:
- `sh -c 'echo hello'` — parses and executes `echo hello`
- `sh -c ''` — exits with status 0 immediately
- `sh -c 'cmd1; cmd2'` — executes cmd1 then cmd2, exit status is from cmd2

The ref to `system()` (functions/system.html) is informative only — it describes the C library function that has similar semantics. The shell itself does not implement `system()`; this reference just clarifies the intended parsing behavior.
