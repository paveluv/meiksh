## Extracted Text

> If the shell is interactive, the subshell shall behave as a non-interactive shell in all respects except: The expansion of the special parameter '-' may continue to indicate that it is interactive. The set -n option may be ignored.

## Context

**Duplicate of SHALL-19-13-007: interactive shell's subshell is non-interactive with exceptions.**

Same obligation as SHALL-19-13-007. If the parent shell is interactive, a subshell **shall** behave as non-interactive, except:
- `$-` expansion **may** still include `i`
- `set -n` **may** be ignored in the subshell

See SHALL-19-13-007 for full context and implementation guidance.
