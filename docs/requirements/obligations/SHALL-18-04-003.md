## Extracted Text

> The OPERANDS section describes the utility operands, and how they affect the actions of the utility. Apparent disagreements between functionality descriptions in the OPERANDS and DESCRIPTION (or EXTENDED DESCRIPTION) sections shall be resolved in favor of the OPERANDS section. If an operand naming a file can be specified as '-', which means to use the standard input instead of a named file, this is explicitly stated in this section. Unless otherwise stated, the use of multiple instances of '-' to mean standard input in a single command produces unspecified results. Unless otherwise stated, the standard utilities that accept operands shall process those operands in the order specified in the command line. Default Behavior: When this section is listed as "None.", it means that the implementation need not support any operands.

## Context

For OPERANDS handling in shell builtins: (1) The OPERANDS section takes precedence over DESCRIPTION for any apparent conflicts. (2) If a builtin accepts `-` as a file operand meaning stdin, this is only valid when explicitly stated. (3) Using multiple `-` operands for stdin in a single command produces **unspecified** results unless stated otherwise. (4) Builtins **shall** process operands in the order specified on the command line unless stated otherwise. (5) When OPERANDS is listed as "None.", the builtin need not support any operands. Reference: XCU 1.4.
