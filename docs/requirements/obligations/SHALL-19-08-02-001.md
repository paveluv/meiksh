## Extracted Text

> The exit status of a command shall be determined as follows:

## Context

**Behavior:** This is the introductory statement for the exit status determination rules in section 2.8.2. It establishes that the shell MUST use a specific ordered set of rules (enumerated in the sub-obligations SHALL-19-08-02-002 through SHALL-19-08-02-009) to determine the exit status of every command it executes.

**Applies to:** All command types (simple commands, pipelines, compound commands, functions, builtins) in all shell modes. The exit status determination must be applied after a command completes, and the result stored in `$?`.

**Implementation detail:** The exit status resolution is a priority-ordered chain: (1) command not found → 127, (2) found but not executable → 126, (3) terminated by signal → 128+signum (implementation-defined encoding), (4) otherwise → WEXITSTATUS(wait_status). The shell must implement this chain in its command execution path.

**Spec reference:** Section 2.8.2, line 1699 in `utilities/V3_chap02.html`.
