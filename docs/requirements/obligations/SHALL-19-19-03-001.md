> The shell shall tokenize (see 2.3 Token Recognition) the contents of the file, parse the tokens (see 2.10 Shell Grammar), and execute the resulting commands in the current environment. It is unspecified whether the commands are parsed and executed as a program (as for a shell script) or are parsed as a single compound_list that is executed after the entire file has been parsed.

## Context

**Behavior:** The `.` (dot) special built-in reads and executes commands from a file in the **current** environment (not a subshell). The shell must:
1. Tokenize the file contents (per 2.3 Token Recognition).
2. Parse the tokens (per 2.10 Shell Grammar).
3. Execute the resulting commands in the current environment.

It is **unspecified** whether the commands are parsed and executed as a program (incrementally, like a script) or parsed as a single compound_list that is executed after the entire file has been parsed.

**Key difference from script execution:** The dot command runs in the current environment, so variable assignments, function definitions, and `cd` calls persist after the file is sourced.

**Applies to:** All shell modes. This is a special built-in.
