## Extracted Text

> The LC_CTYPE category shall define character classification, case conversion, and other character attributes. In addition, a series of characters can be represented by three adjacent <period> characters representing an ellipsis symbol ("..."). The ellipsis specification shall be interpreted as meaning that all values between the values preceding and following it represent valid characters. The ellipsis specification shall be valid only within a single encoded character set; that is, within a group of characters of the same size. An ellipsis shall be interpreted as including in the list all characters with an encoded value higher than the encoded value of the character preceding the ellipsis and lower than the encoded value of the character following the ellipsis.

## Context

**Behavior:** The `LC_CTYPE` locale category defines:
1. Character classification (upper, lower, alpha, digit, space, punct, cntrl, etc.)
2. Case conversion mappings (toupper, tolower)
3. Other character attributes

In the locale definition, ellipsis notation (`...`) between two characters includes all characters with encoding values between them. The ellipsis is valid only within characters of the same byte size.

**Shell relevance:** The shell depends heavily on `LC_CTYPE` for:
- Character classification in bracket expressions (`[[:alpha:]]`, `[[:digit:]]`, `[[:space:]]`, etc.)
- `IFS` processing (what constitutes whitespace)
- Case conversion in `${var^^}`, `${var,,}` expansions
- Multi-byte character boundary detection during tokenization
- The `[:alpha:]`, `[:alnum:]`, etc. POSIX character classes used in pattern matching

The shell accesses this through C library functions (`isalpha()`, `isupper()`, `towlower()`, etc.), not by parsing the locale definition.
