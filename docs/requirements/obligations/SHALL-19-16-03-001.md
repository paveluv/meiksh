## Extracted Text

> If n is specified, the break utility shall exit from the nth enclosing for, while, or until loop. If n is not specified, break shall behave as if n was specified as 1. Execution shall continue with the command immediately following the exited loop. The application shall ensure that the value of n is a positive decimal integer. If n is greater than the number of enclosing loops, the outermost enclosing loop shall be exited. If there is no enclosing loop, the behavior is unspecified.

## Context

**Behavior:** The `break` special built-in exits from enclosing `for`, `while`, or `until` loops:

1. If `n` is specified, `break` exits from the **nth** enclosing loop (counting from the innermost).
2. If `n` is not specified, it behaves as `break 1` (exit the innermost loop).
3. Execution continues with the command immediately following the exited loop.
4. The application must ensure `n` is a **positive decimal integer** (≥ 1).
5. If `n` is greater than the number of enclosing loops, the **outermost** enclosing loop is exited.
6. If there is **no** enclosing loop, the behavior is **unspecified**.

**Exit status:** Zero, unless `n` is invalid (not a positive decimal integer), in which case it is an error.

**Applies to:** All shell modes.
