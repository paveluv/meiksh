## Extracted Text

> Remove Smallest Prefix Pattern. The word shall be expanded to produce a pattern. The parameter expansion shall then result in parameter, with the smallest portion of the prefix matched by the pattern deleted. If present, word shall not begin with an unquoted '#'.

## Context

**Behavior:** The `${parameter#word}` expansion (Remove Smallest Prefix Pattern):
1. Expand `word` to produce a pattern (using pattern matching notation per 2.14).
2. Match the pattern against the **prefix** (beginning) of the value of `parameter`.
3. Find the **smallest** (shortest) matching prefix.
4. Return the value of `parameter` with that smallest prefix removed.

**Constraint:** If `word` is present, it must not begin with an unquoted `#`. This avoids ambiguity with the `##` (largest prefix) form. A quoted `#` at the start is permitted.

**Inputs/Outputs:**
- `x="/usr/local/bin"; echo ${x#*/}` → `usr/local/bin` (removes shortest prefix matching `*/`, which is `/`)
- `x="file.tar.gz"; echo ${x#*.}` → `tar.gz` (removes `file.`, the shortest prefix ending in `.`)

**Spec reference:** See 2.6.2 Parameter Expansion in utilities/V3_chap02.html, lines 982–985.
