## Extracted Text

> When not inside a bracket expression, the following shall match a single character:

## Context

**Behavior:** When not inside a bracket expression, the following constructs shall match a single character: (1) a BRE ordinary character, (2) a BRE special character or `]` preceded by an unescaped backslash, (3) a period (`.`), and (4) a bracket expression (which matches a single character or collating element).

**Shell relevance:** While the shell uses pattern matching notation (not BRE), the analogous single-character matchers in shell patterns are: (1) literal characters, (2) `?` (matches any single character, analogous to `.` in BRE), (3) bracket expressions `[...]`. The shell's bracket expression rules derive from section 9.3.5 shared between REs and shell patterns.
