## Extracted Text

> A character can be represented as a hexadecimal constant. A hexadecimal constant shall be specified as the escape character followed by an 'x' followed by two hexadecimal digits. Each constant shall represent a byte value. Multi-byte values can be represented by concatenated constants specified in byte order with the last constant specifying the least significant byte of the character. For example: \x63;\xe7;\x63\x68 "\x4d\x61\x79"

## Context

**Behavior:** Duplicate of hexadecimal constant rules from SHALL-07-03-016. Hex constants in locale definition files use escape character + `x` + 2 hex digits, each representing a byte. Multi-byte characters use concatenated constants.

**Shell relevance:** The shell does not parse locale definition files. This is `localedef` input encoding syntax. No shell implementation action needed.
