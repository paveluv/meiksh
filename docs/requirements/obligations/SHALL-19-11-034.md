## Context

**Background job completion/termination notification (parent obligation):** When a background job completes normally or is terminated by a signal, an **interactive** shell shall write a status message to stderr. The format is as described by the `jobs` utility (without `-l`), e.g., `[1]+  Done                    sleep 5` for completion or `[1]+  Terminated              sleep 5` for signal termination. See `utilities/jobs.html` for format details.

The timing of this message depends on `set -b` (see SHALL-19-11-035 through SHALL-19-11-038). The shell detects background job completion via `waitpid()` with `WNOHANG` in its event loop or prompt processing.
