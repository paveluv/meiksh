## Extracted Text

> If an option-argument is missing:: If the first character of optstring is a <colon>, the shell variable specified by name shall be set to the <colon> character and the shell variable OPTARG shall be set to the option character found.

## Context

**Behavior:** When `getopts` detects that a required option-argument is missing AND the first character of `optstring` is `:` (silent error reporting mode), the shell must:
1. Set the variable named by the `name` operand to the literal character `:`.
2. Set `OPTARG` to the option character whose argument is missing.

**Condition:** This applies only when `optstring` begins with `:`. If `optstring` does not begin with `:`, the behavior is governed by SHALL-20-53-03-012 instead.

**Example:** `getopts ":a:" opt` — if `-a` is provided without an argument, `opt` is set to `:` and `OPTARG` is set to `a`.

**Exit status:** The exit status shall be zero (this is not an error in getopts processing per se).

**Spec reference:** POSIX.1-2024, `getopts` DESCRIPTION, lines 69-71 of `utilities/getopts.html`.
