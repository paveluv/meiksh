## Extracted Text

> Certain characters have special functions on input or output or both. These functions are summarized as follows:: Special character on input, which is recognized if the ICANON flag is set. When received, all the bytes waiting to be read are immediately passed to the process without waiting for a <newline>, and the EOF is discarded. Thus, if there are no bytes waiting (that is, the EOF occurred at the beginning of a line), a byte count of zero shall be returned from the read(), representing an end-of-file indication. If ICANON is set, the EOF character shall be discarded when processed.

## Context

**Behavior:** The EOF special character (typically ^D):
1. Recognized on input only when ICANON is set in `c_lflag`.
2. When received, all bytes waiting in the current line are immediately passed to the process **without** waiting for a newline. The EOF character itself is discarded.
3. If no bytes are waiting (EOF at start of line) → `read()` returns 0, indicating end-of-file.
4. The EOF character shall be **discarded** when processed (never appears in read data).

**Conditions:** ICANON must be set in `c_lflag`.

**Shell relevance:** This is how Ctrl-D works to signal EOF to the shell. When the user presses ^D at the beginning of a line, the shell receives a zero-length read, which it interprets as end-of-input and exits (for non-login shells) or prompts "Use 'exit' to leave the shell" (implementation choice). When ^D is pressed mid-line, the partial line is delivered to the shell. The shell's `read` builtin also relies on this mechanism. Note: VEOF may alias VMIN (see MAY-11-02-06-001), so mode switching must preserve this value.

**Spec reference:** POSIX.1-2024, §11.1.9 Special Characters (tag_11_01_09), `basedefs/V1_chap11.html` lines 235–238.
