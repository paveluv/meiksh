## Extracted Text

> [Redirection to or from filename] The expansions specified in 2.7 Redirection shall occur. As specified there, exactly one field can result (or the result is unspecified), and there are additional requirements on pathname expansion.

## Context

**Grammar Rule 2 (Redirection filename word):** When a TOKEN appears as the target of a redirection operator (`<`, `>`, `>>`, `<>`, `>|`, `<&`, `>&`), the word expansions specified in Section 2.7 "Redirection" (see `utilities/V3_chap02.html#tag_19_07`) shall be applied. These expansions include tilde expansion, parameter expansion, command substitution, arithmetic expansion, and quote removal.

After expansion, the result must be **exactly one field**; if it produces zero or more than one field, the result is unspecified (the shell should treat this as an error). Additionally, pathname expansion has special constraints in redirection context: it must produce exactly one match to be valid.

The implementation must ensure that the word following a redirection operator is classified as WORD and subjected to the redirection-specific expansion rules at execution time.
