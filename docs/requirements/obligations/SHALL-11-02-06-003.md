## Extracted Text

> If the value of one of the changeable special control characters (see 11.1.9 Special Characters) is _POSIX_VDISABLE, that function shall be disabled; that is, no input data is recognized as the disabled special character. If ICANON is not set, the value of _POSIX_VDISABLE has no special meaning for the VMIN and VTIME entries of the c_cc array.

## Context

**Behavior:**
1. Setting a changeable special control character to `_POSIX_VDISABLE` shall **disable** that character's function. No input byte will be recognized as that special character.
2. In non-canonical mode (ICANON not set), `_POSIX_VDISABLE` has **no special meaning** for VMIN and VTIME — these are numeric parameters, not character matchers, so disabling them via `_POSIX_VDISABLE` is not meaningful.

**Conditions:** Applies to VINTR, VQUIT, VERASE, VKILL, VEOF, VEOL, VSUSP (and VSTART/VSTOP if changeable). `_POSIX_VDISABLE` is typically 0 or 0xff.

**Shell relevance:** The `stty` builtin must support disabling special characters (e.g., `stty intr undef` or `stty susp ^-`). The shell implements this by setting the corresponding `c_cc` entry to `_POSIX_VDISABLE`. The shell's line editor may want to disable certain special characters (e.g., disable SUSP to allow ^Z as a line editor command). VMIN and VTIME must be treated as numeric values, not characters, regardless of `_POSIX_VDISABLE`.

**Spec reference:** POSIX.1-2024, §11.2.6 Special Control Characters (tag_11_02_06), `basedefs/V1_chap11.html` lines 1357–1360. See also §11.1.9.
