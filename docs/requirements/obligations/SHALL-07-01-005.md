## Extracted Text

> For functions that do not take a locale object as an argument, the current locale for the thread can be set by calling uselocale() or the global locale for the process can be set by calling setlocale(). Such functions shall use the current locale of the calling thread if one has been set for that thread; otherwise, they shall use the global locale.

## Context

**Behavior:** Duplicate of the core locale-selection rule from SHALL-07-01-003. Functions without an explicit locale argument must use the thread-local locale if set, otherwise the global locale.

**Shell relevance:** Same as SHALL-07-01-003. The shell's `setlocale(LC_ALL, "")` at startup initializes the global locale. Functions like `isalpha()`, `mbrtowc()`, `strcoll()`, and `wcwidth()` used by the shell will then operate in the correct locale. The refs to `functions/` pages are C API documentation — not shell-relevant files to extract.

**Spec reference:** Section 7.1, lines 58-61 in `basedefs/V1_chap07.html`.
