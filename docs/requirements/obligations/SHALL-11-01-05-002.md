## Extracted Text

> If the O_NONBLOCK flag is clear, then the read request shall be blocked until data is available or a signal has been received. If the O_NONBLOCK flag is set, then the read request shall be completed, without blocking, in one of three ways:: If there is enough data available to satisfy the entire request, the read() shall complete successfully and shall return the number of bytes read.

## Context

**Behavior:** Non-blocking terminal read, case 1: If O_NONBLOCK is set and sufficient data is available to fill the entire read buffer, read() shall succeed and return the number of bytes read.

**Conditions:** O_NONBLOCK set on terminal fd, and input queue contains >= requested bytes.

**Shell relevance:** When the shell performs non-blocking reads (e.g., for line editing input checking), this case applies when data is readily available.

**Spec reference:** XBD 11.1.5 "Input Processing and Reading Data" (tag_11_01_05), `basedefs/V1_chap11.html`, lines 138–139.
