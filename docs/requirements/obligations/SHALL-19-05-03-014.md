## Extracted Text

> The following variables shall affect the execution of the shell:: Determine the behavior of range expressions, equivalence classes, and multi-character collating elements within pattern matching.

## Context

This defines the semantics of the `LC_COLLATE` variable.

**Required behavior:**
- `LC_COLLATE` determines the behavior of range expressions (e.g., `[a-z]`), equivalence classes (e.g., `[[=a=]]`), and multi-character collating elements within pattern matching.
- This affects pathname expansion (globbing) patterns and `case` statement patterns that use bracket expressions with ranges or collation-dependent features.

**Conditions:** Applies whenever the shell performs pattern matching (2.6.6 Pathname Expansion, `case` patterns, `${parameter%pattern}`, etc.) that involves collation-sensitive bracket expressions.

**Implementation note:** The shell should use the C library's locale-aware collation functions (e.g., `strcoll`, `fnmatch` with locale) and respect the current `LC_COLLATE` setting.

See 2.5.3 "Shell Variables" (`LC_COLLATE`) in `utilities/V3_chap02.html`.
