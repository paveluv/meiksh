## Extracted Text

> A utility program shall be either an executable file, such as might be produced by a compiler or linker system from computer source code, or a file of shell source code, directly interpreted by the shell. The program may have been produced by the user, provided by the system implementor, or acquired from an independent distributor.

## Context

This defines what constitutes a "utility" for the shell's command search and execution:

1. **Executable files**: Binary executables (ELF, Mach-O, etc.) found via `PATH` search. The shell executes these via `fork()`/`exec()`.
2. **Shell scripts**: Text files containing shell source code, directly interpreted by the shell. The shell detects these (via `#!` shebang line or by attempting to execute them as shell commands when `exec()` fails).

The shell's command search and execution mechanism (XCU §2.9.1.4) must handle both types. When executing a command, the shell first tries `exec()`, and if that fails with ENOEXEC, may attempt to interpret the file as a shell script.

See: `basedefs/V1_chap04.html` §4.25 Utility; `utilities/V3_chap02.html` §2.9.1.4 Command Search and Execution.
