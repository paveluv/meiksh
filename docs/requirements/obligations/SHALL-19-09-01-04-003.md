## Extracted Text

> If a simple command has a command name and an optional list of arguments after word expansion (see 2.9.1.1 Order of Processing), the following actions shall be performed:: If the command name contains at least one <slash>, the shell shall execute a non-built-in utility as described in 2.9.1.6 Non-built-in Utility Execution.

## Context

**Behavior:** If the command name contains at least one `/` character (e.g., `./script`, `/usr/bin/foo`, `subdir/cmd`), the shell MUST execute it as a non-built-in utility per 2.9.1.6. No PATH search, no builtin/function lookup — the name is treated as a direct pathname.

**Applies to:** All commands with slashes in the name, in all shell modes.

**Implementation detail:** When the command name contains `/`, skip the entire command search algorithm and go directly to the non-builtin execution path (fork + exec with the name as the path). The 2.9.1.6 rules for ENOEXEC fallback to shell script execution still apply.

**Spec reference:** Section 2.9.1.4, lines 1932-1933 in `utilities/V3_chap02.html`. Cross-ref: 2.9.1.6 Non-built-in Utility Execution.
