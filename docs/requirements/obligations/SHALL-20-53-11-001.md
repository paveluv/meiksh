## Extracted Text

> Whenever an error is detected and the first character in the optstring operand is not a <colon> (':'), a diagnostic message shall be written to standard error with the following information in an unspecified format:

## Context

**Behavior:** In verbose error mode (when `optstring` does NOT begin with `:`), `getopts` must write a diagnostic message to stderr whenever it detects an error. The format of the message is unspecified, but must include specific information as detailed in SHALL-20-53-11-002 through SHALL-20-53-11-007.

**Errors that trigger diagnostics:**
1. An option character not in `optstring` is encountered.
2. A required option-argument is missing.

**Condition:** Only when `optstring[0] != ':'`. In silent mode (`:` prefix), no diagnostic is written to stderr.

**Required information in the diagnostic (unspecified format):**
- The invoking program name (value of `$0` or `basename "$0"`).
- The type of error (unknown option vs. missing argument).
- The invalid option character.

**Spec reference:** POSIX.1-2024, `getopts` STDERR, line 169 of `utilities/getopts.html`.
