## Extracted Text

> If neither -f nor -v is specified, name refers to a variable; if a variable by that name does not exist, it is unspecified whether a function by that name, if any, shall be unset.

## Context

**Behavior:** When neither `-f` nor `-v` is specified for `unset`, the name refers to a **variable**. If no variable by that name exists, it is **unspecified** whether a function by that name (if any) is unset.

**Implementation choice:** meiksh should document whether bare `unset name` also removes functions when no variable exists. Common behavior is to only unset variables.

**Applies to:** All shell modes.
