## Extracted Text

> Enclosing characters in double-quotes ("") shall preserve the literal value of all characters within the double-quotes, with the exception of the characters backquote, <dollar-sign>, and <backslash>, as follows:: `

## Context

**Behavior:** Inside double-quotes, the backquote character (`` ` ``) retains its special meaning as the introducer for the legacy form of command substitution (`` `...` ``). This is one of the three exceptions to the general double-quote literal-preservation rule.

**Applies to:** All shell modes. When the tokenizer/expander encounters `` ` `` inside double-quotes, it must recognize it as the start of a command substitution, not as a literal character.

**Implementation detail:** The tokenizer must track backquote nesting when inside double-quotes. The rules for what constitutes the command inside `` `...` `` are specified in SHALL-19-02-03-005/009. The result of the command substitution replaces the `` `...` `` sequence during word expansion.

**Spec reference:** Section 2.2.3, line 132 in `utilities/V3_chap02.html`. See also 2.6.3 Command Substitution for full expansion rules.
