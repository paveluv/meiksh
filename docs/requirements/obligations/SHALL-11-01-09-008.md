## Extracted Text

> Certain characters have special functions on input or output or both. These functions are summarized as follows:: Special character on both input and output, which is recognized if the IXON (output control) or IXOFF (input control) flag is set. Can be used to resume output that has been suspended by a STOP character. If IXON is set, the START character shall be discarded when processed.

## Context

**Behavior:** The START special character (typically ^Q):
1. Recognized on both input and output when IXON (output control) or IXOFF (input control) is set in `c_iflag`.
2. When recognized with IXON set, it resumes terminal output that was previously suspended by a STOP character.
3. If IXON is set, the START character shall be **discarded** when processed (not placed in input queue).

**Conditions:** IXON and/or IXOFF must be set in `c_iflag`.

**Shell relevance:** ^Q resumes output after ^S. Like STOP, this can interfere with the shell's line editor (e.g., ^Q might be desired for a quoted-insert in emacs mode). The shell may disable IXON during line editing. The `stty` builtin must support `start` character settings. See also SHALL-11-01-09-007 (STOP character).

**Spec reference:** POSIX.1-2024, §11.1.9 Special Characters (tag_11_01_09), `basedefs/V1_chap11.html` line 254.
