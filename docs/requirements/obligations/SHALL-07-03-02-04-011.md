## Extracted Text

> An ellipsis symbol as a weight shall be interpreted to mean that each character in the sequence shall have unique weights, equal to the relative order of their character in the character collation sequence. The use of the ellipsis as a weight shall be treated as an error if the collating element is neither an ellipsis nor the special symbol UNDEFINED.

## Context

When an ellipsis (`...`) is used as a weight operand (rather than as a collating identifier), it assigns each character in the range unique weights matching their collation-order position. This is only valid when the collating element itself is also an ellipsis or UNDEFINED — using an ellipsis weight on any other element is an error. The shell is affected because this weight assignment rule determines how characters in ellipsis-filled ranges compare, which in turn affects bracket expression range matching in pattern matching.

See: `basedefs/V1_chap07.html` §7.3.2.4 Collation Order.
