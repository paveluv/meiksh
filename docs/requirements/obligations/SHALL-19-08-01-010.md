## Extracted Text

> Non-Interactive Shell: Redirection error with compound commands -- shall not exit

## Context

A non-interactive shell SHALL NOT exit when a redirection error occurs with a compound command. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required:**
- When the shell is non-interactive and a redirection attached to a compound command (`if`, `while`, `for`, `case`, `{...}`, `(...)`, etc.) fails, the shell SHALL write a diagnostic to stderr but SHALL NOT exit.
- A shell diagnostic message IS required.
- The compound command is not executed, but the script continues with the next command.

**Example:** `while true; do echo hi; done >/no/such/dir` — the redirection fails, the while loop is not executed, but the script continues.
