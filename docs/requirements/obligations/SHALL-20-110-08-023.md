## Extracted Text

> The following environment variables shall affect the execution of sh:: [UP] Determine a pathname of the user's mailbox file for purposes of incoming mail notification. If this variable is set, the shell shall inform the user if the file named by the variable is created or if its modification time has changed. Informing the user shall be accomplished by writing a string of unspecified format to standard error prior to the writing of the next primary prompt string. Such check shall be performed only after the completion of the interval defined by the MAILCHECK variable after the last such check. The user shall be informed only if MAIL is set and MAILPATH is not set.

## Context

**Behavior:** MAIL provides incoming mail notification for interactive shells:

1. **When MAIL is set and MAILPATH is not set:** The shell shall monitor the file named by MAIL for creation or modification time changes.
2. **Notification method:** When a change is detected, the shell shall write an unspecified-format notification string to standard error **before** writing the next primary prompt (PS1).
3. **Check interval:** The shell shall only check after the MAILCHECK interval (in seconds) has elapsed since the last check. It does NOT check every prompt — only when the interval has passed.
4. **Precedence:** The user shall be informed only if MAIL is set AND MAILPATH is not set. If MAILPATH is set, it takes precedence and MAIL is ignored.

**Applies to:** Interactive shells only (User Portability [UP] option).

**Implementation detail:** The shell must track the last-checked time and the last-known modification time of the MAIL file. On each primary prompt, check if MAILCHECK seconds have elapsed since the last check; if so, stat the file and compare modification times.
