## Extracted Text

> The ordered list of output fields so produced, which might be empty, shall replace the list of input fields.

## Context

**Behavior:** After the field splitting algorithm processes all input fields, the resulting ordered list of output fields **replaces** the original input field list. This output field list:
- May be **empty** (if all input fields were empty or contained only IFS delimiters)
- May have **fewer fields** than the input (fields were merged or empty fields removed)
- May have **more fields** than the input (fields were split into multiple output fields)
- Maintains the **same ordering** as the input fields they originated from

This output field list then becomes the input for the next expansion stage: pathname expansion (2.6.6), unless `set -f` is in effect.

**Spec reference:** See 2.6.5 Field Splitting in utilities/V3_chap02.html, line 1287.
