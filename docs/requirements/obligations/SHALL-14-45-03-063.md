## Context

The `<signal.h>` header shall define the `siginfo_t` type as a structure ([CX] extension) with at least the following members: `int si_signo` (signal number), `int si_code` (signal code indicating why the signal was generated), `int si_errno` ([XSI] — errno value associated with the signal), `pid_t si_pid` ([CX] — sending process ID), `uid_t si_uid` ([CX] — real user ID of sending process), `void *si_addr` ([CX] — address that caused fault), `int si_status` ([CX] — exit value or signal), and `union sigval si_value` ([CX] — signal value). For `meiksh`, `siginfo_t` is relevant when using `SA_SIGINFO` handlers for `SIGCHLD` to obtain child PID and exit status, and for `SIGINT`/`SIGTERM` to identify the sender. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 579–597).
