## Extracted Text

> The following environment variables shall affect the execution of getopts:: This variable shall be used by the getopts utility as the index of the next argument to be processed.

## Context

**Behavior:** `OPTIND` shall be used by getopts as the 1-based index of the next argument to be processed in the parameter list. On each invocation, getopts reads `OPTIND` to determine where to resume parsing, and updates it after processing.

**Initialization:** The shell initializes `OPTIND` to 1 when it starts.

**Updates by getopts:**
- After successfully parsing an option with argument: set to the index of the next element to search.
- After end of options (exit status 1): set to the index of the first operand, or param count + 1.
- In other cases: encodes information needed for the next invocation (value may be unspecified).

**Spec reference:** POSIX.1-2024, `getopts` ENVIRONMENT VARIABLES, line 156 of `utilities/getopts.html`.
