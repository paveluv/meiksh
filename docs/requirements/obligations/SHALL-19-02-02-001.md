## Extracted Text

> Enclosing characters in single-quotes ('') shall preserve the literal value of each character within the single-quotes. A single-quote cannot occur within single-quotes.

## Context

**Single-quotes preserve literal value of all enclosed characters; no single-quote inside single-quotes.**

Enclosing characters in single-quotes (`'...'`) **shall** preserve the literal value of every character within the quotes. No expansion, substitution, or special character interpretation occurs inside single-quotes.

A single-quote **cannot** occur within single-quotes — there is no escape mechanism inside single-quotes. To include a literal single-quote in a single-quoted context, the application must end the single-quote, add an escaped or double-quoted single-quote, and restart single-quoting. For example: `'can'\''t'` or `'can'"'"'t'`.

Implementation: when the tokenizer encounters an opening `'`, consume all characters literally until the next `'`. No character inside has special meaning. If EOF is reached before a closing `'`, it is a syntax error.

See 2.2.2 Single-Quotes in utilities/V3_chap02.html.
