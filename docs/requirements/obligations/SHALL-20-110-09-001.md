## Extracted Text

> The sh utility shall take the standard action for all signals (see 1.4 Utility Description Defaults) with the following exceptions.

## Context

**Behavior:** The `sh` utility must take the standard action for all signals (as described in 1.4 Utility Description Defaults — typically the default signal disposition) with specific exceptions listed in the subsequent obligations (SHALL-20-110-09-002 through -008). The "standard action" means that unless otherwise specified, signals are handled as defined by the system default (e.g., SIGPIPE terminates, SIGHUP terminates, etc.).

**The exceptions** that override the default are: SIGINT handling during line editing (interactive), SIGQUIT/SIGTERM ignored (interactive), and SIGTTIN/SIGTTOU/SIGTSTP handling depending on `-m` option.

**Applies to:** The `sh` utility itself (both interactive and non-interactive, with exceptions noted per-obligation).

**Spec reference:** ASYNCHRONOUS EVENTS section in `utilities/sh.html`, line 262. Cross-ref: `utilities/V3_chap01.html#tag_18_04` (1.4 Utility Description Defaults).
