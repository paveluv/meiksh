## Extracted Text

> If si_code is equal to CLD_EXITED, then si_status holds the exit value of the process; otherwise, it is equal to the signal that caused the process to change state. The exit value in si_status shall be equal to the full exit value (that is, the value passed to _exit(), _Exit(), or exit(), or returned from main()); it shall not be limited to the least significant eight bits of the value.

## Context

For `SIGCHLD`, when `si_code` equals `CLD_EXITED`, the `si_status` member of `siginfo_t` holds the full exit value of the child process (the value passed to `_exit()`, `_Exit()`, or `exit()`, or returned from `main()`). It shall NOT be limited to the least significant eight bits. When `si_code` is not `CLD_EXITED` (e.g., `CLD_KILLED`, `CLD_STOPPED`), `si_status` holds the signal number that caused the child to change state. For `meiksh`, this is significant: if using `SA_SIGINFO` handlers for `SIGCHLD`, the shell can retrieve the full exit status directly from `si_status` rather than using `WEXITSTATUS()` macros on the `waitpid()` status (which is limited to 8 bits on some systems). This enables proper handling of exit values > 255 if the platform supports it. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 1100–1105).
