## Extracted Text

> If a symbolic link is encountered during pathname resolution, the behavior shall depend on whether the pathname component is at the end of the pathname and on the function being performed. If all of the following are true, then pathname resolution is complete:: This is the last pathname component of the pathname.

## Context

This is the first of three conditions under which a symbolic link is NOT followed during pathname resolution: the symlink must be the last (final) component of the pathname. For the shell, this matters for operations like `test -L path` (which checks if the final component is a symlink), `rm path` (which removes the symlink itself), and similar operations. If the symlink is an intermediate component (not the last), it is always followed regardless. This is part of the three-condition rule from SHALL-04-16-005.

See: `basedefs/V1_chap04.html` §4.16 Pathname Resolution.
