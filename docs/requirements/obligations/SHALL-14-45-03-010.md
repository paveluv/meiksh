## Extracted Text

> The <signal.h> header shall define the following data types:: [CX] As described in <sys/types.h>.

## Context

This confirms `pid_t` (CX extension) is defined as specified in `<sys/types.h>`. It is a signed integer type capable of representing process IDs. The shell must use `pid_t` (not plain `int`) for all process ID variables to ensure portability and correct range. This is a system header requirement that the shell depends on.
