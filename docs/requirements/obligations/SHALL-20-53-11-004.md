## Extracted Text

> Whenever an error is detected and the first character in the optstring operand is not a <colon> (':'), a diagnostic message shall be written to standard error with the following information in an unspecified format:: If an option requiring an option-argument is found, but an option-argument is not found, this error shall be identified and the invalid option character shall be identified in the message.

## Context

**Behavior:** When a required option-argument is missing (verbose mode, no leading `:`), the diagnostic message to stderr must:
1. Identify the error type (missing/required argument).
2. Include the option character whose argument is missing.

**Example formats (historical, not mandated):**
- `"script: option requires an argument -- b"`

The exact format is unspecified, but the error type and option character must be present.

**Spec reference:** POSIX.1-2024, `getopts` STDERR, lines 185-186 of `utilities/getopts.html`.
