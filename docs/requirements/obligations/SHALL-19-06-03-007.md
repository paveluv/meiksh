## Extracted Text

> With both the backquoted and $(commands) forms, the commands string shall be tokenized (see 2.3 Token Recognition) and parsed (see 2.10 Shell Grammar). It is unspecified whether the commands string is parsed and executed incrementally as a program (as for a shell script), or is parsed as a single compound_list that is executed after the string has been completely parsed. In addition, it is unspecified whether the terminating ')' of the $(commands) form can result from alias substitution. With the $(commands) form any syntactically correct program can be used for commands, except that:: The behavior is unspecified if the terminating ')' is not present in the token containing the command substitution; that is, if the ')' is expected to result from alias substitution.

## Context

**Behavior:** The closing `)` of a `$(commands)` command substitution must be physically present in the source token. If the `)` would only appear through alias expansion (i.e., an alias expands to text containing the closing parenthesis), the behavior is **unspecified**.

**Implementation guidance:** The parser should expect the closing `)` to be in the literal source text. Relying on alias expansion to provide the closing delimiter is not portable and need not be supported.

**Spec reference:** See 2.6.3 Command Substitution in utilities/V3_chap02.html, lines 1135–1136.
