## Extracted Text

> [CX] The <signal.h> header shall define the pthread_t, size_t, and uid_t types as described in <sys/types.h>.

## Context

The `<signal.h>` header must provide `pthread_t`, `size_t`, and `uid_t` type definitions (CX extension). For `meiksh`, `uid_t` is relevant for signal-related operations that involve user IDs (e.g., `kill` builtin sending signals to processes by UID). The shell does not directly use `pthread_t` but relies on `<signal.h>` including these types for correct compilation. This is a system header requirement rather than a shell behavioral requirement—the shell depends on the platform providing these types.
