## Extracted Text

> where the fields shall be as follows:: The associated command that was given to the shell.

## Context

**Behavior:** The `<command>` field shall contain the command text as it was given to the shell when the background job was started. For example, if the user typed `sleep 100 &`, the command field would be `sleep 100`.

**Implementation guidance:** The shell should store the original command text (or a reasonable representation) when creating a background job, for later display by `jobs`.

**Spec reference:** POSIX.1-2024, `jobs` STDOUT, line 167 of `utilities/jobs.html`.
