## Extracted Text

> The exit status of an OR list shall be the exit status of the last command that is executed in the list.

## Context

An OR list connects pipelines with `||`. The shell evaluates pipelines left-to-right; if any pipeline returns a zero exit status, execution of subsequent pipelines in the OR list stops. The exit status of the entire OR list must be the exit status of **the last command that was actually executed**.

Examples:
- `false || false || true` — exit status is 0 (from `true`, the last command executed)
- `true || false` — exit status is 0 (from `true`; `false` was never executed)
- `false || false` — exit status is 1

See 2.9.3.7 OR Lists in utilities/V3_chap02.html#tag_19_09_03_08.
