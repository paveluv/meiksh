## Extracted Text

> Unless otherwise stated, the exit status of a command shall be that of the last simple command executed by the command. There shall be no limit on the size of any shell command other than that imposed by the underlying system (memory constraints, {ARG_MAX}, and so on).

## Context

**Behavior:** Two rules are established: (1) Unless a specific section states otherwise, the exit status of any compound or complex command MUST be the exit status of the last simple command executed within it. This applies to if/then/else, while, for, case, brace groups, subshells, etc. (2) There MUST be no shell-imposed limit on the size of any command — only underlying OS limits (memory, ARG_MAX, etc.) apply.

**Applies to:** All compound commands, pipelines, lists, and function bodies. The "last simple command" rule means, for example: `{ cmd1; cmd2; }` has the exit status of `cmd2`; `if false; then cmd1; else cmd2; fi` has the exit status of `cmd2`.

**Implementation detail:** For compound commands, the shell should propagate `$?` from the last executed simple command naturally. For the size limit, the shell must not impose fixed-size buffers on command text — use dynamic allocation. The OS-level `ARG_MAX` limit on exec arguments is outside shell control.
