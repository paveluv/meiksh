## Extracted Text

> The <signal.h> header shall define the following symbolic constants for the values of sigev_notify:: A notification function is called to perform notification.

## Context

This describes the semantics of `SIGEV_THREAD`: when the event occurs, a notification function (specified by `sigev_notify_function`) is called in a new thread, using thread attributes from `sigev_notify_attributes`. For `meiksh`, this threading mechanism is unlikely to be used since shells typically handle signals in the main thread via signal handlers. This is a system header semantics requirement.
