## Extracted Text

> When the shell is using standard input and it invokes a command that also uses standard input, the shell shall ensure that the standard input file pointer points directly after the command it has read when the command begins execution. It shall not read ahead in such a manner that any characters intended to be read by the invoked command are consumed by the shell (whether interpreted by the shell or not) or that characters that are not read by the invoked command are not seen by the shell. When the command expecting to read standard input is started asynchronously by an interactive shell, it is unspecified whether characters are read by the command or interpreted by the shell.

## Context

**Behavior:** When the shell reads commands from standard input and invokes a command that also uses stdin, the shell shall ensure that the stdin file pointer is positioned directly after the last command the shell has read when the invoked command begins execution. The shell must not read ahead beyond what it needs, so that:
- Characters intended for the invoked command are not consumed by the shell.
- Characters not read by the invoked command remain available for the shell to see afterward.

**Exception:** When a command expecting stdin is started asynchronously (`&`) by an interactive shell, it is unspecified whether characters are read by the command or interpreted by the shell.

**Implementation:** This row constrains observable stdin handling: the shell must not read ahead in a way that steals characters from the command it is about to execute.
