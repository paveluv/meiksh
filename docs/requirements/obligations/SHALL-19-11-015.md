> When a foreground job is created, or a background job is brought into the foreground by the fg utility, if the shell has a controlling terminal it shall set the foreground process group ID associated with the terminal as follows:

## Context

**Setting foreground process group for jobs (parent obligation):** When a foreground job is created or a background job is brought to the foreground via `fg` (see `utilities/fg.html`), if the shell has a controlling terminal, it shall set the terminal's foreground process group ID. The specific value depends on whether the job was originally a foreground or background job (see SHALL-19-11-016 through SHALL-19-11-019 for the two cases).

This obligation establishes that `tcsetpgrp()` must be called in these situations. The shell must track each job's original creation mode (foreground vs background) to determine the correct process group to set.
