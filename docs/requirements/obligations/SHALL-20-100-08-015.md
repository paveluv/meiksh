## Extracted Text

> The following environment variables shall affect the execution of read:: Provide the prompt string that an interactive shell shall write to standard error when a line ending with a <backslash> <newline> is read and the -r option was not specified.

## Context

**Behavior required:** Descriptive text for PS2: it provides the prompt string that the interactive shell writes to stderr when `read` encounters a line ending with `\<newline>` (and `-r` is not specified).

**Condition:** Interactive shell, terminal stdin, no `-r`, line ends with `\<newline>`.

**Implementation detail:** When all conditions for continuation prompting are met (see SHALL-20-100-03-003):
1. The `\<newline>` pair is consumed (line continuation).
2. The value of PS2 is written to stderr as a prompt.
3. `read` continues reading the next physical line.

PS2 defaults to `"> "` per Section 2.5.3.
