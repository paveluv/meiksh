## Extracted Text

> If the n operand is invalid or is greater than "$#", this may be treated as an error and a non-interactive shell may exit; if the shell does not exit in this case, a non-zero exit status shall be returned and a warning message shall be written to standard error. Otherwise, zero shall be returned.

## Context

**Behavior:** Exit status rules for the `shift` special built-in:

1. **Invalid n operand or n > $#:**
   - This **may** be treated as an error (not required).
   - A non-interactive shell **may** exit (not required — this is discretionary).
   - If the shell does NOT exit, it shall:
     - Return a non-zero exit status.
     - Write a warning message to standard error.
2. **Valid shift (n <= $# and n is a valid unsigned decimal integer):** The exit status shall be zero.
3. **Default n:** If n is not specified, it defaults to 1.

**Applies to:** All shell modes. The may-exit behavior applies specifically to non-interactive shells.

**Implementation guidance:** For meiksh, a reasonable choice is to always return non-zero and write a diagnostic on invalid `shift`, and optionally exit in non-interactive mode (for POSIX strictness). The warning message should indicate the invalid operand or that n exceeds the parameter count.
