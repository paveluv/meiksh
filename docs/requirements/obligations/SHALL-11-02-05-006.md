## Extracted Text

> If ICANON is set, canonical processing shall be enabled. This enables the erase and kill edit functions, and the assembly of input characters into lines delimited by NL, EOF, and EOL, as described in 11.1.6 Canonical Mode Input Processing.

## Context

**Behavior:** When ICANON is set in `c_lflag`, canonical input processing is enabled. This enables:
1. The ERASE and KILL edit functions for line editing.
2. Assembly of input characters into lines delimited by NL, EOF, and EOL characters (see §11.1.6 Canonical Mode Input Processing).

**Relevance to meiksh:** The `stty` builtin must support `icanon`/`-icanon`. The shell typically disables ICANON when its own line editor is active (to get character-at-a-time input), and re-enables it when running external commands that expect canonical input. This is one of the most important terminal mode flags for the shell.

**Spec reference:** POSIX.1-2024, §11.2.5 Local Modes (tag_11_02_05), `basedefs/V1_chap11.html` lines 1182–1184. See also §11.1.6.
