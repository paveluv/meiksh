## Extracted Text

> In canonical mode input processing, terminal input is processed in units of lines. A line is delimited by a <newline> character (NL), an end-of-file character (EOF), or an end-of-line (EOL) character. See 11.1.9 Special Characters for more information on EOF and EOL. This means that a read request shall not return until an entire line has been typed or a signal has been received. Also, no matter how many bytes are requested in the read() call, at most one line shall be returned. It is not, however, necessary to read a whole line at once; any number of bytes, even one, may be requested in a read() without losing information.

## Context

**Behavior:** In canonical (cooked) mode, terminal input is line-oriented:
- Lines are delimited by NL, EOF, or EOL characters
- read() shall not return until a complete line is available or a signal arrives
- At most one line is returned per read() call, regardless of buffer size
- Partial-line reads are permitted (requesting fewer bytes than a line) without data loss

**Conditions:** Applies when ICANON is set in the terminal's c_lflag. This is the normal mode for shell interactive input.

**Shell relevance:** The shell relies on canonical mode for basic line input. The `read` builtin and interactive command reading use canonical mode by default. When implementing line editing (e.g., emacs/vi mode), the shell switches to non-canonical mode.
