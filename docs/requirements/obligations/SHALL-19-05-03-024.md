## Extracted Text

> The following variables shall affect the execution of the shell:: A string formatted as described in XBD 8. Environment Variables, used to effect command interpretation; see 2.9.1.4 Command Search and Execution.

## Context

This defines the semantics of the `PATH` variable.

**Required behavior:**
- `PATH` is a colon-separated list of directory pathnames used for command search and execution.
- The format is defined in XBD 8 "Environment Variables": `pathname1:pathname2:...`
- An empty component (e.g., leading colon `:pathname`, trailing colon `pathname:`, or double colon `path1::path2`) represents the current directory.
- The shell uses `PATH` during command search (2.9.1.4 "Command Search and Execution") to locate external commands that are not builtins, functions, or specified with a `/`.

**Conditions:** Applies whenever the shell searches for a command to execute. PATH is consulted left-to-right; the first matching executable file is used.

**Implementation note:** The shell must re-evaluate PATH for each command execution (or invalidate any PATH cache when PATH changes). The `hash` builtin may cache PATH lookups but must respect PATH changes.

See 2.5.3 "Shell Variables" (`PATH`), 2.9.1.4 "Command Search and Execution" in `utilities/V3_chap02.html`, and XBD 8 "Environment Variables" in `basedefs/V1_chap08.html`.
