## Extracted Text

> The minimum of either the number of bytes requested or the number of bytes currently available shall be returned without waiting for more bytes to be input. If no characters are available, read() shall return a value of zero, having read no data.

## Context

**Behavior:** Non-canonical mode Case D (MIN=0, TIME=0):
1. `read()` returns immediately with `min(requested_bytes, available_bytes)`.
2. If no bytes are available → `read()` returns 0 (not an error).
3. This is a pure non-blocking poll of the input queue.

**Conditions:** Applies when ICANON is not set and `c_cc[VMIN] == 0` and `c_cc[VTIME] == 0`.

**Shell relevance:** This mode provides non-blocking terminal reads without using O_NONBLOCK. The shell could use this for polling whether input is available (e.g., to check for typeahead while performing other operations). A return of 0 means no data available, not EOF — this must be distinguished from canonical-mode EOF.

**Spec reference:** POSIX.1-2024, §11.1.7.4 Case D: MIN=0, TIME=0 (tag_11_01_07_04), `basedefs/V1_chap11.html` lines 206–207.
