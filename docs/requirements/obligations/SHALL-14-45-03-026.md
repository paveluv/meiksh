## Extracted Text

> The <signal.h> header shall define the following macros that are used to refer to the signals that occur in the system. Signals defined here begin with the letters SIG followed by an uppercase letter. The macros shall expand to positive integer constant expressions with type int and distinct values [CX] less than the value of {NSIG_MAX} defined in <limits.h>. The value 0 is reserved for use as the null signal (see kill()). Additional implementation-defined signals may occur in the system.

## Context

All standard signal macros (SIGABRT, SIGTERM, SIGINT, etc.) must be defined as positive integer constant expressions of type `int` with distinct values. Signal value 0 is reserved as the null signal for `kill()` (used to check if a process exists). Signal values must be less than `NSIG_MAX`. The implementation may define additional signals beyond the standard set. For `meiksh`, the shell must: (1) map signal names to numbers and vice versa for `kill`, `trap`, and `wait` builtins, (2) handle signal 0 specially in the `kill` builtin (test for process existence), (3) handle implementation-defined signals in `kill -l` and `trap` listings.
