## Extracted Text

> When a job consisting of a single asynchronous AND-OR list is created, it shall form a background job and the associated process ID shall be that of a child process that is made a process group leader, with all other processes (if any) that the shell creates to execute the AND-OR list initially having this process ID as their process group ID.

## Context

**Background job process group setup:** When a background job (single async AND-OR list) is created, the shell shall:
1. Fork a child process to execute the AND-OR list.
2. Make that child a process group leader (`setpgid(child_pid, child_pid)`).
3. All other processes created to execute the AND-OR list shall initially have this child's PID as their process group ID (`setpgid(grandchild_pid, child_pid)`).
4. The job's associated process ID is that of this process group leader child.

Both the parent shell and the child should call `setpgid()` to avoid race conditions (the child calls `setpgid(0, 0)` on itself, the parent calls `setpgid(child, child)`). The background job is not given terminal access—it runs without being the foreground process group.
