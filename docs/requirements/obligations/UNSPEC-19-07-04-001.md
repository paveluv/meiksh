## Context

Specifies the file descriptor and file type for here-documents. See 2.7.4 Here-Document in utilities/V3_chap02.html.

**Behavior required:**
- The here-document content is provided on the file descriptor designated by `n` in `[n]<<word`.
- If `n` is omitted, the here-document is provided on standard input (fd 0).

**Unspecified aspect:**
- It is unspecified whether the fd is opened as a regular file (e.g., a temp file) or some other file type (e.g., a pipe).
- Portable applications cannot rely on the fd being seekable.

**Implementation choices for meiksh:**
- Use a temporary file: allows seeking, simpler for large documents, but requires tmpdir management.
- Use a pipe: no temp file cleanup needed, but not seekable and may block if buffer fills.
- Common practice: most shells use a temp file (created in `$TMPDIR` or `/tmp`).
