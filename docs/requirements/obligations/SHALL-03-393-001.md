## Extracted Text

> A thread-safe function shall avoid data races with other calls to the same function, and with calls to any other thread-safe functions, by multiple threads. Each function defined in the System Interfaces volume of POSIX.1-2024 is thread-safe unless explicitly stated otherwise. Examples are any "pure" function, a function which holds a mutex locked while it is accessing static storage, or objects shared among threads.

## Context

**Definition only — no shell relevance.** Defines "thread-safe" as a function that avoids data races with other calls from multiple threads. All POSIX System Interfaces functions are thread-safe unless explicitly stated otherwise.

**Shell impact:** None for the shell command language. If `meiksh` uses multiple threads internally, it must call only thread-safe functions from concurrent threads. This is an internal implementation quality concern, not a shell language requirement. No action required.
