> The following operands shall be supported:

## Context

This is a structural preamble obligation: the `sh` utility must support the operands defined in the OPERANDS section. The specific operands are: `-` (single hyphen-minus), `argument`, `command_file`, `command_name`, and `command_string`. Each is detailed in subsequent obligations (SHALL-20-110-05-002 through SHALL-20-110-05-016).

The shell's argument parsing must correctly distinguish between options (like `-c`, `-s`, `-i`) and operands, following the Utility Syntax Guidelines (XBD 12.2). The `--` end-of-options marker separates options from operands.

See utilities/sh.html#tag_20_110_05 for the full OPERANDS section.
