## Extracted Text

> When the word time is recognized as a reserved word in circumstances where it would, if it were not a reserved word, be the command name (see 2.9.1.1 Order of Processing) of a simple command that would execute the time utility in a manner other than one for which time states that the results are unspecified, the behavior shall be as specified for the time utility.

## Context

**Behavior:** When `time` is recognized as a reserved word in a position where it would otherwise be the command name of a simple command that would execute the `time` utility (and the usage is not one for which the `time` utility says results are unspecified), the shell must behave as specified by the `time` utility.

**In practice:** If `time` is recognized as a reserved word AND it appears in command-name position, and the resulting command would be a valid invocation of the external `time` utility, then the shell must provide the `time` functionality (measuring execution time of the following pipeline). This allows the shell to implement `time` as a keyword rather than relying on an external utility.

**Applies to:** All shell modes, but only when `time` is actually recognized as a reserved word (which is implementation-optional per UNSPEC-19-04-001).

**Implementation detail:** If the shell chooses to recognize `time` as a reserved word, it must implement the `time` utility's behavior as a built-in feature. This includes: executing the following pipeline, measuring its real/user/system time, and writing timing information to stderr in the POSIX format. See `utilities/time.html` for the full specification.

**Spec reference:** Section 2.4, lines 428-432 in `utilities/V3_chap02.html`. Cross-refs: 2.9.1.1 Order of Processing, `utilities/time.html`.
