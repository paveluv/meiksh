## Extracted Text

> An evaluation A happens before an evaluation B if A is sequenced before B or A inter-thread happens before B. The implementation shall ensure that a cycle in the "happens before" relation never occurs.

## Context

This is a C-level memory model requirement ensuring acyclicity of the happens-before relation. It is not relevant to shell language semantics. For a Rust-based shell, Rust's type system and memory model provide these guarantees. No shell-language-level action is required.

See: `basedefs/V1_chap04.html` §4.15.1.1 Data Races.
