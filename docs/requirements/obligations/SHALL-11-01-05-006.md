## Extracted Text

> If there is not enough data available to satisfy the entire request, the read() shall complete successfully, having read as much data as possible, and shall return the number of bytes it was able to read.

## Context

**Behavior:** Duplicate of SHALL-11-01-05-003 (standalone extraction). Non-blocking partial read returns whatever data is available when the full request cannot be satisfied.

**Conditions:** O_NONBLOCK set, 0 < available data < requested count.

**Shell relevance:** The shell must handle short reads correctly.

**Spec reference:** XBD 11.1.5 "Input Processing and Reading Data" (tag_11_01_05), `basedefs/V1_chap11.html`, lines 142–143.
