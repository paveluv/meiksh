## Extracted Text

> A <backslash> that is not quoted shall preserve the literal value of the following character, with the exception of a <newline>. If a <newline> immediately follows the <backslash>, the shell shall interpret this as line continuation. The <backslash> and <newline> shall be removed before splitting the input into tokens. Since the escaped <newline> is removed entirely from the input and is not replaced by any white space, it cannot serve as a token separator.

## Context

**Backslash escape character: preserves literal value, except newline triggers line continuation.**

An unquoted `\` (backslash) **shall** preserve the literal value of the following character, with one exception:

- **Newline after backslash**: If a `<newline>` immediately follows the `\`, the shell **shall** interpret this as line continuation. Both the `\` and the `<newline>` **shall** be removed before splitting the input into tokens. The removed `\<newline>` is NOT replaced by any whitespace, so it cannot serve as a token separator.

Examples:
- `\$` → literal `$` (no parameter expansion)
- `\\` → literal `\`
- `\<newline>` → line continuation (the two characters vanish, joining the next line)
- `echo hel\<newline>lo` → `echo hello` (one token)

Implementation: during tokenization, when an unquoted `\` is encountered, check if the next character is `<newline>`. If yes, discard both and continue reading. If no, mark the next character as literal (quoted) and consume both.

See 2.2.1 Escape Character (Backslash) in utilities/V3_chap02.html.
