## Extracted Text

> When not inside a bracket expression, the interpretation of an ordinary character preceded by an unescaped <backslash> is undefined, except for:: The ']' character; "\]" shall match a ']' character

## Context

**Behavior:** In BRE context, outside a bracket expression, `\]` shall match a literal `]` character. The interpretation of an ordinary character preceded by an unescaped backslash is undefined except for specifically listed cases.

**Shell relevance:** In shell pattern matching, backslash-escaping of `]` outside a bracket expression is governed by shell quoting rules (section 2.2), not BRE rules. Inside a bracket expression, `]` has special meaning (closes the expression). This BRE-specific escaping rule does not directly apply to shell pattern matching, but the bracket expression rules (section 9.3.5) shared with shell patterns describe `]` handling within bracket expressions.
