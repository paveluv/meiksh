## Extracted Text

> If a <backslash>-escape sequence represents a single-quote character (for example \'), that sequence shall not terminate the dollar-single-quote sequence.

## Context

**Behavior:** If a backslash-escape sequence inside `$'...'` produces a single-quote character (most notably `\'`), that single-quote must NOT terminate the `$'...'` construct. Only an unescaped single-quote terminates it.

**Applies to:** All shell modes during `$'...'` tokenization and escape processing.

**Implementation detail:** During tokenization, when scanning for the end of a `$'...'` string, the tokenizer must track escape sequences. If it encounters `\` followed by `'`, it must recognize this as an escaped single-quote (producing a literal `'` in the output) and continue scanning for the actual terminating `'`. The tokenizer should NOT use a simple "find next `'`" approach; it must be escape-aware.

**Example:** `$'it\'s here'` should produce the string `it's here` — the `\'` produces a literal `'` and the construct continues until the final unescaped `'`.

**Spec reference:** Section 2.2.4, line 234 in `utilities/V3_chap02.html`.
