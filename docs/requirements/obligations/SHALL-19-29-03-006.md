## Extracted Text

> Signals that were ignored on entry to a non-interactive shell cannot be trapped or reset, although no error need be reported when attempting to do so. An interactive shell may reset or catch signals ignored on entry. Traps shall remain in place for a given shell until explicitly changed with another trap command.

## Context

**Behavior:** Signal inheritance and trap persistence:
1. Signals that were **ignored** on entry to a **non-interactive** shell cannot be trapped or reset (no error need be reported for trying).
2. An **interactive** shell **may** reset or catch signals that were ignored on entry.
3. Traps remain in place for a given shell until explicitly changed with another `trap` command.

**Implementation:** On startup, check the disposition of each signal. If SIG_IGN for a non-interactive shell, do not allow `trap` to change it. For interactive shells, allowing override is permitted.

**Applies to:** Differs for interactive vs non-interactive shells.
