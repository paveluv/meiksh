## Extracted Text

> The following environment variables shall affect the execution of sh:: [UP] Determine a decimal number representing the limit to the number of previous commands that are accessible. If this variable is unset, an unspecified default greater than or equal to 128 shall be used. The maximum number of commands in the history list is unspecified, but shall be at least 128. An implementation may choose to access this variable only when initializing the history file, as described under HISTFILE . Therefore, it is unspecified whether changes made to HISTSIZE after the history file has been initialized are effective.

## Context

**Behavior:** HISTSIZE controls the number of previous commands accessible in the history list:

1. **When set:** Its value shall be interpreted as a decimal number limiting the number of accessible previous commands.
2. **When unset:** The shell shall use an unspecified default that is >= 128.
3. **Minimum:** The maximum number of commands in the history list shall be at least 128, regardless of HISTSIZE's value.
4. **Initialization:** The implementation may read HISTSIZE only once during history file initialization (as described under HISTFILE). Changes to HISTSIZE after initialization may have no effect (this is unspecified).

**Applies to:** Interactive shells (User Portability [UP] option).

**Implementation guidance:** A reasonable default when HISTSIZE is unset would be 512 or 1000 (must be >= 128). The shell should parse HISTSIZE as a decimal integer; non-numeric values have unspecified behavior.
