## Extracted Text

> The following environment variables shall affect the execution of read:: IFS

## Context

**Behavior required:** The `IFS` environment variable must affect the execution of the `read` builtin. `IFS` determines the field separators used to split the input line into fields for variable assignment.

**Condition:** During field splitting of the input read from stdin.

**Implementation detail:** `IFS` is the primary variable controlling `read`'s field splitting behavior:
- If `IFS` is set to empty string: no splitting, entire line goes to first var.
- If `IFS` is unset: defaults to `<space><tab><newline>`.
- If `IFS` is set to non-empty: its characters are used as field delimiters per the modified field splitting algorithm.

See SHALL-20-100-03-004 through SHALL-20-100-03-009 for the splitting rules.
