## Extracted Text

> For the purposes of job control, a list that includes more than one asynchronous AND-OR list shall be treated as if it were split into multiple separate lists, each ending with an asynchronous AND-OR list.

## Context

**Multiple async lists in one list — splitting:** If a list contains more than one asynchronous AND-OR list (e.g., `cmd1 & cmd2 & cmd3`), it shall be treated as if split into multiple separate lists, each ending with an asynchronous AND-OR list. So `cmd1 & cmd2 & cmd3` is treated as three separate lists: `cmd1 &`, `cmd2 &`, and `cmd3` (the last being synchronous if not `&`-terminated). Each `&`-terminated portion creates its own background job.

The implementation must detect multiple `&` separators in a single list and create separate jobs for each async segment.
