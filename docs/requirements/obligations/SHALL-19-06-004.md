## Extracted Text

> Pathname expansion (see 2.6.6 Pathname Expansion) shall be performed, unless set -f is in effect.

## Context

This defines word expansion step 3: pathname expansion (globbing).

**Required behavior:**
- Pathname expansion (as defined in 2.6.6) must be performed on each field resulting from step 2, unless globbing is disabled via `set -f` (noglob).
- Unquoted pattern characters (`*`, `?`, `[`) trigger pathname expansion: the field is matched against filesystem entries and replaced with the sorted list of matching pathnames.
- If `set -f` is in effect, this step is skipped entirely — pattern characters are treated as literal.

**Conditions:** Only performed on words in field-splitting contexts (command arguments). Not performed in:
- Variable assignments
- `case` patterns (which use pattern matching but not pathname expansion)
- Here-document bodies
- Words entirely within double-quotes

See 2.6 "Word Expansions" (step 3), 2.6.6 "Pathname Expansion", and the `set` built-in (`-f` option) in `utilities/V3_chap02.html`.
