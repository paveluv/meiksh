## Extracted Text

> The process ID associated with the asynchronous AND-OR list shall become known in the current shell execution environment; see 2.13 Shell Execution Environment. This process ID shall remain known until any one of the following occurs (and, unless otherwise specified, may continue to remain known after it occurs).

## Context

**Behavior:** The process ID of the asynchronous AND-OR list MUST become known in the current shell execution environment (see 2.13). This is the PID stored in `$!`. The PID MUST remain known until one of the specified conditions occurs (e.g., waited on, or certain job-related events). Even after a condition occurs, the PID MAY continue to remain known.

**Applies to:** All `cmd &` constructs. The PID is the process ID of the subshell created for the async AND-OR list.

**Implementation detail:** After forking the background process, store its PID as `$!` (the `!` special parameter) and add it to the shell's known-process list. This list is used by `wait` and job control. The PID remains in the list until it is explicitly waited on or the job completion is reported.

**Spec reference:** Section 2.9.3.1, lines 2147-2148 in `utilities/V3_chap02.html`. Cross-ref: 2.13 Shell Execution Environment.
