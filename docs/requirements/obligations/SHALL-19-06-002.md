## Extracted Text

> Tilde expansion (see 2.6.1 Tilde Expansion), parameter expansion (see 2.6.2 Parameter Expansion), command substitution (see 2.6.3 Command Substitution ), and arithmetic expansion (see 2.6.4 Arithmetic Expansion) shall be performed, beginning to end. See item 5 in 2.3 Token Recognition.

## Context

This defines word expansion step 1: the four interleaved expansions.

**Required behavior:**
- The following four expansions must be performed as step 1, processing the word from beginning to end (left-to-right):
  1. **Tilde expansion** (2.6.1): Expand unquoted `~` prefixes to home directories.
  2. **Parameter expansion** (2.6.2): Expand `$name`, `${name}`, `${name:-word}`, etc.
  3. **Command substitution** (2.6.3): Expand `$(command)` and `` `command` ``.
  4. **Arithmetic expansion** (2.6.4): Expand `$((expression))`.
- These four are performed "beginning to end" — meaning as the shell scans the word left-to-right, it handles whichever expansion it encounters first. Nested expansions are resolved recursively.
- This step corresponds to "item 5 in 2.3 Token Recognition" — the token recognition phase identifies where these expansions begin.

**Conditions:** Applies to all words subject to expansion (command words, arguments, assignment values, here-document bodies, etc., as appropriate per context).

See 2.6 "Word Expansions" (step 1), 2.6.1–2.6.4, and 2.3 "Token Recognition" in `utilities/V3_chap02.html`.
