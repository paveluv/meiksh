## Extracted Text

> If the exec command fails, a non-interactive shell shall exit from the current shell execution environment; [UP] an interactive shell may exit from a subshell environment but shall not exit if the current shell environment is not a subshell environment.

## Context

**Behavior:** When `exec` with a utility operand **fails** (e.g., command not found, permission denied):
- **Non-interactive shell:** must exit from the current shell execution environment.
- **Interactive shell:** may exit from a subshell environment, but must **not** exit if the current environment is not a subshell environment. The interactive shell continues running.

**Implementation:** After a failed `execve()`, check if the shell is interactive. If non-interactive, exit. If interactive and in a subshell, may exit; if interactive and in the main shell, report the error and continue.

**Applies to:** Differs for interactive vs non-interactive shells.
