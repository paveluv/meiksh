## Extracted Text

> All implementation-provided locales (either preinstalled or provided as locale definitions which can be installed later) shall define a collation sequence that has a total ordering of all characters unless the locale name has an '@' modifier indicating that it has a special collation sequence (for example, @icase could indicate that each upper and lowercase character pair collates equally).

## Context

**Behavior:** All implementation-provided locales must define a total ordering of all characters in their collation sequence, UNLESS the locale name contains an `@` modifier indicating a special collation (e.g., `en_US.UTF-8@icase` for case-insensitive collation). A total ordering means no two distinct characters compare as equal under `strcoll()`.

**Shell relevance:** This is important for the shell's pattern matching:
- With a total ordering, bracket expression ranges like `[a-z]` have well-defined, unambiguous behavior — every character is either in the range or not.
- With `@` modifier locales that relax total ordering (e.g., case-insensitive), `[a-z]` might match uppercase letters too.
- The shell should be aware that `@` modifier locales can have non-total orderings.
