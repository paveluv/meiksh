## Extracted Text

> writes the characters 2>a to standard output. The optional number, redirection operator, and word shall not appear in the arguments provided to the command to be executed (if any).

## Context

Redirections are syntactic constructs consumed by the shell and must NOT be passed to the executed command. See 2.7 Redirection in utilities/V3_chap02.html.

**Behavior required:**
- When building the argument list (`argv`) for a command, the shell must strip out the optional fd number `n`, the redirection operator, and the target `word` from the arguments.
- These redirection components are processed by the shell to set up file descriptors but never appear in the command's argument vector.

**Example:** `echo 2>a` — the command receives only `echo` in its argv (no `2>a`). `echo hello >file` — the command receives `echo hello`, and stdout is redirected to `file`.

**Implementation note:** During command execution setup, after parsing redirections, the argument list passed to `execve` (or the builtin's args) must have all redirection tokens removed.
