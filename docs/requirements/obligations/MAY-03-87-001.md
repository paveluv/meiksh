## Extracted Text

> A terminal that is associated with a session. Each session may have at most one controlling terminal associated with it, and a controlling terminal is associated with exactly one session. Certain input sequences from the controlling terminal cause signals to be sent to all processes in the foreground process group associated with the controlling terminal.

## Context

**Definition — direct shell relevance.** Defines "controlling terminal" as a terminal associated with a session. Each session has at most one controlling terminal. Certain input sequences from the controlling terminal cause signals to be sent to all processes in the foreground process group (e.g., Ctrl+C → SIGINT, Ctrl+Z → SIGTSTP).

**Shell impact:** The shell's job control implementation depends heavily on the controlling terminal concept. When the shell starts interactively, it establishes itself as the session leader (or inherits the session) and associates with a controlling terminal. Job control signals (SIGTSTP, SIGTTIN, SIGTTOU) are sent to foreground process groups via the controlling terminal. The shell must manage foreground/background process groups using `tcsetpgrp()`. When no controlling terminal exists (e.g., `nohup`, daemons), job control is unavailable.
