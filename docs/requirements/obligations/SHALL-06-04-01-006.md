## Extracted Text

> A utility that divides, truncates, or extracts substrings from statefully encoded data shall produce output that contains locking shifts at the beginning or end of the resulting data, if appropriate, to retain correct state information.

## Context

**Behavior:** Duplicate of the core requirement from SHALL-06-04-01-004. When substring extraction produces output from state-dependent encoded data, the output must include appropriate shift codes to maintain valid encoding state.

**Shell relevance:** Same as SHALL-06-04-01-004. Affects parameter expansion substring operations (`${var:offset:length}`, `${var#pattern}`, etc.) when the locale uses a locking-shift encoding.
