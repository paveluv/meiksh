## Context

The `fg` builtin must support exactly one optional operand: `job_id`. If no operand is given, `fg` operates on the current job (most recently suspended, backgrounded, or started as a background job). See SHALL-20-45-05-002 through SHALL-20-45-05-004 for the `job_id` operand details. Reference: POSIX.1-2024 §fg, OPERANDS (tag_20_45_05).
