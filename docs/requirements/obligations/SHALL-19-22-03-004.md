## Extracted Text

> A trap action on EXIT shall be executed before the shell terminates, except when the exit utility is invoked in that trap action itself, in which case the shell shall exit immediately. It is unspecified whether setting a new trap action on EXIT during execution of a trap action on EXIT will cause the new trap action to be executed before the shell terminates.

## Context

**Behavior:** EXIT trap execution rules:

1. A trap action on EXIT must be executed **before** the shell terminates.
2. **Exception:** If `exit` is invoked within the EXIT trap action itself, the shell must exit **immediately** (no infinite recursion).
3. It is **unspecified** whether setting a **new** trap action on EXIT during execution of an EXIT trap causes the new trap to be executed before termination.

**Implementation:** When the shell is about to exit, check if an EXIT trap is set. If so, execute the trap action. During trap execution, if `exit` is called again, skip any further trap processing and exit immediately.

**Applies to:** All shell modes.
