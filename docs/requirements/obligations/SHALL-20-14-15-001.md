## Extracted Text

> The working directory shall remain unchanged.

## Context

**Behavior:** When cd encounters an error (any error that prevents successful completion), the working directory must remain unchanged. The shell must not partially change the directory — if any step of the cd algorithm fails, the process's current working directory must be exactly the same as it was before cd was invoked.

**Error conditions this applies to:** Directory not found, permission denied, empty string operand, PATH_MAX exceeded, OLDPWD unset when using `-`, and any other error in the cd algorithm.

**Implementation detail:** If the shell calls `chdir()` and it succeeds but a subsequent step fails (unlikely but possible), the shell should call `chdir()` back to the original directory to maintain this guarantee. In practice, errors should be detected before calling `chdir()`.

**Spec reference:** POSIX.1-2024, `utilities/cd.html`, section CONSEQUENCES OF ERRORS (tag_20_14_15), line 259.
