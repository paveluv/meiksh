## Extracted Text

> The following commands shall be recognized in command mode:: Move forwards through the command history, searching for the specified pattern, beginning with the next command line. Patterns use the pattern matching notation described in 2.14 Pattern Matching Notation, except that the '^' character shall have special meaning when it appears as the first character of pattern. In this case, the '^' is discarded and the characters after the '^' shall be matched only at the beginning of a line. Commands in the command history shall be treated as strings, not as filenames. If the pattern is not found, the current command line shall be unchanged and the terminal shall be alerted. If it is found in a following line, the current command line shall be set to that line and the cursor shall be set to the fist character of the new command line. If pattern is empty, the last non-empty pattern provided to / or ? shall be used. If there is no previous non-empty pattern, the terminal shall be alerted and the current command line shall remain unchanged.

## Context

**Behavior:** In vi command mode, `?pattern<newline>` searches FORWARD (toward newer entries) through the command history for a line matching `pattern`, starting from the next command line. Uses the same pattern matching rules as `/` (see 2.14 Pattern Matching Notation in `utilities/V3_chap02.html#tag_19_14`), including the `^` anchor at pattern start.

**Details:**
- If not found: alert terminal, command line unchanged.
- If found: set current command line to that history entry, cursor on first character.
- If pattern is empty: reuse the last non-empty pattern from `/` or `?`. If there is no previous pattern, alert terminal, no change.

**Spec reference:** vi Line Editing Command Mode in `utilities/sh.html`, line 655–663. See `utilities/V3_chap02.html#tag_19_14` for pattern matching notation.
