## Context

**Behavior:** The `&&` and `||` operators MUST have equal precedence and MUST be evaluated with left associativity. This means `a || b && c` is parsed as `(a || b) && c`, NOT as `a || (b && c)`. Example: `false || echo bar && echo baz` outputs both "bar" and "baz" because `false || echo bar` succeeds (exit 0), then `&& echo baz` also runs.

**Applies to:** All AND-OR lists in all shell modes.

**Implementation detail:** The parser should build a flat left-associative chain of AND-OR operators, not a tree with different precedence levels for `&&` vs `||`. During execution, evaluate left-to-right: for `&&`, execute the right side only if the left exited 0; for `||`, execute the right side only if the left exited non-zero.

**Spec reference:** Section 2.9.3, lines 2102-2103 in `utilities/V3_chap02.html`.
