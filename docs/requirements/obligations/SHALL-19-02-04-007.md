## Extracted Text

> A sequence of characters starting with a <dollar-sign> immediately followed by a single-quote ($') shall preserve the literal value of all characters up to an unescaped terminating single-quote ('), with the exception of certain <backslash>-escape sequences, as follows:: \e yields an <ESC> character.

## Context

**Behavior:** Inside `$'...'`, the escape sequence `\e` yields the ESC (escape) character.

**Applies to:** All shell modes during `$'...'` escape processing.

**Implementation detail:** Emit byte 0x1B (ESC). Note: this is a POSIX 2024 addition; `\e` is not a standard C escape. If the locale does not have an encoding for ESC, the result is implementation-defined (see SHALL-19-02-04-019), but the implementation must not emit invalid characters.
