## Extracted Text

> The <signal.h> header shall also define the following symbolic constants:: SA_SIGINFO

## Context

The `<signal.h>` header shall define the symbolic constant `SA_SIGINFO`. This flag is used in the `sa_flags` member of `struct sigaction`. When set, the signal handler receives extra information via a `siginfo_t` structure (the handler signature uses `sa_sigaction` instead of `sa_handler`). For `meiksh`, this may be used with `SIGCHLD` to obtain the child's PID, exit status, and signal information directly from the `siginfo_t` rather than calling `waitpid()` in the handler. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html line 531).
