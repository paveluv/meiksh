## Extracted Text

> If end-of-file is detected before a terminating logical line delimiter is encountered, the variables specified by the var operands shall be set as described above and the exit status shall be 1.

## Context

**Behavior required:** When EOF is reached before a line delimiter (newline or `-d delim` character) is found:
1. The variables are STILL set using whatever partial input was read (using the same splitting and assignment rules).
2. The exit status is 1 (not 0).

**Condition:** EOF on stdin before a terminating delimiter is encountered.

**Example:** If stdin contains `"hello"` with no trailing newline:
- `read var` sets `var="hello"` and returns exit status 1.
- `read a b` with input `"one two"` (no newline) sets `a="one"`, `b="two"`, returns 1.

**Implementation detail:** After reading bytes until EOF:
1. Process the accumulated input as a logical line (minus delimiter, which is absent).
2. Perform field splitting and variable assignment normally.
3. Return exit status 1 instead of 0.

This is distinct from errors (which return >1) — EOF specifically returns exactly 1.
