## Extracted Text

> Some standard utilities and functions support case-insensitive regular expression matching. When this type of matching is in effect, the matching process shall be modified as described in 4.1 Case Insensitive Comparisons.

## Context

**Behavior:** When case-insensitive RE matching is in effect, the matching process shall be modified as described in section 4.1 (Case Insensitive Comparisons).

**Shell relevance:** The shell does not natively support case-insensitive pattern matching in POSIX mode (there is no standard shell option for it). However, if `meiksh` implements extensions like `nocasematch` (bash-like), the case folding must follow the rules in section 4.1. In standard POSIX mode, all shell pattern matching is case-sensitive.

**Spec reference:** Section 9.2, lines 100-101 in `basedefs/V1_chap09.html`; section 4.1 Case Insensitive Comparisons.
