## Extracted Text

> The following commands shall be recognized in command mode:: Move to the start of the next word. If the cursor was positioned on the last character of the line, the terminal shall be alerted and the cursor shall not be advanced. If the count is larger than the number of words after the cursor, this shall not be considered an error; the cursor shall advance to the last character on the line.

## Context

**Behavior:** In vi command mode, `w` moves the cursor to the start of the next word. A "word" uses vi-style word boundaries: sequences of alphanumeric/underscore characters, or sequences of other non-blank characters (punctuation), are each a word, separated by blanks.

**Edge cases:**
- If cursor is on the last character: alert the terminal (bell), do NOT advance.
- If count exceeds the number of words after cursor: NOT an error; cursor advances to the last character on the line.

**Implementation:** Implement vi-style word boundary detection. Move forward `count` words. Clamp at line end. This is a motion command.

**Spec reference:** vi Line Editing Command Mode in `utilities/sh.html`, line 470–472. See `utilities/vi.html` for the word definition.
