## Extracted Text

> When a function is executed, it shall have the syntax-error properties described for special built-in utilities in the first item in the enumerated list at the beginning of 2.15 Special Built-In Utilities.

## Context

**Behavior:** When a function is executed, syntax errors within the function body must have the same consequences as syntax errors in special built-in utilities: specifically, a syntax error in a function causes a non-interactive shell to exit. This is the first item in the enumerated list at the beginning of 2.15 Special Built-In Utilities.

**Implementation:** When a syntax error is detected during function execution, the shell must treat it as a fatal error for non-interactive shells (exit the shell). Interactive shells should report the error and continue.

**Applies to:** All shell modes, but the exit-on-error behavior specifically affects non-interactive shells.

**Spec reference:** Section 2.9.5 Function Definition Command in `utilities/V3_chap02.html`, lines 2392-2393. Cross-ref: 2.15 Special Built-In Utilities.
