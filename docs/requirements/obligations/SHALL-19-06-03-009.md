## Extracted Text

> The results of command substitution shall not be processed for further tilde expansion, parameter expansion, command substitution, or arithmetic expansion.

## Context

**Behavior:** After a command substitution is expanded (stdout captured, trailing newlines stripped, substituted into the word), the resulting text must NOT undergo any further recursive expansion. Specifically, the substitution result must not be re-scanned for:
- Tilde expansion (`~`)
- Parameter expansion (`$var`, `${...}`)
- Command substitution (`$(...)`, `` `...` ``)
- Arithmetic expansion (`$((...))`)

The result IS still subject to:
- Field splitting (if unquoted and IFS is set)
- Pathname expansion (globbing, if unquoted and `set -f` is not in effect)
- Quote removal

**Rationale:** This prevents uncontrolled recursive expansion of user data. Command output containing `$`, `~`, or backticks must be treated as literal text.

**Spec reference:** See 2.6.3 Command Substitution in utilities/V3_chap02.html, line 1138.
