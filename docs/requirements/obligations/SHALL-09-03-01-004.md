## Extracted Text

> When not inside a bracket expression, the following shall match a single character:: a <period>

## Context

**Behavior:** A period (`.`) outside a bracket expression is a BRE that shall match any single character in the supported character set except NUL.

**Shell relevance:** The shell equivalent is the `?` pattern character, which matches any single character except NUL (and, in filename expansion, does not match a leading `.` in a filename component unless the `.` is explicitly in the pattern). The period (`.`) has no special meaning in shell patterns — it is a literal character in shell pattern matching context.
