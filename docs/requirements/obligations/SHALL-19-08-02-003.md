## Extracted Text

> The exit status of a command shall be determined as follows:: Otherwise, if the command name is found, but it is not an executable utility, the exit status shall be 126.

## Context

**Behavior:** When a command name resolves to a file that exists but is not an executable utility (e.g., a regular file without execute permission, or a directory), the shell MUST set the exit status to exactly 126. The shell must also write a diagnostic error message to standard error.

**Applies to:** Commands where the file is found on disk (either via PATH search or a slash-containing path) but `execl()` would fail with EACCES or similar. Also applies when the file exists but is not a regular file (e.g., a directory).

**Error handling:** Write a "permission denied" or "not executable" style diagnostic to stderr.

**Implementation detail:** This applies when the file exists (stat succeeds) but either lacks execute permission or is not a regular file. Set `$?` to 126. This check happens before the ENOEXEC fallback to shell script execution.

**Spec reference:** Section 2.8.2, line 1702 in `utilities/V3_chap02.html`.
