## Extracted Text

> Since the termios structure may include additional members, and the standard members may include both standard and non-standard modes, the structure should never be initialized directly by the application as this may cause the terminal to behave in a non-conforming manner. When opening a terminal device (other than a pseudo-terminal) that is not already open in any process, it should be opened with the O_TTY_INIT flag before initializing the structure using tcgetattr() to ensure that any non-standard elements of the termios structure are set to values that result in conforming behavior of the terminal interface.

## Context

**Behavior (advisory):** The termios structure may have implementation-specific additional members. Applications should not initialize it from scratch — instead, use tcgetattr() to read the current state, then modify specific fields. For new terminal opens (not pseudo-terminals), O_TTY_INIT should be used to ensure non-standard members have conforming values.

**Conditions:** Advisory ("should"), not mandatory. Applies when modifying terminal attributes.

**Shell relevance:** The shell must always use tcgetattr() to read the current terminal state before modifying it. Never zero-initialize a termios structure and use tcsetattr() — this could corrupt implementation-specific fields.
