## Extracted Text

> A non-negative integer value defining the default column width for any printable character not listed by one of the WIDTH keywords. If no WIDTH_DEFAULT keyword is included in the charmap, the default character width shall be 1.

## Context

**Behavior:** The `WIDTH_DEFAULT` keyword in a charmap file sets the default column width for printable characters not explicitly listed in WIDTH declarations. If `WIDTH_DEFAULT` is absent, the default width is 1.

**Shell relevance:** The shell uses character display widths via `wcwidth()`. The default width of 1 for unlisted characters is the platform guarantee. For the shell's line editing, this means any printable character without a specific width entry is treated as occupying one column position.

**Spec reference:** Section 6.4, lines 1987-1989 in `basedefs/V1_chap06.html`.
