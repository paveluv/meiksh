## Extracted Text

> The following exit values shall be returned:: A non-interactive shell detected an error other than command_file not found, command_file not executable, or an unrecoverable read error while reading commands (except from the file operand of the dot special built-in); including but not limited to syntax, redirection, or variable assignment errors.

## Context

**Behavior:** The `sh` utility shall return exit status 1-125 when a non-interactive shell detects an error that is NOT one of these specific cases:
- command_file not found (which gets 127)
- command_file not executable / ENOEXEC (which gets 126)
- Unrecoverable read error while reading commands (which gets 128)

**Included errors (non-exhaustive):**
- Syntax errors in the script
- Redirection errors (e.g., cannot open file for redirection)
- Variable assignment errors (e.g., readonly variable assignment)
- Other shell errors per 2.8.1 Consequences of Shell Errors

**Exception:** Read errors from the file operand of the `dot` (`.`) special built-in are excluded from the "unrecoverable read error" category — those would fall into the 1-125 range or be handled by the dot built-in itself.

**Applies to:** Non-interactive shells only. Interactive shells do not exit on these errors.

**Cross-reference:** See `utilities/dot.html` for the dot built-in specification.
