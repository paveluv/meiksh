## Extracted Text

> Special character on input, which is recognized if the ICANON flag is set. Erases the last character in the current line; see 11.1.6 Canonical Mode Input Processing. It shall not erase beyond the start of a line, as delimited by an NL, EOF, or EOL character. If ICANON is set, the ERASE character shall be discarded when processed.

## Context

**Behavior:** The ERASE special character (typically ^? or ^H) — duplicate/alternate extraction of the same spec text as SHALL-11-01-09-003:
1. Recognized on input only when ICANON is set in `c_lflag`.
2. Erases the last character in the current (un-delimited) line.
3. Shall not erase beyond the start of a line (delimited by NL, EOF, or EOL).
4. The ERASE character shall be **discarded** when processed.

**Conditions:** ICANON must be set in `c_lflag`.

**Shell relevance:** See SHALL-11-01-09-003 for full details. Terminal driver provides ERASE in canonical mode; the shell's line editor implements its own in non-canonical mode.

**Spec reference:** POSIX.1-2024, §11.1.9 Special Characters (tag_11_01_09), `basedefs/V1_chap11.html` lines 228–229. See also §11.1.6.
