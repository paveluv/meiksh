## Extracted Text

> The largest file descriptor number supported in shell redirections is implementation-defined; however, all implementations shall support at least 0 to 9, inclusive, for use by the application.

## Context

The shell must support file descriptor numbers 0 through 9 (inclusive) at minimum for use in redirections. The maximum supported fd number is implementation-defined. See 2.7 Redirection in utilities/V3_chap02.html.

**Behavior required:**
- Redirections using fd numbers 0–9 must always work (e.g., `3>file`, `4<input`, `5<&3`).
- The shell may support higher fd numbers (e.g., `10>file`), but this is not required by POSIX.
- If a user specifies an fd number beyond the implementation's maximum, the shell should treat it as an error.

**Decision for meiksh:** Document the maximum supported fd number. Common choices:
- Support only 0–9 (strict POSIX minimum)
- Support up to the system's `OPEN_MAX` or `ulimit -n` limit (what most modern shells do)
