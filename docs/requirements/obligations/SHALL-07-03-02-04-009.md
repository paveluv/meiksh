## Extracted Text

> Weights shall be expressed as characters (in any of the forms specified in 7.3 Locale Definition), <collating-symbol>s, <collating-element>s, an ellipsis, or the special symbol IGNORE. A single character, a <collating-symbol>, or a <collating-element> shall represent the relative position in the character collating sequence of the character or symbol, rather than the character or characters themselves. Thus, rather than assigning absolute values to weights, a particular weight is expressed using the relative order value assigned to a collating element based on its order in the character collation sequence.

## Context

Collation weights in the LC_COLLATE order section are expressed using symbolic references to the collation order rather than absolute numeric values. A weight can be: a character literal, a collating-symbol, a collating-element, an ellipsis, or IGNORE. When a character or symbol is used as a weight, it represents the relative collation position of that character/symbol, not the character itself. This means weights are relative references into the collation sequence. The shell is affected because this weight system determines equivalence classes (characters with the same primary weight are in the same class, usable via `[=x=]` in bracket expressions) and the multi-level comparison behavior used by system collation functions.

See: `basedefs/V1_chap07.html` §7.3 Locale Definition, §7.3.2.4 Collation Order.
