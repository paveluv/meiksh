## Extracted Text

> First command1 shall be executed. If its exit status is zero, command2 shall be executed, and so on, until a command has a non-zero exit status or there are no more commands left to execute. The commands are expanded only if they are executed.

## Context

**Behavior:** In an AND list (`command1 && command2 && ...`), the shell must execute commands sequentially from left to right. After executing `command1`:
- If its exit status is **zero** (success), execute `command2`.
- If its exit status is **non-zero** (failure), skip all remaining commands in the AND list.
- Continue this pattern for each subsequent `&&`-joined command until one fails or all have executed.

**Lazy evaluation:** Commands to the right of a failing command are never expanded or executed. Word expansions, redirections, and side effects of skipped commands must not occur.

**Exit status:** The exit status of the AND list is the exit status of the last command that was actually executed.

**Interaction with `-e`:** When `set -e` is active, AND list commands other than the last are executed with `-e` ignored (the failure of a non-last command does not cause the shell to exit).

**Applies to:** All shell modes.
