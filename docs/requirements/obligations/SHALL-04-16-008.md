## Extracted Text

> If a symbolic link is encountered during pathname resolution, the behavior shall depend on whether the pathname component is at the end of the pathname and on the function being performed. If all of the following are true, then pathname resolution is complete:: The function is required to act on the symbolic link itself, or certain arguments direct that the function act on the symbolic link itself.

## Context

This is the third of three conditions under which a symbolic link is NOT followed: the operation must specifically act on the symbolic link itself (e.g., `lstat()` rather than `stat()`, or the `test -L`/`test -h` operators). For the shell:

- `test -L file` / `test -h file`: must act on the symlink itself (using `lstat`), not follow it.
- `cd -P`: when resolving the physical path, uses the resolved (followed) target but is aware of symlinks.
- Most other shell file operations (redirections, `.` sourcing) follow symlinks by default.

The OS kernel handles this distinction based on which syscall the shell uses (e.g., `lstat` vs. `stat`, `O_NOFOLLOW` flag).
