> If the file is indicated by a pathname pathname, the file descriptor shall be determined by the equivalent of the following operation defined in the System Interfaces volume of POSIX.1-2024:

## Context

When a reference is made to "the contents of a file" identified by a pathname, the file descriptor shall be determined by the equivalent of `fildes = open(pathname, O_RDONLY)`. The contents are then defined as all data obtained by performing `while (read(fildes, buf, nbytes) > 0)` on that file descriptor. For `meiksh`, this defines what "file contents" means when the spec refers to reading a file: the shell opens the file read-only and reads until EOF. This applies to operations like the `.` (dot) command, `source`, input redirection (`< file`), and command substitution reading from files. See POSIX Utilities Chapter 1, Section 1.1.1.7 (utilities/V3_chap01.html lines 731–741).
