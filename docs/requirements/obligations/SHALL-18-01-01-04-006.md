## Extracted Text

> When a file that does not exist is created, the following features defined in the System Interfaces volume of POSIX.1-2024 shall apply unless the utility or function description states otherwise:: The last data access, last data modification, and last file status change timestamps of the file shall be updated as specified in XBD 4.12 File Times Update.

## Context

When a new file is created, the last data access, last data modification, and last file status change timestamps of the file shall be updated as specified in XBD 4.12 File Times Update. Additionally, the last data modification and last file status change timestamps of the parent directory shall be marked for update. For `meiksh`, this is handled by the OS when the shell creates files via `open()` / `creat()` for redirections. The shell does not need to explicitly set timestamps. See POSIX Utilities Chapter 1, Section 1.1.1.4 item 4 (utilities/V3_chap01.html lines 100–101).
