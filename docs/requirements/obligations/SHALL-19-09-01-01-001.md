## Extracted Text

> When a given simple command is required to be executed (that is, when any conditional construct such as an AND-OR list or a case statement has not bypassed the simple command), the following expansions, assignments, and redirections shall all be performed from the beginning of the command text to the end:

## Context

**Behavior:** When a simple command is actually executed (not bypassed by conditional constructs like AND-OR lists or case), the shell MUST perform expansions, assignments, and redirections in a specific ordered sequence from beginning to end of the command text. The steps are: (1) identify variable assignments and redirections, (2) expand non-assignment/non-redirection words to find the command name, (3) perform redirections, (4) expand and perform variable assignments.

**Applies to:** All simple commands that are actually executed. Commands bypassed by conditionals (e.g., the right side of `false && cmd`) are NOT subject to these steps.

**Conditions:** This is the introductory "shall" for the processing order. The detailed steps are in SHALL-19-09-01-01-002 through SHALL-19-09-01-01-008.

**Implementation detail:** The command executor must implement a clear multi-phase pipeline: parse → identify assignments/redirections → expand command name → expand arguments → redirect → assign. The order matters because redirections can affect command substitutions in assignments.

**Spec reference:** Section 2.9.1.1, lines 1733-1734 in `utilities/V3_chap02.html`.
