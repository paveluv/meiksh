## Extracted Text

> Interactive Shell: Expansion error -- shall not exit

## Context

An interactive shell SHALL NOT exit when an expansion error occurs. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required:**
- When the shell is interactive and an expansion error occurs, the shell SHALL write a diagnostic to stderr but SHALL NOT exit.
- A shell diagnostic message IS required.
- The shell returns to the prompt.

**Example:** In an interactive shell, `echo ${x!y}` produces an error message but the shell continues.
