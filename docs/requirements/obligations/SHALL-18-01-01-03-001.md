## Extracted Text

> The file access control mechanism described by XBD 4.7 File Access Permissions shall apply to all files on an implementation conforming to this volume of POSIX.1-2024.

## Context

The file access control mechanism described by XBD 4.7 File Access Permissions shall apply to all files on a conforming implementation. This means every file operation (read, write, execute, search) is subject to permission checks based on the process's effective user ID, effective group ID, and supplementary group IDs compared against the file's owner, group, and permission bits. For `meiksh`, this affects all file operations the shell performs: opening files for redirection (`>`, `>>`, `<`), sourcing scripts (`.` command), executing commands via `PATH` search, `cd` to directories, etc. When permission is denied, the shell must report appropriate errors. See POSIX Utilities Chapter 1, Section 1.1.1.3 (utilities/V3_chap01.html line 81) and XBD 4.7.
