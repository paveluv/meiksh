## Extracted Text

> If the command_name is the same as the name of one of the special built-in utilities, the special properties in the enumerated list at the beginning of 2.15 Special Built-In Utilities shall not occur. In every other respect, if command_name is not the name of a function, the effect of command (with no options) shall be the same as omitting command, except that command_name does not appear in the command word position in the command command, and consequently is not subject to alias substitution (see 2.3.1 Alias Substitution) nor recognized as a reserved word (see 2.4 Reserved Words).

## Context

**Behavior:** `command` modifies how special built-ins and command words are treated:

1. **Special built-in suppression:** When `command special_builtin` is used, the special properties of that built-in (from section 2.15) do NOT apply. Specifically:
   - Variable assignment errors do not cause the shell to exit.
   - Syntax/redirection errors do not cause the shell to exit.
   - Variable assignments from the command prefix do not persist after the command completes.
   The special built-in still executes, but without its "special" error-handling behavior.

2. **Alias and reserved word suppression:** The command_name after `command` is NOT in the "command word position" of the original line, so:
   - It is not subject to alias substitution (section 2.3.1).
   - It is not recognized as a reserved word (section 2.4).
   For example, `command if` would search for a utility called "if", not start a conditional.

3. **Otherwise same behavior:** If command_name is not a function, `command cmd` behaves identically to just `cmd` (with the exceptions above).

**Spec reference:** POSIX.1-2024, `utilities/command.html`, section DESCRIPTION (tag_20_22_03), lines 42-48.
