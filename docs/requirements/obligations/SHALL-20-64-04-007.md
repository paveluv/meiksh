## Extracted Text

> The following options shall be supported:: -signal_name

## Context

**Behavior:** The `kill` built-in must support the `-signal_name` option form (XSI extension). This allows specifying the signal as a bare option, e.g., `kill -HUP pid`. This is equivalent to `-s signal_name` (see SHALL-20-64-04-008).

**Conditions:** This is an XSI option. The same case-independent matching and SIG-prefix-stripping rules apply as for `-s signal_name`. This form may exceed the single-character option convention from XBD 12.2 Utility Syntax Guidelines.
