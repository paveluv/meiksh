## Extracted Text

> A character that in the output stream indicates that printing should start at the beginning of the next line. It is the character designated by '\n' in the C language. It is unspecified whether this character is the exact sequence transmitted to an output device by the system to accomplish the movement to the next line.

## Context

**Definition — direct shell relevance.** Defines the newline character `<newline>` as `\n` (the C language `'\n'`). It indicates printing should start at the beginning of the next line. Whether this byte is the exact terminal sequence is unspecified.

**Shell impact:** Newline is fundamental to shell operation:
- It terminates commands (equivalent to `;` as a command separator)
- It is the line delimiter in shell scripts and interactive input
- It terminates lines read by the `read` builtin
- It separates records in command substitution output (trailing newlines stripped)
- It appears in escape sequences: `printf '\n'`, `echo`, `$'\n'`
- Here-documents use newline as the line terminator
- The IFS variable's default value includes newline

The shell must handle `\n` (byte 0x0A) correctly in all these contexts.
