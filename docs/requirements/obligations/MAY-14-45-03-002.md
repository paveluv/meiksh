## Extracted Text

> [CX] Implementations may support additional si_code values not included in this list, may generate values included in this list under circumstances other than those described in this list, and may contain extensions or limitations that prevent some values from being generated. Implementations do not generate a different value from the ones described in this list for circumstances described in this list.

## Context

This [CX] provision grants implementations flexibility regarding `si_code` values: (1) implementations may support additional `si_code` values beyond those listed in the standard, (2) implementations may generate listed values under circumstances other than those described, and (3) implementations may have extensions or limitations that prevent some values from being generated. However, implementations must not generate a different value from the standard list for circumstances that the standard specifically describes. For `meiksh`, this means the shell should handle recognized `si_code` values (`CLD_EXITED`, `CLD_KILLED`, `CLD_DUMPED`, `CLD_STOPPED`, `CLD_CONTINUED`) and have a fallback for unrecognized codes. The shell must not assume all listed values will be generated on all platforms. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 1018–1022).
