## Extracted Text

> In the context of programmatic message passing, an object to which messages can be added and removed. Messages may be removed in the order in which they were added or in priority order.

## Context

**Definition only — no shell relevance.** Defines "message queue" as an IPC object to which messages can be added and removed, either in FIFO order or by priority. This is a System V / POSIX IPC concept.

**Shell impact:** None. The shell does not interact with message queues. The `ipcs` and `ipcrm` utilities (external) manage IPC objects. No action required.
