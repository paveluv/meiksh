## Extracted Text

> Each collating-identifier shall consist of either a character (in any of the forms defined in 7.3 Locale Definition), a <collating-element>, a <collating-symbol>, an ellipsis, or the special symbol UNDEFINED. The order in which collating elements are specified determines the character order sequence, such that each collating element shall compare less than the elements following it.

## Context

Each collating identifier in the LC_COLLATE collation order section must be one of: a character literal (in any form from §7.3), a collating-element name (multi-character sequence treated as a unit), a collating-symbol (abstract ordering position), an ellipsis (range fill), or the UNDEFINED symbol (catch-all for unspecified characters). The order of appearance defines the collation sequence: earlier entries sort before later entries. The shell is affected because this ordering determines how bracket expression ranges in pattern matching work (e.g., whether `[a-z]` includes accented characters depends on the collation order). The shell relies on system collation functions that implement this ordering.

See: `basedefs/V1_chap07.html` §7.3 Locale Definition, §7.3.2.4 Collation Order.
