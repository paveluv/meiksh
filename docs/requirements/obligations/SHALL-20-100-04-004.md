## Extracted Text

> The following options shall be supported:: If delim consists of one single-byte character, that byte shall be used as the logical line delimiter. If delim is the null string, the logical line delimiter shall be the null byte. Otherwise, the behavior is unspecified.

## Context

**Behavior required:** The `-d delim` option defines the logical line delimiter:

1. **Single-byte delim:** If `delim` is exactly one single-byte character, that byte replaces newline as the line terminator. `read` reads until that byte is encountered (or EOF).
2. **Empty string delim:** If `delim` is `""` (null string/zero length), the null byte (`\0`) becomes the line delimiter. This allows reading null-terminated input.
3. **Multi-byte or multi-character delim:** Behavior is unspecified. The implementation may reject it, use only the first byte, or handle it in any way.

**Condition:** When `-d delim` is specified.

**Implementation detail:** Read bytes from stdin one at a time, comparing each to the delimiter byte. For empty delim, compare against `\0`. Stop reading when the delimiter is found or EOF is reached. The delimiter byte itself is removed from the input before processing.
