## Extracted Text

> If there are no operands and the -c option is not specified, the -s option shall be assumed.

## Context

Default input mode: if the `sh` invocation has **no operands** (no `command_file`, no `argument` values) and the **`-c` option is not specified**, the shell must behave as if `-s` was specified — i.e., read commands from standard input.

This means:
- `sh` with no arguments → reads from stdin (equivalent to `sh -s`)
- `sh -e` with no file → reads from stdin with errexit enabled
- `sh -c 'echo hi'` → does NOT imply `-s`; reads from command_string

This is the primary rule that determines the shell's input source when invoked without an explicit command file or command string.

See utilities/sh.html#tag_20_110_04.
