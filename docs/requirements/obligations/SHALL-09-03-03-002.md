## Extracted Text

> A BRE special character has special properties in certain contexts. Outside those contexts, or when preceded by an unescaped <backslash>, such a character is a BRE that matches the special character itself. The BRE special characters and the contexts in which they have their special meaning are as follows:: The <asterisk> shall be special except when used: In a bracket expression As the first character of an entire BRE (after an initial '^', if any) Immediately following a "\|" escape sequence (after an initial '^', if any), if the implementation does not match the escape sequence "\|" to the literal character '|'. As the first character of a subexpression (after an initial '^', if any); see 9.3.6 BREs Matching Multiple Characters

## Context

**Behavior:** The asterisk (`*`) shall be special in BRE context except when: (1) inside a bracket expression, (2) as the first character of the entire BRE (after initial `^`), (3) immediately following a `\|` escape sequence (if `\|` is not treated as literal `|`), or (4) as the first character of a subexpression (after initial `^`). In these non-special positions, `*` matches itself literally.

**Shell relevance:** In shell pattern matching, `*` is always special outside bracket expressions (matches zero or more characters). Inside bracket expressions, `*` loses its special meaning and matches a literal `*`. The BRE-specific exceptions (first position, after `\|`) do not apply to shell patterns because shell `*` has different semantics (glob wildcard vs. repetition operator).
