## Extracted Text

> The alias utility shall create or redefine alias definitions or write the values of existing alias definitions to standard output. An alias definition provides a string value that shall replace a command name when it is encountered. For information on valid string values, and the processing involved, see 2.3.1 Alias Substitution.

## Context

The `alias` builtin **shall** create or redefine alias definitions, or write existing alias definitions to stdout. An alias definition maps an alias name to a string value that replaces the command name during alias substitution (XCU 2.3.1). When invoked with `alias-name=string` operands, it creates/updates those aliases. When invoked with bare `alias-name` operands, it writes their definitions. When invoked with no operands, it writes all alias definitions. The replacement string value is substituted for the alias name when the name appears as the first word of a simple command, subject to the alias substitution rules in XCU 2.3.1. Reference: alias utility DESCRIPTION, XCU 2.3.1.
