## Extracted Text

> This variable shall represent a decimal integer >0 used to indicate the user's preferred width in column positions for the terminal screen or window; see 3.75 Column Position. If this variable is unset or null, the number of columns shall be set according to the terminal window size (see XSH tcgetwinsize()); if the terminal window size cannot be obtained, the implementation determines the number of columns, appropriate for the terminal or window, in an unspecified manner. When COLUMNS is set, the number of columns in the terminal window size and any terminal-width information implied by TERM are overridden. Users and conforming applications should not set COLUMNS unless they wish to override the system selection and produce output unrelated to the terminal characteristics. Users should not need to set this variable in the environment unless there is a specific reason to override the implementation's default behavior, such as to display data in an area arbitrarily smaller than the terminal or window.

## Context

**Behavior:** The `COLUMNS` environment variable shall represent a decimal integer >0 indicating the user's preferred terminal width in column positions. If unset or null, the width shall be determined from the terminal window size (via `tcgetwinsize()`); if that fails, the implementation determines the width in an unspecified manner. When `COLUMNS` is set, it overrides the terminal window size and any width implied by `TERM`.

**Shell relevance:** The shell uses `COLUMNS` for: (1) line editing and command-line wrapping in interactive mode, (2) the `select` built-in's column formatting, (3) tab completion display. The shell must check `COLUMNS` on startup and update it when the terminal window is resized (SIGWINCH). When `COLUMNS` is explicitly set by the user, the shell must honor that value even if it doesn't match the actual terminal width. The shell should initialize `COLUMNS` from `tcgetwinsize()` during startup if not already set.
