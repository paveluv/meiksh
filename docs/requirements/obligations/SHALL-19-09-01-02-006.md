## Extracted Text

> Variable assignments shall be performed as follows:: If the command name is a function that is not a standard utility implemented as a function, variable assignments shall affect the current execution environment during the execution of the function. It is unspecified: Whether or not the variable assignments persist after the completion of the function Whether or not the variables gain the export attribute during the execution of the function Whether or not export attributes gained as a result of the variable assignments persist after the completion of the function (if variable assignments persist after the completion of the function)

## Context

**Behavior:** When the command name is a user-defined function (NOT a standard-utility-implemented-as-function), variable assignments MUST affect the current execution environment DURING the function's execution. Three things are UNSPECIFIED: (1) whether assignments persist after the function returns, (2) whether the variables gain the export attribute during function execution, and (3) whether export attributes persist after return.

**Applies to:** `VAR=value my_function` where `my_function` is a user-defined shell function.

**Implementation detail:** meiksh must make the assignments visible during function execution. The common behavior (bash, dash, etc.) is for prefix assignments to user functions to persist after return, but this is not required by POSIX. meiksh should decide and document its behavior. The simplest approach is to set variables in the current environment (persistent).

**Spec reference:** Section 2.9.1.2, lines 1787-1792 in `utilities/V3_chap02.html`.
