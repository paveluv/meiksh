## Extracted Text

> The following tokens are processed (in addition to those string constants shown in the grammar):: In a BRE, one of the character sequences: \^ \. \* \[ \] \$ \\ On implementations where the escape sequences "\?", "\+", and "\|" match the literal characters '?', '+', and '|', respectively, QUOTED_CHAR shall also include: \? \+ \| In an ERE, one of the character sequences: \^ \. \[ \] \$ \( \) \| \* \+ \? \{ \} \\

## Context

The `QUOTED_CHAR` token defines which backslash-escape sequences produce literal character matches. In BRE mode, the mandatory QUOTED_CHAR sequences are: `\^`, `\.`, `\*`, `\[`, `\]`, `\$`, `\\`. If the implementation also supports `\?`, `\+`, `\|` as literal escapes in BREs, those must also be included. In ERE mode, the QUOTED_CHAR sequences are: `\^`, `\.`, `\[`, `\]`, `\$`, `\(`, `\)`, `\|`, `\*`, `\+`, `\?`, `\{`, `\}`, `\\`. Each of these must match the literal character following the backslash. The ERE engine must recognize all 14 ERE QUOTED_CHAR sequences and treat them as literal character atoms.
