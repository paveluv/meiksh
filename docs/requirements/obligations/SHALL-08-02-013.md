## Extracted Text

> The values of locale categories shall be determined by a precedence order; the first condition met below determines the value:

## Context

**Behavior:** The values of locale categories shall be determined by a precedence order. The shell must evaluate the following conditions in order and use the first one that applies: (1) `LC_ALL` if defined and non-null, (2) the specific `LC_*` variable for the category if defined and non-null, (3) `LANG` if defined and non-null, (4) the implementation-defined default locale.

**Shell relevance:** This precedence governs every locale-sensitive operation in the shell: character classification (`LC_CTYPE`), collation in pattern matching and bracket expressions (`LC_COLLATE`), message formatting (`LC_MESSAGES`), numeric formatting in `printf` (`LC_NUMERIC`). The shell must implement this precedence correctly when initializing its own locale and when setting up the environment for child processes.
