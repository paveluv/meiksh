## Extracted Text

> Once a process ID that is known in the current shell execution environment (see 2.13 Shell Execution Environment) has been successfully waited for, it shall be removed from the list of process IDs that are known in the current shell execution environment. If the process ID is associated with a background job, the corresponding job shall also be removed from the list of background jobs.

## Context

**Behavior:** After `wait` successfully waits for a known PID:

1. **PID removal:** The PID shall be removed from the shell's list of known process IDs. Subsequent `wait` calls for the same PID will treat it as unknown (returning 127).
2. **Job removal:** If the PID is associated with a background job (job control), the corresponding job entry shall also be removed from the shell's job table.

**Conditions:** This cleanup happens after the process has terminated and been successfully waited for. It applies to both explicit `wait $pid` calls and implicit waiting via `wait` with no operands.

**Implication:** Once a PID has been waited for, its exit status is no longer available. The shell discards the status information.
