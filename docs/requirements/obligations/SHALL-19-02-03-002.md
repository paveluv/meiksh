## Extracted Text

> Enclosing characters in double-quotes ("") shall preserve the literal value of all characters within the double-quotes, with the exception of the characters backquote, <dollar-sign>, and <backslash>, as follows:: $

## Context

**Behavior:** Inside double-quotes, the `$` character retains its special meaning as the introducer for parameter expansion (`$name`, `${name}`), command substitution (`$(...)`), and arithmetic expansion (`$(( ... ))`). This is one of the three exceptions to the general rule that double-quotes preserve literal values.

**What is NOT retained:** The `$` does NOT retain its special meaning for introducing dollar-single-quotes (`$'...'`). Inside double-quotes, `$'` is treated as a literal dollar-sign followed by a literal single-quote—it does not trigger escape-sequence processing.

**Applies to:** All shell modes, during expansion of double-quoted words.

**Implementation detail:** When the expander encounters `$` inside a double-quoted context, it must check what follows: `{` → parameter expansion, `(` → command substitution (or `((` → arithmetic expansion), a name/digit/special-param char → simple parameter expansion. If `'` follows, the `$` is literal (no dollar-single-quote processing inside double-quotes).

**Spec reference:** Section 2.2.3, line 110 in `utilities/V3_chap02.html`. See also 2.6.2 Parameter Expansion, 2.6.3 Command Substitution, 2.6.4 Arithmetic Expansion, 2.2.4 Dollar-Single-Quotes.
