## Context

**Behavior:** Introductory statement establishing that when a simple command has a command name (and optional arguments) after word expansion, a specific set of search and execution actions MUST be performed. The detailed rules are in SHALL-19-09-01-04-002 through SHALL-19-09-01-04-005, covering the two main cases: commands without slashes (PATH search) and commands with slashes (direct execution).

**Applies to:** All simple commands that produce a command name after step 2 of 2.9.1.1.

**Implementation detail:** The command search is the central dispatch mechanism of the shell. It must check, in order: special builtins → reserved names → functions → intrinsic utilities → PATH search → error. For slash-containing names, skip directly to execution.

**Spec reference:** Section 2.9.1.4, line 1812 in `utilities/V3_chap02.html`. Cross-ref: 2.9.1.1 Order of Processing.
