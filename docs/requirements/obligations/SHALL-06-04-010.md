## Extracted Text

> The character set mapping definitions shall be all the lines immediately following an identifier line containing the string "CHARMAP" starting in column 1, and preceding a trailer line containing the string "END CHARMAP" starting in column 1. Empty lines and lines containing a <comment_char> in the first column shall be ignored. ## Context

**Behavior:** Charmap file structure: character mapping definitions appear between `CHARMAP` and `END CHARMAP` delimiters (both starting in column 1). Within this section, empty lines and comment lines (starting with the comment character) are ignored. Each data line is in one of two forms: (1) a single symbolic name with its encoding, or (2) a range of symbolic names with a starting encoding.

**Shell relevance:** The shell does not parse charmap files. This describes the charmap file format consumed by `localedef`. No shell implementation action needed.
