## Extracted Text

> A sequence of characters starting with a <dollar-sign> immediately followed by a single-quote ($') shall preserve the literal value of all characters up to an unescaped terminating single-quote ('), with the exception of certain <backslash>-escape sequences, as follows:: \ddd yields the byte whose value is the octal value ddd (one to three octal digits).

## Context

**Behavior:** Inside `$'...'`, the escape sequence `\ddd` (backslash followed by 1-3 octal digits) yields the byte whose value is the octal number `ddd`.

**Applies to:** All shell modes during `$'...'` escape processing.

**Implementation detail:** After `\`, if the next character is an octal digit (0-7), consume up to 3 octal digits total (see SHALL-19-02-04-017 for termination). Convert to a byte value. If the octal value exceeds 255 (i.e., `\400` or above), behavior is unspecified (see UNSPEC-19-02-04-004). If the value is 0 (`\0`, `\00`, `\000`), behavior regarding null inclusion is unspecified (see UNSPEC-19-02-04-003).

**Example:** `$'\101'` → byte 0x41 → `A`; `$'\012'` → byte 0x0A → newline.

**Spec reference:** Section 2.2.4, lines 213-214 in `utilities/V3_chap02.html`.
