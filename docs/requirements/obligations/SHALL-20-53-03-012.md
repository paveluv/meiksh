## Extracted Text

> If an option-argument is missing:: Otherwise, the shell variable specified by name shall be set to the <question-mark> character, the shell variable OPTARG shall be unset, and a diagnostic message shall be written to standard error. This condition shall be considered to be an error detected in the way arguments were presented to the invoking application, but shall not be an error in getopts processing; a diagnostic message shall be written as stated, but the exit status shall be zero.

## Context

**Behavior:** When `getopts` detects that a required option-argument is missing AND `optstring` does NOT begin with `:` (verbose error reporting mode), the shell must:
1. Set the variable named by `name` to the literal character `?`.
2. Unset `OPTARG`.
3. Write a diagnostic message to stderr (format is unspecified but must include program name and the option character — see SHALL-20-53-11-001 through SHALL-20-53-11-007).

**Exit status:** Despite detecting a missing argument, the exit status shall be **zero**. This is considered an application-level error (user passed wrong arguments), not a getopts processing error.

**Condition:** This is the default behavior when `optstring` does not begin with `:`. The silent mode (`:` prefix) is covered by SHALL-20-53-03-011.

**Spec reference:** POSIX.1-2024, `getopts` DESCRIPTION, lines 73-76 of `utilities/getopts.html`.
