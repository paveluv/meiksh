## Extracted Text

> If the -p option is not specified and the first operand is an unsigned decimal integer, the shell shall treat all operands as conditions, and shall reset each condition to the default value. Otherwise, if the -p option is not specified and there are operands, the first operand shall be treated as an action and the remaining as conditions.

## Context

**Behavior:** Operand parsing for `trap` (when `-p` is not specified):
1. If the first operand is an **unsigned decimal integer**, treat **all** operands as conditions (signal names/numbers), and reset each to the default action.
2. Otherwise, treat the **first operand** as an action and the remaining operands as conditions.

This means `trap 1 2 3` resets signals 1, 2, and 3 to defaults (because `1` is a number), while `trap 'echo hi' INT` sets INT's action to `echo hi`.

**Applies to:** All shell modes. This is a special built-in.
