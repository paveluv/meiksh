## Extracted Text

> [XSI] If IEXTEN is set, the ERASE, KILL, and EOF characters can be escaped by a preceding <backslash> character, in which case no special function shall occur.

## Context

**Behavior:** [XSI extension] When IEXTEN is set in `c_lflag`:
1. A backslash ('\') preceding an ERASE, KILL, or EOF character shall **escape** it.
2. An escaped ERASE, KILL, or EOF character shall **not** perform its special function — it is treated as a literal data character.
3. The backslash itself is consumed (acts as an escape character).

**Conditions:** IEXTEN must be set in `c_lflag`. ICANON must also be set (since ERASE, KILL, and EOF are canonical-mode characters). This is an XSI extension, not base POSIX.

**Shell relevance:** This allows users to type literal ^D, ^U, ^H etc. in canonical mode by preceding them with backslash. The shell's line editor (non-canonical mode) handles escaping differently. If meiksh claims XSI conformance, it must support this behavior when acting as the terminal driver (unlikely — this is typically a kernel-level feature). For `stty` and terminal configuration, the shell should be aware that IEXTEN enables this escaping.

**Spec reference:** POSIX.1-2024, §11.1.9 Special Characters (tag_11_01_09), `basedefs/V1_chap11.html` lines 271–273. XSI extension.
