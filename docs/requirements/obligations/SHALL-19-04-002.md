## Extracted Text

> This recognition shall only occur when none of the characters is quoted and when the word is used as:

## Context

**Behavior:** Reserved word recognition has two prerequisites that BOTH must be true:
1. **No quoting:** None of the characters in the word is quoted (no backslash-escaping, no single-quotes, no double-quotes, no dollar-single-quotes around or within the word).
2. **Appropriate position:** The word appears in one of the specific grammatical positions listed in sibling obligations SHALL-19-04-003 through SHALL-19-04-006.

If either condition fails, the word is treated as an ordinary WORD token (e.g., a command name or argument), even if it matches a reserved word string.

**Applies to:** All shell modes during parsing.

**Implementation detail:** The quoting check is done on the raw token from the tokenizer (before quote removal). If the token contains ANY quoting characters, it cannot be a reserved word. This is why `\if`, `'if'`, `"if"` are all treated as ordinary words, not the `if` reserved word.
