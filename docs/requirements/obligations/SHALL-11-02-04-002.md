## Extracted Text

> The following functions are provided for getting and setting the values of the input and output baud rates in the termios structure: cfgetispeed(), cfgetospeed(), cfsetispeed(), and cfsetospeed(). The effects on the terminal device shall not become effective and not all errors need be detected until the tcsetattr() function is successfully called.

## Context

**Behavior:** The functions `cfgetispeed()`, `cfgetospeed()`, `cfsetispeed()`, and `cfsetospeed()` get/set baud rate values in the `termios` structure. Changes to baud rates shall not take effect on the terminal device until `tcsetattr()` is successfully called. Not all errors need be detected until `tcsetattr()` is called.

**Relevance to meiksh:** The `stty` builtin uses these functions (or equivalent) to get/set baud rates. The shell must call `tcsetattr()` after modifying baud rate values in the `termios` structure for changes to take effect.

**Spec reference:** POSIX.1-2024, §11.2.4 Control Modes (tag_11_02_04), `basedefs/V1_chap11.html` lines 1060–1064.
