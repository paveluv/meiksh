## Extracted Text

> The shell has several programming constructs that are "compound commands", which provide control flow for commands. Each of these compound commands has a reserved word or control operator at the beginning, and a corresponding terminator reserved word or operator at the end. In addition, each can be followed by redirections on the same line as the terminator. Each redirection shall apply to all the commands within the compound command that do not explicitly override that redirection.

## Context

**Behavior:** When a compound command (`for`, `case`, `if`, `while`, `until`, brace group `{ }`, or subshell `( )`) is followed by redirections on the same line as the terminator reserved word (`done`, `esac`, `fi`, `}`, or `)`), each such redirection must apply to **every command** within the compound command that does not explicitly override that redirection.

**Example:** In `for i in 1 2 3; do echo $i; done > output.txt`, the `> output.txt` applies to every `echo $i` invocation inside the loop. But if a command inside the loop has its own redirection (e.g., `echo $i > other.txt`), that command's stdout redirection overrides the compound-level one for that specific command.

**Implementation:** When executing a compound command with trailing redirections, the shell must apply those redirections before entering the compound body, and restore file descriptors after the compound command completes. Individual commands within the body that have their own redirections for the same file descriptors override the compound-level ones for their duration.

**Applies to:** All compound commands: `for`, `while`, `until`, `case`, `if`, `{ }`, `( )`.
