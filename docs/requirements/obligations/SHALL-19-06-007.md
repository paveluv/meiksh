## Extracted Text

> Tilde expansions, parameter expansions, command substitutions, arithmetic expansions, and quote removals that occur within a single word shall expand to a single field, except as described below. The shell shall create multiple fields or no fields from a single word only as a result of field splitting, pathname expansion, or the following cases:: Parameter expansion of the special parameters '@' and '*', as described in 2.5.2 Special Parameters, can create multiple fields or no fields from a single word.

## Context

This is the first exception to the single-field rule: `$@` and `$*` can create multiple fields or no fields from a single word.

**Required behavior:**
- Parameter expansion of the special parameters `@` and `*` (as defined in 2.5.2 "Special Parameters") can produce multiple fields from a single word, or zero fields.
- This is an inherent property of `$@` and `$*` — they are the only parameter expansions that can split a single word into multiple fields outside of field splitting and pathname expansion.
- The specific rules for `$@` are in SHALL-19-05-02-002; for `$*` in SHALL-19-05-02-003.

**Conditions:** Applies in both splitting and non-splitting contexts (though the exact behavior differs — `"$@"` in double-quotes produces separate fields without field splitting; `"$*"` in double-quotes produces one joined field).

See 2.6 "Word Expansions" and 2.5.2 "Special Parameters" in `utilities/V3_chap02.html`.
