## Extracted Text

> If the current character is not quoted and can be used as the first character of a new operator, the current token (if any) shall be delimited. The current character shall be used as the beginning of the next (operator) token.

## Context

**Behavior:** Token recognition Rule 6: If the current character is unquoted and can be the first character of an operator, two things happen:
1. The current token (if any) is delimited (completed and emitted).
2. The current character becomes the beginning of a new operator token.

**Operator-starting characters:** `|`, `&`, `;`, `<`, `>`, `(`, `)`, newline. These are the characters that can begin shell operators. Note that `(` and `)` are single-character operators that are always immediately delimited.

**Applies to:** All shell modes during tokenization.

**Implementation detail:** This rule only fires if the current character is NOT quoted (not inside single-quotes, double-quotes, dollar-single-quotes, or preceded by an escape backslash). If a word token is in progress, it is finalized. The tokenizer then enters "building operator" state with the current character as the first character of the new operator.

**Example:** In `echo hello|cat`, when the tokenizer reaches `|`, the word `hello` is delimited, and `|` starts a new operator token.

**Spec reference:** Section 2.3, rule 6, lines 290-291 in `utilities/V3_chap02.html`.
