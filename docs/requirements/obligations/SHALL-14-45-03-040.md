## Extracted Text

> The <signal.h> header shall also define the following symbolic constants:: SA_ONSTACK

## Context

The `<signal.h>` header shall define the symbolic constant `SA_ONSTACK`. This is an [XSI] flag used in the `sa_flags` member of `struct sigaction`. When set, it causes signal delivery to occur on an alternate signal stack (established via `sigaltstack()`). For `meiksh`, this flag is generally not needed since shells do not typically use alternate signal stacks. The shell must recognize this constant exists in `<signal.h>` but need not use it. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 521–523).
