## Extracted Text

> Where a utility described in the Shell and Utilities volume of POSIX.1-2024 as conforming to these guidelines is required to accept, or not to accept, the operand '-' to mean standard input or output, this usage is explained in the OPERANDS section. Otherwise, if such a utility uses operands to represent files, it is implementation-defined whether the operand '-' stands for standard input (or standard output), or for a file named -.

## Context

For utilities/builtins that use operands to represent files: when the specific utility's OPERANDS section does not explicitly define the meaning of `-` as an operand, it is implementation-defined whether `-` means standard input/output or a file literally named `-`. The shell must document its choice for each affected builtin. Builtins where `-` is explicitly documented (e.g., `cat -` means stdin) must follow their documentation. For builtins where `-` is not documented, `meiksh` must choose and document one behavior: either treat `-` as stdin/stdout or as a literal filename.
