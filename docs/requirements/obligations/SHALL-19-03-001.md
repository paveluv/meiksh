## Extracted Text

> The shell shall read its input in terms of lines. (For details about how the shell reads its input, see the description of sh.) The input lines can be of unlimited length. These lines shall be parsed using two major modes: ordinary token recognition and processing of here-documents.

## Context

**Behavior:** The shell must read its input in terms of lines and support unlimited line length. Two major parsing modes exist:
1. **Ordinary token recognition** — the default mode, applying the tokenization rules numbered 1-10 in section 2.3.
2. **Here-document processing** — activated when an `io_here` token (`<<` or `<<-`) is recognized by the grammar.

**Input reading:** For details on how input is read (interactive line editing, `-c` string, stdin, script file), see the `sh` utility description (`utilities/sh.html`). The tokenizer operates on one line at a time, but line continuation (`\<newline>`) joins lines before tokenization.

**Applies to:** All shell modes (interactive, non-interactive, `-c` string, script files).

**Implementation detail:** The lexer/tokenizer must handle arbitrarily long lines without fixed buffer limits. It must maintain state to know whether it is in here-document mode or ordinary token mode.

**Spec reference:** Section 2.3 Token Recognition, lines 237-238 in `utilities/V3_chap02.html`. Cross-ref: `utilities/sh.html`.
