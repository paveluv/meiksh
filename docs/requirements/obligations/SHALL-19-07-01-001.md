## Extracted Text

> Input redirection shall cause the file whose name results from the expansion of word to be opened for reading on the designated file descriptor, or standard input if the file descriptor is not specified.

## Context

Defines input redirection behavior using the `<` operator. See 2.7.1 Redirecting Input in utilities/V3_chap02.html.

**Behavior required:**
- The syntax `[n]<word` opens the file named by the expansion of `word` for reading.
- The file is opened on the file descriptor designated by `n`.
- If `n` is not specified, the file is opened on standard input (fd 0).
- The `word` undergoes the standard redirection word expansions (tilde, parameter, command substitution, arithmetic, quote removal — but NOT pathname expansion in non-interactive shells; see SHALL-19-07-004).

**Error conditions:**
- If the file does not exist or cannot be opened for reading, this is a redirection error (see SHALL-19-07-005).
- The consequences depend on the command type per 2.8.1.

**System call:** Implemented via `open(word_expanded, O_RDONLY)` or equivalent.
