## Extracted Text

> If more than one "<<" or "<<-" operator is specified on a line, the here-document associated with the first operator shall be supplied first by the application and shall be read first by the shell.

## Context

Defines the ordering of multiple here-documents on a single command line. See 2.7.4 Here-Document in utilities/V3_chap02.html.

**Behavior required:**
- When more than one `<<` or `<<-` operator appears on a single line, the here-document bodies must be supplied (and read by the shell) in left-to-right order.
- The first `<<`/`<<-` operator's here-document body appears first in the input, followed by the second, and so on.

**Example:**
```sh
cat <<EOF1; cat <<EOF2
body of first here-doc
EOF1
body of second here-doc
EOF2
```
The shell reads the body for `EOF1` first, then the body for `EOF2`.

**Parser implication:** When multiple here-document operators are parsed on one line, the parser must queue them and read their bodies in order after the newline that terminates the command line.
