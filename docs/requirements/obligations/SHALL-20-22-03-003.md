## Extracted Text

> When the -v or -V option is used, the command utility shall provide information concerning how a command name is interpreted by the shell.

## Context

**Behavior:** When invoked with `-v` or `-V`, `command` must switch from its normal "execute a command" mode to an informational mode. Instead of executing the command_name, it provides information about how the shell would interpret that name:
- `-v`: Writes the pathname or command form (concise, machine-friendly).
- `-V`: Writes a human-readable description of the command type and location.

Neither `-v` nor `-V` causes the command_name to be executed.

**Spec reference:** POSIX.1-2024, `utilities/command.html`, section DESCRIPTION (tag_20_22_03), line 49.
