## Extracted Text

> The standard error shall be used only for diagnostic messages.

## Context

The `break` special builtin (section 2.16 in utilities/V3_chap02.html#tag_19_16) must use standard error **only** for diagnostic messages (errors, warnings). It must not write any non-diagnostic output to stderr. The `break` builtin produces no stdout output.

Diagnostic messages that may be written to stderr include:
- Invalid operand (e.g., `break foo` where `foo` is not a number)
- `break` used outside a loop context (implementation-specific behavior)
- Operand `n` is less than 1

See 2.16 break in utilities/V3_chap02.html#tag_19_16.
