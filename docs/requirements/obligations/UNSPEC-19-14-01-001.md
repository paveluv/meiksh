## Extracted Text

> In a pattern, or part of one, where a shell-quoting <backslash> cannot be used to preserve the literal value of a character that would otherwise be treated as special:: For the shell only, it is unspecified whether or not a <backslash> character inside a bracket expression preserves the literal value of the following character.

## Context

**Backslash inside bracket expressions in non-shell-quoting pattern contexts: unspecified for the shell.**

For the shell specifically (as opposed to other utilities like `find` or functions like `fnmatch()`), when a backslash appears inside a bracket expression (e.g., `[\a]`) in a pattern context where shell quoting cannot be used, it is **unspecified** whether the backslash preserves the literal value of the following character.

This means `[\!]` might match either `\` and `!`, or just `!`, depending on the implementation. The implementation may choose either interpretation; portable scripts should avoid backslash inside bracket expressions in non-shell-quoting contexts.
