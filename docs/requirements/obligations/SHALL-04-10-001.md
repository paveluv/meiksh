## Extracted Text

> For a filename to be portable across implementations conforming to POSIX.1-2024, it shall consist only of the portable filename character set as defined in 3.265 Portable Filename Character Set.

## Context

The portable filename character set (defined in basedefs §3.265) consists of: uppercase letters A-Z, lowercase letters a-z, digits 0-9, period `.`, hyphen `-`, and underscore `_`. While the shell must be able to handle any valid filename the OS supports, this obligation defines which characters are guaranteed portable. This is primarily an informational constraint for the shell: the shell should not restrict filenames to this set (that would be non-conforming), but documentation and test suites can rely on this character set for portable test cases. The shell's pathname expansion, tab completion, and filename handling must work correctly with all valid filenames, not just portable ones.

See: `basedefs/V1_chap03.html` §3.265 Portable Filename Character Set.
