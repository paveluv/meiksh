## Extracted Text

> If the characters in the tilde-prefix following the <tilde> form a portable login name, the tilde-prefix shall be replaced by a pathname of the initial working directory associated with the login name. The pathname shall be obtained as if by using the getpwnam() function as defined in the System Interfaces volume of POSIX.1-2024. If the system does not recognize the login name, the results are unspecified.

## Context

**Behavior:** When the tilde-prefix `~username` contains a valid portable login name, the shell must replace the entire tilde-prefix with the home directory pathname of that user. The home directory must be obtained via the system user database (as if by calling `getpwnam(username)` and using the `pw_dir` field).

**Conditions:** Applies when tilde expansion encounters `~username` where `username` passes the portable login name validation (per SHALL-19-06-01-002).

**Inputs/Outputs:**
- Input: `~username` where `username` is a valid portable login name
- Output: the tilde-prefix is replaced by the home directory path from the user database (e.g., `/home/username`)

**Error handling:** If the system does not recognize the login name (i.e., `getpwnam()` returns no match), the results are unspecified. The implementation may leave `~username` unexpanded, expand to empty, or produce an error.

**Spec reference:** See 2.6.1 Tilde Expansion in utilities/V3_chap02.html, lines 761–763. The `getpwnam()` function is defined in the System Interfaces volume (functions/getpwnam.html) — this is an OS interface the shell calls, not a shell builtin.
