## Extracted Text

> A failure to open or create a file shall cause a redirection to fail.

## Context

Any failure to open or create the target file in a redirection must be treated as a redirection error. See 2.7 Redirection in utilities/V3_chap02.html.

**Behavior required:**
- If the `open()` or `creat()` system call fails for the redirection target (e.g., permission denied, directory does not exist, read-only filesystem), the redirection fails.
- When a redirection fails, the shell must write a diagnostic message to stderr.
- The consequences of the redirection failure depend on the command context (special built-in, compound command, function, or other utility) — see 2.8.1 Consequences of Shell Errors (tag_19_08_01).
- The command associated with the failed redirection is not executed.

**Error handling:** The shell should report the filename and the system error (e.g., via `strerror(errno)`) in the diagnostic message. The exit status for a failed redirection is typically non-zero (implementation practice: 1).
