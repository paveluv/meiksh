## Extracted Text

> The following codes are used in Actions when Creating a File that Already Exists:: Fail. The attempt to create the new file shall fail and the utility shall either continue with its operation or exit immediately with an exit status that indicates an error occurred, depending on the description of the utility.

## Context

Action code "F" (Fail): when the table indicates Fail, the attempt to create the new file shall fail and the utility shall either continue with its operation or exit immediately with an error exit status, depending on the utility description. For `meiksh`, this applies when `> file` targets a file of an incompatible type (e.g., trying to redirect to an existing directory). The shell should report an error and either continue with the next command or abort, depending on context (in a script, the command fails but the script may continue). See POSIX Utilities Chapter 1, Section 1.1.1.4 action code F (utilities/V3_chap01.html line 655).
