## Extracted Text

> The exit status of an AND list shall be the exit status of the last command that is executed in the list.

## Context

An AND list connects pipelines with `&&`. The shell evaluates pipelines left-to-right; if any pipeline returns a non-zero exit status, execution of subsequent pipelines in the AND list stops. The exit status of the entire AND list must be the exit status of **the last command that was actually executed**.

Examples:
- `true && true && false` — exit status is 1 (from `false`, the last command executed)
- `false && true` — exit status is 1 (from `false`; `true` was never executed)
- `true && true` — exit status is 0

See 2.9.3.5 AND Lists in utilities/V3_chap02.html#tag_19_09_03_06.
