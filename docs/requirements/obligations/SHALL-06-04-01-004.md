## Extracted Text

> If the implementation supports a character encoding of this type, any of the standard utilities that describe character (versus byte) or text-file manipulation shall have the following characteristics:: A utility that divides, truncates, or extracts substrings from statefully encoded data shall produce output that contains locking shifts at the beginning or end of the resulting data, if appropriate, to retain correct state information.

## Context

**Behavior:** When extracting substrings from state-dependent encoded data, the shell must include appropriate locking shift codes at the beginning and/or end of the result to maintain correct encoding state. Without this, extracted substrings could be misinterpreted.

**Shell relevance:** This affects parameter expansion operations that extract substrings: `${var:offset:length}`, `${var#pattern}`, `${var%pattern}`, `${var##pattern}`, `${var%%pattern}`. If the active locale uses a locking-shift encoding, the result of these operations must include necessary shift codes to be a well-formed encoded string. In practice, using `mbrtowc`/`wcrtomb` for substring extraction handles this.
