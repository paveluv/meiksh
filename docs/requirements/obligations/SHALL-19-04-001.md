## Context

**Behavior:** The shell must recognize the following words as reserved words (keywords with special grammatical meaning):

`!`, `{`, `}`, `case`, `do`, `done`, `elif`, `else`, `esac`, `fi`, `for`, `if`, `in`, `then`, `until`, `while`

These reserved words are used to construct compound commands and control flow. They have special meaning to the shell grammar (see 2.9 Shell Commands) and must be recognized by the parser when they appear in the appropriate contexts (see sibling obligations SHALL-19-04-002 through 006 for the positional rules).

**Applies to:** All shell modes during parsing/grammar processing.

**Implementation detail:** The parser must have a reserved word table containing exactly these 16 words. After tokenization and alias substitution, if a token matches one of these words AND meets the positional criteria, it is classified as the corresponding reserved word token type rather than a generic WORD.
