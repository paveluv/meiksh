## Extracted Text

> The following variables shall affect the execution of the shell:: Set by the shell to the decimal value of its parent process ID during initialization of the shell. In a subshell (see 2.13 Shell Execution Environment), PPID shall be set to the same value as that of the parent of the current shell. For example, echo $PPID and (echo $PPID ) would produce the same value.

## Context

This defines the semantics of the `PPID` variable.

**Required behavior:**
- The shell must set `PPID` to the decimal representation of its parent process ID during initialization (via `getppid()`).
- In a subshell (see 2.13 "Shell Execution Environment"), `PPID` must retain the same value as in the parent of the current (top-level) shell — NOT the PID of the shell itself.
- Example: both `echo $PPID` and `(echo $PPID)` must produce the same value — the parent PID of the original shell.

**Conditions:** Set once during initialization, preserved in subshells. Read-only in the sense that POSIX expects the shell to maintain it, though POSIX does not explicitly forbid user assignment (behavior would be implementation-defined).

**Key distinction from `$$`:** `$$` is the shell's own PID; `PPID` is the shell's parent's PID. Both are preserved unchanged in subshells.

See 2.5.3 "Shell Variables" (`PPID`) and 2.13 "Shell Execution Environment" in `utilities/V3_chap02.html`.
