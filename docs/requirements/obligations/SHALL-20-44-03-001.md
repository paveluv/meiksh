## Extracted Text

> The fc utility shall list, or shall edit and re-execute, commands previously entered to an interactive sh.

## Context

The `fc` builtin must provide two modes of operation: (1) listing commands from the command history (with `-l`), and (2) editing and re-executing previous commands (without `-l`, or with `-s` for quick re-execution). `fc` only operates on commands entered to an interactive shell — history is not maintained for non-interactive shells. The `fc` builtin is an intrinsic utility (required to be built into the shell) and must operate within the current shell execution environment to access and modify the command history list. It is part of the [UP] User Portability option. Reference: POSIX.1-2024 §fc, DESCRIPTION (tag_20_44_03).
