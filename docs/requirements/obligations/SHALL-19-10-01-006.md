## Context

A generic TOKEN must be further classified based on syntactic context into one of: WORD, NAME, ASSIGNMENT_WORD, or a reserved word (e.g., `if`, `then`, `else`, `fi`, `do`, `done`, `case`, `esac`, `while`, `until`, `for`, `in`, `{`, `}`, `!`). The grammar productions are annotated with rule numbers (Rules 1–9 from Section 2.10.2) that specify how to perform this context-dependent reclassification.

When the parser encounters a TOKEN in a position where an annotated grammar production could apply, it must apply the corresponding rule to determine the final token type. This is crucial: the same character sequence (e.g., `in`) may be a reserved word in one context (after `for x`) and a plain WORD in another (as a command argument).

The implementation must track parser state to know which grammar productions are currently viable, and apply the appropriate reclassification rule.
