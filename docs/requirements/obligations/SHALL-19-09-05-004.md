## Extracted Text

> The compound-command shall be executed whenever the function name is specified as the name of a simple command (see 2.9.1.4 Command Search and Execution). The operands to the command temporarily shall become the positional parameters during the execution of the compound-command; the special parameter '#' also shall be changed to reflect the number of operands. The special parameter 0 shall be unchanged. When the function completes, the values of the positional parameters and the special parameter '#' shall be restored to the values they had before the function was executed. If the special built-in return (see return) is executed in the compound-command, the function completes and execution shall resume with the next command after the function call.

## Context

**Behavior:** Function execution semantics:

1. **Invocation:** The function's compound-command is executed whenever the function name appears as the name of a simple command (found during command search, see 2.9.1.4).
2. **Positional parameters:** The operands (arguments) to the function call temporarily become the positional parameters (`$1`, `$2`, ...) during function execution. The special parameter `$#` is updated to reflect the number of function arguments.
3. **`$0` unchanged:** The special parameter `$0` must NOT be changed during function execution — it retains the shell's name or script name.
4. **Restore on return:** When the function completes (normally or via `return`), the positional parameters and `$#` must be restored to the values they had before the function was called.
5. **`return` built-in:** If the `return` special built-in is executed within the function body, the function completes immediately and execution resumes with the next command after the function call. The exit status is the value specified by `return`, or `$?` if no value is given.

**Applies to:** All shell modes.
