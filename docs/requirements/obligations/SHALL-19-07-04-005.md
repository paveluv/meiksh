## Extracted Text

> If any part of word is quoted, not counting double-quotes outside a command substitution if the here-document is inside one, the delimiter shall be formed by performing quote removal on word, and the here-document lines shall not be expanded. Otherwise:: All lines of the here-document shall be expanded, when the redirection operator is evaluated but after the trailing delimiter for the here-document has been located, for parameter expansion, command substitution, and arithmetic expansion. If the redirection operator is never evaluated (because the command it is part of is not executed), the here-document shall be read without performing any expansions.

## Context

When the here-document delimiter is unquoted, the body lines must be expanded. See 2.7.4 Here-Document in utilities/V3_chap02.html.

**Behavior required:**
- All lines of the here-document body SHALL be expanded for: parameter expansion (`$var`, `${var}`), command substitution (`$(cmd)`, `` `cmd` ``), and arithmetic expansion (`$((expr))`).
- Expansions are performed WHEN the redirection operator is evaluated, but AFTER the trailing delimiter has been located (i.e., the shell first reads the entire here-document body, then expands it).
- If the redirection operator is never evaluated (e.g., the command is inside an `if` branch that is not taken, or a function that is never called), the here-document SHALL be read/parsed but NOT expanded.

**Timing detail:** The "locate delimiter first, then expand" rule means the expansion cannot affect where the delimiter is found. The body is captured as raw text first, then expansions are applied at execution time.

**Not expanded:** Tilde expansion, field splitting, and pathname expansion are NOT performed on here-document content.
