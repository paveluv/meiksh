## Extracted Text

> If the file is not a directory: If the link count becomes zero: If no process has the file open, the space occupied by the file shall be freed and the file shall no longer be accessible. If one or more processes have the file open, the file contents shall be preserved until all references to the file have been closed. If the link count is not reduced to zero, the last file status change timestamp shall be marked for update.

## Context

Duplicate extraction of SHALL-18-01-01-05-006: for non-directory files, if link count reaches zero and no process has the file open, space is freed; if processes have it open, contents are preserved until all references close. If link count doesn't reach zero, status change timestamp is updated. For `meiksh`, this enables the open-then-unlink pattern used for secure temporary files (e.g., here-documents). See POSIX Utilities Chapter 1, Section 1.1.1.5 item 5 (utilities/V3_chap01.html lines 711–721).
