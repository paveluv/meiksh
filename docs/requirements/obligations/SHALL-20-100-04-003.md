## Extracted Text

> The following options shall be supported:: -d delim

## Context

**Behavior required:** The `read` builtin must support the `-d delim` option. This option specifies a custom logical line delimiter instead of the default newline.

**Condition:** When `-d` is specified on the `read` command line.

**Implementation detail:** The `-d` option takes one argument `delim`. The behavior of `delim` is defined in SHALL-20-100-04-004: if `delim` is a single byte, that byte becomes the line delimiter; if `delim` is the empty string, the null byte is the delimiter; otherwise behavior is unspecified.

**Spec reference:** Section OPTIONS of `utilities/read.html`, line 111.
