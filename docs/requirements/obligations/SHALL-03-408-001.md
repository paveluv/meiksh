## Extracted Text

> A non-negative integer that is used to identify a system user. When the identity of a user is associated with a process, a user ID value is referred to as a real user ID, an effective user ID, or a saved set-user-ID. The value (uid_t)-1 shall not be a valid user ID, but does have a defined use in some interfaces defined in this standard.

## Context

**Definition — low shell relevance.** Defines "user ID" as a non-negative integer of type `uid_t` identifying a system user. User IDs appear as real, effective, and saved set-user-ID. The value `(uid_t)-1` SHALL NOT be a valid user ID.

**Shell impact:** The shell exposes user IDs through:
- `$UID` (bash extension, not POSIX) or `id -u` (external utility)
- The `$EUID` variable (extension)
- File permission checks during command execution and redirections
- The `kill` builtin (signal delivery permissions depend on UIDs)

For POSIX compliance, the shell inherits UIDs from its parent process and passes them to children. The shell does not set UIDs directly (that requires `setuid()` which is not a shell operation).
