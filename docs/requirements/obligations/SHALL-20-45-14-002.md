## Extracted Text

> If fg does not move a job into the foreground, the following exit value shall be returned:: >0

## Context

When `fg` fails, it must return an exit status greater than 0. Failure conditions include: job_id not found, job control is disabled, no jobs exist, or the specified job cannot be foregrounded. The specific value above 0 is not mandated. Reference: POSIX.1-2024 §fg, EXIT STATUS (tag_20_45_14).
