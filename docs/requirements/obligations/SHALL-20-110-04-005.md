## Extracted Text

> The following additional options shall be supported:: Read commands from the command_string operand. Set the value of special parameter 0 (see 2.5.2 Special Parameters) from the value of the command_name operand and the positional parameters ($1, $2, and so on) in sequence from the remaining argument operands. No commands shall be read from the standard input.

## Context

When `sh -c command_string [command_name [argument...]]` is invoked, the shell must:

1. **Read commands from `command_string`**: parse and execute the string as shell commands.
2. **Set `$0`** from the `command_name` operand (see 2.5.2 Special Parameters in utilities/V3_chap02.html#tag_19_05_02).
3. **Set positional parameters** `$1`, `$2`, etc. from remaining `argument` operands.
4. **Not read from stdin**: No commands shall be read from standard input, regardless of whether stdin is a terminal.

Example: `sh -c 'echo $0 $1' myscript arg1` → outputs `myscript arg1`, `$0` = `myscript`, `$1` = `arg1`.

The `-c` option is mutually exclusive with reading from a `command_file`. If `-c` is given, no file operand is used for commands.

See utilities/sh.html#tag_20_110_04.
