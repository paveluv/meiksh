## Extracted Text

> the result is unspecified. If a '$' that is neither within single-quotes nor escaped by a <backslash> is immediately followed by a <space>, <tab>, or a <newline>, or is not followed by any character, the '$' shall be treated as a literal character.

## Context

This defines the behavior of a bare `$` character that doesn't introduce an expansion.

**Required behavior:**
- If a `$` that is neither within single-quotes nor escaped by a backslash is immediately followed by:
  - A `<space>`, `<tab>`, or `<newline>`, OR
  - Is not followed by any character (end of input/word)
  ...then the `$` must be treated as a **literal character** — it is not an expansion introducer.
- If `$` is followed by a character that doesn't form a valid expansion (not a name, digit, special parameter character, `(`, `{`, etc.), the result is unspecified.

**Conditions:** Applies during the parameter expansion recognition phase (step 1 of word expansion). The shell must distinguish between `$` that introduces an expansion and `$` that is a literal.

**Testing notes:** `echo $` should print `$`; `echo $ foo` should print `$ foo`; `echo "$"` should print `$`.

See 2.6 "Word Expansions" in `utilities/V3_chap02.html`.
