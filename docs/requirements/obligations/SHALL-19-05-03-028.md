## Extracted Text

> The following variables shall affect the execution of the shell:: [UP] The processing of the PS1 shell variable shall be supported if the system supports the User Portability Utilities option. Each time an interactive shell is ready to read a command, the value of this variable shall be subjected to parameter expansion (see 2.6.2 Parameter Expansion) and exclamation-mark expansion (see below). Whether the value is also subjected to command substitution (see 2.6.3 Command Substitution) or arithmetic expansion (see 2.6.4 Arithmetic Expansion) or both is unspecified. After expansion, the value shall be written to standard error. The expansions shall be performed in two passes, where the result of the first pass is input to the second pass. One of the passes shall perform only the exclamation-mark expansion described below. The other pass shall perform the other expansion(s) according to the rules in 2.6 Word Expansions. Which of the two passes is performed first is unspecified. The default value shall be "$ ". For users who have specific additional implementation-defined privileges, the default may be another, implementation-defined value. Exclamation-mark expansion: The shell shall replace each instance of the <exclamation-mark> character ('!') with the history file number (see Command History List) of the next command to be typed. An <exclamation-mark> character escaped by another <exclamation-mark> character (that is, "!!") shall expand to a single <exclamation-mark> character.

## Context

This defines the full semantics of the `PS1` variable. Tagged `[UP]` (User Portability Utilities option).

**Required behavior:**
- Processing of `PS1` must be supported (meiksh targets UP compliance).
- Each time an interactive shell is ready to read a command, the value of `PS1` is expanded and the result is written to standard error.
- **Expansion rules:**
  - PS1 must be subjected to parameter expansion (2.6.2).
  - PS1 must be subjected to exclamation-mark expansion (see below).
  - Whether command substitution (2.6.3) and/or arithmetic expansion (2.6.4) are also performed is unspecified (implementation choice — meiksh should support them for compatibility).
  - Expansions are performed in **two passes**: one pass does exclamation-mark expansion only; the other pass does the other expansions per 2.6 rules. The order of the two passes is unspecified.
- **Exclamation-mark expansion:**
  - Each unescaped `!` in the PS1 value is replaced with the history file number of the next command to be typed.
  - `!!` (two consecutive exclamation marks) expands to a single literal `!`.
- **Default value:** `"$ "`. For privileged users (e.g., root), the default may be another implementation-defined value (traditionally `"# "`).
- **Output:** The expanded prompt is written to standard error (fd 2).

**Conditions:** Interactive shells only. Non-interactive shells do not process PS1.

See 2.5.3 "Shell Variables" (`PS1`), 2.6.2–2.6.4 in `utilities/V3_chap02.html`, and "Command History List" in `utilities/sh.html`.
