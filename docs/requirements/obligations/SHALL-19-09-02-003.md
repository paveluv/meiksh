## Extracted Text

> The standard output of command1 shall be connected to the standard input of command2. The standard input, standard output, or both of a command shall be considered to be assigned by the pipeline before any redirection specified by redirection operators that are part of the command (see 2.7 Redirection).

## Context

**Behavior:** Two rules about pipe connections: (1) The stdout of command1 MUST be connected to the stdin of command2 in a pipeline. (2) The pipe-assigned stdin/stdout of a command is considered assigned BEFORE any explicit redirections in that command are applied. This means `cmd1 | cmd2 >file` redirects cmd2's stdout to file (overriding the pipe), and `cmd1 >file | cmd2` means cmd1's stdout goes to file (not the pipe), so cmd2 gets nothing on stdin from cmd1.

**Applies to:** All pipelines with explicit redirections on pipeline members.

**Implementation detail:** Set up the pipe connections first (before processing redirection operators), then apply the command's own redirections on top. This ensures that user-specified redirections override the pipe assignments.

**Spec reference:** Section 2.9.2, lines 2023-2025 in `utilities/V3_chap02.html`. Cross-ref: 2.7 Redirection.
