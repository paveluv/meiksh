## Extracted Text

> Provide a default value for the internationalization variables that are unset or null. (See XBD 8.2 Internationalization Variables for the precedence of internationalization variables used to determine the values of locale categories.)

## Context

**Behavior:** LANG provides a default value for the internationalization variables (LC_COLLATE, LC_CTYPE, LC_MESSAGES, etc.) when they are unset or null. The precedence hierarchy is defined in XBD 8.2:
1. LC_ALL (if set to non-empty, overrides everything)
2. The specific LC_* variable for the category
3. LANG (as the fallback default)

**Applies to:** All shell modes (interactive and non-interactive).

**Details:** The shell uses locale settings for:
- Character classification (LC_CTYPE): determining what constitutes letters, digits, blanks, etc. in pattern matching and word operations.
- Collation (LC_COLLATE): range expressions and equivalence classes in pattern matching.
- Diagnostic messages (LC_MESSAGES): format and language of error messages written to stderr.

**Cross-reference:** See `basedefs/V1_chap08.html#tag_08_02` for the normative definition of internationalization variable precedence.
