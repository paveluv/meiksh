## Extracted Text

> The following operands shall be supported:: first, last

## Context

The `fc` builtin must support `first` and `last` operands that select the range of commands from the history to list or edit. These operands can take three forms: (1) `[+]number` — a positive command number (as displayed by `fc -l`). (2) `-number` — a negative offset meaning the command executed that many commands ago (e.g., `-1` is the previous command). (3) `string` — a string prefix matching the most recently entered command that begins with that string. The accessible range is determined by `HISTSIZE`. See SHALL-20-44-05-003 for the full behavioral details including defaults. Reference: POSIX.1-2024 §fc, OPERANDS (tag_20_44_05).
