## Extracted Text

> Each symbolic name specified in Portable Character Set shall be included in the file. Each character in Portable Character Set (each row in the table) shall be mapped to a unique coding value. For each character in Non-Portable Control Characters that exists in the character set described by the file, the character's symbolic name(s) from Non-Portable Control Characters and the character's single-byte encoding value shall be included in the file.

## Context

**Behavior:** The charmap file (character set description file) must include every symbolic name from the Portable Character Set table, each mapped to a unique encoding value. Additionally, for any non-portable control characters supported by the character set, their symbolic names and single-byte encodings must be present.

**Shell relevance:** The shell does not parse charmap files directly. However, the shell relies on the system's charmap being correct — the Portable Character Set (ASCII letters, digits, and special characters used in shell syntax like `$`, `|`, `&`, `;`, `<`, `>`, etc.) must all have unique, well-defined encodings. This ensures the shell's tokenizer can safely compare single bytes for these characters in any supported locale.

**Conditions:** Applies to any charmap file used by the system for locale generation. The shell depends on this guarantee being satisfied by the platform.
