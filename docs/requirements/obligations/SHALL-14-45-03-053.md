## Extracted Text

> The <signal.h> header shall also define the following symbolic constants:: [XSI] Process is executing on an alternate signal stack.

## Context

This is the [XSI] extension defining the semantics of `SS_ONSTACK`: when the `ss_flags` member of a `stack_t` returned by `sigaltstack()` has this value, it indicates the process is currently executing on the alternate signal stack. This is a read-only status indicator. For `meiksh`, this is informational and not directly relevant to shell signal handling operations. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html lines 544–545).
