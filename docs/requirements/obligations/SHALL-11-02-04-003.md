## Extracted Text

> The CSIZE bits shall specify the number of transmitted or received bits per byte. If ISTRIP is not set, the value of all the other bits is unspecified. If ISTRIP is set, the value of all but the 7 low-order bits shall be zero, but the value of any other bits beyond CSIZE is unspecified when read. CSIZE shall not include the parity bit, if any. If CSTOPB is set, two stop bits shall be used; otherwise, one stop bit. For example, at 110 baud, two stop bits are normally used.

## Context

**Behavior:** The CSIZE bits in `c_cflag` specify the number of data bits per byte (CS5, CS6, CS7, CS8). If ISTRIP is not set, the value of bits beyond the character size is unspecified. If ISTRIP is set, bits beyond the 7 low-order bits shall be zero but bits beyond CSIZE are unspecified when read. CSIZE does not include the parity bit. If CSTOPB is set, two stop bits are used; otherwise, one stop bit.

**Relevance to meiksh:** The `stty` builtin must support setting cs5, cs6, cs7, cs8, and cstopb/-cstopb. The shell must correctly map these `stty` operands to the appropriate CSIZE and CSTOPB flag values.

**Spec reference:** POSIX.1-2024, §11.2.4 Control Modes (tag_11_02_04), `basedefs/V1_chap11.html` line 1066.
