## Extracted Text

> The standard input shall be used only if one of the following is true:: The script executes one or more commands that require input from standard input (such as a read command that does not redirect its input).

## Context

**Behavior:** Even when the shell is reading commands from a script file (not stdin), individual commands within that script may still read from standard input. For example, the `read` builtin without input redirection reads from stdin.

**Conditions:** This condition means stdin is "used" by the script's commands, not by the shell's command parser. The shell must not interfere with or consume stdin data that is intended for child commands.

**Applies to:** Commands like `read`, `cat` (without file args), or any command that reads stdin. The shell itself is reading its command stream from a file, but stdin is still available to child processes.

**Spec reference:** STDIN section of `utilities/sh.html`, line 140–142. See also `utilities/read.html` for the `read` builtin.
