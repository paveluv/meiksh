## Extracted Text

> When a file that does not exist is created, the following features defined in the System Interfaces volume of POSIX.1-2024 shall apply unless the utility or function description states otherwise:: The group ID of the file shall be set to the effective group ID of the calling process or the group ID of the directory in which the file is being created.

## Context

When a new file is created, the group ID shall be set to either the effective group ID of the calling process or the group ID of the directory in which the file is created (implementation-defined which one). For `meiksh`, the group ownership of files created by redirection depends on the underlying OS behavior (BSD-style inherits directory group, System V-style uses effective group ID). The shell does not control this directly. See POSIX Utilities Chapter 1, Section 1.1.1.4 item 2 (utilities/V3_chap01.html line 90).
