## Extracted Text

> POSIX.1-2024 places only the following requirements on the encoded values of the characters in the portable character set:: A null character, NUL, which has all bits set to zero, shall be in the set of characters.

## Context

The NUL character (byte value 0x00) must exist in the character set. For the shell:

1. **String termination**: Shell variables and arguments are NUL-terminated C strings internally. The shell cannot store NUL characters within variable values — NUL terminates the string.
2. **Input processing**: NUL bytes in input data are typically ignored or cause truncation. The `read` builtin, command substitution `$()`, and here-documents cannot preserve NUL bytes.
3. **Pathname handling**: Pathnames cannot contain NUL bytes (NUL is the only character forbidden in POSIX filenames besides `/` in components).

The shell implementation must be aware that NUL is the string terminator and cannot appear as data within shell strings.
