## Extracted Text

> The following environment variables shall affect the execution of cd:: A <colon>-separated list of pathnames that refer to directories. The cd utility shall use this list in its attempt to change the directory, as described in the DESCRIPTION. An empty string in place of a directory pathname represents the current directory. If CDPATH is not set, it shall be treated as if it were an empty string.

## Context

**Behavior:** CDPATH semantics for `cd`:

1. **Format:** Colon-separated list of directory pathnames (e.g., `:/home/user:/usr/local`).
2. **Usage:** When resolving a relative directory operand (that doesn't start with `.` or `..`), cd iterates through each CDPATH entry. For each non-null entry, it tests if `entry/directory` names an existing directory. For null (empty) entries, it tests `./directory`.
3. **Empty entry:** An empty string between colons (or at start/end) represents the current directory (`.`).
4. **Unset CDPATH:** If CDPATH is not set at all, it must be treated as if it were an empty string — meaning only the current directory is searched, equivalent to no CDPATH search.

**Conditions:** This applies during step 5 of the cd algorithm, only for relative operands not starting with `.` or `..`.

**Spec reference:** POSIX.1-2024, `utilities/cd.html`, section ENVIRONMENT VARIABLES (tag_20_14_08), lines 179-181.
