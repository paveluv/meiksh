## Extracted Text

> A string assigned to special parameter 0 when executing the commands in command_string. If command_name is not specified, special parameter 0 shall be set to the value of the first argument passed to sh from its parent (for example, argv[0] for a C program), which is normally a pathname used to execute the sh utility.

## Context

When `sh -c command_string command_name [argument...]` is invoked, the `command_name` operand is a string that is assigned to special parameter `$0` for the duration of executing commands in `command_string`.

- If `command_name` **is specified**: `$0` is set to `command_name`.
- If `command_name` **is not specified** (i.e., `sh -c command_string` with no further operands): `$0` must be set to the value of the first argument passed to `sh` from its parent process (typically `argv[0]`, the pathname used to execute `sh`).

Example: `sh -c 'echo $0' myscript` outputs `myscript`.

See utilities/sh.html#tag_20_110_05 and 2.5.2 Special Parameters in utilities/V3_chap02.html#tag_19_05_02.
