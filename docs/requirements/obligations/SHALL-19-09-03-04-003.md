## Extracted Text

> If job control is enabled, the AND-OR lists shall form all or part of a foreground job that can be controlled as described in 2.11 Job Control.

## Context

**Behavior:** If job control is enabled, sequential AND-OR lists MUST form all or part of a foreground job that can be controlled as described in 2.11 Job Control. This means the shell places them in the terminal's foreground process group and they can be suspended/resumed with job control signals (SIGTSTP, SIGCONT).

**Applies to:** Sequential AND-OR lists when job control is enabled (`set -m`). In non-interactive shells or when job control is disabled, this does not apply.

**Implementation detail:** When job control is active, sequential lists run in the foreground process group. The shell must manage process groups correctly so that `Ctrl+Z` can suspend the foreground job and `fg`/`bg` can resume it.

**Spec reference:** Section 2.9.3.2, lines 2186-2187 in `utilities/V3_chap02.html`. Cross-ref: 2.11 Job Control.
