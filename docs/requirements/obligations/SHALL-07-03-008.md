## Extracted Text

> The comment character shall default to the <number-sign> ('#'). Blank lines and lines containing the <comment character> in the first position shall be ignored.

## Context

**Behavior:** In locale definition files, the default comment character is `#`. Blank lines and lines starting with the comment character are ignored.

**Shell relevance:** The shell does not parse locale definition files. This is `localedef` input syntax. No shell implementation action needed.
