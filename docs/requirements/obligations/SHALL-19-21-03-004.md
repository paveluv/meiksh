## Extracted Text

> If the exec command fails and the shell does not exit, any redirections associated with the exec command that were successfully made shall take effect in the current shell execution environment.

## Context

**Behavior:** If `exec` fails and the shell does **not** exit (interactive shell case), any redirections associated with the `exec` command that were **successfully made** must take effect in the current shell execution environment. Partially completed redirections persist even though the exec itself failed.

**Example:** `exec nonexistent_cmd >out.txt` — if `nonexistent_cmd` is not found but `>out.txt` was successfully opened, the stdout redirection to `out.txt` persists in the current shell.

**Applies to:** Interactive shell (where exec failure doesn't cause exit).
