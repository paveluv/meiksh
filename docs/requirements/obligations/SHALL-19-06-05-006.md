## Extracted Text

> If the IFS variable is unset, then for the purposes of this section, but without altering the value of the variable, its value shall be considered to contain the three single-byte characters <space>, <tab>, and <newline> from the portable character set, all of which are IFS white-space characters.

## Context

**Behavior:** When IFS is **unset** (not merely empty), the shell must behave for field splitting purposes as if IFS were set to `" \t\n"` (space, tab, newline). This is NOT the same as setting IFS — the variable remains unset, and `${IFS-default}` would still substitute `default`. This default only applies to the field splitting algorithm.

**Key distinction from `IFS=""`:** When IFS is empty, no field splitting occurs (SHALL-19-06-05-003). When IFS is unset, field splitting occurs using the default space/tab/newline delimiters.

**All three default characters are IFS white space:** They behave as coalescing field terminators (leading/trailing ones are ignored, runs of them count as one delimiter).

**Spec reference:** See 2.6.5 Field Splitting in utilities/V3_chap02.html, lines 1234–1236.
