## Extracted Text

> The read utility shall read a single logical line from standard input into one or more shell variables.

## Context

**Behavior required:** The `read` builtin must read exactly one logical line from standard input and split it into one or more shell variables specified as operands.

**Key concepts:**
- "Logical line": A sequence of bytes terminated by the line delimiter (newline by default, or the delimiter specified with `-d`). When backslash-newline continuation is active (no `-r`), multiple physical lines joined by `\<newline>` form one logical line.
- "Standard input": `read` always reads from fd 0.
- "One or more shell variables": At least one `var` operand must be specified.

**Condition:** Every invocation of the `read` builtin.

**Implementation detail:** The `read` builtin reads byte-by-byte from stdin until it encounters the line delimiter or EOF. It then processes the line (backslash handling, field splitting, variable assignment) per the subsequent obligations.
