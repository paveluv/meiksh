## Extracted Text

> The following rules shall be used to construct EREs matching multiple characters from EREs matching a single character:: When an ERE matching a single character or an ERE enclosed in parentheses is followed by the special character <asterisk> ('*'), together with that <asterisk> it shall match what zero or more consecutive occurrences of the ERE would match. For example, the ERE "b*c" matches the first character in the string "cabbbcde", and the ERE "b*cd" matches the third to seventh characters in the string "cabbbcdebbbbbbcdbc". And, "[ab]*" and "[ab][ab]" are equivalent when matching the string "ab".

## Context

The `*` operator must match zero or more consecutive occurrences of the preceding single-character ERE or parenthesized ERE. It applies to the immediately preceding atom. The match is greedy (leftmost longest) by default. Examples: `b*c` matches "c" in "cabbbcde" (zero `b`s followed by `c`); `b*cd` matches "bbbcd" in "cabbbcdebbbbbbcdbc". The `*` operator includes the possibility of matching zero occurrences (the empty string for the repeated part), which distinguishes it from `+`.
