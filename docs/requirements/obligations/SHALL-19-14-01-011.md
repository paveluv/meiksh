## Extracted Text

> When unquoted, unescaped, and not inside a bracket expression, the following three characters shall have special meaning in the specification of patterns:: An <asterisk> is a pattern that shall match multiple characters, as described in 2.14.2 Patterns Matching Multiple Characters.

## Context

**`*` matches multiple characters as described in 2.14.2.**

An unquoted, unescaped `*` not inside a bracket expression **shall** match multiple characters. The precise semantics are in 2.14.2 Patterns Matching Multiple Characters (utilities/V3_chap02.html):
- `*` alone matches any string including the empty string (SHALL-19-14-02-001/004)
- `*` uses greedy matching — it matches the greatest possible number of characters that still allows the rest of the pattern to match (SHALL-19-14-02-003/006)
- In filename expansion: `*` does not match `/` and does not match a leading `.` (see 2.14.3)

See SHALL-19-14-02-* obligations for the detailed multi-character matching rules.
