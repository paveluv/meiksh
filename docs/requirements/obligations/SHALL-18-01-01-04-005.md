## Extracted Text

> When a file that does not exist is created, the following features defined in the System Interfaces volume of POSIX.1-2024 shall apply unless the utility or function description states otherwise:: If the file is a regular file, the permission bits of the file shall be set to: S_IROTH | S_IWOTH | S_IRGRP | S_IWGRP | S_IRUSR | S_IWUSR (see the description of File Modes in XBD 14. Headers , <sys/stat.h>) except that the bits specified by the file mode creation mask of the process shall be cleared. If the file is a directory, the permission bits shall be set to: S_IRWXU | S_IRWXG | S_IRWXO except that the bits specified by the file mode creation mask of the process shall be cleared.

## Context

When a new file is created: if it is a regular file, permission bits shall be set to `S_IROTH | S_IWOTH | S_IRGRP | S_IWGRP | S_IRUSR | S_IWUSR` (0666) except that bits specified by the process's file mode creation mask (umask) are cleared. If it is a directory, permission bits shall be set to `S_IRWXU | S_IRWXG | S_IRWXO` (0777) except that umask bits are cleared. For `meiksh`, when performing output redirection (`> file`), the shell calls `open()` with mode 0666, and the OS applies the umask. The shell's `umask` builtin controls the file mode creation mask. See POSIX Utilities Chapter 1, Section 1.1.1.4 item 3 (utilities/V3_chap01.html lines 92–98).
