## Extracted Text

> The following environment variables shall affect the execution of sh:: This variable shall represent an absolute pathname of the current working directory. Assignments to this variable may be ignored.

## Context

**Behavior:** PWD has two requirements:

1. **Shall represent an absolute pathname:** PWD shall always contain an absolute pathname of the current working directory. The shell must maintain this invariant — updating PWD whenever the current directory changes (e.g., via the `cd` built-in).
2. **Assignments may be ignored:** User assignments to PWD (e.g., `PWD=/tmp`) may be ignored by the shell. The implementation is permitted to discard or override such assignments to maintain the correct current working directory value.

**Applies to:** All shell modes (interactive and non-interactive).

**Details:** The shell should set PWD at startup from the inherited environment (if valid) or by calling `getcwd()`. The `cd` built-in is responsible for updating PWD. The value of PWD may contain symbolic link components (logical path) unless `cd -P` was used.

**Error handling:** If the shell cannot determine the current working directory, the behavior for PWD is unspecified.
