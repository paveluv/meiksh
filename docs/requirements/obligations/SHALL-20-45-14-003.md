## Extracted Text

> If fg does not move a job into the foreground, the following exit value shall be returned:: An error occurred.

## Context

This is the description for exit status >0 of `fg`: "An error occurred." This applies when `fg` cannot move a job into the foreground. Possible error conditions include: invalid or unrecognized job_id, no jobs in the job table, job control not enabled (set +m), or the job has already terminated. The implementation must write a diagnostic message to stderr before returning the error exit status. Reference: POSIX.1-2024 §fg, EXIT STATUS (tag_20_45_14).
