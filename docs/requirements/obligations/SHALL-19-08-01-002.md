## Extracted Text

> Non-Interactive Shell: Shell language syntax error -- shall exit

## Context

A non-interactive shell SHALL exit when a shell language syntax error occurs. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required:**
- When the shell is non-interactive (e.g., running a script, `sh -c`, or stdin is not a terminal), a syntax error causes the shell to write a diagnostic to stderr and exit with a non-zero status.
- Syntax errors include: invalid tokens, unmatched quotes, missing required keywords (`do`/`done`, `then`/`fi`, etc.), invalid grammar constructs.
- A shell diagnostic message IS required for this error type.

**Exit status:** Typically 2 for syntax errors (common convention), though POSIX does not mandate a specific value beyond "non-zero".
