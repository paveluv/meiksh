## Extracted Text

> The initial values of all control characters are implementation-defined.

## Context

**Behavior:** The initial values of all special control characters in the `c_cc` array (VINTR, VQUIT, VERASE, VKILL, VEOF, VEOL, VSUSP, VSTART, VSTOP, VMIN, VTIME, etc.) are implementation-defined when a terminal device is first opened.

**Conditions:** Applies to the initial state of a terminal after `open()`. Common default values on most systems include: VINTR=^C, VQUIT=^\, VERASE=^? or ^H, VKILL=^U, VEOF=^D, VSUSP=^Z, VSTART=^Q, VSTOP=^S.

**Shell relevance:** When meiksh starts up on a terminal, it should not assume any particular control character values. The shell must call `tcgetattr()` to discover the current values. If the shell needs to restore terminal state on exit, it must save and restore these values. The `stty` builtin must be able to query and display these values. meiksh must document its chosen initial control character values as its implementation-defined behavior.

**Spec reference:** POSIX.1-2024, §11.2.6 Special Control Characters (tag_11_02_06), `basedefs/V1_chap11.html` line 1356.
