## Extracted Text

> The extended regular expression (ERE) notation and construction rules shall apply to utilities defined as using extended regular expressions; any exceptions to the following rules are noted in the descriptions of the specific utilities using EREs.

## Context

The shell's ERE engine (used by utilities such as `grep -E`, `awk`, and the `[[ string =~ ere ]]` conditional construct) must implement the ERE notation and construction rules defined in POSIX Section 9.4. All ERE rules from subsections 9.4.1 through 9.4.9 apply by default to any utility documented as using EREs. Deviations from these rules are only permitted where the specific utility's description explicitly notes an exception. The implementation must not invent additional ERE semantics beyond what is specified, and must not omit any required ERE behavior unless a per-utility exception applies.
