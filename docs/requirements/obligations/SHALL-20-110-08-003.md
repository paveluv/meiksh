## Extracted Text

> The following environment variables shall affect the execution of sh:: [UP] This variable, when and only when an interactive shell is invoked, shall be subjected to parameter expansion (see 2.6.2 Parameter Expansion) by the shell, and the resulting value shall be used as a pathname of a file containing shell commands to execute in the current environment. The file need not be executable. If the expanded value of ENV is not an absolute pathname, the results are unspecified. ENV shall be ignored if the real and effective user IDs or real and effective group IDs of the process are different. The file specified by ENV need not be processed if the file can be written by any user other than the user identified by the real (and effective) user ID of the shell process.

## Context

**Behavior:** When an interactive shell is invoked, the shell shall:
1. Subject the value of ENV to parameter expansion (see 2.6.2 Parameter Expansion in `utilities/V3_chap02.html#tag_19_06_02`).
2. Use the resulting expanded value as a pathname of a file containing shell commands.
3. Execute those commands in the current shell environment (not a subshell).

**Conditions:**
- This applies **only** when an interactive shell is invoked. Non-interactive shells shall not process ENV.
- The file need not be executable (no execute permission required; only read permission).
- If the expanded value of ENV is not an absolute pathname, behavior is unspecified — the implementation may choose any behavior.
- **Security restriction:** ENV shall be ignored entirely if the real and effective user IDs differ OR the real and effective group IDs differ (i.e., setuid/setgid invocations).
- **Optional security:** The file specified by ENV need not be processed if it can be written by any user other than the user identified by the real (and effective) user ID of the shell process.

**Error handling:** No specific error behavior is mandated if the ENV file does not exist or cannot be read — the shell simply does not source it.

**Feature gate:** This is a [UP] (User Portability) option feature.
