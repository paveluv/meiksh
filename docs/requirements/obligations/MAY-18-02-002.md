## Extracted Text

> It is not guaranteed that the application can actually reach the specified limit of an implementation in any given case, or at all, as a lack of virtual memory or other resources may prevent this. The limit value indicates only that the implementation does not specifically impose any arbitrary, more restrictive limit.

## Context

Reaching the specified implementation limit is not guaranteed in every case. Resource constraints (virtual memory, disk space, process table limits, etc.) may prevent the application from reaching the stated limit. The limit value only indicates that the implementation does not impose an arbitrary restriction more restrictive than the stated value. The shell need not guarantee that the full limit is always reachable — only that no artificial ceiling below the limit exists. Reference: XCU 1.2.
