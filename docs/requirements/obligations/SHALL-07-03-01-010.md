## Extracted Text

> Define characters to be classified as control characters. In the POSIX locale, no characters in classes alpha or print shall be included. In a locale definition file, no character specified for the keywords upper, lower, alpha, digit, punct, graph, print, or xdigit shall be specified.

## Context

**Behavior:** The `cntrl` keyword defines the set of control characters:
- In the POSIX locale: no overlap with `alpha` or `print` classes.
- Constraint: No character in `cntrl` may also be in `upper`, `lower`, `alpha`, `digit`, `punct`, `graph`, `print`, or `xdigit`.
- In practice, in the POSIX locale this includes bytes 0x00-0x1F and 0x7F (DEL).

**Shell relevance:** The shell uses the `cntrl` class for:
- `[[:cntrl:]]` bracket expression in pattern matching.
- Control characters affect terminal handling in interactive mode.
