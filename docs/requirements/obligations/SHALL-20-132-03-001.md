## Context

**Behavior:** The `umask` builtin shall set the file mode creation mask (umask) of the current shell execution environment to the value specified by the `mask` operand. The mask affects the initial permission bits of files subsequently created by the shell or its child processes.

**Scope:** The change applies to the current shell execution environment as defined in Section 2.13. This means the umask value persists for the lifetime of the current shell unless changed again. All subsequent file creation operations (e.g., via redirection `>`, `touch`, `mkdir`) will have their permission bits modified by this mask.

**Mask semantics:** The file mode creation mask is bitwise-ANDed with the complement of the requested permissions when creating files. For example, a mask of 022 clears the group-write and other-write bits from new files.

**Subshell isolation:** When `umask` is called in a subshell `(umask 002)` or a separate utility execution environment (e.g., via `nohup umask ...` or `find . -exec umask ... \;`), changes do not propagate back to the parent shell environment.
