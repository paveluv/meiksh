## Extracted Text

> (Hyphen.) Expands to the current option flags (the single-letter option names concatenated into a string) as specified on invocation, by the set special built-in command, or implicitly by the shell. It is unspecified whether the -c and -s options are included in the expansion of "$-". The -i option shall be included in "$-" if the shell is interactive, regardless of whether it was specified on invocation.

## Context

This defines the expansion behavior of the special parameter `$-` (hyphen/minus).

**Required behavior:**
- `$-` must expand to a string of single-letter option flags that are currently enabled, concatenated together (e.g., `himBH`).
- Options come from three sources: (1) flags specified on the command line at invocation (`sh -e -x script.sh`), (2) flags toggled by the `set` special built-in (`set -x`), and (3) flags set implicitly by the shell.
- The `-i` option flag **must** be included in `$-` if the shell is interactive, regardless of whether `-i` was explicitly specified on invocation.
- Whether the `-c` and `-s` options are included in the expansion is unspecified (implementation choice).

**Conditions:** Applies in all execution contexts. `$-` is read-only; it cannot be assigned to directly.

**Implementation note:** The shell must maintain a set of active option flags and serialize them into a string for `$-` expansion. The order of letters in the string is not specified by POSIX.

See 2.5.2 "Special Parameters" (`-`) and the `set` special built-in in `utilities/V3_chap02.html`.
