## Extracted Text

> It is unspecified whether environment variables that were passed to the invoking shell when it was invoked itself, but were not used to initialize shell variables (see 2.5.3 Shell Variables) because they had invalid names, are included in the invoked utility's environment.

## Context

**Invalid-name environment variables: inheritance to child is unspecified.**

When the shell is invoked, it receives environment variables from its parent. Some may have names that are not valid shell variable names (e.g., names containing `=`, starting with digits, or containing characters other than `[A-Za-z0-9_]`). These invalid-name variables cannot be used to initialize shell variables (see 2.5.3 Shell Variables in utilities/V3_chap02.html).

It is **unspecified** whether these invalid-name environment variables are included when the shell constructs the environment for a child utility. The implementation may choose to:
- Pass them through to children (preserving the full inherited environment), or
- Drop them (only passing shell variables with the export attribute)

Either behavior is conforming. The simpler approach is usually to pass them through by preserving the original `environ` entries that weren't consumed as shell variables.
