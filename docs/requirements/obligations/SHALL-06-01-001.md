## Extracted Text

> Conforming implementations shall support one or more coded character sets. Each supported locale shall include the portable character set, which is the set of symbolic names for characters in Portable Character Set. This is used to describe characters within the text of POSIX.1-2024. The first eight entries in Portable Character Set and all characters in Non-Portable Control Characters are defined in the ISO/IEC 6429:1992 standard. The rest of the characters in Portable Character Set are defined in the ISO/IEC 10646-1:2020 standard.

## Context

The shell must operate correctly with at least one coded character set, and every locale the shell supports must include the portable character set. The portable character set includes all ASCII letters (A-Z, a-z), digits (0-9), and common punctuation/control characters that the shell uses as syntax characters (e.g., `$`, `|`, `&`, `;`, `(`, `)`, `<`, `>`, `{`, `}`, `!`, `#`, `*`, `?`, `[`, `]`, `~`, `=`, `%`, `@`, `^`, `+`, `-`, `,`, `.`, `/`, `\`, `'`, `"`, `` ` ``, space, tab, newline).

For the shell, this means that all shell syntax characters are guaranteed to be present and have defined representations in every supported locale. The shell can rely on the portable character set being available regardless of the locale setting.
