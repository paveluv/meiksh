## Extracted Text

> When used in circumstances where reserved words are recognized (described above), the following words may be recognized as reserved words, in which case the results are unspecified except as described below for time:

## Context

**Behavior:** In reserved-word-recognition positions, the following words MAY be recognized as reserved words, with unspecified results (except for `time`):

`[[`, `]]`, `function`, `namespace`, `select`, `time`

If the implementation recognizes any of these (except `time`) as reserved words, the resulting behavior is entirely unspecified—the implementation may do anything.

**Applies to:** All shell modes during parsing.

**Implementation choice:** For a strict POSIX shell, the safest approach is to NOT recognize these as reserved words (treat them as ordinary WORD tokens). However, the implementation MAY recognize them for extensions (e.g., `[[...]]` conditional expressions, `function` keyword for function definitions, `select` for menu loops). If any are recognized, the behavior should be documented.

**`time` exception:** The word `time` has specific defined behavior when recognized as a reserved word—see SHALL-19-04-007.

**Spec reference:** Section 2.4, lines 404-405 in `utilities/V3_chap02.html`.
