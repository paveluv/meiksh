## Extracted Text

> The <signal.h> header shall also define the following symbolic constants:: SA_NOCLDSTOP

## Context

The `<signal.h>` header shall define the symbolic constant `SA_NOCLDSTOP`. This flag is used in the `sa_flags` member of `struct sigaction` when installing a handler for `SIGCHLD`. When set, the system will not generate `SIGCHLD` when child processes stop (and, with XSI, when stopped children continue). For `meiksh`, this flag is relevant to job control: the shell typically wants to be notified when children stop (for `jobs`, `fg`, `bg` builtins), so it would generally NOT set `SA_NOCLDSTOP`. However, the shell must be aware of this flag's semantics. See POSIX `<signal.h>` DESCRIPTION (basedefs/signal.h.html line 516).
