## Extracted Text

> Implementations may allow other characters within alias names as an extension.

## Context

**Optional extension — direct shell relevance.** The standard alias name character set consists of characters from the portable filename character set (uppercase/lowercase letters, digits, period, underscore, hyphen) that are not purely numeric. Implementations MAY allow additional characters in alias names as an extension.

**Shell impact:** The `alias` builtin and alias expansion logic must validate alias names. The minimal conforming set is `[A-Za-z0-9._-]` (not purely digits). `meiksh` may choose to accept additional characters (e.g., `!`, `@`, `%`) but is not required to. The alias name validation function should be configurable or clearly document which extension characters are accepted.
