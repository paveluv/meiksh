## Extracted Text

> If the current character is an unquoted <blank>, any token containing the previous character is delimited and the current character shall be discarded.

## Context

**Behavior:** Token recognition Rule 7: If the current character is an unquoted blank (space or tab), any token containing the previous character is delimited, and the blank character itself is discarded (not included in any token).

**Applies to:** All shell modes during tokenization.

**Blank characters:** Space (0x20) and tab (0x09) are the blank characters. Newline is NOT a blank—it is an operator.

**Implementation detail:** When an unquoted blank is encountered:
- If a word or operator token is being built, it is delimited and emitted.
- The blank itself is consumed and discarded.
- Multiple consecutive blanks are each individually discarded (they don't create empty tokens because of the "empty token discard" rule in SHALL-19-03-003).

**Example:** `echo   hello` — the three spaces between `echo` and `hello` each trigger this rule. `echo` is delimited by the first space; the second and third spaces find no in-progress token and are discarded.

**Spec reference:** Section 2.3, rule 7, lines 294-295 in `utilities/V3_chap02.html`.
