## Extracted Text

> When determining whether a command name is a declaration utility, an implementation may use only lexical analysis. It is unspecified whether assignment context will be used if the command name would only become recognized as a declaration utility after word expansions.

## Context

**Behavior:** When determining whether a command name is a declaration utility (for the purpose of choosing assignment vs. regular expansion in step 2), the shell MAY use only lexical analysis — i.e., check the literal text before expansion. It is UNSPECIFIED whether assignment context applies if the command name only becomes a declaration utility after word expansion (e.g., `cmd=export; $cmd VAR=val`).

**Applies to:** Step 2 of simple command processing when deciding expansion mode for arguments.

**Implementation detail:** meiksh should determine declaration-utility status from the literal first word before expansion. This is simpler and is what all major shells do. If the first word is literally `export`, `readonly`, or another declaration utility, use assignment expansion for assignment-like arguments. If the word is a variable that expands to a declaration utility name, the behavior is implementation's choice — meiksh may treat it as a regular command.
