## Extracted Text

> Non-Interactive Shell: Other utility (not a special built-in) error -- shall not exit

## Context

A non-interactive shell SHALL NOT exit when a non-special-built-in utility encounters an error. See 2.8.1 Consequences of Shell Errors in utilities/V3_chap02.html.

**Behavior required:**
- When the shell is non-interactive and a regular (non-special) built-in or external utility encounters an error, the shell SHALL NOT exit.
- The shell continues executing the next command in the script.
- A shell diagnostic message is NOT required (note 3) — the utility itself writes its own diagnostic.

**What this covers:** Errors from regular built-ins (e.g., `cd` to non-existent directory, `read` errors) and external commands (e.g., `ls` on non-existent file). The shell's exit behavior is unaffected; only the command's exit status is set to non-zero.

**Contrast:** This differs from special built-in errors (SHALL-19-08-01-004) where the non-interactive shell DOES exit.
