## Extracted Text

> Specify that the shell is interactive; see below. An implementation may treat specifying the -i option as an error if the real user ID of the calling process does not equal the effective user ID or if the real group ID does not equal the effective group ID.

## Context

This is an **optional** (MAY) behavior related to the `-i` option. The implementation **may** treat specifying `-i` as an error if:

- The **real user ID** of the calling process does not equal the **effective user ID**, OR
- The **real group ID** does not equal the **effective group ID**

This is a security safeguard for setuid/setgid shell invocations. If the shell binary has the setuid or setgid bit set, an attacker could use `-i` to force interactive mode and potentially exploit the elevated privileges. The implementation may reject this.

If `meiksh` chooses to implement this check, it should call `getuid()`/`geteuid()` and `getgid()`/`getegid()` during option parsing and produce a diagnostic error if the IDs differ and `-i` is specified.
