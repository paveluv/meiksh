## Extracted Text

> When the -v or -V options are specified, the following exit values shall be returned:: The command_name could not be found or an error occurred.

## Context

**Behavior:** Exit status >0 for `command -v`/`-V` means the command_name could not be found in any category (not an alias, function, built-in, reserved word, or executable in PATH), or some other error occurred during the lookup. No output is written to stdout in the "not found" case for `-v`; for `-V` the behavior on not-found is not explicitly specified but typically an error message is produced.

**Spec reference:** POSIX.1-2024, `utilities/command.html`, section EXIT STATUS (tag_20_22_14), line 200.
