## Extracted Text

> A loop shall enclose a break or continue command if the loop lexically encloses the command. A loop lexically encloses a break or continue command if the command is:: Not in the body of a function whose function definition command (see 2.9.5 Function Definition Command) is contained in a compound-list associated with the loop.

## Context

**Behavior:** Third condition for lexical enclosure: the `break`/`continue` command must **not** be in the body of a function whose function definition command is contained in a compound-list associated with the loop.

**Example:** In `for i in 1 2 3; do f() { break; }; f; done`, the `break` inside function `f` does NOT break the `for` loop, because `break` is in a function body whose definition is within the loop. The function creates a boundary — `break` in a called function does not propagate to enclosing loops of the caller.

**Applies to:** All shell modes.
