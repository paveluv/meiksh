## Extracted Text

> The initial input control value after open() is implementation-defined.

## Context

**Behavior:** The initial value of the `c_iflag` (input control) field after `open()` on a terminal device is implementation-defined. POSIX does not mandate default values for input mode flags such as BRKINT, ICRNL, IGNBRK, IGNCR, IGNPAR, INLCR, INPCK, ISTRIP, IXANY, IXOFF, IXON, or PARMRK.

**Relevance to meiksh:** meiksh must document its chosen initial input control values for terminals it opens. When the shell starts in interactive mode and sets up the terminal, it should explicitly configure the input modes it requires rather than relying on any particular default.

**Spec reference:** POSIX.1-2024, §11.2.2 Input Modes (tag_11_02_02), `basedefs/V1_chap11.html` line 535.
