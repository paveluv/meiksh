## Extracted Text

> If IEXTEN is set, implementation-defined functions shall be recognized from the input data. It is implementation-defined how IEXTEN being set interacts with ICANON, ISIG, IXON, or IXOFF. If IEXTEN is not set, implementation-defined functions shall not be recognized and the corresponding input characters are processed as described for ICANON, ISIG, IXON, and IXOFF.

## Context

**Behavior:** When IEXTEN is set in `c_lflag`, implementation-defined functions shall be recognized from input data. How IEXTEN interacts with ICANON, ISIG, IXON, and IXOFF is implementation-defined. When IEXTEN is not set, implementation-defined functions shall not be recognized and the corresponding input characters are processed normally according to ICANON, ISIG, IXON, and IXOFF.

**Relevance to meiksh:** The `stty` builtin must support `iexten`/`-iexten`. meiksh must document what implementation-defined functions (if any) it recognizes when IEXTEN is set and how they interact with other flags. The shell's line editor should be aware that IEXTEN may cause certain input characters to be consumed by the terminal driver rather than delivered to the shell.

**Spec reference:** POSIX.1-2024, §11.2.5 Local Modes (tag_11_02_05), `basedefs/V1_chap11.html` line 1188.
