## Extracted Text

> [UP] Provide a list of pathnames and optional messages separated by <colon> characters. If this variable is set, the shell shall inform the user if any of the files named by the variable are created or if any of their modification times change. (See the preceding entry for MAIL for descriptions of mail arrival and user informing.) Each pathname can be followed by '%' and a string that shall be subjected to parameter expansion and written to standard error when the modification time changes. If a '%' character in the pathname is preceded by a <backslash>, it shall be treated as a literal '%' in the pathname. The default message is unspecified. The MAILPATH environment variable takes precedence over the MAIL variable.

## Context

**Behavior:** This is a duplicate extraction of the MAILPATH variable description (same source lines as SHALL-20-110-08-027). The full behavior specification is identical:

1. **Format:** Colon-separated list of `pathname[%message]` entries.
2. **Monitoring:** Inform user when files are created or modification times change.
3. **Custom messages:** `%message` is parameter-expanded and written to stderr.
4. **Escaping:** `\%` is a literal percent in the pathname.
5. **Default message:** Unspecified.
6. **Precedence:** MAILPATH takes precedence over MAIL.

**Feature gate:** [UP] (User Portability) option.
