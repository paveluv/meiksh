## Extracted Text

> Follow link. Unless otherwise specified, the symbolic link shall be followed as specified for pathname resolution, and the operation performed shall be as if the target of the symbolic link (after all resolution) had been named. If the target of the symbolic link does not exist, it shall be as if that nonexistent target had been named directly.

## Context

Duplicate extraction of SHALL-18-01-01-04-019: action code "FL" (Follow Link) — symlinks are followed per pathname resolution. The operation proceeds on the target. If the target doesn't exist, it's as if the nonexistent target were named directly. For `meiksh`, `open()` follows symlinks by default, so this is standard behavior for shell redirections. See POSIX Utilities Chapter 1, Section 1.1.1.4 action code FL (utilities/V3_chap01.html line 658).
