## Extracted Text

> When the shell is first invoked, the shell variable OPTIND shall be initialized to 1. Each time getopts is invoked, it shall place the value of the next option found in the parameter list in the shell variable specified by the name operand and the shell variable OPTIND shall be set as follows:

## Context

The shell must initialize `OPTIND` to 1 at shell startup. On each `getopts` invocation: (1) The next option character from the parameter list is stored in the shell variable named by the `name` operand. (2) `OPTIND` is updated to encode the parsing position for the next invocation. The specific `OPTIND` value depends on the parsing result — see companion obligations SHALL-20-53-03-003 through SHALL-20-53-03-008 for the three cases (successful parse with option-argument, end of options, and all other cases). Reference: POSIX.1-2024 §getopts, DESCRIPTION (tag_20_53_03).
