## Extracted Text

> For the purposes of this section, when a field is said to be delimited, then the candidate field, as generated below shall become an output field. When the algorithm transforms a candidate into an output field it shall be appended to the current list of output fields.

## Context

**Behavior:** This defines the terminology for the field splitting algorithm:
- A **candidate field** is the accumulator that collects bytes during the splitting process.
- When the algorithm says a field is "delimited" (a delimiter is encountered or certain conditions are met), the current candidate field becomes a finalized **output field**.
- Output fields are appended to the output list in order, maintaining sequential processing order.

**Implementation significance:** The splitting algorithm maintains a "candidate field" buffer. When a delimiter is found and the delimiting conditions are satisfied (per SHALL-19-06-05-012), the candidate is flushed to the output list and a new empty candidate begins.

**Spec reference:** See 2.6.5 Field Splitting in utilities/V3_chap02.html, lines 1247–1249.
