## Extracted Text

> The <collating-symbol> shall be a symbolic name, enclosed between angle brackets ('<' and '>'), and shall not duplicate any symbolic name in the current charmap file (if any), or any other symbolic name defined in this collation definition. A <collating-symbol> defined via this keyword is only recognized within the LC_COLLATE category.

## Context

**Behavior:** Collating symbol naming rules:
- Must be a symbolic name enclosed in angle brackets.
- Must be unique — no duplicates from the charmap file or other collation definitions.
- Scoped to LC_COLLATE only — not recognized in other locale categories.

**Shell relevance:** The shell does not parse locale definition files. Collating symbols are internal to the collation definition. No shell implementation action needed.

**Spec reference:** Section 7.3.2.2, lines 2470-2473 in `basedefs/V1_chap07.html`.
