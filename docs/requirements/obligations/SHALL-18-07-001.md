## Extracted Text

> As described in 2.9.1.4 Command Search and Execution, intrinsic utilities are not subject to a PATH search during command search and execution. The utilities named in Intrinsic Utilities shall be intrinsic utilities.

## Context

Intrinsic utilities (also called "special builtins" in some contexts, but here meaning utilities that are not subject to `PATH` search) **shall** be found and executed without searching `PATH`, as described in XCU 2.9.1.4 (Command Search and Execution). The following utilities **shall** be intrinsic: `alias`, `bg`, `cd`, `command`, `false`, `fc`, `fg`, `getopts`, `jobs`, `kill`, `newgrp`, `pwd`, `read`, `true`, `umask`, `unalias`, `wait`. These are always found before any `PATH` lookup. Unlike special builtins (which have additional error-handling semantics), intrinsic utilities have guaranteed availability without `PATH` but otherwise behave like regular commands. Reference: XCU 1.7, XCU 2.9.1.4, XCU Table "Intrinsic Utilities."
