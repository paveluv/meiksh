## Extracted Text

> When expanding the words in a command about to be executed that are used with variable assignments or redirections, it is unspecified whether the expansions are carried out in the current execution environment or in the environment of the command about to be executed.

## Context

This documents an unspecified behavior regarding the execution environment for certain expansions during command preparation.

**Unspecified behavior:**
- When expanding words in a command about to be executed that are used with **variable assignments** or **redirections** (as opposed to the command name and arguments), it is unspecified whether the expansions are carried out in:
  - The current shell execution environment, OR
  - The environment of the command about to be executed.

**Practical impact:**
- For a command like `VAR=$(cmd) someutility`, the expansion of `$(cmd)` for the assignment `VAR=...` could happen in either environment.
- For redirections like `someutility > ${file}`, the expansion of `${file}` could happen in either environment.
- This matters when the command to be executed is a special built-in (which runs in the current shell environment) vs. an external command (which runs in a child process).

**Implementation choice:** meiksh should document which environment it uses. Most shells perform all expansions in the current shell environment for simplicity and compatibility.

See 2.6 "Word Expansions" in `utilities/V3_chap02.html`.
