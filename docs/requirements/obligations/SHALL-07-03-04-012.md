## Extracted Text

> The operand is a string containing the symbol that shall be used as the decimal delimiter (radix character) in numeric, non-monetary formatted quantities. This keyword cannot be omitted and cannot be set to the empty string. In contexts where standards limit the decimal_point to a single byte, the result of specifying a multi-byte operand shall be unspecified.

## Context

The `decimal_point` value in LC_NUMERIC is mandatory and non-empty — every locale must define a decimal delimiter for non-monetary numbers. If a multi-byte character is used, behavior is unspecified in single-byte contexts. The shell is directly affected: the `printf` builtin must use this value when outputting floating-point numbers (`%f`, `%e`, `%g`). The shell's arithmetic parser always uses `'.'` (POSIX/C locale), but output formatting must honor the active LC_NUMERIC decimal point. In the POSIX locale, the decimal point is always `'.'`.

See: `basedefs/V1_chap07.html` §7.3.4 LC_NUMERIC.
