## Extracted Text

> If the -l option is specified:: For non-job-control background jobs (if supported), a field containing the process ID associated with the job shall be inserted before the <state> field. Also, more processes created to execute the job may be output on separate lines, using only the process ID and <command> fields.

## Context

**Behavior:** When `jobs -l` is specified and the shell supports non-job-control background jobs, the output for such jobs:
1. A process ID field (not process group ID) shall be inserted before the `<state>` field.
2. Additional processes may optionally be listed on separate lines with PID and command.

**Applicability:** Non-job-control background jobs only exist when job control is disabled (`set +m`). If the shell does not support background jobs without job control, this requirement does not apply.

**Spec reference:** POSIX.1-2024, `jobs` STDOUT, lines 176-179 of `utilities/jobs.html`.
