## Extracted Text

> The last process to close a terminal device file shall cause any output to be sent to the device and shall cause any input to be discarded. If HUPCL is set in the control modes and the communications port supports a disconnect function, the terminal device shall perform a disconnect.

## Context

**Behavior:** When the last process closes a terminal device file:
1. Any pending output shall be sent (flushed) to the device.
2. Any pending input shall be discarded.
3. If HUPCL is set in `c_cflag` and the port supports disconnect, the terminal shall perform a disconnect (hangup).

**Conditions:** Only applies when the **last** file descriptor referencing the terminal device is closed across all processes.

**Shell relevance:** When the shell exits and it holds the last reference to the controlling terminal, this causes the terminal to flush output and hang up. The HUPCL flag controls whether closing the terminal triggers a hangup signal to other processes. This is relevant for login shells where the terminal should disconnect when the shell exits. The shell doesn't directly control this behavior — it's handled by the terminal driver — but the shell should be aware of it for proper cleanup semantics.

**Spec reference:** POSIX.1-2024, §11.1.11 Closing a Terminal Device File (tag_11_01_11), `basedefs/V1_chap11.html` line 285.
