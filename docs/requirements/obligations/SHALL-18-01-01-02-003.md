## Extracted Text

> The following functionality of the fork() function defined in the System Interfaces volume of POSIX.1-2024 shall be available on all systems conforming to this volume of POSIX.1-2024:: A process shall be able to create a new process with all of the attributes referenced in 1.1.1.1 Process Attributes, determined according to the semantics of a call to the fork() function defined in the System Interfaces volume of POSIX.1-2024 followed by a call in the child process to one of the exec functions defined in the System Interfaces volume of POSIX.1-2024.

## Context

A process shall be able to create a new child process with all process attributes from Section 1.1.1.1 (Current Working Directory, File Mode Creation Mask, File Descriptors, Environment Variables, Real/Effective User/Group IDs, etc.) determined by the semantics of `fork()` followed by `exec()`. For `meiksh`, this is the fundamental mechanism for running external commands: the shell `fork()`s, the child inherits process attributes (including file descriptors for redirections, environment for exported variables, umask, working directory), then `exec()`s the target command. The shell must ensure proper setup of these attributes between `fork()` and `exec()`. See POSIX Utilities Chapter 1, Section 1.1.1.2 (utilities/V3_chap01.html lines 75–78).
