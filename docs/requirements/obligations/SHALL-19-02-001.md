## Context

**Characters that must be quoted to represent themselves.**

The application **shall** quote the following characters if they are to represent themselves (i.e., be treated as literal data rather than shell syntax):

```
|  &  ;  <  >  (  )  $  `  \  "  '  <space>  <tab>  <newline>
```

These are always special in the shell language. Without quoting, each triggers specific shell behavior (pipes, backgrounding, command separation, redirection, subshells, expansion, escaping, quoting, and word splitting).

Additionally, the following characters might need quoting under certain circumstances (they are sometimes special depending on context):

```
*  ?  [  ]  ^  -  !  #  ~  =  %  {  ,  }
```

The shell's tokenizer/parser must recognize all of the first set as syntactically significant when unquoted. The quoting mechanisms (backslash, single-quotes, double-quotes, dollar-single-quotes) and here-documents suppress this special treatment.

See 2.2 Quoting in utilities/V3_chap02.html.
