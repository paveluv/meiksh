> Arithmetic expansion provides a mechanism for evaluating an arithmetic expression and substituting its value. The format for arithmetic expansion shall be as follows:

## Context

**Behavior:** The shell must recognize the arithmetic expansion syntax `$(( expression ))`. When this form is encountered during word expansion, the shell must evaluate the arithmetic expression and substitute the resulting numeric value (as a decimal string) in place of the `$((...))` construct.

**Conditions:** Arithmetic expansion is recognized during word expansion in all contexts where expansions are performed — unquoted, double-quoted, and here-documents. It is NOT recognized inside single-quotes.

**Inputs/Outputs:**
- Input: `$((expression))` where `expression` is an arithmetic expression
- Output: the decimal string representation of the evaluated expression replaces `$((expression))`
- Example: `echo $((2+3))` → `5`
