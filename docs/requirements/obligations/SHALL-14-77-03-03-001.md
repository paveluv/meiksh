## Extracted Text

> The <termios.h> header shall define the following symbolic constants for use as flags in the c_oflag field. The c_oflag field specifies the system treatment of output.

## Context

**Behavior:** `<termios.h>` **shall** define symbolic constants for the `c_oflag` field (output modes). These include at minimum: OPOST (post-process output), and on XSI-conformant systems additional flags such as ONLCR, OCRNL, ONOCR, ONLRET, OFILL, NLDLY, CRDLY, TABDLY, BSDLY, VTDLY, FFDLY.

**Conditions:** Must be defined as bitmask constants. OPOST is the master switch for output processing.

**Shell relevance:** meiksh uses output mode flags for terminal configuration:
- OPOST: enables output post-processing (NL→CR-NL translation, tab expansion, etc.)
- ONLCR: map NL to CR-NL on output (essential for proper terminal display)
The `stty` builtin must support these flag names. The shell should ensure OPOST and ONLCR are set for normal interactive output.

**Spec reference:** POSIX.1-2024, `<termios.h>` Output Modes (tag_14_77_03_03), `basedefs/termios.h.html` lines 250–251.
