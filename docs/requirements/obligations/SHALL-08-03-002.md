## Extracted Text

> The system shall initialize this variable at the time of login to be a pathname of the user's home directory. See <pwd.h>.

## Context

**Behavior:** The system shall initialize `HOME` at login time to be a pathname of the user's home directory (from the password database, see `<pwd.h>`).

**Shell relevance:** The shell uses `HOME` for: (1) tilde expansion (`~` expands to `$HOME`), (2) the `cd` built-in with no arguments (changes to `$HOME`), (3) locating shell startup files (e.g., `~/.profile`). The shell must not modify `HOME` itself but must propagate it to child processes. If `HOME` is unset, tilde expansion for `~` produces unspecified results and `cd` with no arguments should produce an error.

**Spec reference:** Section 8.3 "Other Environment Variables", lines 673-674 in `basedefs/V1_chap08.html`.
