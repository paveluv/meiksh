## Extracted Text

> In a locale definition file, the operands are strings, formatted as indicated by the grammar in 7.4 Locale Definition Grammar. For some keywords, the strings can contain only integers. Keywords that are not provided, or integer keywords set to -1, can be used to indicate that the value is not available in the locale. String values set to the empty string ("") can be used to indicate that the value is available and is an empty string, or that the value is not available. The following keywords shall be recognized:: An integer set to a value indicating the positioning of the negative_sign for a negative formatted monetary quantity.

## Context

The `n_sign_posn` value is an integer (0-4) controlling where the negative sign appears relative to the currency symbol and value for negative monetary quantities, using the same interpretation as `p_sign_posn`. The shell does not directly format monetary values, but must correctly propagate the LC_MONETARY environment variable.

See: `basedefs/V1_chap07.html` §7.3.3 LC_MONETARY.
