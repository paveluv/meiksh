## Extracted Text

> [XSI] Specify a non-negative decimal integer, signal_number, representing the signal to be used instead of SIGTERM, as the sig argument in the effective call to kill(). The correspondence between integer values and the sig value used is shown in the following list. The effects of specifying any signal_number other than those listed below are undefined. 0 0 1 SIGHUP 2 SIGINT 3 SIGQUIT 6 SIGABRT 9 SIGKILL 14 SIGALRM 15 SIGTERM If the first argument is a negative integer, it shall be interpreted as a -signal_number option, not as a negative pid operand specifying a process group.

## Context

**Behavior:** Duplicate extraction of the `-signal_number` option description (same content as SHALL-20-64-04-010). XSI extension. The defined number-to-signal mapping: 0→0, 1→SIGHUP, 2→SIGINT, 3→SIGQUIT, 6→SIGABRT, 9→SIGKILL, 14→SIGALRM, 15→SIGTERM. Other numbers have undefined effects.

**Disambiguation:** A leading negative integer argument is always parsed as `-signal_number`, not as a negative pid. Use `--` to pass negative pids.

**Spec reference:** Section OPTIONS (tag_20_64_04) in `utilities/kill.html`, lines 88–114. XSI extension.
