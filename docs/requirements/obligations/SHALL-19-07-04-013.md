## Extracted Text

> When a here-document is read from a terminal device and the shell is interactive, it shall write the contents of the variable PS2, processed as described in 2.5.3 Shell Variables, to standard error before reading each line of input until the delimiter has been recognized.

## Context

Defines PS2 prompting for interactive here-document input from a terminal. See 2.7.4 Here-Document in utilities/V3_chap02.html and 2.5.3 Shell Variables (tag_19_05_03).

**Behavior required:**
- When a here-document is read from a terminal device AND the shell is interactive, the shell SHALL write the contents of the `PS2` variable to standard error before reading each line of input.
- This prompting continues until the delimiter line has been recognized.
- The `PS2` value is processed for prompt expansions as described in 2.5.3 Shell Variables (e.g., `!` for history number, etc., if the implementation supports prompt expansion).

**Conditions:**
- Only applies when the shell is interactive (not when reading from a script file or pipe).
- Only applies when input is from a terminal device (not redirected stdin).
- The prompt goes to stderr (fd 2), not stdout.

**Default:** `PS2` defaults to `"> "` (greater-than followed by a space).
