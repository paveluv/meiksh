## Extracted Text

> Once the input is empty, the candidate shall become an output field if and only if it is not empty.

## Context

**Behavior:** This is a duplicate extraction of the same termination rule documented in SHALL-19-06-05-013. When the field splitting algorithm has consumed all input bytes:
- Non-empty candidate → becomes the final output field
- Empty candidate → discarded (no output field produced)

This ensures no trailing empty field is generated from trailing IFS delimiters.

**Spec reference:** See 2.6.5 Field Splitting in utilities/V3_chap02.html, line 1285.
