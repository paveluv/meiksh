## Extracted Text

> Otherwise, the characters in the tilde-prefix following the <tilde> shall be treated as a possible login name from the user database. If these characters do not form a portable login name (see the description of the LOGNAME environment variable in XBD 8.3 Other Environment Variables), the results are unspecified.

## Context

**Behavior:** When the tilde-prefix is `~username` (i.e., `~` followed by one or more characters before the first `/` or end of word), the shell must treat those characters as a candidate login name. The login name must conform to the "portable login name" character set defined in XBD 8.3 (alphanumerics, period, underscore, hyphen; first character not hyphen).

**Conditions:** Applies whenever tilde expansion encounters a tilde-prefix containing characters after `~`. The characters following `~` up to the first unquoted `/` (or end of word, or `:` in assignment context) form the candidate login name.

**Inputs/Outputs:**
- Input: `~username` where `username` is one or more characters
- Output: if `username` is a portable login name, processing continues to SHALL-19-06-01-003 for the lookup; if not portable, behavior is unspecified

**Error handling:** If the characters do not form a portable login name (e.g., contain `$`, `'`, `"`, `\`, or `/`), the results are unspecified. The implementation may leave the tilde-prefix unexpanded or produce an error.

**Spec reference:** See 2.6.1 Tilde Expansion in utilities/V3_chap02.html, lines 741–743. For the definition of portable login name characters, see basedefs/V1_chap08.html#tag_08_03.
