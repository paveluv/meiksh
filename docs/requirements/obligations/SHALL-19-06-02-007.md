## Extracted Text

> Assign Default Values. If parameter is unset or null, quote removal shall be performed on the expansion of word and the result (or an empty string if word is omitted) shall be assigned to parameter. In all cases, the final value of parameter shall be substituted. Only variables, not positional parameters or special parameters, can be assigned in this way.

## Context

**Behavior:** The `${parameter:=word}` expansion (Assign Default Values):
- If `parameter` is **unset or null**: expand `word` (with tilde expansion, parameter expansion, command substitution, arithmetic expansion), then perform **quote removal** on the result, then **assign** that result to `parameter`. The final value of `parameter` (after assignment) is substituted into the expansion.
- If `parameter` is **set and non-null**: substitute the value of `parameter`; `word` is not expanded and no assignment occurs.
- In **all cases**, the expansion result is the final value of `parameter`.

**Restrictions:** Only ordinary shell variables can be assigned this way. Positional parameters (`$1`, `$2`, etc.) and special parameters (`$?`, `$#`, `$@`, etc.) cannot be targets of `:=`. If attempted, the shell should produce an error.

**No-colon variant:** `${parameter=word}` only tests for unset (not null) — see SHALL-19-06-02-010.

**Inputs/Outputs:**
- `${var:=hello}` → if `var` is unset or empty, assigns `var=hello` and substitutes `hello`
- `${1:=value}` → error: cannot assign to positional parameter
