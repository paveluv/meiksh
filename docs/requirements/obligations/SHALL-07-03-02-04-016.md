## Extracted Text

> This example is interpreted as follows:: The UNDEFINED means that all characters not specified in this definition (explicitly or via the ellipsis) shall be ignored when comparing primary weights, and have individual secondary weights based on their ordinal encoded values.

## Context

This is an illustrative example from the LC_COLLATE specification showing how UNDEFINED works in a multi-level collation definition. Characters not explicitly listed or covered by an ellipsis are given IGNORE as their primary weight (they are skipped during primary-level comparison) and individual secondary weights based on code point values (they can be distinguished at the secondary level). The shell is affected because this determines how unspecified characters behave in collation-based operations: they are effectively invisible at the primary comparison level but distinguishable at secondary levels, which affects bracket expression range matching and equivalence class membership.

See: `basedefs/V1_chap07.html` §7.3.2.4 Collation Order.
