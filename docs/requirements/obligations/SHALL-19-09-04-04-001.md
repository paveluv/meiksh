## Extracted Text

> If there is at least one item in the list of items, the exit status of a for command shall be the exit status of the last compound-list executed. If there are no items, the exit status shall be zero.

## Context

The `for` loop (`for name in word...; do compound-list; done`) iterates over a list of items. The exit status rules are:

1. **If there is at least one item in the list**: the exit status is the exit status of the **last `compound-list` executed** — i.e., the body of the last iteration.
2. **If there are no items** (the expansion of the word list produces zero fields): the exit status is **zero**, and the body is never executed.

Examples:
- `for x in a b c; do false; done` — exit status is 1 (body ran 3 times, last was `false`)
- `for x in; do echo hi; done` — exit status is 0 (no items, body never ran)
- Empty list via expansion: `for x in $(true); do echo hi; done` — if `$(true)` produces no output, exit status is 0

See 2.9.4.3 The for Loop in utilities/V3_chap02.html#tag_19_09_04_03.
