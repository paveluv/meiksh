## Extracted Text

> After a token has been categorized as type TOKEN (see 2.10.1 Shell Grammar Lexical Conventions), including (recursively) any token resulting from an alias substitution, the TOKEN shall be subject to alias substitution if all of the following conditions are true:: Either the TOKEN is being considered for alias substitution because it follows an alias substitution whose replacement value ended with a <blank> (see below) or the TOKEN could be parsed as the command name word of a simple command (see 2.10 Shell Grammar), based on this TOKEN and the tokens (if any) that preceded it, but ignoring whether any subsequent characters would allow that.

## Context

**Behavior:** Alias substitution condition 5 (position check): The TOKEN must be in a position where it could be a command name. Specifically, either:
- (a) The TOKEN follows an alias substitution whose replacement value ended with a blank (space/tab), triggering "next-word alias checking" (see SHALL-19-03-01-007), OR
- (b) The TOKEN could be parsed as the command name word of a simple command, based on the current TOKEN and any preceding tokens, ignoring what follows.

**"Command name position" means:** The first word of a simple command, after any preceding variable assignments and redirections. Per the grammar in 2.10, this is where a command name appears.

**Applies to:** All shell modes during alias processing.

**Implementation detail:** The parser/lexer must track whether the current token position is one where a command name could appear. This typically means: the first word after a newline/semicolon/pipe/`&&`/`||`/`(`/`{`/control-structure keyword, or after preceding assignment words. If the token is in argument position (not command name position), alias substitution does not apply unless triggered by a blank-ending alias.

**Example:** In `x=1 cmd arg`, only `cmd` is in command-name position. `arg` is not (unless `cmd`'s alias ended with a blank).

**Spec reference:** Section 2.3.1, lines 331-334 in `utilities/V3_chap02.html`. Cross-ref: 2.10 Shell Grammar.
