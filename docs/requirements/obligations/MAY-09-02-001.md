## Extracted Text

> The interfaces specified in POSIX.1-2024 do not permit the inclusion of a NUL character in an RE or in the string to be matched. If during the operation of a standard utility a NUL is included in the text designated to be matched, that NUL may designate the end of the text string for the purposes of matching.

## Context

**Behavior:** POSIX interfaces do not permit NUL characters in an RE or in the string being matched. If a NUL is included in the text designated to be matched during a standard utility operation, the NUL may designate the end of the text string for matching purposes.

**Shell relevance:** The shell operates on NUL-terminated C strings internally. Shell variables cannot contain NUL characters. Pattern matching in the shell (glob, `case`) will never encounter NUL within the strings being matched because the shell's string handling inherently terminates at NUL. No special handling is needed — this is naturally satisfied.
