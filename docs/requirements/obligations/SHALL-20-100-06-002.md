## Extracted Text

> If the -d delim option is specified and delim is the null string, the standard input shall contain zero or more bytes (which need not form valid characters).

## Context

**Behavior required:** When `-d ''` is specified (null string delimiter, meaning the null byte `\0` is the delimiter):
- The standard input may contain zero or more bytes.
- The bytes need not form valid characters.
- Null bytes ARE allowed in the input (they serve as the line delimiter).

**Condition:** When `-d ''` (empty string delim) is specified.

**Implementation detail:** When the delimiter is `\0`, the implementation must be able to handle null bytes in the input stream (since `\0` is the delimiter). This means the implementation cannot use C string functions that treat `\0` as a terminator for the input buffer — it must read byte-by-byte and compare against `\0`. This mode is useful for processing null-delimited input (e.g., from `find -print0`).
