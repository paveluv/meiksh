## Extracted Text

> If the command name is a function that is not a standard utility implemented as a function, variable assignments shall affect the current execution environment during the execution of the function. It is unspecified: Whether or not the variable assignments persist after the completion of the function Whether or not the variables gain the export attribute during the execution of the function Whether or not export attributes gained as a result of the variable assignments persist after the completion of the function (if variable assignments persist after the completion of the function)

## Context

**Behavior:** Standalone restatement of SHALL-19-09-01-02-006. For user-defined functions, prefix variable assignments MUST affect the current environment during execution. Persistence after return, export attribute behavior, and export attribute persistence are all UNSPECIFIED.

**Applies to:** `VAR=value user_function` commands.

**Implementation detail:** See SHALL-19-09-01-02-006.

**Spec reference:** Section 2.9.1.2, lines 1787-1792 in `utilities/V3_chap02.html`.
