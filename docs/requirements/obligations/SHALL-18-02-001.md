## Extracted Text

> The values specified in Utility Limit Minimum Values represent the lowest values conforming implementations shall provide and, consequently, the largest values on which an application can rely without further enquiries, as described below. These values shall be accessible to applications via the getconf utility (see getconf).

## Context

The shell **shall** respect the utility limit minimum values defined in XCU Table "Utility Limit Minimum Values." These represent the smallest values a conforming implementation must provide. Key limits include: `{LINE_MAX}` (minimum 2048 bytes — maximum line length for input), `{EXPR_NEST_MAX}`, and others. These values **shall** be accessible via the `getconf` utility. The shell must ensure its internal processing (line buffering, argument handling, etc.) supports at least these minimum values. Reference: XCU 1.2, XCU Table "Utility Limit Minimum Values."
