## Extracted Text

> When getopts successfully parses an option that takes an option-argument (that is, a character followed by <colon> in optstring, and exit status is 0), the value of OPTIND shall be the integer index of the next element of the parameter list (if any; see OPERANDS below) to be searched for an option character. Index 1 identifies the first element of the parameter list.

## Context

This obligation (duplicate extraction of the OPTIND-after-option-argument rule) specifies that after successfully consuming an option WITH its option-argument, `OPTIND` must point to the next parameter element (1-based index). If the option was `-b` and the argument was in the next parameter (`-b value`), OPTIND advances past both. If the option-argument was concatenated (`-bvalue`), OPTIND advances past just that parameter. The key invariant: OPTIND always points to the next parameter to examine, and index 1 is the first element. Reference: POSIX.1-2024 §getopts, DESCRIPTION (tag_20_53_03).
