## Extracted Text

> The positional parameters ($1, $2, and so on) shall be set to arguments, if any.

## Context

When `sh` is invoked with operands that include `argument` values (the remaining arguments after `command_file` or after `command_name` in `-c` mode), the shell must set the positional parameters `$1`, `$2`, etc. to those argument values in order.

For `sh script.sh arg1 arg2`: `$1` is `arg1`, `$2` is `arg2`.
For `sh -c 'echo $1' myname arg1 arg2`: `$0` is `myname`, `$1` is `arg1`, `$2` is `arg2`.

If no argument operands are provided, the positional parameters are empty and `$#` is 0.

See utilities/sh.html#tag_20_110_05 and 2.5.2 Special Parameters in utilities/V3_chap02.html#tag_19_05_02.
