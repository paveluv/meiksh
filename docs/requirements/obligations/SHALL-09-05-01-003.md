## Extracted Text

> The following tokens are processed (in addition to those string constants shown in the grammar):: For basic regular expressions, one of the following special characters: . Anywhere except inside bracket expressions \ Anywhere except inside bracket expressions [ Anywhere except inside bracket expressions ^ When used as an anchor (see 9.3.8 BRE Expression Anchoring) $ When used as an anchor * Anywhere except first in an entire RE, anywhere in a bracket expression, directly following "\(", directly following an anchoring '^' For extended regular expressions, shall be one of the following special characters found anywhere except inside bracket expressions: ^ . [ $ ( ) | * + ? { \ The close-parenthesis shall be considered special in this context only if matched with a preceding open-parenthesis.

## Context

The `SPEC_CHAR` token defines which characters are special in BRE and ERE modes. For BREs: `.` (special outside bracket expressions), `\` (special outside bracket expressions), `[` (special outside bracket expressions), `^` (special as anchor), `$` (special as anchor), `*` (special everywhere except: first in RE, inside brackets, after `\(`, after anchoring `^`). For EREs: all of `^`, `.`, `[`, `$`, `(`, `)`, `|`, `*`, `+`, `?`, `{`, `\` are special outside bracket expressions. The `)` is only special when matched with a preceding `(`. The RE lexer must classify these characters as SPEC_CHAR in the appropriate contexts and as ordinary characters elsewhere.
