## Extracted Text

> For each pid operand, the kill utility shall perform actions equivalent to the kill() function defined in the System Interfaces volume of POSIX.1-2024 called with the following arguments:: The sig argument is the value specified by the -s option, -signal_number option, or the -signal_name option, or by SIGTERM, if none of these options is specified.

## Context

**Behavior:** The signal to send **shall** be determined as follows (in order of precedence):
1. If `-s signal_name` is specified: use that signal (e.g., `-s KILL`, `-s HUP`)
2. If `-signal_number` is specified: use that signal number (e.g., `-9`, `-15`)
3. If `-signal_name` is specified (XSI form): use that signal (e.g., `-KILL`, `-HUP`)
4. If none of the above is specified: default to **SIGTERM**

**Conditions:** Only one signal specification method should be used. The signal name can be specified with or without the "SIG" prefix.

**Shell relevance:** The `kill` builtin must:
1. Parse `-s signal_name` option (required by base POSIX)
2. Parse `-signal_number` shorthand (e.g., `kill -9`)
3. Parse `-signal_name` shorthand (e.g., `kill -KILL`) [XSI]
4. Default to SIGTERM when no signal is specified
5. Map signal names to numbers using a signal name table
6. Accept both `SIGTERM` and `TERM` forms

**Spec reference:** POSIX.1-2024, `kill` DESCRIPTION (tag_20_64_03), `utilities/kill.html` lines 52–54.
