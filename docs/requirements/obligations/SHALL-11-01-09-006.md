## Extracted Text

> Certain characters have special functions on input or output or both. These functions are summarized as follows:: If the ISIG flag is set, receipt of the SUSP character shall cause a SIGTSTP signal to be sent to all processes in the foreground process group for which the terminal is the controlling terminal, and the SUSP character shall be discarded when processed.

## Context

**Behavior:** The SUSP special character (typically ^Z):
1. Recognized on input only when ISIG is set in `c_lflag`.
2. When received, a SIGTSTP signal shall be sent to **all** processes in the foreground process group for which this terminal is the controlling terminal.
3. The SUSP character itself shall be **discarded** when processed (not placed in input queue).

**Conditions:** ISIG must be set in `c_lflag`. The terminal must be the controlling terminal.

**Shell relevance:** This is the mechanism behind Ctrl-Z for job control. When a foreground command is running and the user presses ^Z, SIGTSTP is sent to the foreground process group, which by default stops (suspends) the processes. The shell (which is in its own process group and not in the foreground group at that point) does not receive the signal. The shell then detects that the foreground job has stopped (via waitpid/WUNTRACED) and moves itself back to the foreground, reporting the stopped job. The shell must support this for job control (`set -m`).

**Spec reference:** POSIX.1-2024, §11.1.9 Special Characters (tag_11_01_09), `basedefs/V1_chap11.html` line 246.
