## Extracted Text

> The function is named fname; the application shall ensure that it is a name (see XBD 3.216 Name) and that it is not the name of a special built-in utility. An implementation may allow other characters in a function name as an extension. The implementation shall maintain separate name spaces for functions and variables.

## Context

**Behavior:** Function definition naming rules and namespace separation:

1. **Name validation:** The function name (`fname`) must be a valid Name as defined in XBD 3.216 (starts with `[A-Za-z_]`, followed by `[A-Za-z0-9_]*`). The shell must reject function names that are names of special built-in utilities (`break`, `continue`, `:`, `.`, `eval`, `exec`, `exit`, `export`, `readonly`, `return`, `set`, `shift`, `times`, `trap`, `unset`).
2. **Extension:** An implementation **may** allow other characters in function names as an extension (but meiksh should document if it does).
3. **Separate namespaces:** The shell must maintain separate namespaces for functions and variables. A function named `foo` and a variable named `foo` must coexist independently. Unsetting one must not affect the other (unless using `unset -f` vs `unset -v`).

**Applies to:** All shell modes.
