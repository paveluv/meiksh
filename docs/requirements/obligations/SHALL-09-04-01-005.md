## Extracted Text

> A bracket expression shall match a single character or a single collating element. An ERE matching a single character enclosed in parentheses shall match the same as the ERE without parentheses would have matched.

## Context

Two rules are stated here: (1) A bracket expression (`[...]`) must match exactly one character or one multi-character collating element from the set defined by the bracket expression. It never matches more or fewer than one element. (2) Enclosing a single-character ERE in parentheses does not alter what it matches. For example, `(a)` matches the same strings as `a`, and `(.)` matches the same strings as `.`. Parentheses only add grouping/subexpression semantics; they do not change the matching behavior of the enclosed atom. The ERE engine must ensure bracket expressions are always single-element matchers and parenthesized single-character EREs are equivalent to their unparenthesized form.
