## Extracted Text

> A contiguous region of bytes within a file, all having the value of zero. Not all bytes with the value zero need belong to a hole; however, all seekable files shall have a virtual hole starting at the current size of the file. A hole is typically created via truncate(), or if an lseek() call has been made to position beyond the end of a file and data subsequently written at that point, although it is up to the implementation to define when sparse files can be created and with what granularity for the size of holes.

## Context

**Definition only — no shell relevance.** Defines "hole" as a contiguous region of zero bytes in a file, typically created by `truncate()` or writing past EOF after `lseek()`. All seekable files SHALL have a virtual hole starting at the current file size. Hole granularity is implementation-defined.

**Shell impact:** None directly. The shell may create files with holes via redirections (e.g., `exec 3<>file` followed by seeking, though the shell has no native seek command). Sparse file handling is an OS concern. No action required.
