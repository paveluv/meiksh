## Extracted Text

> All file system operations that read or search a directory or that modify the contents of a directory (for example creating, unlinking, or renaming a file) shall operate atomically. That is, each operation shall either have its entire effect and succeed, or shall not affect the file system and shall fail. Furthermore, these operations shall be serializable; that is, the state of the file system and of the results of each operation shall always be values that would be obtained if the operations were executed one after the other.

## Context

**Definition — low shell relevance.** States that all directory read/search/modify operations (create, unlink, rename) SHALL be atomic and serializable. Each operation either fully succeeds or has no effect.

**Shell impact:** The shell relies on this atomicity guarantee when performing filename expansion (globbing reads directories), `cd` (searches directories), and command lookup in `$PATH` (reads directories). The shell does not need to implement atomicity — it is guaranteed by the filesystem. This means the shell can trust that directory listings are consistent during globbing.

**Spec reference:** basedefs/V1_chap04.html, section 4.4.
