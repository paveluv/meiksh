## Context

**Behavior:** `trap` with no operands (or `-p` only) outputs trap information:

1. **Without `-p`:** Writes commands for conditions **not in default state** (including signals ignored on entry to a non-interactive shell).
2. **With `-p`:** Writes commands for **all** conditions, except it is unspecified whether SIGKILL/SIGSTOP are included.
3. **In a subshell** (when the single-trap optimization is not applied and no trap commands with operands have been executed since entry): the output must list the trap commands from **before** the subshell was entered (parent's traps).
4. Otherwise, the output lists the **currently** associated commands.
5. The format is: `trap -- action condition`

**Applies to:** All shell modes.
