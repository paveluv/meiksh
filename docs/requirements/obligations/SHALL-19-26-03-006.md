## Extracted Text

> Set the export attribute for all variable assignments. When this option is on, whenever a value is assigned to a variable in the current shell execution environment, the export attribute shall be set for the variable. This applies to all forms of assignment, including those made as a side-effect of variable expansions or arithmetic expansions, and those made as a result of the operation of the cd, getopts, or read utilities. Note: As discussed in 2.9.1 Simple Commands, not all variable assignments happen in the current execution environment. When an assignment happens in a separate execution environment the export attribute is still set for the variable, but that does not affect the current execution environment.

## Context

**Behavior:** The `-a` (`allexport`) option: when enabled, the export attribute is automatically set for **every** variable assignment in the current shell execution environment. This applies to all forms of assignment including:
- Direct assignments (`VAR=value`)
- Side-effect assignments from variable expansions (`${var:=default}`)
- Arithmetic expansion side effects
- Assignments from `cd`, `getopts`, or `read` utilities

**Important:** Assignments in separate execution environments (subshells for pipeline elements, etc.) also get the export attribute, but that does not affect the parent environment.

**Applies to:** All shell modes.
