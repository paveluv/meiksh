## Extracted Text

> Certain characters have special functions on input or output or both. These functions are summarized as follows:: Special character on input, which is recognized if the ISIG flag is set. Generates a SIGQUIT signal which is sent to all processes in the foreground process group for which the terminal is the controlling terminal. If ISIG is set, the QUIT character shall be discarded when processed.

## Context

**Behavior:** The QUIT special character (typically ^\):
1. Recognized on input only when ISIG is set in `c_lflag`.
2. When recognized, generates a SIGQUIT signal sent to **all** processes in the foreground process group for which this terminal is the controlling terminal.
3. The QUIT character itself shall be **discarded** when processed (not placed in the input queue).

**Conditions:** ISIG must be set in `c_lflag`. The terminal must be the controlling terminal for the affected process group.

**Shell relevance:** SIGQUIT typically causes a core dump in the default handler. The shell itself usually ignores SIGQUIT (interactive shells ignore it per POSIX shell spec). Foreground child processes that haven't changed their SIGQUIT disposition will receive it and typically terminate with a core dump. The shell must be aware of this signal for job status reporting (e.g., "Quit (core dumped)").

**Spec reference:** POSIX.1-2024, §11.1.9 Special Characters (tag_11_01_09), `basedefs/V1_chap11.html` line 224.
