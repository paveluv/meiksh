## Extracted Text

> Produce symbolic output.

## Context

**Behavior:** When the `-S` option is specified, `umask` shall produce symbolic output on stdout in the format `u=%s,g=%s,o=%s\n`. The three values are combinations of letters from the set {r, w, x}, where the presence of a letter indicates the corresponding bit is **clear** (allowed) in the file mode creation mask.

**Example:** If the mask is 022 (octal), the symbolic output is `u=rwx,g=rx,o=rx` because the write bit is set (blocked) for group and other, so 'w' is absent for those categories.

**Conditions:** This only affects the output when `umask` is invoked in query mode (no mask operand specified). When a mask operand is provided, no output is produced regardless of `-S`.
