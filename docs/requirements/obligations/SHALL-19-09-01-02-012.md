## Extracted Text

> If any of the variable assignments attempt to assign a value to a variable for which the readonly attribute is set in the current shell environment (regardless of whether the assignment is made in that environment), a variable assignment error shall occur. See 2.8.1 Consequences of Shell Errors for the consequences of these errors.

## Context

**Behavior:** If any variable assignment in a simple command attempts to assign to a variable with the `readonly` attribute set in the current shell environment, a variable assignment error MUST occur. This applies regardless of whether the assignment would have been made in the current environment or only exported temporarily. The consequences of such errors are defined in 2.8.1 Consequences of Shell Errors.

**Applies to:** All simple commands with variable assignments where the target variable has the readonly attribute. Example: `readonly X=1; X=2 cmd` must produce an error.

**Error handling:** Per 2.8.1, in a non-interactive shell, a variable assignment error from a special builtin causes the shell to exit. For other commands in a non-interactive shell, the command fails but the shell continues. In an interactive shell, the shell always continues.

**Implementation detail:** Before performing any assignment (step 4), check if the target variable is readonly. If so, emit an error to stderr and follow the 2.8.1 error handling rules.

**Spec reference:** Section 2.9.1.2, lines 1797-1799 in `utilities/V3_chap02.html`. Cross-ref: 2.8.1 Consequences of Shell Errors.
