## Extracted Text

> Certain characters have special functions on input or output or both. These functions are summarized as follows:: Special character on both input and output, which is recognized if the IXON (output control) or IXOFF (input control) flag is set. Can be used to suspend output temporarily. It is useful with CRT terminals to prevent output from disappearing before it can be read. If IXON is set, the STOP character shall be discarded when processed.

## Context

**Behavior:** The STOP special character (typically ^S):
1. Recognized on both input and output when IXON (output control) or IXOFF (input control) is set in `c_iflag`.
2. When recognized with IXON set, it suspends terminal output (XON/XOFF flow control).
3. If IXON is set, the STOP character shall be **discarded** when processed (not placed in input queue).

**Conditions:** IXON and/or IXOFF must be set in `c_iflag`.

**Shell relevance:** XON/XOFF flow control can interfere with the shell's line editor if ^S and ^Q are consumed by the terminal driver. Many terminals and terminal emulators have IXON enabled by default. The shell may want to disable IXON when entering its line editor so that ^S can be used for forward search (as in emacs mode). The `stty` builtin must support `ixon`/`-ixon` and `stop` character settings.

**Spec reference:** POSIX.1-2024, §11.1.9 Special Characters (tag_11_01_09), `basedefs/V1_chap11.html` line 250.
