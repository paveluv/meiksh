## Extracted Text

> Some functions, such as catopen() and those related to text domains, may reference various environment variables and a locale category of a specific locale to access files they need to use.

## Context

**Behavior (optional):** Functions like `catopen()` and text domain functions may reference environment variables and locale categories to locate message catalogs and other locale-dependent files. This is permissive — these functions are allowed but not required to do so.

**Shell relevance:** If the shell implements internationalized messages via `catopen()`/`catgets()` or `gettext()`, those functions may use `NLSPATH`, `LC_MESSAGES`, `LANG`, etc. to find message catalog files. This is an implementation choice. The ref to `functions/catopen.html` is a C API page — not a shell-relevant file to extract.
