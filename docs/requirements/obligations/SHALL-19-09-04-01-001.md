## Extracted Text

> The format for grouping commands is as follows:: Execute compound-list in a subshell environment; see 2.13 Shell Execution Environment. Variable assignments and built-in commands that affect the environment shall not remain in effect after the list finishes. If a character sequence beginning with "((" would be parsed by the shell as an arithmetic expansion if preceded by a '$', shells which implement an extension whereby "((expression))" is evaluated as an arithmetic expression may treat the "((" as introducing as an arithmetic evaluation instead of a grouping command. A conforming application shall ensure that it separates the two leading '(' characters with white space to prevent the shell from performing an arithmetic evaluation.

## Context

**Behavior:** The `( compound-list )` grouping command must execute the compound-list in a **subshell environment** (see 2.13 Shell Execution Environment). This means:
1. Variable assignments made inside `( ... )` must not persist after the subshell exits.
2. Built-in commands that modify the shell environment (e.g., `cd`, `export`, `set`) must not affect the parent shell.
3. The subshell inherits copies of all variables, functions, traps, umask, working directory, etc.

**Arithmetic extension ambiguity:** If a sequence beginning with `((` could be parsed as an arithmetic expansion when preceded by `$`, and the shell implements the `((expression))` arithmetic evaluation extension, the shell **may** treat `((` as arithmetic evaluation rather than a grouping command. A conforming application must separate the two `(` characters with whitespace to avoid ambiguity (e.g., `( (cmd) )`).

**Applies to:** All shell modes.
