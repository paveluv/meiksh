## Extracted Text

> The terminating logical line delimiter (if any) shall be removed from the input. Then, if the shell variable IFS (see 2.5.3 Shell Variables) is set, and its value is an empty string, the resulting data shall be assigned to the variable named by the first var operand, and the variables named by other var operands (if any) shall be set to the empty string. No other processing shall be performed in this case.

## Context

**Behavior required:** After reading the logical line:

1. **Remove the terminating delimiter:** Strip the trailing newline (or `-d delim` character) from the input. If EOF was reached without a delimiter, there is no delimiter to remove.

2. **Special case when IFS is set to empty string (`IFS=''`):**
   - The entire input (minus delimiter) is assigned to the FIRST var operand with no field splitting.
   - All remaining var operands (if any) are set to the empty string.
   - No backslash processing (post-splitting removal) is performed in this case. No field splitting occurs.

**Condition:** This special case applies when `IFS` is set AND its value is exactly the empty string (zero-length). This is distinct from `IFS` being unset.

**Implementation detail:** Check `IFS` early. If `IFS` is set to `""`:
- Assign the entire (delimiter-stripped) line to `$1` var
- Set remaining vars to `""`
- Skip field splitting entirely
