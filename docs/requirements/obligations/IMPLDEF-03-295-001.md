## Extracted Text

> A file system that has implementation-defined characteristics restricting modifications.

## Context

**Definition only — low shell relevance.** Defines "read-only file system" as one with implementation-defined characteristics restricting modifications.

**Shell impact:** The shell may encounter EROFS errors when attempting redirections (`>`, `>>`) or creating temporary files on read-only file systems. The shell should propagate such errors with appropriate messages. No special handling beyond standard errno reporting is needed.
