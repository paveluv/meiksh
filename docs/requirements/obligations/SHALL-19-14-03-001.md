## Extracted Text

> The rules described so far in 2.14.1 Patterns Matching a Single Character and 2.14.2 Patterns Matching Multiple Characters are qualified by the following rules that apply when pattern matching notation is used for filename expansion:: The <slash> character in a pathname shall be explicitly matched by using one or more <slash> characters in the pattern; it shall neither be matched by the <asterisk> or <question-mark> special characters nor by a bracket expression. <slash> characters in the pattern shall be identified before bracket expressions; thus, a <slash> cannot be included in a pattern bracket expression used for filename expansion. If a <slash> character is found following an unescaped <left-square-bracket> character before a corresponding <right-square-bracket> is found, the open bracket shall be treated as an ordinary character. For example, the pattern "a[b/c]d" does not match such pathnames as abd or a/d. It only matches a pathname of literally a[b/c]d.

## Context

**Slash handling in filename expansion patterns.**

During pathname expansion (globbing), the `/` character in pathnames **shall** be explicitly matched — it has special treatment:

1. `/` **shall not** be matched by `*`, `?`, or any bracket expression `[...]`.
2. `/` characters in the pattern **shall** be identified before processing bracket expressions. Therefore `/` cannot appear inside a bracket expression used for filename expansion.
3. If a `/` is found after an unescaped `[` but before the corresponding `]`, the `[` **shall** be treated as an ordinary character (not introducing a bracket expression).

Example: the pattern `a[b/c]d` does NOT match `abd` or `a/d`. It only matches the literal filename `a[b/c]d` because the `/` inside brackets invalidates the bracket expression, making `[` a literal.

Implementation: when parsing patterns for filename expansion, split on `/` first to get path components, then match each component independently. Each component's pattern matcher must not allow `?`, `*`, or `[...]` to match `/`.
