## Extracted Text

> The following options shall be supported:: Write a string to standard output that indicates the pathname or command that will be used by the shell, in the current shell execution environment (see 2.13 Shell Execution Environment), to invoke command_name, but do not invoke command_name. Executable utilities, regular built-in utilities, command_names including a <slash> character, and any implementation-provided functions that are found using the PATH variable (as described in 2.9.1.4 Command Search and Execution), shall be written as absolute pathnames. Shell functions, special built-in utilities, regular built-in utilities not associated with a PATH search, and shell reserved words shall be written as just their names. An alias shall be written as a command line that represents its alias definition. Otherwise, no output shall be written and the exit status shall reflect that the name was not found.

## Context

**Behavior:** Full specification of the `-v` option output. `command -v name` must write to stdout based on the category of `name` in the current shell execution environment:

1. **Executable utilities** (found via PATH search): Write the absolute pathname (e.g., `/usr/bin/ls`).
2. **Regular built-in utilities** (associated with a PATH search, e.g., found in a PATH directory): Write the absolute pathname.
3. **command_names containing a `/`** character: Write the absolute pathname.
4. **Implementation-provided functions found via PATH**: Write the absolute pathname.
5. **Shell functions**: Write just the function name (e.g., `myfunc`).
6. **Special built-in utilities** (break, colon, continue, dot, eval, exec, exit, export, readonly, return, set, shift, times, trap, unset): Write just the name.
7. **Regular built-in utilities NOT associated with a PATH search** (e.g., cd, echo, test when built-in): Write just the name.
8. **Shell reserved words** (if, then, else, fi, for, while, etc.): Write just the name.
9. **Aliases**: Write the alias definition as a command line (e.g., `alias ll='ls -la'`).
10. **Not found**: Write nothing to stdout and set exit status to indicate not found (>0).
