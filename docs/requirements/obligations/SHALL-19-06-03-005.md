## Extracted Text

> With both the backquoted and $(commands) forms, the commands string shall be tokenized (see 2.3 Token Recognition) and parsed (see 2.10 Shell Grammar). It is unspecified whether the commands string is parsed and executed incrementally as a program (as for a shell script), or is parsed as a single compound_list that is executed after the string has been completely parsed. In addition, it is unspecified whether the terminating ')' of the $(commands) form can result from alias substitution. With the $(commands) form any syntactically correct program can be used for commands, except that:: If the commands string consists solely of redirections, the results are unspecified.

## Context

**Behavior:** In the `$(commands)` form of command substitution, if the entire `commands` string consists only of redirections (e.g., `$(> /dev/null)` or `$(< file)`), the results are **unspecified**.

**Implementation guidance:** The shell need not support or produce consistent behavior for command substitutions whose body is purely redirections with no actual command. The implementation may treat this as empty output, may execute the redirections in the subshell (potentially creating files), or may produce an error. Since this is unspecified, meiksh should document its chosen behavior.
