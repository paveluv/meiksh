## Extracted Text

> The interpretation of a pathname component is dependent on the value of {NAME_MAX} and _POSIX_NO_TRUNC associated with the path prefix of that component. If any pathname component is longer than {NAME_MAX}, the implementation shall consider this an error.

## Context

When the shell resolves pathnames (for commands, redirections, `cd`, etc.), if any single component of the path exceeds the file system's `{NAME_MAX}` limit, the operation must fail with an error (typically `ENAMETOOLONG`). The shell itself does not need to check this — the OS kernel enforces this during pathname resolution syscalls. However, the shell should propagate such errors correctly to the user (e.g., "No such file or directory" or "File name too long" diagnostic).

See: `basedefs/V1_chap04.html` §4.16 Pathname Resolution.
