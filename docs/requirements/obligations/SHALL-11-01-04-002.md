## Extracted Text

> If a process is in the foreground process group of its controlling terminal, write operations shall be allowed as described in 11.1.8 Writing Data and Output Processing. Attempts by a process in a background process group to write to its controlling terminal shall cause the process group to be sent a SIGTTOU signal unless one of the following special cases applies: if TOSTOP is not set, or if TOSTOP is set and the process is ignoring the SIGTTOU signal or the writing thread is blocking the SIGTTOU signal, the process is allowed to write to the terminal and the SIGTTOU signal is not sent. If TOSTOP is set, the process group of the writing process is orphaned, the writing process is not ignoring the SIGTTOU signal, and the writing thread is not blocking the SIGTTOU signal, the write() shall return -1, with errno set to [EIO] and no signal shall be sent.

## Context

**Behavior:** Terminal write access control:
- Foreground process group → writes are allowed normally
- Background process group attempting to write:
  - TOSTOP not set → write is allowed (no signal)
  - TOSTOP set → SIGTTOU sent to process group (default: stop)
  - TOSTOP set but process ignoring/blocking SIGTTOU → write is allowed
  - TOSTOP set, orphaned process group, not ignoring/blocking SIGTTOU → write() returns -1/EIO

**Error handling:** Orphaned background process group with TOSTOP → write returns -1 with errno=EIO.

**Shell relevance:** The shell must handle SIGTTOU correctly. It should ignore SIGTTOU itself. Background jobs may be stopped by SIGTTOU if TOSTOP is set. The shell's `stty` builtin controls the TOSTOP flag.
