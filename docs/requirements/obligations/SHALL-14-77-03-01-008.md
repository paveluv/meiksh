## Extracted Text

> The above types shall be all unsigned integer types.

## Context

**Behavior:** The types `cc_t`, `speed_t`, and `tcflag_t` **shall** all be unsigned integer types. This guarantees well-defined behavior for bitwise operations (no sign extension) and ensures that all values are non-negative.

**Conditions:** Required by `<termios.h>`.

**Shell relevance:** meiksh can safely use bitwise operations (OR, AND, complement, shift) on `tcflag_t` values without concern about sign-related undefined behavior. Comparisons with `cc_t` values (e.g., checking for `_POSIX_VDISABLE`) work predictably since the type is unsigned.

**Spec reference:** POSIX.1-2024, `<termios.h>` The termios Structure (tag_14_77_03_01), `basedefs/termios.h.html` line 51.
