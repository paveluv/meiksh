## Extracted Text

> The following exit values shall be returned:: >1

## Context

**Behavior:** `getopts` shall return an exit status greater than 1 when a getopts processing error occurs. This is distinct from application-level errors (unknown option, missing argument) which return 0.

**Conditions that produce >1:**
- Failure to set `OPTIND`, `OPTARG`, or the `name` variable because it is marked readonly.
- Any other internal getopts processing failure.

**Spec reference:** POSIX.1-2024, `getopts` EXIT STATUS, line 208 of `utilities/getopts.html`.
