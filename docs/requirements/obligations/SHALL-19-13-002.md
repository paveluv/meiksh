## Extracted Text

> If the utility is a shell script, traps caught by the shell shall be set to the default values and traps ignored by the shell shall be set to be ignored by the utility; if the utility is not a shell script, the trap actions (default or ignore) shall be mapped into the appropriate signal handling actions for the utility

## Context

**Trap disposition inheritance for external utilities and shell scripts.**

When forking a child to execute a utility:

1. **If the utility is a shell script**: Traps that the shell has *caught* (i.e., has a trap action set via `trap`) **shall** be reset to their default actions (SIG_DFL) in the child. Traps that are *ignored* (SIG_IGN) **shall** remain ignored in the child shell script.

2. **If the utility is NOT a shell script** (binary executable): The trap actions (either default or ignore) **shall** be mapped to the corresponding signal handling actions. Caught traps become SIG_DFL; ignored traps become SIG_IGN. (There is no concept of "caught" for an exec'd binary — only default or ignore.)

Implementation: Before `exec()`, iterate over the signal table. For each signal with a trap handler, call `sigaction()` to reset to SIG_DFL. For signals set to SIG_IGN, leave them as SIG_IGN. This applies regardless of whether the target is a script or binary.
