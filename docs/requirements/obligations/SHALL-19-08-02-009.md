## Extracted Text

> Otherwise, the exit status shall be the value obtained by the equivalent of the WEXITSTATUS macro applied to the status obtained by the wait() function (as defined in the System Interfaces volume of POSIX.1-2024). Note that for C programs, this value is equal to the result of performing a modulo 256 operation on the value passed to _Exit(), _exit(), or exit() or returned from main().

## Context

**Behavior:** For normally-terminated commands, the exit status MUST be `WEXITSTATUS(wait_status)`, i.e., the low 8 bits of the value passed to exit(). This is a standalone restatement of SHALL-19-08-02-005.

**Applies to:** The normal termination case after waitpid() where WIFEXITED is true. The most common exit status path.

**Implementation detail:** Same as SHALL-19-08-02-005. Use `WEXITSTATUS()` macro on the wait status. The refs to wait/exit/etc. are OS-level functions, not shell-level commands.

**Spec reference:** Section 2.8.2, lines 1706-1710 in `utilities/V3_chap02.html`.
