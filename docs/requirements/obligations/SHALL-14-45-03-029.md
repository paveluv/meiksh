## Extracted Text

> [CX] The storage occupied by sa_handler and sa_sigaction may overlap, and a conforming application shall not use both simultaneously.

## Context

The `sa_handler` and `sa_sigaction` members of the `sigaction` structure may share storage (typically implemented as a union). A conforming application must not set both simultaneously—only one should be used per `sigaction()` call. For `meiksh`, when configuring signal handlers: if using the simple handler (`sa_handler`), do not read `sa_sigaction`, and vice versa. When using `SA_SIGINFO` flag, set `sa_sigaction`; otherwise, set `sa_handler`. The shell must be consistent about which member it uses for each signal configuration.
