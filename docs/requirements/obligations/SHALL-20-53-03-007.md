## Extracted Text

> When getopts reports end of options (that is, when exit status is 1), the value of OPTIND shall be the integer index of the next element of the parameter list (if any).

## Context

This obligation (duplicate extraction of the OPTIND-at-end-of-options rule) specifies that when `getopts` returns exit status 1 (end of options), `OPTIND` must be set to the index of the next parameter element. This is typically the index of the first non-option argument (operand), or `$# + 1` (or `param count + 1`) if all parameters were options. The `name` variable is set to `?` at end-of-options. This value of `OPTIND` allows `shift $((OPTIND - 1))` to remove all processed options. Reference: POSIX.1-2024 §getopts, DESCRIPTION (tag_20_53_03).
