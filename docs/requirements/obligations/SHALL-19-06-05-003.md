## Extracted Text

> If the IFS variable is set and has an empty string as its value, no field splitting shall occur. However, if an input field which contained the results of an expansion is entirely empty, it shall be removed. Note that this occurs before quote removal; any input field that contains any quoting characters can never be empty at this point. After the removal of any such fields from the input, the possibly modified input field list shall become the output.

## Context

**Behavior:** When `IFS=""` (set to empty string), field splitting is disabled with one exception:

1. **No splitting:** No IFS-based field splitting occurs at all — the field passes through as-is.
2. **Empty field removal:** If an input field is **entirely empty** AND it contained expansion results, that field must be removed (producing zero fields instead of one empty field). This handles cases like `$empty_var` where the variable is empty — the expansion result is empty, and the field is removed.
3. **Quote-protected fields survive:** This removal happens **before** quote removal, so any field that contains quoting characters (even quoting an empty value, like `""` or `''`) cannot be empty at this stage and is preserved.

**Inputs/Outputs (with `IFS=""`):**
- `x=""; echo $x` → no output (empty field removed)
- `x=""; echo "$x"` → one empty line (quoting protects the field)
- `x="a b c"; echo $x` → `a b c` (no splitting, spaces preserved)
