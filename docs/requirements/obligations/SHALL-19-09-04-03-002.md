## Extracted Text

> First, the list of words following in shall be expanded to generate a list of items. Then, the variable name shall be set to each item, in turn, and the compound-list executed each time. If no items result from the expansion, the compound-list shall not be executed. Omitting:

## Context

**Behavior:** Execution of `for name in word... ; do compound-list ; done`:
1. The word list following `in` must be expanded (tilde expansion, parameter expansion, command substitution, arithmetic expansion, field splitting, pathname expansion) to generate a list of items.
2. The variable `name` is set to each item in turn, and the compound-list is executed for each iteration.
3. If expansion produces **no items** (empty list), the compound-list must **not** be executed at all.

**Omitting `in word...`:** When the `in word...` clause is omitted, the for loop iterates over the positional parameters (`"$@"`). See SHALL-19-09-04-03-003.

**Exit status:** The exit status of the `for` loop is the exit status of the last command executed in the compound-list. If the list of items is empty and the compound-list is never executed, the exit status is zero.

**Applies to:** All shell modes.
