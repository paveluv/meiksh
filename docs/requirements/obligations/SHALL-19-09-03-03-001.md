## Extracted Text

> The exit status of an asynchronous AND-OR list shall be zero.

## Context

When an AND-OR list (a sequence of pipelines connected by `&&` or `||`) is executed **asynchronously** (i.e., terminated with `&`), the shell must immediately set the exit status of the entire asynchronous list to **zero**, regardless of what the individual pipelines within the list eventually return.

This is because the shell does not wait for the asynchronous list to complete before continuing. The `$?` value visible to subsequent commands must be 0 after launching the async list.

Example: `false && true &` — the exit status of this async AND-OR list is 0, even though `false` would normally cause the AND-OR list to stop with exit status 1.

See 2.9.3.1 Asynchronous AND-OR Lists in utilities/V3_chap02.html#tag_19_09_03_02 for the async execution model.
