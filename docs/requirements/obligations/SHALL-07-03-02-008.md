## Extracted Text

> User-defined ordering of collating elements. Each collating element shall be assigned a collation value defining its order in the character (or basic) collation sequence. This ordering is used by regular expressions and pattern matching and, unless collation weights are explicitly specified, also as the collation weight to be used in sorting.

## Context

**Behavior:** Duplicate of the core ordering rule from SHALL-07-03-02-003. Each collating element has a collation value defining its basic sequence position. This ordering governs regular expressions and pattern matching, and serves as the default sorting weight.

**Shell relevance:** Same as SHALL-07-03-02-003. The basic collation sequence directly affects bracket expression ranges (`[a-z]`) in the shell's pattern matching. The shell must use the locale's collation order, not encoding order, for range evaluation (except in the POSIX locale where they are identical).
