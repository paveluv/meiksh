## Extracted Text

> Use Alternative Value. If parameter is unset or null, null shall be substituted; otherwise, the expansion of word (or an empty string if word is omitted) shall be substituted.

## Context

**Behavior:** The `${parameter:+word}` expansion (Use Alternative Value) — the logical inverse of `:-`:
- If `parameter` is **unset or null**: substitute **null** (empty string); `word` is NOT expanded.
- If `parameter` is **set and non-null**: expand `word` and substitute the result. If `word` is omitted, substitute an empty string.

This is useful for conditionally including text only when a variable is set, e.g., `${DEBUG:+--verbose}`.

**No-colon variant:** `${parameter+word}` only tests for unset (not null) — see SHALL-19-06-02-010.

**Inputs/Outputs:**
- `${var:+yes}` → if `var` is set and non-null, yields `yes`; otherwise yields empty
- `${var+yes}` (no colon) → if `var` is set (even to empty), yields `yes`; only yields empty if truly unset

**Error handling:** No error conditions.

**Spec reference:** See 2.6.2 Parameter Expansion in utilities/V3_chap02.html, lines 819–821.
