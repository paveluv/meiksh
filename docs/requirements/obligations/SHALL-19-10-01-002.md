## Extracted Text

> If the token is an operator, the token identifier for that operator shall result.

## Context

When the lexer produces a token that matches one of the shell operators (control operators: `&`, `&&`, `(`, `)`, `;`, `;;`, `;&`, `;;&`, `\n`, `|`, `||`; and redirection operators: `<`, `>`, `<<`, `>>`, `<&`, `>&`, `<>`, `<<-`, `>|`), the token identifier must be that specific operator token. The lexer must recognize multi-character operators as single tokens (e.g., `&&` is one token, not two `&` tokens). The operator token types are used directly by the grammar productions.

This is the first classification rule and takes priority: if the token is an operator, it is classified as that operator regardless of context.
