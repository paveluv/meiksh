## Extracted Text

> An attribute of a new process identifying the parent of the process. The parent process ID of a process is the process ID of its creator, for the lifetime of the creator. After the creator's lifetime has ended, the parent process ID is the process ID of an implementation-defined system process.

## Context

**Definition — moderate shell relevance.** Defines "parent process ID" as an attribute of a new process. It equals the creator's PID during the creator's lifetime. After the creator terminates, the parent PID becomes an implementation-defined system process (typically PID 1, init/systemd).

**Shell impact:** The shell is the parent process of all commands it executes. The `$PPID` special parameter reports the parent PID of the shell itself. When the shell forks child processes, those children have the shell's PID as their parent process ID. If the shell dies, orphaned children are reparented to the init process. The `wait` builtin can only wait for the shell's own children.
