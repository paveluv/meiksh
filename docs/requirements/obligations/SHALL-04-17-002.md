## Extracted Text

> A process ID shall not be reused by the system until the process lifetime ends. In addition, if there exists a process group whose process group ID is equal to that process ID, the process ID shall not be reused by the system until the process group lifetime ends. A process that is not a system process shall not have a process ID of 1.

## Context

The shell stores process IDs for background jobs, the `$!` special parameter (PID of last background process), and the `$$` special parameter (PID of the shell itself). This guarantee ensures:

1. **`$!` validity**: The PID stored in `$!` remains valid for `wait` and `kill` operations until the process terminates and is waited for.
2. **Job control safety**: PIDs tracked in the shell's job table will not be reused while the process or its process group is still alive.
3. **PID 1 reserved**: The shell process itself (when not running as init) will never have PID 1. PID 1 is reserved for the system init process.

The shell relies on `waitpid()` to reap child processes, after which the PID may be reused. The shell must ensure it doesn't reference stale PIDs after reaping.

See: `basedefs/V1_chap04.html` §4.17 Process ID Reuse.
