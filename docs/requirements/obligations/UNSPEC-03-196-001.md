## Extracted Text

> The unspecified activity by which a user gains access to the system. Each login is associated with exactly one login name.

## Context

**Definition — low shell relevance.** Defines "login" as the unspecified activity by which a user gains access to the system. Each login is associated with exactly one login name.

**Shell impact:** The shell is often the user's login program (login shell). A login shell is identified by a `-` prefix on `argv[0]` (e.g., `-sh`). Login shells execute profile files (`/etc/profile`, `~/.profile`). The concept of "login" informs when the shell reads these startup files. The `HOME`, `LOGNAME`, and `USER` environment variables are set during login.
