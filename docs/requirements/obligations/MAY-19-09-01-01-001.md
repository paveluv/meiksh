## Extracted Text

> In the preceding list, the order of steps 3 and 4 may be reversed if no command name results from step 2 or if the command name matches the name of a special built-in utility; see 2.15 Special Built-In Utilities.

## Context

**Behavior:** The order of steps 3 (redirections) and 4 (variable assignments) MAY be reversed in two cases: (1) when no command name results from step 2, or (2) when the command name matches a special built-in utility (see 2.15). This means for `VAR=x >file` or `VAR=x readonly Y`, the shell is permitted to perform variable assignments before redirections or vice versa.

**Applies to:** Simple commands with no command name (pure assignments with redirections) and simple commands invoking special built-in utilities.

**Implementation detail:** meiksh may choose either ordering for these cases. The conventional choice is to perform redirections first (step 3 then 4), which matches the numbered order. The flexibility mainly matters for edge cases where assignment expansion triggers side effects that interact with redirections.

**Spec reference:** Section 2.9.1.1, lines 1755-1756 in `utilities/V3_chap02.html`. Cross-ref: 2.15 Special Built-In Utilities.
