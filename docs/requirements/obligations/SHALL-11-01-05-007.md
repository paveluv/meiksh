## Extracted Text

> If there is no data available, the read() shall return -1, with errno set to [EAGAIN].

## Context

**Behavior:** Duplicate of SHALL-11-01-05-004 (standalone extraction). Non-blocking read with empty input queue returns -1/EAGAIN.

**Conditions:** O_NONBLOCK set, no data available.

**Error handling:** Return -1, errno=EAGAIN.

**Shell relevance:** The shell must treat EAGAIN as a non-fatal condition and retry.

**Spec reference:** XBD 11.1.5 "Input Processing and Reading Data" (tag_11_01_05), `basedefs/V1_chap11.html`, lines 146–147.
