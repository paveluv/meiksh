## Extracted Text

> When a directory that is the root directory or current working directory of any process is removed, the effect is implementation-defined. If file access permissions deny access, the requested operation shall fail. Otherwise, when a file is removed:: Its directory entry shall be removed from the file system.

## Context

When a file is removed, its directory entry shall be removed from the file system (step 1 of file removal). The filename is unlinked from the directory. For `meiksh`, this is handled by the OS when utilities like `rm` call `unlink()` or `rmdir()`. The shell does not directly perform this operation but may be affected when files it references are removed. See POSIX Utilities Chapter 1, Section 1.1.1.5 item 1 (utilities/V3_chap01.html line 698).
