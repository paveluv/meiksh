## Extracted Text

> The following variables shall affect the execution of the shell:: [UP] The processing of the PS2 shell variable shall be supported if the system supports the User Portability Utilities option. Each time the user enters a <newline> prior to completing a command line in an interactive shell, the value of this variable shall be subjected to parameter expansion (see 2.6.2 Parameter Expansion). Whether the value is also subjected to command substitution (see 2.6.3 Command Substitution) or arithmetic expansion (see 2.6.4 Arithmetic Expansion) or both is unspecified. After expansion, the value shall be written to standard error. The default value shall be "> ".

## Context

This defines the full semantics of the `PS2` variable (continuation prompt). Tagged `[UP]`.

**Required behavior:**
- Processing of `PS2` must be supported (meiksh targets UP compliance).
- Each time the user enters a `<newline>` prior to completing a command line in an interactive shell (i.e., the shell needs more input to complete a compound command, here-document, quoted string, etc.), the value of `PS2` is expanded and written to standard error.
- **Expansion rules:**
  - PS2 must be subjected to parameter expansion (2.6.2).
  - Whether command substitution (2.6.3) and/or arithmetic expansion (2.6.4) are also performed is unspecified (implementation choice).
- **Default value:** `"> "`.
- **Output:** Written to standard error (fd 2).

**Conditions:** Interactive shells only. Displayed when:
- An incomplete compound command needs closing (`if`/`then`/`fi`, `while`/`do`/`done`, etc.)
- A here-document delimiter hasn't been reached
- A line continuation with `\<newline>` occurs
- An unclosed quote is detected

See 2.5.3 "Shell Variables" (`PS2`) and 2.6.2–2.6.4 in `utilities/V3_chap02.html`.
