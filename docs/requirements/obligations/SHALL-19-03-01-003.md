## Extracted Text

> After a token has been categorized as type TOKEN (see 2.10.1 Shell Grammar Lexical Conventions), including (recursively) any token resulting from an alias substitution, the TOKEN shall be subject to alias substitution if all of the following conditions are true:: The TOKEN is a valid alias name (see XBD 3.10 Alias Name).

## Context

**Behavior:** Alias substitution condition 2: The TOKEN must be a valid alias name as defined in XBD 3.10 Alias Name. Per that definition, a valid alias name consists of characters from the portable character set that are not control characters, are not a blank, and are not any of: `{`, `}`, `=`, `!`, `%`, `#`, `^`, `(`, `)`, `-`. Additionally, the name must not be empty. (In practice, alias names are typically alphanumeric strings, potentially with underscores, plus some punctuation.)

**Applies to:** All shell modes during alias processing.

**Implementation detail:** Before looking up the alias, validate that the token text matches the alias name character rules from `basedefs/V1_chap03.html#tag_03_10`. This prevents tokens containing operator characters or other invalid alias-name characters from being looked up.
