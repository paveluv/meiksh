## Extracted Text

> The following tokens are processed (in addition to those string constants shown in the grammar):: Represents a numeric constant. It shall be an integer in the range 0 <=DUP_COUNT <={RE_DUP_MAX}. This token is only recognized when the context of the grammar requires it. At all other times, digits not preceded by a <backslash> character are treated as ORD_CHAR.

## Context

The `DUP_COUNT` token in the RE grammar represents a numeric constant used in interval expressions (`{m}`, `{m,}`, `{m,n}`). It must be an integer in the range 0 to RE_DUP_MAX (at least 255). The lexer must only recognize digit sequences as `DUP_COUNT` when the grammar context expects it (i.e., inside `{...}` interval expressions). In all other contexts, digit characters not preceded by a backslash must be treated as ordinary characters (`ORD_CHAR`) that match themselves literally. This prevents digit sequences in normal ERE text from being misinterpreted as interval counts.
