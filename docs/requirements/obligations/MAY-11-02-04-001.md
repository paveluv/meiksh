## Extracted Text

> If the object for which the control modes are set is not an asynchronous serial connection, some of the modes may be ignored; for example, if an attempt is made to set the baud rate on a network connection to a terminal on another host, the baud rate need not be set on the connection between that terminal and the machine to which it is directly connected.

## Context

**Behavior (optional):** If the terminal object is not an asynchronous serial connection (e.g., it is a network connection to a remote terminal or a pseudo-terminal), some control modes may be ignored. For example, setting the baud rate on a networked terminal connection need not affect the baud rate on the remote end's physical connection.

**Relevance to meiksh:** The `stty` builtin may accept control mode settings (like baud rate) for non-serial connections without error, even if they have no effect. This is permissive — the shell should not reject valid `c_cflag` values just because the terminal is a pty or network terminal.

**Spec reference:** POSIX.1-2024, §11.2.4 Control Modes (tag_11_02_04), `basedefs/V1_chap11.html` line 1081.
