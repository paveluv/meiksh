## Extracted Text

> If a process that the shell is waiting for is part of a foreground job that was started as a foreground job and is stopped by a catchable signal (SIGTSTP, SIGTTIN, or SIGTTOU):: Otherwise, the shell shall create a suspended job consisting of a set of commands, from within the list comprising the foreground job, that is unspecified except that the set shall include at least the pipeline to which the stopped process belongs. Commands in the foreground job that have not already completed and are not included in the suspended job shall be discarded.

## Context

**Foreground job suspension — complex pipeline case (parent obligation):** When the stopped process is in a foreground job and the currently executing AND-OR list does NOT consist of a single simple-command-only pipeline (e.g., it contains compound commands, functions, or a multi-pipeline AND-OR list), the shell shall create a suspended job from an **unspecified** set of commands from within the list, with the only guarantee being that the set includes at least the pipeline containing the stopped process.

Commands not yet completed and not included in the suspended job shall be discarded. This gives the implementation flexibility in how much of the list to preserve when suspending complex constructs. See SHALL-19-11-025 for the detail version.
