## Extracted Text

> A bracket expression shall match a single character or a single collating element.

## Context

**Behavior:** A bracket expression shall match a single character or a single collating element. The full bracket expression rules are in section 9.3.5.

**Shell relevance:** Bracket expressions in shell patterns (`[abc]`, `[a-z]`, `[[:alpha:]]`, etc.) must match exactly one character or one collating element from the input. The shell uses bracket expressions identically to how they appear in BREs, with modifications described in section 9.3.5 for shell pattern context (e.g., `!` instead of `^` for negation in some contexts).
