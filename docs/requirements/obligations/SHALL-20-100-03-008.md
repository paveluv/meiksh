## Extracted Text

> The input to the algorithm shall be the logical line (minus terminating delimiter) that was read from standard input, and shall be considered as a single initial field, all of which resulted from expansions, with any escaped byte and the preceding <backslash> escape character treated as if they were the result of a quoted expansion, and all other bytes treated as if they were the results of unquoted expansions.

## Context

**Behavior required:** This is the same text as SHALL-20-100-03-006, extracted as a standalone obligation. The input to the `read` field splitting algorithm is the logical line with the delimiter removed, treated as a single initial field where:

- Escaped bytes (byte preceded by unescaped `\` when `-r` not active) are treated as quoted (not subject to IFS splitting)
- All other bytes are treated as unquoted (subject to IFS splitting)

**Condition:** During the field splitting phase of `read` when `IFS` is unset or non-empty.

**Implementation detail:** The field splitting implementation must distinguish between "escaped" and "non-escaped" bytes in the input. Escaped bytes must not be treated as IFS delimiters even if they match an IFS character. The backslash-byte pair is treated as a single quoted unit for splitting purposes, and the backslash is removed after splitting.
