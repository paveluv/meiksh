## Extracted Text

> The syntax of the shell command language has an ambiguity for expansions beginning with "$((", which can introduce an arithmetic expansion or a command substitution that starts with a subshell. Arithmetic expansion has precedence; that is, the shell shall first determine whether it can parse the expansion as an arithmetic expansion and shall only parse the expansion as a command substitution if it determines that it cannot parse the expansion as an arithmetic expansion. The shell need not evaluate nested expansions when performing this determination. If it encounters the end of input without already having determined that it cannot parse the expansion as an arithmetic expansion, the shell shall treat the expansion as an incomplete arithmetic expansion and report a syntax error. A conforming application shall ensure that it separates the "$(" and '(' into two tokens (that is, separate them with white space) in a command substitution that starts with a subshell. ## Context

**Behavior:** The `$((` ambiguity resolution rule:

1. **Arithmetic takes precedence:** When the parser encounters `$((`, it must first attempt to parse it as an arithmetic expansion `$(( expression ))`. Only if parsing as arithmetic fails should it fall back to treating it as `$( (subshell) )` (command substitution containing a subshell).
2. **No nested expansion evaluation required:** The shell does not need to evaluate nested expansions within the candidate arithmetic expression when making this determination — syntactic analysis is sufficient.
3. **End-of-input behavior:** If the shell reaches end of input while still trying to determine whether `$((` is arithmetic or command-substitution, it must treat it as an **incomplete arithmetic expansion** and report a syntax error.
4. **Application responsibility:** Conforming applications must separate `$(` and `(` with whitespace when intending a command substitution that starts with a subshell: `$( (commands) )` instead of `$((commands))`.

**Inputs/Outputs:**
- `$((1+2))` → arithmetic expansion, yields `3`
- `$( (echo hello) )` → command substitution containing a subshell, yields `hello`
- `$((echo hello))` → parsed as arithmetic first; if that fails, may fall back to command sub; behavior depends on implementation
