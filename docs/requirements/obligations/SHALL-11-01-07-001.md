## Extracted Text

> In non-canonical mode input processing, input bytes are not assembled into lines, and erase and kill processing shall not occur. The values of the MIN and TIME members of the c_cc array are used to determine how to process the bytes received. POSIX.1-2024 does not specify whether the setting of O_NONBLOCK takes precedence over MIN or TIME settings. Therefore, if O_NONBLOCK is set, read() may return immediately, regardless of the setting of MIN or TIME. Also, if no data is available, read() may either return 0, or return -1 with errno set to [EAGAIN].

## Context

**Behavior:** In non-canonical mode (ICANON not set):
1. Input bytes are **not** assembled into lines — each byte is available individually.
2. Erase and kill processing **shall not** occur — ERASE and KILL characters are passed through as data.
3. The `c_cc[VMIN]` and `c_cc[VTIME]` values control when `read()` is satisfied (see Cases A–D in §11.1.7.1–11.1.7.4).
4. The interaction between `O_NONBLOCK` and MIN/TIME is **unspecified** — `read()` may return immediately with 0 or -1/EAGAIN when O_NONBLOCK is set, regardless of MIN/TIME.

**Conditions:** Applies when ICANON is not set in `c_lflag`.

**Shell relevance:** This is the primary mode for the shell's interactive line editor (emacs/vi mode). The shell clears ICANON and sets VMIN=1, VTIME=0 to get single-character input. The shell must handle ERASE/KILL itself since the terminal driver won't process them. The shell should avoid setting O_NONBLOCK on the terminal fd when using MIN/TIME, as the interaction is unspecified.

**Spec reference:** POSIX.1-2024, §11.1.7 Non-Canonical Mode Input Processing (tag_11_01_07), `basedefs/V1_chap11.html` lines 172–177.
