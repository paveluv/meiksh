## Extracted Text

> The LC_NUMERIC category shall define the rules and symbols that are used to format non-monetary numeric information. This information is available through the localeconv() function.

## Context

The LC_NUMERIC locale category defines the decimal point character, thousands separator, and digit grouping for non-monetary numeric formatting. The shell is directly affected: the `printf` builtin uses LC_NUMERIC for the `%f`, `%e`, `%g` floating-point conversions and the `%'d` grouping flag. Shell arithmetic (`$(( ))`) uses the POSIX/C locale internally for parsing but the `printf` builtin must honor LC_NUMERIC for output. The shell must correctly set and propagate the LC_NUMERIC environment variable, and its `printf` builtin must use the locale's decimal point when formatting floating-point numbers.
