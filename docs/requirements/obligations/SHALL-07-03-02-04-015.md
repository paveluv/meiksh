## Extracted Text

> This example is interpreted as follows:: All characters between <space> and 'a' shall have the same primary equivalence class and individual secondary weights based on their ordinal encoded values.

## Context

This is an illustrative example from the LC_COLLATE specification showing how collation weight assignment works. In the example, characters between `<space>` and `'a'` are given the same primary weight (placing them in one equivalence class at the primary level) but individual secondary weights based on their code point values (allowing them to be distinguished at the secondary level). The shell is affected because this pattern of weight assignment is common in locale definitions and determines how equivalence classes and multi-level collation work for pattern matching and string comparison.

See: `basedefs/V1_chap07.html` §7.3.2.4 Collation Order.
