## Extracted Text

> Implementations that do not support changing the START and STOP characters may ignore the character values in the c_cc array indexed by the VSTART and VSTOP subscripts when tcsetattr() is called, but shall return the value in use when tcgetattr() is called.

## Context

**Behavior:** For implementations that do not support changing START and STOP characters:
1. `tcsetattr()` **may** ignore values written to `c_cc[VSTART]` and `c_cc[VSTOP]`.
2. `tcgetattr()` **shall** return the actual values currently in use for VSTART and VSTOP.

**Conditions:** This applies only to implementations that choose not to support changing START/STOP (per SHALL-11-01-09-019, whether START/STOP are changeable is implementation-defined).

**Shell relevance:** The `stty` builtin should attempt to set START/STOP when requested by the user. If the implementation silently ignores the change, `stty` should still read back and display the actual values (via `tcgetattr()`). meiksh should document whether it supports changing START/STOP characters.

**Spec reference:** POSIX.1-2024, §11.2.6 Special Control Characters (tag_11_02_06), `basedefs/V1_chap11.html` lines 1353–1355.
