## Extracted Text

> When a given simple command is required to be executed (that is, when any conditional construct such as an AND-OR list or a case statement has not bypassed the simple command), the following expansions, assignments, and redirections shall all be performed from the beginning of the command text to the end:: Each variable assignment shall be expanded for tilde expansion, parameter expansion, command substitution, arithmetic expansion, and quote removal prior to assigning the value.

## Context

**Behavior:** Step 4 of simple command processing: Each variable assignment (saved in step 1) MUST be expanded with tilde expansion, parameter expansion, command substitution, arithmetic expansion, and quote removal BEFORE the value is assigned. Notably, field splitting and pathname expansion are NOT performed on assignment values.

**Applies to:** Variable assignments in simple commands (e.g., `VAR=value cmd` or standalone `VAR=value`). The expansion set is the same as "assignment expansion" — tilde expansion occurs after `=` and after unquoted `:`.

**Implementation detail:** When processing `VAR=value`, expand the `value` part with the restricted expansion set (no field splitting, no globbing). The expanded string becomes the variable's value. This is different from the expansion applied to regular command arguments.
