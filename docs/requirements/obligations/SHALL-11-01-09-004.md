## Extracted Text

> Certain characters have special functions on input or output or both. These functions are summarized as follows:: Special character on input, which is recognized if the ICANON flag is set. Deletes the entire line, as delimited by an NL, EOF, or EOL character. If ICANON is set, the KILL character shall be discarded when processed.

## Context

**Behavior:** The KILL special character (typically ^U):
1. Recognized on input only when ICANON is set in `c_lflag`.
2. Deletes the **entire** current line (all un-delimited data back to the last NL, EOF, or EOL delimiter).
3. The KILL character itself shall be **discarded** when processed (not placed in input queue).
4. No effect if the current line is empty.

**Conditions:** ICANON must be set in `c_lflag`. Only affects un-delimited data in the input queue.

**Shell relevance:** This is the terminal driver's built-in "kill line" in canonical mode. In canonical mode, ^U erases the entire input line. In non-canonical mode (shell line editor), the shell must implement its own kill-line functionality. The `stty kill` command allows users to change which character triggers KILL.

**Spec reference:** POSIX.1-2024, §11.1.9 Special Characters (tag_11_01_09), `basedefs/V1_chap11.html` line 232.
