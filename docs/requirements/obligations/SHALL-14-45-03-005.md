## Extracted Text

> The <signal.h> header shall define the following data types:: sig_atomic_t

## Context

The `<signal.h>` header must define `sig_atomic_t`. This is an integer type that can be accessed atomically even in the presence of asynchronous signal interrupts. For `meiksh`, `sig_atomic_t` should be used for any global flag variables that are set inside signal handlers and read outside them (e.g., a `volatile sig_atomic_t got_sigchld` flag). Using this type ensures correct behavior without requiring additional synchronization.
