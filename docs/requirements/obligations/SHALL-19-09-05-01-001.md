## Extracted Text

> The exit status of a function definition shall be zero if the function was declared successfully; otherwise, it shall be greater than zero. The exit status of a function invocation shall be the exit status of the last command executed by the function.

## Context

Function definitions and invocations have separate exit status rules:

**Function definition** (`fname() compound-command`):
- If the function is declared successfully (valid name, valid body syntax), the exit status is **zero**.
- If the declaration fails (e.g., syntax error in the body, attempting to define a function with a reserved word name), the exit status must be **greater than zero**.

**Function invocation** (calling a previously defined function by name):
- The exit status is the exit status of the **last command executed** within the function body.
- If the function body contains a `return n` command, the exit status is `n`.
- If the function body contains `return` without an argument, the exit status is the exit status of the last command executed before `return`.

Examples:
- `f() { true; }` — definition exit status is 0
- `f() { false; }; f` — invocation exit status is 1
- `f() { return 42; }; f` — invocation exit status is 42

See 2.9.5 Function Definition Command in utilities/V3_chap02.html#tag_19_09_05 and the `return` builtin at 2.25 in utilities/V3_chap02.html#tag_19_25.
