## Extracted Text

> When a directory that is the root directory or current working directory of any process is removed, the effect is implementation-defined. If file access permissions deny access, the requested operation shall fail. Otherwise, when a file is removed:: If the file is a directory that is not empty, the last file status change timestamp shall be marked for update.

## Context

If the removed file is a directory that is not empty, the last file status change timestamp shall be marked for update. Note: this item applies when a non-empty directory removal is attempted — the removal of the directory entry triggers a status change timestamp update even though the directory still contains files (the removal attempt may fail or succeed depending on the utility). For `meiksh`, this is OS-level behavior not directly relevant to shell implementation. See POSIX Utilities Chapter 1, Section 1.1.1.5 item 4 (utilities/V3_chap01.html line 709).
