## Extracted Text

> The exit status of the if command shall be the exit status of the then or else compound-list that was executed, or zero, if none was executed.

## Context

The `if` command (`if condition; then consequent; [elif condition; then consequent;]... [else alternate;] fi`) selects a branch based on the exit status of condition compound-lists. The exit status rules are:

1. **If a `then` or `else` branch was executed**: the exit status is the exit status of that branch's compound-list.
2. **If no branch was executed** (all conditions failed and there is no `else` clause): the exit status is **zero**.

Examples:
- `if true; then false; fi` — exit status is 1 (then-branch ran `false`)
- `if false; then true; fi` — exit status is 0 (no branch executed, no else)
- `if false; then true; else false; fi` — exit status is 1 (else-branch ran `false`)
- `if false; then true; elif true; then false; fi` — exit status is 1 (second then-branch)

See 2.9.4.7 The if Conditional Construct in utilities/V3_chap02.html#tag_19_09_04_07.
